Sample connector and test tools

This section describes Onyx CTI's sample connector and test tools (driver and applet). You can use the sample connector to simulate a functioning connector, providing the communication between Onyx CTI and the simulated CTI system components (test driver and test applet).

The following diagram depicts a high-level view of the architecture of the sample connector, test applet, and test driver, and illustrates their interactions with Onyx CTI.

Sample connector

The Onyx CTI sample connector provides the communication between Onyx CTI and the simulated CTI system components (the Onyx CTI test applet and the Onyx CTI test driver) through the ICTIConnector interface. All methods in the ICTIConnector interface are implemented by the sample connector. Business logic in the sample connector code file explains how to control agent states and call states based on users' actions.

We recommend that you use a copy of the sample connector code file as your starting point when building a new connector. The business logic in the file contains code comments to facilitate removal of select fragments and other customization tasks.

Full path of the sample connector code file: YourOEPwebsite\cti\test\cticonnector.js where YourOEPwebsite is your OEP installation directory.

Test applet

The Onyx CTI test applet is a JavaScript class that simulates a CTI system's client-side applet, and that provides the mechanism for the Onyx CTI test driver to communicate with the Onyx CTI sample connector. The test applet acts as the synchronous CTI system.

The Onyx CTI test applet represents a simulated CTI system that leverages a Java applet for the client-side components of the system. The test applet is not an actual Java applet.

The test applet maintains the agent’s state and the agent's collection of calls. Logic in the test applet determines how to execute events in the sample connector, and how to set and retrieve data for the agent, calls, lists, and CTI system information. Full path of the test applet file: YourOEPwebsite\cti\test\ctiapplet.js where YourOEPwebsite is your OEP installation directory.

Onyx CTI does not have specific client-side requirements for your CTI system beyond the implementation of the ICTIConnector interface required to build your custom connector. The test applet is provided for simulation purposes only.

Test driver

The Onyx CTI test driver provides an HTML-based interface that simulates the functionality of an Automatic Call Distributor (ACD) server system for a single user or agent. This interface includes controls for changing the state of an agent, creating new ACD calls, providing the CTI system summary and detail information, and creating sample lists of agents, queue sequences, and available phone numbers. Path of the test driver files (ctitester*.*): YourOEPwebsite\cti\test\ where YourOEPwebsite is your OEP installation directory.

View an example of the test driver interface

To display the test driver interface:

  1. Install Onyx CTI using the Onyx Computer Telephony Integration Implementation Guide.

    2. The sample connector and test tools are automatically installed.

  2. Click the Open CTI Test Driver button on the Onyx CTI toolbar displayed in OEP.
