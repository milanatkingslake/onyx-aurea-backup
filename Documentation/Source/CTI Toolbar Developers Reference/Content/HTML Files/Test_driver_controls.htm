Test driver controls

This topic explains how to display the test driver interface and describes its controls.

To display the test driver interface:

  1. Install Onyx CTI using the Onyx Computer Telephony Integration Implementation Guide.

    2. The sample connector and test tools are automatically installed.

  2. Click the Open CTI Test Driver button on the Onyx CTI toolbar displayed in OEP.

View an example of the test driver interface

The controls and display items in the Onyx CTI test driver interface are described in the following table.

Control or display item Description

Agent Status

Displays the current status for the associated agent:

  • ID (agent ID)
  • Station ID
  • Name (agent name)
  • State (agent state)
  • Idle Reason (reason selected for being in the Idle state)

Call Status

Displays the status of the current calls:

  • ID (call ID)
  • State (call state)
  • Type (call type)
  • Destination
  • Show Call Xml: when clicked, displays the status of the current calls in XML. Example: <call called="3285" callType="2" callState="2" source="ACD" queue="1" destination="111" connectedCall="1" activeCall="1"/>

Text appended to the "Call Status" title indicates whether the call is currently selected or has not yet been answered:

  • "Call Status (Active)" indicates the currently selected call.
  • "Call Status (Offering)" indicates an incoming call that has not yet been answered.

Links to the right of the "Call Status" title simulate user interactions with the call via an external device, such as a phone. The links allow the user to answer an offered call, hang up (disconnect) a currently selected call, or complete a warm conference or warm transfer.

Agent Status Controls

Includes controls related to the agent state.

Executes an event on the test applet to notify Onyx CTI of changes to the agent status.

Agent State

Sets the agent state. Available states: Ready, Idle, or Wrap Up.

An Idle Reason is required for the Idle state.

Idle Reason

Sets the reason for being in the Idle state.

This control is available only when the Idle state is selected.

ACD Call Data Controls

Includes controls related to calls from the ACD (Automatic Call Distributor).

Sends a call with the entered data (in the Call Actions, Call Search, CTI System Summary, and CTI System Details sections) to the test applet and executes an event on the test applet to notify Onyx CTI that a call is being sent to the agent.

Call Actions

Includes controls to simulate display of OEP records ("screen pops") when the agent answers a call. For each call, you can display one of each of the following types of OEP records: customer, incident, and script.

Existing Customer ID

Sets the ID of an existing customer record to display when the call is answered.

Existing Incident ID

(Displayed when the New Incident check box is cleared.)

Sets the ID of an existing incident record to display when the call is answered.

New Incident

When selected, creates an incident record (using the selected options) when the call is answered.

Category

(Displayed when the New Incident check box is selected.)

Sets the category to be used for the new incident record. This setting is optional. Available options are Sales, Service, and Support.

OwnerId

(Displayed when the New Incident check box is selected.)

Sets the owning customer record by primary ID. This setting is required. An alert is displayed if you leave this setting blank when attempting to send a call.

Existing Script ID

Sets the ID of an existing script to display when the call is answered.

New Script Request

When selected, creates a script record (using the selected options) when the call is answered.

Script Mode

(Displayed when the New Incident check box is selected.)

Includes the following options:

  • Insert Delivery executes the script identified by the selected ID.
  • Script Chooser displays the script chooser.

This setting is required.

Script Id

(Displayed when the New Incident check box is selected.)

Sets the primary ID of the new script. This setting is required when you select Insert Delivery.

Owner Id

(Displayed when the New Incident check box is selected.)

Sets the primary ID of the owning record (customer or incident). This setting is optional.

Owner Type

(Displayed when the New Incident check box is selected.)

Sets the type of the owning record (customer or incident). This setting is optional.

Call Search

Includes controls for filters used in Quick Search (automated display of OEP records) as part of an ACD call. Because filters are run last, they supersede the Existing Customer ID setting.

Filter Field

Sets the name of the field to be used in Quick Search as part of an ACD call. Available options are Phone Number and Email.

Filter Value

Sets the value to be used in Quick Search as part of an ACD call.

CTI System Summary

Includes controls to simulate the summary information sent to Onyx CTI from the CTI system.

Status Summary XML

Defines the CTI system summary information.

Push Interval

Sets the frequency (number of seconds since the last transmission) at which the test driver sends ("pushes") the summary information through the sample connector to Onyx CTI. (The test driver sends the information only when the test driver interface is loaded.) The default frequency of five seconds is used if the box is left empty.

CTI System Details

Includes controls to simulate the detailed information sent to Onyx CTI from the CTI system.

System Details XML

Defines the CTI system information displayed in the CTI System Details window that is accessed from the Onyx CTI toolbar.

Lists

Includes controls to simulate the CTI system's lists of agents, queue items, and available phone numbers. These lists are used to populate the Place Call, Transfer, and Conference windows accessed from the Onyx CTI toolbar.

Agent List

Defines the list of agents in XML.

Queue List

Defines the list of queue items in XML.

Available Numbers List

Defines the list of available phone numbers in XML.
