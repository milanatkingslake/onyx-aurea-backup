Enumeration file

Each connector requires its own unique enumeration file to define the supported enumeration values for agent states, call states, and error return codes.

We recommend that you use a copy of the default enumeration file as your starting point when building a new connector.

To confirm that your changes appear as expected, delete the files in Internet Explorer's Temporary Internet Files folder.

Full path of the default enumeration file: YourOEPwebsite\cti\cti_enummessages.xml where YourOEPwebsite is your OEP installation directory.

View example enumeration file

<enums>

   <agentStateEnums>

      <enum value="0" messageName="agentStateUnknown" />

      <enum value="1" messageName="agentStateReady" />

      <enum value="2" messageName="agentStateIdle" />

      <enum value="3" messageName="agentStateOnCall" />

      <enum value="4" messageName="agentStateWrapUp" />

   </agentStateEnums>

   <callStateEnums>

      <enum value="0" messageName="callStateNoCall" />

      <enum value="1" messageName="callStateOutdial" />

      <enum value="2" messageName="callStateAcdIn" />

      <enum value="3" messageName="callStateConsulting" />

      <enum value="4" messageName="callStateConsulted" />

      <enum value="6" messageName="callStateConference" />

      <enum value="7" messageName="callStateOnHoldACD" />

      <enum value="8" messageName="callStateOnHoldOut" />

      <enum value="9" messageName="callStateConsultConference" />

      <enum value="10" messageName="callStateConsultTransfer" />

   </callStateEnums>

   <returnCodeEnums>

      <enum value="0" messageName="CtiReturnSuccess" />

      <enum value="-1" messageName="CtiReturnMethodError" />

      <enum value="-2" messageName="CtiReturnNoCall" />

      <enum value="-3" messageName="CtiReturnGenericError" />

      <enum value="-4" messageName="CtiReturnNotLoggedOn" />

      <enum value="-5" messageName="CtiReturnInvalidCallStatus" />

   </returnCodeEnums>

</enums>

The following table describes the elements of the enumeration file:

Element Description

enum

Root element.

agentStateEnums

Contains the collection of agent state enumeration definitions. Each state that the agent can be in must be defined in this collection.

callStateEnums

Contains the collection of call state enumeration definitions. Each state that the call can be in must be defined in this collection.

returnCodeEnums

Contains the collection of return code enumeration definitions. Any error code that is returned by the connector must be defined in this collection.

enum

Contains the definition information for an agent, call, or return code enumeration.

The following table describes the attributes of the enumeration file:

Attribute Description

value

Identifies the enumeration definition.

messageName

Lists the resource string ID for the enumeration definition.

Resource string IDs are also listed in the resource string files (name attribute). The resource string files define the text displayed in the Onyx CTI interface. For example, the default enumeration file and the related default resource string file (cti_toolbar.xml) both list the "callStateConsulting" resource string ID; the resource string file defines the text "Consulting" for display in the Onyx CTI interface.
