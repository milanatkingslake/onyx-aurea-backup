Configuration file

Onyx CTI requires multiple copies of the configuration filethe main configuration file plus a configuration file for each connector.

  • The main configuration file specifies the default custom connector. Other settings in this file are used only when the configuration file for the default connector cannot be found; otherwise, these other settings are ignored.
  • Each connector-specific configuration file defines the custom connector's supported behavior and functionality for the Onyx CTI toolbar. The default connector specified in this file is ignored.

We recommend that you use a copy of the default configuration file as your starting point when building a new connector.

To confirm that your changes appear as expected, delete the files in Internet Explorer's Temporary Internet Files folder.

Full path of the default configuration file (also the main configuration file): YourOEPwebsite\cti\cti_config.xml, where YourOEPwebsite is your OEP installation directory.

The configuration file includes sections for the following types of meta-data:

  • Default connector
  • General
  • Toolbars
  • Agent states
  • Call states

The following table describes the elements of the configuration file, including related attributes:

Element Description

defaultConnector

Root element of the default connector section. In the main configuration file, specifies the primaryId of the default connector. The default connector is used if the database does not specify a connector for the logged-in agent. In connector-specific configuration files, this setting is ignored. For more information on agent-connector mappings, see the ctiAgentAssociation object.

general

Root element of the general section. Contains settings that enable or disable functionality in the toolbar and that specify certain configurable data used by Onyx CTI.

debugMode

Enables or disables the display of the Open CTI Test Driver toolbar button, which displays the test driver interface. (default value: 0)
Available values:
0 - disable
1 - enable

autoAnswer

Enables or disables automatic answering of incoming calls by Onyx CTI. When enabled, agents are prompted to accept or reject each call. When disabled, each call that is routed to the agent is automatically accepted and the caller and agent are connected without any prompt. (default value: 0)
Available values:
0 - disable
1 - enable

displayLogon

Enables or disables the display of the Log On window when the Logon toolbar button is clicked. Disabling the display of this window is helpful when the environment has been customized to remove the requirement for a password and user name. (default value: 1)
Available values:
0 - disable
1 - enable

populateMakeCall

Enables or disables the automatic population of the Place Call window with the primary phone number from the currently open customer page.
Available values:
0 - disable
1 - enable

allowedPhoneChars

Contains a list of characters allowed in external phone numbers. This list of values is used to validate a phone number before the number is sent to the connector. The values in this list are considered valid and are allowed.

oepLanguageVersion

Specifies the language of the OEP language-specific code base. Onyx CTI uses this setting to make appropriate calls to the common code.

toolbars

Root element of the toolbars section. Contains meta-data for the Onyx CTI toolbar and its buttons. Elements and attributes in this section are from the common OEP toolbar control.

control

Defines the Onyx CTI toolbar. Do not change this element when configuring or customizing the toolbar.

control

Contains the settings for each control on the Onyx CTI toolbar.
Attributes:
visible - Specifies whether or not the control is visible. Used for commandContainer types of controls only.
Available values:
0 (invisible) or
1 (visible).
interface - Defines the control element as a toolbar or command.
type - Specifies the type of control.
Available values:
commandContainer - Empty container.  This control can be used for layout spacing purposes or any valid HTML elements can be added to the toolbar by adding them as child elements of this control.
commandButton - Button that has only one state (no toggling).
commandMultiState - Button that has multiple states for toggling (such as hold and resume).
Id - Specifies the control ID.  Must correspond to the controlId attribute of the control element in the agentStates and callStates sections of this file.
resourceId - Not used by the Onyx CTI toolbar.

state

Contains the settings for the specified parent control state.
Attributes: value - Identifies the controls state using a unique numeric identifier. There is no limit to the number of states a control can have.
tipResource - Lists the resource string ID for the identified control state. Resource string IDs are also listed in the resource string files (name attribute). The resource string files define the text displayed in the Onyx CTI interface.
Image - Specifies the file name corresponding to the control state's image. The image file must be placed in the following folder: YourOEPwebsite\cti\images\toolbar\, where YourOEPwebsite is your OEP installation directory.

agentStates

Root element of the agent states section. Contains the collection of agentState elements controlling the availability and visibility of the Onyx CTI toolbar buttons according to the agent's current state. Onyx CTI uses this meta-data to render the toolbar whenever the agent state changes; therefore, each agent state defined in the enumeration file must be represented in this section.

agentState

Contains the collection of control elements for the identified agentState element.
Attributes:
state - Specifies the agent state. Must correspond to a state definition in the enumeration file.

control

Contains the settings for the identified control when the agent is in the identified state. Attributes:
controlId - Specifies the control ID.  Must correspond to the Id attribute of the control element in the toolbars section of this file.
enabled - Enables or disables the control.
Available values:
0 (disable) or
1 (enable).
visible - Specifies whether or not the control is visible.
Available values:
0 (invisible) or
1 (visible).

callStates

Root element of the call states section. Contains the collection of callState elements controlling the availability and visibility of the Onyx CTI toolbar buttons according to the call's current state. Onyx CTI uses this meta-data to render the toolbar whenever the call state changes; therefore, each call state defined in the enumeration file must be represented in this section.

callState

Contains the collection of control elements for the identified callState element.
Attributes:
state - Identifies the call state. Must correspond to a state definition in the enumeration file.
hold - Displays the appropriate button for holding calls or resuming held calls, depending on the call state.
Available values:
0 (displays the Hold Current Call button) or
1 (displays the Resume Current Call button).

control

Contains the settings for the identified control when the call is in the identified state. Attributes:
controlId - Specifies the control ID.  Must correspond to the Id attribute of the control element in the toolbars section of this file.
enabled - Enables or disables the control. Available values: 0 (disable) or 1 (enable).
visible - Specifies whether or not the control is visible. Available values: 0 (invisible) or 1 (visible).
