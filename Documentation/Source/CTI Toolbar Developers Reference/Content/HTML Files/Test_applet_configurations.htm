Test applet configurations

You can configure the test applet by editing its configuration file YourOEPwebsite\cti\test\ctiapplet_config.xml, where YourOEPwebsite is your OEP installation directory. By editing this file, you avoid changing the test applet code.

This topic describes the following configurations:

Setting agent state upon logon

You can configure the test applet to automatically set the specified agent state when the agent logs on to Onyx CTI.

To automatically set the agent state upon logon

  • Specify the desired value for the postLoginAgentState configuration setting in the loginConfig section of the ctiapplet_config.xml file.
    • Value Description (agent state) Notes

    1

    Ready

    --

    2

    Idle

    If you set the agent state to Idle, we recommend also setting the Idle Reason. For code descriptions, The numeric values for this setting can be found in the %OEP ROOT%\cti\cti_config.xml file.

    4

    Wrap Up

    --

Controlling automatic call distribution

You can configure the test applet to automatically distribute calls to the agent when the agent is in the Ready state. You can also configure the delay from logon to call distribution and the filter settings for QuickSearch.

To control automatic call distribution

  • Specify the desired values for the desired configuration settings in the defaultCallConfig section of the ctiapplet_config.xml file.

    • Configuration setting Description

    enable

    Enables or disables distribution of the call when the agent is in the Ready state. Available values: 0 (does not distribute the call) and 1 (distributes the call).

    delay

    Time delay, in milliseconds, between the agent's changing to the Ready state and the distribution of the call.

    filterField

    Field name for use in QuickSearch.

    Available values:

    emailAddress

    phoneNumber

    filterValue

    Value to filter on in QuickSearch.

Defining the default data

You can define the default data (such as the CTI system details and the list of agents). This default data is used when data cannot be retrieved from the test driver.

To define the default data:

  • Specify the desired values for the desired configuration settings in the defaultData section of the ctiapplet_config.xml file.

    • For details on each configuration setting (such as the number of IVR prompts available), see the ctiapplet_config.xml file provided with Onyx CTI.

    Configuration setting Description

    summaryInfo

    Specifies the summary system information returned from the ICtiConnector.getStatusSummary method when data cannot be retrieved from the test driver. Values correspond to the data child elements.

    detailInfo

    Specifies the detailed system information returned from the ICtiConnector.getStatusDetails method when data cannot be retrieved from the test driver. Values correspond to the data child elements.

    agentList

    Specifies the agent list returned from the ICtiConnector.getAgentList method when data cannot be retrieved from the test driver. Values correspond to the agentName child elements.

    queueList

    Specifies the queue list returned from the ICtiConnector.getQueueList method when data cannot be retrieved from the test driver. Values correspond to the queueName child elements.

    freqPhoneList

    Specifies the list of frequent phone numbers returned from the ICtiConnector.getFreqPhoneList method when data cannot be retrieved from the test driver. Values correspond to the Name child elements.

Simulating failure of selected methods

You can configure the test applet to simulate failure of selected methods to test Onyx CTI's error handling.

The test applet's configuration file ctiapplet_config.xml lists all applet methods (commented out) for your convenience. When you uncomment a method to simulate its failure, the method returns a CTI_RETURN_METHOD_ERROR.

To simulate method failure:

  • Uncomment the desired methods in the methodFailureConfig section of the ctiapplet_config.xml file.
