CallData XML structure

The CallData Structure contains the details for a specific call.

The connector must send this XML structure in response to the following events: onCallOffered, onCallDataOffered, and OnCallStatusChange.

Example:

<call callId=1234 callState=1 callType=1 source=ACD queue=TechSupport destination=5551234567 filterField= filterValue=>)

   <actions>

   </actions>

</call>
Element Description

call

Root element. Contains the call information.

actions

Contains a collection of action elements, using the ActionData XML structure. Absent for calls without associated actions.

The following table describes the attributes of the CallData XML:

Attribute Description

callId

Specifies the ID of the call.

callState

Specifies the current state of the call. Integer value corresponding to a call state enumeration value.

callType

Specifies the type of call.
Valid values:
1 for internal call
2 for external call

source

Specifies the source of the call (such as ACD or Internal). You can enter values that fit your implementation.

queue

Specifies the queue ID from which the call originated. (Incoming calls are assigned to a queue by the CTI system.)  

destination

Specifies the destination of the call (such as an agent ID, phone number, or queue).

filterField

Specifies the name of a field to be used in a search of OEP records. This attribute is used to automate display of OEP records.
Valid values:
phoneNumber
emailAddress

filterValue

Specifies the value to match in a search of OEP records (used in conjunction with the filterField attribute). This attribute is used to automate display of OEP records.
