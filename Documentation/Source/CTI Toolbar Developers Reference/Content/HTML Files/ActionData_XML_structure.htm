Action Data XML

The ActionData XML structure contains the instructions for automatic display of the indicated customer, incident, and/or script records with the identified call.

If there are actions associated with the call, the connector must send this XML structure, as a child of the call element in the CallData structure, in response to the onCallDataOffered event.

Example:

 <actions>

   <action action=0 type=1 id= idType= category=3 ownerId=12345 ownerIdType=1 filterField= filterValue= scriptid= scriptmode=ownerType=/>

   <action action=1 type=6 id= idType= category=3 ownerId=12345 ownerIdType=1 filterField= filterValue= scriptid= scriptmode=ownerType=/>

   ...

</actions>

The following table describes the elements of the ActionData XML:

Element Description

actions

Root element. Contains the collection of action elements.

action

Contains the instruction for automatic display of OEP records.

The following table describes the attributes of the ActionData XML:

Attribute Description

action

Specifies what the action should do.
Valid values:
0 Open a new record form
1 Load an existing record
2 Execute a search

type

Specifies the type of record involved in the action.
Valid values:
0 a customer record
1 an incident record
2 a work ticket record
3 a script record
4 an e-mail record

id

Specifies the unique identifier of the record to retrieve. Optional if the action attribute is 0 or 2.

idType

Specifies the type of ID represented by the id attribute. Required if the action attribute is set to 2.
Valid values:
0 for the primary ID of the record
1 for the secondary ID of the record

category

Specifies the category of record for this action. Required if the action attribute is set to 1. This attribute is combined with the type attribute to identify the type of customer or incident record to insert.
Valid values when the type attribute is 0 (customer record):
0 an individual record
1 a company record
Valid values when the type attribute is 1 (incident record):
1 a support incident
2 a service incident
3 a sales incident

ownerId

Specifies the unique identifier of the customer record that will own the newly created incident. Required if the action attribute is set to 0 and the type attribute is set to 1 (open an incident record).

ownerIdType

Specifies the type of ID represented by the ownerId attribute. Valid values:
0 for the primary ID of the record
1 for the secondary id of the record

filterField

Specifies the name of the property to be used in search of OEP records. Required if the action attribute is set to 2.
Valid values:
0 Secondary ID
1 First Name
2 Last Name
3 Company Name
4 Post Code
5 Phone Number

filterValue

Specifies the value to match in the search of OEP records. This attribute is used in conjunction with the filterField attribute.

scriptId

Specifies the ID of the script to execute.

scriptMode

Specifies the script mode.
Valid values:
0 execute the script
1 display the script chooser

ownerType

Specifies the type of record (customer or incident) that will own the new script. Valid values:
0 Assign the script to a customer
1- Assign the script to an incident
