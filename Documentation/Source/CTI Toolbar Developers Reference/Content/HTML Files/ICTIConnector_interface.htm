ICTIConnector interface

The ICTIConnector interface is a JavaScript class that enables communication between Onyx CTI and CTI systems by abstracting the communication layer and exposing a standard set of methods. These methods map calls from Onyx CTI to the components in your CTI system. The custom connector that you build communicates with Onyx CTI using the ICTIConnector interface's methods and events. You can customize these methods and events to use business logic required by your CTI system.

The methods of the ICTIConnector interface are listed below:

Method Description

clearCall

Disconnects the identified call.

conferenceCallCold

Performs a one-step cold conference, in which the incoming call is not put on hold while the conference call is made to a third party.

conferenceCallWarmComplete

Performs the second step in a two-step warm conference by retrieving the held call (the original incoming call) and establishing a three-way conference call among the agent , original incoming caller, and the third party.

conferenceCallWarmStart

Performs the first step in a two-step warm conference by connecting agents for a consultation call.

connectCall

Connects the identified call.

getActiveCall

Retrieves the active call.

getAgentData

Retrieves the agent's AgentData XML structure from the connector.

getAgentList

Retrieves the AgentList XML structure that contains the list of available agents for transferring calls and for conferencing.

getCalls

Retrieves all calls for the logged-on agent.

getFreqPhoneList

Retrieves the list of available numbers for placing or transferring calls.

getIdleReasonList

Retrieves the IdleReasonList XML structure that contains the list of reasons agents can select when changing to the Idle state.

getQueueList

Retrieves the QueueList XML structure that contains information about the queues to which a call can be transferred.

getStatusDetail

Retrieves detailed status information from the CTI system.

getStatusSummary

Retrieves summary information from the CTI system.

holdCall

Places the identified call on hold.

idleAgent

Sets the agents state to Idle.

initialise

Instantiates and initializes all necessary CTI system components.

liveAgent

Sets the agents state to Live.

logoffAgent

Logs the agent off the CTI system.

logonAgent

Logs the agent on to the CTI system.

makeCall

Places an internal or external call.

readyAgent

Sets the agents state to Ready.

resubmitCall

Sends a call back to its originating queue.

setActiveCall

Notifies the connector that the selected call in the Onyx CTI interface has changed.

setCallActionData

Attaches record display instructions to the identified call.

transferCallCold

Performs a one-step cold transfer, in which the call is transferred without any type of discussion between agents.

transferCallWarmComplete

Performs the second step in a two-step warm transfer by transferring the original incoming call to the transferee and ending the call for the transferor.

transferCallWarmStart

Performs the first step in a two-step warm transfer by connecting agents for a consultation call.

unholdCall

Retrieves the identified held call.

wrapAgent

Sets the agents state to Wrap Up.
