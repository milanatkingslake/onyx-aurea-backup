Code files

Each connector requires its own set of code files to enable you to load the connector.

We recommend that you use copies of the default code files from the sample connector folder as your starting point when building a new connector.

To confirm that your changes appear as expected, delete the files in Internet Explorer's Temporary Internet Files folder.

Full paths of the default code files, where YourOEPwebsite is your OEP installation directory:

  • YourOEPwebsite\cti\test\cticonnector.asp
  • YourOEPwebsite\cti\test\cticonnector.js

cticonnector.asp file

The .asp file is responsible for any server-side processing required to set up the CTI system components. During initialization of Onyx CTI, the .asp page is loaded into a hidden frame. The default .asp file contains a reference to the test applet JavaScript file, as well as some basic logic for initializing the applet and loading the test driver interface.

When configuring your custom .asp file for your connector, include any logic required to clean up CTI system resources when Onyx CTI is unloaded, either due to the user closing the browser or navigating away from OEP. Typically this logic is encapsulated in the onBeforeUnload event.

cticonnector.js file

The .js file includes a JavaScript class that contains your implementation of the ICTIConnector interface for your custom connector. During initialization of Onyx CTI, a reference to this file is dynamically added to the Onyx CTI frameset page and the class is instantiated using the entry from the OEDB.

When configuring your custom .js file for your connector, replace the code in each ICTIConnector interface method of your custom .js file with the logic required by your CTI system.

For information about the default cticonnector.js file, see Sample connector and test tools.
