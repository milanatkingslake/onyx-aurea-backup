IdleReasonList XML structure

The IdleReasonList XML structure contains the list of reasons that the agent can select when changing to the Idle state.

The connector must return this structure in response to the getIdleReasonList method.

Example:

<idleReasons>

   <idleReason code=1 reason=Administration />

   <idleReason code=2 reason=Break />

   <idleReason code=3 reason=External Call />

   <idleReason code=4 reason=Front Desk />

   <idleReason code=5 reason=Lunch />

   <idleReason code=6 reason=Meeting />

</idleReasons>

For CTI systems that support Idle reasons, the Idle reason identified by the code (enumeration value) of 1 is selected by default. For CTI systems that do not support Idle reasons, there is no default Idle reason selection.

The following table describes the elements of the IdleReasonList XML:

Element Description

idleReasons

Root element. Contains the collection of idleReason elements.

idleReason

Contains the data for the identified idleReason element.

The following table describes the attributes of the IdleReasonList XML:

Attribute Description

code

Specifies the enumeration value of the Idle Reason.

This value is used by the idleCode attribute in the AgentData XML structure.

reason

Specifies the description of the Idle Reason for display in the Onyx CTI interface.
