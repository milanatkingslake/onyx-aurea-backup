About Hierarchical Relationships

The Search Criteria and the Search Results pages may include fields that are hierarchical, meaning that the values in those fields depend upon values entered in preceding fields. For example, because the values for the field State depend on the value for the field Country, Country is the parent field for the dependent field State. This hierarchy can exist at several levels; therefore, a field can be both a parent field as well as a dependent field. For example, in a hierarchy of Resolution Code 1 - Resolution Code 2 - Resolution Code 3, the field Resolution Code 2 is a parent for the field Resolution Code 3, but a dependent for the field Resolution Code 1.

Hierarchical relationships are applicable only to fields where you select an item from a drop-down list.

Use the following information to understand how to select hierarchical fields in the search criteria panel and in the search result list:

  • The Search Criteria Field Administration page is divided into two columns - Available and Selected.
  • The Search Result Field Administration page is divided into two columns - Available and Selected.
  • The Available column lists all the fields that you can include as Search Criteria or Search Result fields for the selected Search Type. The fields available depend on the permissions you have.
  • The Selected column lists the fields that are already included as Search Criteria or Search Result fields for the selected Search Type.
  • When you move a dependent field from the Available to the Selected column, the parent field is automatically moved to the Selected column. You can see the name of the field being dragged along with an icon next to it, indicating that other fields are being moved automatically along with the selected field.
  • You can move a parent field from the Available to the Selected column without moving the dependent field.
  • When the parent field is in the Selected column, and there exist dependent fields for the parent in the Available column, the parent field is displayed in gray and in italics above the dependent fields to indicate the hierarchical relationship between the fields.
  • When you move a parent field from the Selected to the Available column, all previously selected dependent fields are moved automatically to the Available column. You can see the name of the field being dragged along with an icon next to it, indicating that other fields are being moved automatically along with the selected field.
  • You can move a dependent field from the Selected to the Available column without moving the parent field.
  • Click the Tree View button to view the fields in the order of their hierarchy.
  • Click the List View button to view the fields in the order that they are displayed in the Search Criteria and the Search Results pages.
  • When you move fields from one column to another, the page automatically displays in List View so that you can move the field to the desired position in the list.
