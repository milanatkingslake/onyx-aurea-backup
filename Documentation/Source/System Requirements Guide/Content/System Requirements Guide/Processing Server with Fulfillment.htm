Processing Server with Fulfillment Subsystem

The following requirements, which apply to a server computer dedicated to the Fulfillment subsystem, are the equivalent of a low-end server-class computer. For example, the hardware requirements for the OPS /OES Administration client computer are sufficient to install OPS with the Fulfillment subsystem.

Example Configuration

Delivery method Example hardware configuration Recommended number of subsystems based on processing capacity

Email processing
  • OPS server with local, dedicated OEAS
  • OPS-OEAS server: 4 Xeon processors at 500 MHz with 1 GB of RAM
  • 100-Mbps network connectivity between the OPS-OEAS server and OEDB server
  • Processing capacity: Approximately 16,500 order lines per hour
  • One EMS with local Microsoft Windows SMTP service

Print processing
  • OPS server with local, dedicated OEAS
  • OPS-OEAS server: 4 Xeon processors at 500 MHz with 1 GB of RAM
  • 100-Mbps network connectivity between the OPS-OEAS server and OEDB server
  • Processing capacity: Approximately 4,000 order lines spooled to print queue per hour
  • One Fulfillment subsystem with remote network print server

Concurrent Email and Print Processing (High volume)
  • OPS server with local, dedicated OEAS
  • OPS-OEAS server: 4 Xeon processors at 500 MHz with 1 GB of RAM
  • 100-Mbps network connectivity between the OPS-OEAS server and OEDB server
  • Processing Capacity: Print, approximately 10,000 order lines spooled to print queue per hour. Email approximately 50,000 order lines per hour
  • Five EMSs with local Microsoft Windows SMTP service
  • Five Fulfillment subsystems with:
  • Remote network print server
  • Remote Microsoft Windows SMTP service
