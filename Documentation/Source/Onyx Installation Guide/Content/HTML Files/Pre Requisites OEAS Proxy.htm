Pre-Requisites: OEAS Proxy Creation

  • Visual C++ Runtime (64-bit) needs to be installed on Client Server Application where OEAS Proxy is installed.
  • Enable network COM+ access on the Remote Server Application:
  • Press Start, point to Control Panel, and then click Add or Remove Programs.
  • Click Add/Remove Windows Components.
  • Select Application Server, and then click Details.
  • Click Enable network COM+ access, and then click OK.
  • Click Next, and then click Finish.
  • Restart the computer.

To create a proxy connection:

  1. On the OEAS server, create, name, and share a proxy share folder.
  2. Open Component Services.
  3. Expand Component Services, Computers, My Computer, and COM+ Applications.
  4. Right-click OnyxEASApp, and then select Export.

The COM+ Application Export Wizard appears.

  1. Click Next.

The Application Export Information page of the wizard appears.

  1. Specify the name and the location of the application proxy file.
  • Under the prompt to specify the path and file name for the application proxy file, click Browse.
  • The Export application to File window appears.
  • Navigate to the shared folder that you created previously, and in the File name box type a name for the application proxy file. The file is created with an extension of .msi.
  • The application proxy file includes information that points to the OEAS server on which the application proxy was created.
  • Click Save.

The Export application to File window closes.

The Application Export Information page of the wizard displays the location and file name that you specified in the preceding step.

  1. In the Export as area, select Application proxy - Install on other machines to enable access to this machine, and then click Next.

The last page of the wizard appears.

  1. Click Finish.

In the share, you can view the .Msi file and .Cab files, each with the name you specified for the COM+ application.

The OEAS server and the OES or OPS server are ready to be connected by MSI.

  1. On the OES or OPS server, delete any OEAS COM+ application that already exists.
  • Open Component Services.
  • In the left pane of the Component Services window, open Component Services, Computers, My Computer, and COM+ Applications.
  • If the OnyxEASApp COM+ application is present under COM+ Applications, delete it, and then close Component Services.
  1. In Windows Explorer, navigate to and open the shared folder that you created on the OEAS server.
  2. Double-click the .MSI file.

The .MSI file carries out the MSI installation that connects the two servers in COM+.

  1. Verify the connection with the OEAS server.
  • On the OES or OPS server, open Component Services.
  • In the left pane of the Component Services window, expand Component Services > Computers > My Computer > COM+ Applications.
  • Right-click OnyxEASApp, and then click Properties.
  • In the OnyxEASApp Properties window, click the Activation tab.

The name of the OEAS server appears in the Remote server name box, confirming the application proxy connection is established.
