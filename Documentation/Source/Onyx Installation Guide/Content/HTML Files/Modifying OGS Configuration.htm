Modifying OGS Configuration File

  1. On your OEAS server, navigate to your Onyx installation folder and take a backup of the OnyxWindowsService.exe.config file. The default location of this file is ..\Program Files (x86)\Onyx\AppServer\Applications\Onyx\OnyxGatewayService.
  2. In an XML editor, open the OnyxWindowsService.exe.config file.
  3. Navigate to the <baseAddresses> node and change the baseAddress from http to https. Change the port number to the one you configured for your SSL certificate.
  4. Navigate to the <system.serviceModel><bindings><webHttpBinding> node.
  5. For each existing binding element within <webHttpbinding>, add the element <securitymode="Transport" />.
  • If no binding element is defined under <webHttpbinding>, add a new secure binding element and set the security mode to Transport.

<binding name="WebHttpSecure">

<security mode="Transport" />

</binding>

  1. Under the <Services> element, ensure that each endpoint with webHttpbinding has a value for the bindingConfiguration attribute.
  2. If an endpoint does not have this attribute, then configure it by adding the binding name you created in the previous step.
  3. Save and close the OnyxWindowsService.exe.config file.
  4. Start the OGS service
