Understanding Password Encryption Errors

When you configure a logical application and OGS, run the OGS password encryption utility, which encrypts the Onyx database password and enables Navigator to access the Onyx database.

During this process, the public key file that encrypts the password is copied to the local temp folder. The default path for this file is ..\Users\<username>\AppData\Local\Temp\_ISTMP1.DIR\_ISTMP0.DIR, where <username> is the logged on user ID. Once Setup is complete, the file is automatically deleted from this location.

This table following table lists the error types that can occur during this process.

Error type Description Action to resolve

noPUK

The public key file name is not provided.

Run a fresh copy of the installer. If this does not resolve the issue, contact Avolin Customer Support.

noPUKPermissions

The public key file does not have Read permission.

Navigate to the folder where the public key file is stored, and make the file Read-Only.

noPUKExists

The public key file was not found in the specified folder.

Run a fresh copy of the installer. If this does not resolve the issue, contact Avolin Customer Support.

noPUKData

Key information is not available in the specified public key file.

Contact Avolin Customer Support.

noEncryptData

The utility is unable to encrypt the password provided. A generic error occurred while processing.

Resolve any.NET errors that are displayed. If this does not resolve the issue, contact Avolin Customer Support.

noFileExist

The OGSConfig file is not found.

Contact Avolin Customer Support.

noFileName

The public key file name is not available in the OGS Config file.

Contact Avolin Customer Support.

noPassword

Password not found. Please provide the password.

Contact Avolin Customer Support.

noCFFile

Configuration file path is not provided.

Ensure that \CF is included the parameters in the .exe file.

noCFData

The OGS config file does not have the required data.

Ensure that the OGS file contains the required information. If this does not resolve this issue, contact Avolin Customer Support.

illegalPresent

Unallowed characters present. Passwords can contain only alphanumeric or special characters other than /, <, >, -, ?, ., *, &, ', \", _, comma(,).

Ensure that the password does not contain unallowed characters.

noPVK

The private key file name is not provided in the OGS Config file.

Contact Avolin Customer Support.

noPVKPermissions

The private key file does not have Read permission.

Navigate to the folder where the private key file is stored, and make the file Read-Only.

noPVKExists

The private key file is not found in the specified folder.

Run a fresh copy of the installer. If this does not resolve the issue, contact Avolin Customer Support.

NoKeyLoc

The private key file path is not provided in the OGS Config file.

Provide the correct location for the private key. The default location is your OGS installation folder.

noPVKData

Key information is not available in the specified public key file.

Contact Avolin Customer Support.

NoTextDecrypt

The password cannot be decrypted. A generic error occurred while processing.

Resolve any.NET errors that are displayed. If this does not resolve the issue, contact Avolin Customer Support.

noEncryptPassword

Encrypted password not found. Please provide the encrypted password to decrypt.

The EXTRADATA node is empty in the OGS Config file. Use the passencrypt application from the Utilities folder of the Onyx installation package, and then encrypt the password.

This table lists the errors that can occur when you create your own set of public and private keys.

Error type Description Action to resolve

noPUKGen

The public key cannot be generated.

Resolve any.NET errors that are displayed, and confirm that you have Write permissions to the folder where the file is stored. If this does not resolve the issue, contact Avolin Customer Support.

noPVKGen

The private key cannot be generated.

Resolve any.NET errors that are displayed, and confirm that you have Write permissions to the folder where the file is stored. If this does not resolve the issue, contact Avolin Customer Support.

DesFilePresent

The key already exists in the destination folder.

Rename any existing key files in the folder, and continue with the installation.

noFolderExists

The specified directory does not exist.

Verify the directory path provided.
