Installing Onyx Web Services

Web Services is the Web-based interface for establishing an OEAS session and calling OEAS logical methods. In this procedure you specify whether to use Onyx authentication or Windows authentication, and you name two websites: one for authentication and the other for interaction with OEAS components. You also specify the location to install the Web Services program files.

Be sure to have the following information ready before you begin:

  • The website name for the Onyx Middle Tier Web Service will be the URL that users will use to access web services.
  • The website name for the Onyx Authentication Onyx Web Service. This name should be different from the one you specify for the Middle Tier Web Service.
  • The directory location to install the Web Services program files.

Onyx does not support installing the OEAS and Web Services to the same directory.

To install Onyx Web Services

  1. In your Onyx installation package, navigate to Platform Components > Web Server > Onyx Web Services, and then run OnyxWebServices.exe.

The Welcome window appears.

  1. Click Next.

The Onyx WebService with Onyx Authentication window appears.

  1. Retain the existing OEAS Web Service name and OEAS Authentication Web Service name or change, if required.
  2. Click Next.

The Onyx Webservices with Windows Authentication window appears.

  1. Retain the existing OEAS Web Service name and OEAS Authentication Web Service name or change, if required.
  2. Click Next.

The Destination Folder window appears.

  1. Change the destination folder path or retain the displayed path.
  2. Click Next.

The Start Copying Files window appears.

  1. Verify the details provided in the earlier steps, and then click Next.

The Ready to Install the Program window appears.

  1. Click Install.

After the installation is complete, the Installation Wizard Completed window appears.

  1. Click Finish.

Make sure that in the Application Pool field, Onyx Web Service Pool is the default selection.

To run OWS as windows service, ASP.net v4.0 should be present in IIS 7.0 or the higher version. Otherwise, ISAPI and CGI restriction error occurs. To avoid this, run the following in the command prompt to install ASP.net v4.0 in IIS 7.0 or later. ..\Windows\Microsoft.NET\Framework\v4.0.30319\aspnet_regiis.exe –i
