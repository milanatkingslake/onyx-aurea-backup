Installing Onyx Enterprise Studio

Onyx Enterprise Studio (OES) is the Web-based user interface for Onyx administration tools and design tools. In this procedure, you name two authentication websites: one for Onyx Authentication and the other for Integrated Windows Authentication. You also specify the site ID to connect to the Onyx database, and the location to install OES program files.

If you are installing OES on a different server than the OEAS, you must create an application proxy for communication between the servers. For more information, see Creating a Proxy Connection.

Be sure to have the following information ready before you begin:

  • The name of the logical application that you are installing OES for.
  • If you are installing OES on a different server from the OEAS, the name of the OEAS for which to set up OES.
  • The Onyx Authentication virtual website name, and the Onyx user ID and password used to log on to the website. This is the URL you will use to connect to the OES.
  • If you plan to use Integrated Windows Authentication, the Windows Authentication virtual website name, and the Windows user ID and password used to logon to the website. This is the URL you will use to connect to the OES. The Windows Authentication website name must not be the same as the Onyx Authentication website name.

If you are logged onto a webpage with Integrated Windows Authentication, you cannot use the same browser session to log on to a Onyx webpage using Onyx authentication. To log on to OES (or another Onyx webpage) using Onyx authentication, you must use a new browser instance.

  • The site ID to connect to the Onyx database.

Onyx supports only one site per OEDB. If you create a second site in the OEDB Site table and then run the OES Setup as an individual setup, the OES Setup inserts in the Site ID box the first site ID that it encounters. Make sure that this site ID is the one that you want to administer with OES.

  • The directory location to install the OES program files.

Onyx does not support installing the OEAS and OES to the same directory.

To set up Onyx Enterprise Studio:

  1. On the server where you intend to install OES, log on with an account that has full administrative privileges.

If you're installing OES on a different server than the one that OEAS is set up on, ensure that the log on account has full administrative privileges on both the servers.

  1. In your Onyx installation package, navigate to Platform Components > Web Server > Onyx Enterprise Studio, and then run EnterpriseStudio.exe.

The Welcome window appears.

  1. Click Next.

The Logical Application window appears.

  1. Select the name of the OEAS logical application for which you are installing OES, and then click Next.

The Onyx Authentication Web Site Name window appears.

  1. Retain the default website name (OnyxEnterpriseStudio_Onyx) or type a different name, and then click Next.

The Windows Authentication Web Site Name window appears.

  1. Retain the default website name (OnyxEnterpriseStudio_Windows) or type a different name.
  2. Click Next.

The Web Site Default Site ID window appears.

  1. Accept the default site ID for the OES website or enter a different one.

The default site ID in the Site ID box is the Onyx database site ID. The OES website uses this site ID to connect to the database.

  1. Click Next.

The Destination Folder window appears where you specify the location for the OES files.

  1. Accept the default file location or click Change, and then select a different location.

If any of the Setup files already exist in the selected location, a message appears. You can choose to overwrite the existing files, or you can select a different location.

  1. Click Next.

The Start Copying Files window appears. This window displays the information you had entered in the preceding windows.

  1. Verify that the information displayed is accurate, and then click Next.

To change any of the information displayed, revert to the preceding windows in the wizard and change the information as required.

The Ready to Install the Program window appears.

  1. Click Install.

A series of status boxes track progress as the setup program copies OES files and creates the virtual websites that you specified for both kinds of user authentication.

The Installation Wizard Completed window appears, indicating that the setup has successfully Installed Onyx Enterprise Studio.

  1. Click Finish.
  2. Create a proxy connection for the OES server to communicate with the OEAS server. For information about creating proxy, see Creating Proxy Connection.

Each time you run OES Setup, Setup creates a COM+ application associated with the OES website. You can remove extra COM+ applications from Windows Administrative Tools, under Component Services.
