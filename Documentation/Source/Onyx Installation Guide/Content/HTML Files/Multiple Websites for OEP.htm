Multiple Websites for OEP

  1. Make a copy of EmployeePortal folder from the following location:
  • For 64-bit: ..\Program Files\Onyx
  1. Rename the copied folder to the desired name. For example, OEP.
  2. Open Internet information Server (IIS)
  3. Right-click and then click Add Application.
  • Type the Alias Name to the desired name. For example, OEP_Web
  • Click Select, and then select OEPAppPool.
  • Click Browse to provide the physical path.
  1. Once the application is created, click New Created Application, and then go to the ASP section.
  • Set Enable Parent Path property to True (default is False).
  • Set Session property to False (default is True)
  1. Right-click and then click Add Application.
  • Type the Alias Name to the desired name. For example, OEP_Window.
  • Click Select and then select OEPAppPool.
  • Click Browse to provide the physical path.
  1. Once the application is created, click New Created Application, and then go to the ASP Section.
  • Set the Enable Parent Path property to True (default is False).
  • Set the Session property to False (default is True).
  • Click New Created Application and then go to the Authentication Section.
  • Enable Windows Authentication (default is disabled).
  1. Make the following changes in the OEP folder.
  • OTM_security.asp in common\include
  • Rename the site_id and logical application name
  • public const gsAppName = "Onyx1"
  • public const gnSiteId = 2
  1. Restart IIS.
