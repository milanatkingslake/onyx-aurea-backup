Configuring Onyx File Agent

To configure Onyx File Agent:

  1. Open the Computer Management tool on the server (that is, from where the OFA needs to be installed) and add the OEP web site anonymous user account to the local OnyxFAUsers group.

The OEP anonymous login user is located in the OFA server > IIS Manager/Properties/ ‘Directory Security’ tab /‘Enable Authentication’ section.

  1. Following are the example of the OnyxFAUsers group members added:
  • Administrator (local)
  • Domain\Administrator
  • Domain\OnyxFAUser
  • IUSR
  1. Create an OEP FileShare directory on the server. This is generally named as OEPFileshare.
  2. Navigate to \\utilities\Attachments\FileLinkDirectoryScript, and then copy the createTree.vbs script to the folder you just created.
  3. Run the script to set up the tree structure.
  4. Log on to OES, and then open Table Administration.
  5. Select File Location.
  6. Click on the plus mark (+) button.
  7. Type the Primary ID number.
  8. Type a description for the OEP File Share (can be OEPFileShare).
  9. Type the path for the file share. For example, : \\serverName\OEPFileShare

Use the virtual server name as value when using Virtual Machines.

  1. Click Save.
  2. Right-click the OEPFileShare folder, and then click Sharing and Security.
  3. Select Share this folder.
  4. Click Permissions.
  5. Add OFAUsers (this is a group, automatically created after you have installed OFA).
  6. Add OnyxEASUser.
  7. For all the users, provide Read/Write permissions.
  8. Click OK.
  9. Add the Onyx GatewayService user in the OnyxFAApp step component.
  1. Expand Component services and open to the OnyxFAApp step component.
  2. Under OnyxFAApp expand Roles then expand OnyxFARestricted.
  3. Click OnyxFARestricted -> Users.
  4. Add OGSUser.
  1. Restart the server.

You can now add attachments to OEP.
