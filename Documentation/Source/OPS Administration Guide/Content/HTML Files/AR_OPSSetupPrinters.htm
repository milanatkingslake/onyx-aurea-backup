Set up network printers

Step 3: Set up network printers

One or more printers must be available and configured for use by each print subsystem that you create.

To configure printers for printing order lines, do the following:

  1. Install the network printers for print items.

    2. You must be logged in with the OPSSvcUser domain account when you install the printers. If install a printer while logged on with a different account, the print subsystem will not be able to find it. Inability of the print subsystem to find a printer results in a critical failure.

    OPS does not support the use of a printer installed on the OPS server itself. The printer must be a network printer installed on a separate server.

  2. Add the printers to the Service Printer Info table in Table Administration.

    3. The printers configured in Table Administration are associated with the print literature products in Product Administration. The print subsystem uses this data and the literature product attributes to route the order line to the appropriate printer.

    You configure printers using Table Administration. With this tool, you can:

  • Add new printers
  • Update existing printer names and paths
  • Set printers to active or inactive
  • Delete existing printers

The following table describes the default printer data installed with the OEDB.

Field name

Default data

Description

printerShare

\\servername\printername

Used by the print subsystem to route the print item to the printer. Must be a valid printer path.

printerName

<default_printer_share>

Used when associating a literature product with a printer. Appears in the Literature Information cell in Product Administration.

    See Table Administration Help for further details.

  • To print using Unicode characters, install the proper Unicode fonts on the OPS server.

    • Onyx recommends that you use a dedicated printer with label sheets for printing labels.

To verify that the printers are set up and configured properly:

  1. Log in to the OPS server as OPSSvcUser.
  2. Verify that the printers are installed on the server.
  3. Open Microsoft Word, dismissing all dialogs and adding initials as appropriate.
  4. Print sample templates to configured printers.

    5. If you find any problems, repeat Step 3.
