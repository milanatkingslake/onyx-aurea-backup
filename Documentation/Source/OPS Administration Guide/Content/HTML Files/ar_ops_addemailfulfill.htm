Adding and configuring an email subsystem

Note: When using Basic Authentication SMTP protocol in OEAS configuration utility, see additional configuration on Exchange Server.

After you add the subsystem, you must reload any running instances of OPS Administration for the newly added subsystem to appear in the navigation area. For more information on the subsystem attributes, see Fulfillment subsystem settings.

To add an email subsystem:

  1. In the navigation area, select the Fulfillment Subsystems collection to which you want to add an email subsystem.
  2. Click on the toolbar.

    3. A new subsystem node titled newSubsystem1 (added) appears in the navigation tree. The status remains as added until the service is restarted.  

  3. In the edit area, modify the attributes for the subsystem.

    4. The email settings for email subsystems apply to email order lines, error notifications, and system documents with an email fulfillment type.

    The following table describes settings available in the edit area, under Process Settings.

    Setting or option

    Description

    Name

    A unique name for the subsystem. This is a required field with a maximum of 50 characters

    Description

    A description for this subsystem with a maximum of 255 characters. This field is optional.

    Fulfillment Engine

    A fulfillment engine to use. To create an email subsystem, select Onyx Email Engine (XSL and Microsoft CDO).

    Additional Data

    A semi-colon delimited list of name-value pairs representing data required for sending SMTP messages. These messages contain error notifications.

    Active

    The active or non-active status of the subsystem. The box is checked by default.

    Batch Size

    The number of order lines to process in each batch. This field is required. The valid range is 1 - 2,000.

    Start Time (00:00 to 23:59)

    The start of the subsystem’s operation window for processing. The time must be in 24-hour format with a valid range from 00:00 to 23:59. The default is 00:00 (midnight). This is a required field.

    To configure a subsystem to process 24 hours a day, set both the Start Time and Stop Time to 00:00.

    Stop Time (00:00 to 23:59)

    The end of the subsystem’s operation window for processing. The time must be in 24-hour format with a valid range from 00:00 to 23:59. The default is 00:00 (midnight). This is a required field.

    Standard Priority Run Interval (minutes)

    The number of minutes you want the subsystem to wait between checks of the queue for email order lines with either standard or high priority. This is a required field. The valid range is 1 - 10,080 (1 week).

    High Priority Run Interval (minutes)

    The number of minutes you want the subsystem to wait between checks of the queue for new, high-priority order lines. Standard-priority order lines are ignored. The valid range is 1 - 10,080 (1 week).

    Ignore Start/Stop Time for High Priority Item

    Whether high-priority order lines are processed outside the operation window.

    The following table describes settings available in the edit area, under Email Settings.

    Setting or option

    Description

    Mail Server

    The IP address, name, or Internet address of the server that sends error notifications. Entering a value in this field is necessary for configuring email subsystem and error notifications to be sent. If the SMTP server is local, enter 127.0.0.1 to optimize performance.

    Administrator Email Address

    The complete email address where error notifications are to be sent. Entering a value in this field is necessary for configuring subsystem error notifications to be sent.

    From Email Address

    The complete email address from which the error notifications are to be sent. Entering a value in this field is necessary for configuring subsystem error notifications to be sent.

    From Email Display Name

    The name to be displayed in the From field of the error notification. This field is optional, though Including the subsystem name tells the administrator which subsystem was the source of the error notification.

  4. To save your changes, click .

    5. The subsystem is ready to start.
