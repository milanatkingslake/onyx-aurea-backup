Using OPS Administration

Onyx Processing Server (OPS) Administration is an administration tool that interacts with OPS and OEAS. OPS Administration enables you to configure and manage the OPS service and subsystems. The OPS service controls the subsystems, which perform processing tasks at specified times and time intervals.

Using OPS Administration, you can:

  • Add, configure, and delete subsystems.
  • Start, stop, pause, or restart the OPS service and subsystems.
  • Activate and deactivate subsystems.

The following subsystems are supported for OPS:

  • Fulfillment subsystems (batch, fax, email, and print)

    • Fulfillment subsystems process order lines and List Manager actions. Each fulfillment subsystem is configured either to process list actions,  to print, to fax, or to send email. You can have multiple fulfillment subsystems, each with unique settings.

  • Email monitor subsystem (EMS)

    • An EMS monitors mailboxes for inbound mail at set intervals and assimilates inbound mail into OEAS. Using information such as the sender and recipient email addresses and an internal thread identifier, the EMS associates inbound email with companies, individuals, Onyx users, and any other related customer records.

Before you begin

Before you use OPS Administration, ensure that you have done the following:

  • Created an OPS connection
  • Configured OPS security and user permissions

For more information, see the OPS documentation in the ONYX Installation Guide.
