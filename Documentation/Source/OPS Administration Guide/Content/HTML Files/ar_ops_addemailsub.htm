Adding and configuring email monitor subsystems

For each OPS connection, you can configure multiple EMSs and control them independently of each other. After you have configured an EMS, you can configure a collection of mailboxes for monitoring by the subsystem. For step-by-step procedures, see Adding and configuring monitored mailboxes.

Note: EMSs and mailboxes have a parent/child, one-to-many relationship. Therefore, you can configure a parent subsystem to monitor a collection of child mailboxes. Configuring multiple parent subsystems to monitor a single child mailbox is not supported.

To add a new email monitor subsystem:

  1. In the navigation area of OPS Administration, select the Email Monitor Subsystems collection.
  2. Click .

    3. A new subsystem node titled newSubsystem1 (added*) appears in the navigation tree. The status remains as added until the service is restarted.

  3. In the Edit Email Monitor Subsystem area, modify the attributes for this EMS. For descriptions of the attributes in the edit area, refer to the following:
  • Email monitor subsystems settings
  • Example: Email monitor subsystem
  1. Click to save your changes.
  2. For localization support, map one or more languages to automatically convert to Unicode by configuring the charsetmaps.xml file to use a Unicode page. This file is located in the \Onyx\ProcessingServer\Components directoy. For example:

    <codePage codePageValue="950" friendlyName="Chinese Traditional (Big5)" familyCodePage="950">

    <charsetLabel prefered="1">big5</charsetLabel>

    <charsetLabel>cn-big5</charsetLabel>

    <charsetLabel>csbig5</charsetLabel>

    <charsetLabel>x-x-big5</charsetLabel>

    </codePage>

    3. In this example:

  • The value for the codePageValue parameter is the code page that EMS uses to convert the string to Unicode using code page 950.
  • The charsetLabel element is the charset label in which the email is encoded.

    Several of these might need to be mapped to the same code page.
