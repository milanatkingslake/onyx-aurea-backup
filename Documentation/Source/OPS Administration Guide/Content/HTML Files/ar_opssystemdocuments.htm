System documents

System documents are a specific type of literature product that you configure to be generated automatically when orders are processed by a subsystem. To use OPS system documents, you need first to determine what type of system documents you need.

Two types of system documents are available. Both types can be emailed or printed:

  • Batch system documents

    • Batch system documents are output once per batch of orders. A mailroom operator might use a batch system document for assembling OPS printed documents and physical items for mailing.

  • Order system documents

    • Order system documents are output once per order. You might use an order system document as a packing slip that lists all the items shipped in a customer order.

When deciding whether to use system documents, consider the following:

  • The type of system documents that your organization needs
  • For each system document, whether you need to merge information into the literature product

    • This consideration affects how you customize the template.

  • The fulfillment type for each system document

    • The same fulfillment types available for literature products are available for system documents. The fulfillment type of a system document depends on how you will use the system document.

  • Which subsystems will generate the system document

    • Which subsystems will generate the system document depend on how you will use the system documents and subsystems.
