Subsystem failure

Subsystem failure is classified as either non-critical or critical. When a subsystem encounters a non-critical failure, it creates an entry in the event log and resumes processing with the next item. (In the case of an EMS, the subsystem resumes processing the next inbound email message.)

If a subsystem encounters a critical error, the subsystem stops processing, logs the error, and sends an error notification as appropriate. (Subsystem errors are logged to the connection event log that is created during OPS Configuration.) The subsystem enters an error state, which is indicated in OPS Administration. After the error condition is fixed, you must stop and restart the subsystem for it to resume processing.
