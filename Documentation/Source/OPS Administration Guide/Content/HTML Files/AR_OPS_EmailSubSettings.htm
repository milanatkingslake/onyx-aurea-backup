Email monitor subsystems settings

The important fields for configuration are shown in the following tables.

Setting or option

Example value

Description

Name

Q1 Promotions

Type a unique name for the subsystem. This is a required field with a maximum of 50 characters.

Description

Monitor email responses for Q1 Promotions

Type a description for this subsystem. This field is optional. The description can be a maximum of 255 characters.

Additional Data

View valid values

A semi-colon delimited list of name-value pairs representing data required for sending SMTP messages. These messages contain error notifications.

Active

Checked

Use this checkbox to activate or deactivate a subsystem. The default is Active.

Start Time

00:00

Type the start of the subsystem’s operation window for processing.

Stop Time

23:59

Type the end of the subsystem’s operation window for processing. The time must be in 24-hour format with a valid range from 00:00 to 23:59. The default is 00:00 (midnight). This is a required field.

Run Interval (minutes)

10 minutes

Type how long (in minutes) after the processing cycle completes that you want the subsystem to enter sleep mode before restarting. The valid range is 1 - 10,080 (1 week). This is a required field.

Governor Interval (milliseconds)

100 milliseconds

Within the processing window, type how long (in milliseconds) you want the subsystem to pause between calls to OEAS. This setting ensures that OEAS is not overloaded by subsystem calls to the database. This precaution is necessary when there are end-user applications accessing the same OEAS server that the EMS accesses. The valid range is 0 - 60000.

The following table shows important settings for configuring email notification:

Setting or option

Description

Mail Server

Type the IP address, name, or Internet address (path or URL) of the server that is to send error notifications. Entering a value in this field is necessary for configuring error notifications to be sent. If the SMTP server is local, enter 127.0.0.1 to optimize performance.

Note: If the mail server rejects error notification emails, configure EMS to use the local SMTP server by entering 127.0.0.1 in this field.

Administrator Email Address

Type the complete email address where error notifications are to be sent. Entering a value in this field is necessary for configuring subsystem error notifications.

From Email Address

Type the complete email address from which the error notifications are to be sent (email address displayed). Entering a value in this field is necessary for configuring subsystem error notifications.

From Email Display Name

Type the name to be displayed in the From field of the error notification email. This field is optional, though including the subsystem name tells the administrator which subsystem was the source of an error notification.
