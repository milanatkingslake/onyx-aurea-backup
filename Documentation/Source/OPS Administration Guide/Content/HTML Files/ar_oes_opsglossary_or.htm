Glossary of OPS terms

The following glossary describes the OPS terms used throughout the OPS documentation.

Action batch — An action batch is a request from the List Manager for an action such as creating an incident a list of customers. Each action batch includes a set of OEAS method calls that are executed against a set of customers.

Action batch XML — The action batch XML contains the information that defines which OEAS method calls the batch subsystem executes for an action batch.

Batch queue — A batch queue contains action batch items. The two types of batch queues available in OPS Administration are the New Batches queue and the Completed Batches queue.

Batch subsystem — A batch subsystem processes action batches submitted through List Manager.

Batch system document — A batch system document is a system document that is generated once per batch of order lines. For example, a batch system document may be used by shipping or fulfillment personnel as a pick list.

Completed batches — Completed batches contain action batches that have been processed by OPS and have a status of Completed. You can view completed batches in OPS Administration.

Completed order lines — Completed order lines contain order lines that have been processed by OPS and have a status of Completed. You can view completed order lines in OPS Administration.

Email fulfillment type — Order lines with an email fulfillment type are delivered to customers as email messages. Email fulfillment types include:

  • Email HTML in body
  • Email HTML as attachment
  • Email text in body
  • Email text as attachment
  • Email file as attachment

Email order line — An order line for a literature product with an email fulfillment type. Email order lines are processed by an email subsystem.

Email subsystem — An email subsystem is a fulfillment subsystem configured with an email engine. The subsystem is responsible for processing email order lines.

Email template — An email template is an XSL file used to fulfill email order lines. This customizable template is an XSL stylesheet which has commands that support data merging. Literature products reference the template. When OPS processes an order line for the literature product, OPS uses the template in combination with the data to create a merged output document.

Error notification — The process by which the OPS service or subsystems sends the administrator notification via email when an error occurs. You the OPS service to perform error notification during OPS setup. You use OPS Administration to specify error notification for the subsystems.

Fulfillment queue — A fulfillment queue is a container for literature order lines. The two types of fulfillment queues available in OPS Administration are the New Order Lines queue and Completed Order Lines.

Fulfillment subsystem — Fulfillment subsystems are responsible for processing literature order lines. A fulfillment subsystem is combined with either a print or email engine. Combining a fulfillment subsystem with a print engine makes it a print subsystem. Combining a fulfillment subsystem with an email engine makes it an email subsystem.

Incomplete batch — An incomplete batch results when a subsystem is stopped or fails after the reservation stage. The batch is considered incomplete because there are order lines with the status of In Process remaining in the batch.

List Manager action — A List Manager action is a request to add data for a list of customers. In OEP, the supported actions include inserting an incident or inserting an externalContact. Each List Manager action uses one or more OEAS method calls for each customer to add the requested data. A List Manager action is represented in OEAS by an action batch.

List Manager feature — The List Manager feature is an OEP feature that enables users to create a query for a list of customers or other entities that meet certain criteria. Once the list is retrieved, an action can be applied to the customers. By default, actions are not supported against queries that do not return a list of customers.

Literature feature — The Literature feature is an OEAS feature that enables users to create orders for literature products to be sent to customers.

Literature order (or order) — A literature order is a request created in the Literature feature for one or more literature products to be printed, emailed, or mailed to a customer. A literature order consists of one or more order lines, each representing a literature product.

Literature order line (or order line) — An order line is a request for a literature product sent to a customer. A literature order is made up of one or more order lines. Each order line has a fulfillment type that indicates how it should be fulfilled.

Literature product — A literature product represents a literature item that can be fulfilled by print, email, or mail. A literature product is a standard product with additional attributes such as printer path, file path, label attribute, or fulfillment type.

Mailing labels — You can configure mailing labels to be printed automatically when associated print literature products or physical items are fulfilled. You first create a literature product for the mailing label, then associate the label to other literature products. When an order is processed for the literature product, a mailing label will automatically be generated as well.

Master application — The master application is the OEAS logical application that is used in conjunction with the master site to provide security for the OPS service and OPS Administration.

Master site — The master site is the OEDB site that is used in conjunction with the master application to provide security for the OPS service and OPS Administration.

New batches — New batches are the action batches waiting to be processed by a batch subsystem. All action batches appear in this queue except those with a status of Completed. You can look at the queue in OPS Administration.

New order lines — The queue of order lines waiting to be processed by a print or email subsystem. All order lines appear in this queue except those with a status of Completed. You can look at the queue in OPS Administration.

Operation window — The operation window is the time interval that a subsystem uses to process literature orders or action batches. The time interval is defined by the subsystem’s start time and stop time, which you configure in OPS Administration.

Onyx Processing Server (OPS) — OPS is responsible for processing literature orders and List Manager actions (action batches).

OPS Administration — OPS Administration is the tool with which you configure and monitor the OPS service, subsystems, literature orders lines, and action batches.

OPS connection — An OPS connection defines a link between OPS and an OEAS logical application. You create connections with OPS Configuration.

OPS server — The OPS server is the machine on which OPS is installed.

OPS service — The OPS service is a Windows service that manages the subsystems that process literature order lines and List Manager actions. The service is composed of fulfillment subsystems, batch subsystems, and the OPS Administration tool.

OPS.Administrator — The OPS.Administrator role represents users that are OPS system administrators. Such users are responsible for configuring and maintaining the OPS service and subsystems. By default, this role has access permissions for all OPS Administration resources.

OPS.FulfillmentOperator — The OPS.FulfillmentOperator role represents users that have shipping or fulfillment responsibilities. By default, this role has permission to all OPS resources except OPS subsystem configuration.

OPS.User — The OPS.User role represents users that access OPS Administration only to view the fulfillment or batch queues. By default, this role has view permission to the fulfillment and batch queues, but no permission to other OPS Administration functionality.

Order batch — An order batch refers to a group of order lines that OPS processes in the same time interval.

Order batch XML — The order batch XML contains the source XML input that the templates use for transformation or substitution rules when fulfilling order lines. All the information for the literature products, literature orders, printers, and custom XML data relevant to a batch of orders is included.

Order system document — An order system document is a system document that is generated once per each literature order. One use of an order system document is as a packing list in the shipping department.

Physical item — A physical item is a literature product representing an item that you can mailed to a customer. Examples include company brochures, software CDs, manuals, or other packaged items.

Print fulfillment type — Order lines with a print fulfillment type are automatically printed when OPS processes the order line. Print fulfillment types are:

  • Print document
  • Physical item

Print order line (or print order) — An order line for a literature product with a print fulfillment type. Print order lines are processed by a print subsystem.

Print subsystem — A print subsystem is a fulfillment subsystem configured with a print engine. Print subsystems process print order lines.

Print template — A print template is a Word document with VBA code that supports data merging. A literature product is created that references the template. When an order is placed for the literature product, OPS merges data into the template and routes it to a printer queue for printing.

Resumption — Resuming processing of an incomplete batch is called resumption.

Sample templates — Sample print and email templates are installed by default with OPS. These templates can be customized for your company’s needs.

Subsystems — Subsystems are responsible for processing literature order lines and List Manager actions. Currently, there are two types of subsystems available: fulfillment subsystems and batch subsystems. A fulfillment subsystem is combined with either a print engine or email engine and is responsible for processing order lines. A batch subsystem is responsible for processing action batches. Subsystems are maintained using OPS Administration.

System documents — System documents are literature products configured to be generated when batches of order lines are processed. There are two types of system documents: batch system document and order system document.

Template — A literature product references a template that is used to fulfill an order line. There are two types of templates: print templates and email templates. Sample templates are installed with OPS and can be customized.
