Working with synchronization subsystems

Sync subsystems synchronize Onyx data between the database and the staging server (from which they are synchronized with OEP-Offline clients). When you add or configure a synchronization subsystem, the settings that you specify control these exchanges of data.

In addition to adding and configuring a sync subsystem, you can use OPS Administration to modify the sync subsystem’s settings, activate or deactivate it, or delete it. For more information, see the following topics:

  • Adding and configuring a synchronization subsystem
  • Modifying a synchronization subsystem
  • Deleting a synchronization subsystem
  • Activating or deactivating a synchronization subsystem

It is recommended that offline users regularly consult the Synchronization Issues web page to view, resolve, and re-synchronize their errors. Otherwise, the number synchronization cycles may slow or even halt during the user's session.

As administrator, you can also increase the Maximum Items Downsynced Per Cycle parameter. For example, if this setting is 500 and an OEP-Offline user has 250 synchronization errors, only 250 other items can be processed per cycle. If the number of errors is 500 or more, no processing occurs for the user. To avoid this problem, users should view, resolve, and re-synchronize their errors regularly.
