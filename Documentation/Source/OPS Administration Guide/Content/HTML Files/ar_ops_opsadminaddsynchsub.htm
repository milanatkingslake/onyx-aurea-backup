Adding and configuring a synchronization subsystem

Add a sync subsystem and configure it using the following procedure. To configure an existing sync subsystem, use the latter part of the procedure to edit subsystem attributes, process settings, and notification email settings.

To add a synchronization subsystem:

  1. Open the OPS Administration Web site.
  2. In the navigation area, expand Connections and then expand the connection for which you want to add the subsystem.
  3. To add a synchronization subsystem, highlight Synchronization Subsystems, and then click on the toolbar.
  1. In the Name and Description boxes, type a name and description for the subsystem.
  2. Assign values to name-value pairs in the Additional Data field. This field contains a semi-colon delimited list of name-value pairs representing data required for sending SMTP messages. These messages contain error notifications. View valid values.
  3. Leave the Active attribute selected. If unchecked the subsystem is deactivated.
  4. Under Process Settings, specify the following attributes:
  • Start Time and Stop Time – These attributes, which are both required, define the daily window within which the subsystem synchronizes data between the database and the staging server.
  • The time must be in 24-hour format in the range 00:00 to 23:59. The default is 00:00 (midnight).
  • To configure a subsystem to process 24 hours a day, set both the start time and Stop Time to 00:00.
  • Run Interval (minutes) – This attribute specifies the number of minutes between the start of one synchronization cycle and the start of the next. It is recommended that you specify the shortest interval that the OPS server hardware is capable of handling. However, the valid range is 1-10,080 minutes i.e., 1 week.
  • Governor Interval (milliseconds) – This attribute specifies the number of milliseconds between each OEAS call made by the synchronization subsystem. Zero means there is no governor, and the upper limit of 60000 means there is a one minute pause between OEAS calls. It is recommended that you specify the smallest value that the OEDB server hardware is capable of handling.
  • Maximum Items Downsynced per Cycle – This attribute specifies how many records are downloaded to the staging server during each synchronization cycle, per user. The default is 1000 items. If set to zero, there is no limit to the number of items downsynced per cycle to a user’s folder.
  1. Under Notification Email Settings, in order for the administrator to receive event-notification messages, such as error notifications, specify the following attributes:
  • Mail Server – This attribute identifies the email server used to send event-notification email messages.
  • Administrator Email Address – This attribute specifies the email address of the administrator.
  • From Email Address and From Email Display Name – These attributes specify the address and display names for the messages.
  1. To save changes, click on the toolbar.

After you have saved changes, OPS supplies an ID and alternate ID for the subsystem.
