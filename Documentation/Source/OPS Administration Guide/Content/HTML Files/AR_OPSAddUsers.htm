Step 11: Add additional users for OPS Administration (optional)

This is an optional step, in which you can grant permission to additional users to access OPS Administration. Using Security Administration, you can add users to the default roles described in the following table.

Note: If you have multiple connections, you must grant permission on each connection.

Security Role

Description

Restrictions

OPS.Administrator

This role grants full control over all OPS features. OPS Administrators can administer OPS services, add and configure subsystems, view and change order line information, and monitor action batch queues. This level is intended only for the OPS Administrator.

No restrictions.

OPS.Fulfillment.Operator

This role grants essentially the same access as OPS Administrator except for access to OPS subsystem configuration. Typically, the literature fulfillment or mailroom operator would use this security role.

Cannot configure the subsystems.

OPS.User

This role grants only view permissions to the fulfillment queues and action batch queues. Users can use this permission level to monitor the status of their order line or action batch.

Cannot administer the OPS service, configure subsystems, or change order line information.

For example, if you want a user to be able to view queues, add the user to the OPS User role. See Security Administration Help for further details.
