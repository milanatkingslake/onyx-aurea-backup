Configuring OPS event logging

To enable, disable, or configure event logging for OPS and the sync subsystem, you specify values for the following registry settings (their location is provided in the procedure that follows these descriptions).

  • GlobalLogSettings

    • This value specifies what kind of information is logged to the OPS event log. The possible values are 1 (errors only), 2 (warnings only) or 3 (both errors and warnings).

  • FileOutput

    • This entry specifies the directory (such as C:\OPSLogs) to which the log files are to be written.

  • MessageEventLogLevel

    • This entry specifies the logging level for messages logged to the event log.

    The following table gives the possible decimal values and the kind of information that they generate. You can specify different combinations of types of entries by entering the added value of the types of data that you want to log. For example, one of the most commonly desired combinations is errors, warnings, and debug data. To specify this combination of logging, you would enter 7, the sum of 1, 2, and 4. Another commonly used combination, errors and the timer, is specified by entering 33.

Decimal value

Meaning

1

Error—generates an entry for each error

2

Warning—generates an entry for each warning

4

Debug—generates more information about the errors and warnings that have been logged

8

Info—generates an entry for events such as the starting and stopping of the subsystem

16

Trace—generates an entry that reports that a process is starting or stopping and describes the process

32

Timer—includes the time of each event that generates an entry

63

Everything—generates entries that include all the above data

  • MessageXmlLogLevel

    • If you choose to log events to an XML log file, this entry specifies the desired level of logging. It has the same possible values and meanings as MessageEventLogLevel. Setting a value other than 0 for this entry enables OPS logging to XML-formatted files. Note, however, that logging to an XML file can slow performance.

  • MessageFileLogLevel

    • This entry specifies the desired level of logging for a text log file for the sync subsystem. It has the same possible values and meanings as MessageEventLogLevel.

  • MessageXmlFileName

    • If you choose to log events to an XML file, this entry specifies the name of the file.

  • MessageFileLogName

    • This entry specifies the root and extension of the file name for the sync subsystem text log file.

    OPS builds the file name with the syntax “applicationName-siteid-subsystemPrimaryId_ MessageFileLogNameRoot[_suffix].MessageFileLogNameExtension”.

    In this syntax, “suffix” is an incrementing integer that OPS generates to distinguish between multiple log files.

    Suppose the following:

  • You specify “opssync.log” for MessageFileLogName.
  • The name of the OEAS logical application is “Onyx”.
  • The site ID is 1.
  • The subsystem primary ID is 100001.

    • OPS would name the first two log files as follows:

  • “Onyx-1-100001_opssync.log”
  • “Onyx-1-100001_opssync_0001.log.
  • MessageFileLogMaxCount

    • This entry specifies the number of files saved in addition to the original OPS log file before re-using the oldest file. The default value is 10, meaning that 10 files in addition to the original log file are saved before the oldest file is overwritten with new entries. A value of 0 means that the number of new files is unlimited. Allowing an unlimited number of new files can fill up available space rapidly, depending on how much data is logged.

  • MessageFileLogMaxSize

    • The file size in bytes at which OPS starts a new log file. By default, the value is 5242800 bytes. The possible range of values is unlimited. If you enter 0, the file size is unlimited.

To set up and configure the log file:

  1. Log on to the OPS server using an account with local administrative privileges.
  2. Open the registry editor Regedit.exe.
  3. In the left pane of the Registry Editor, navigate to and select the following key:

    4. HKEY_LOCAL_MACHINE\SOFTWARE\ONYX\ProcessingServer\Service\Applications\application_name\OnyxLog

    where application_name is the name of the OEAS logical application that the OPS server access.

  4. According to your business needs, change the values for the entries described in the preceding list and table.
  5. Close the Registry Editor.
