Configuring Summary Pages

The fields and actions available on Summary pages depend on the configuration within the XML file that corresponds to the page. Use this information to configure XML files for Company, Individual , and Incident Summary pages.

Values you enter in the XML file are case-sensitive.

To configure a Summary page:

  1. Open the XML file corresponding to the Customer Summary page that you plan to modify. For a list of XML files and the screens that they correspond to, see About XML files.
  2. Configure attributes for each element within the XML file.
  3. Configure attributes for each group within the XML file. You can add or remove groups on this screen.
  4. Configure attributes for each field within each group.
  5. Configure attributes for each item in the More group. This group contains navigation menu items that open other OEP screens.
  6. Save and close the XML file.
  7. Log on to OEP, launch the Summary page from Navigator, and verify your changes.

Elements on a Summary page

Use the following table to configure properties for each group and field displayed on a Summary page.

Element

Attribute

Data Type

Description

EntityId

-

Numeric

Do not change this value. This node displays the number corresponding to the entity for which the details are displayed.

EntityName

-

String

Do not change this value. This node displays the name of the LBO that corresponds to the entity.

PrimaryId

-

-

Do not change this value. This is a placeholder for the primary ID of the LBO that corresponds to the Details screen. This value is retrieved when the Summary page is loaded.

PageTitle

-

String

Enter the text that will appear as the title for the Summary page.

Group

-

-

This element marks a section on the Summary page. Each group has a header and a detail section. You can add or remove groups on the Summary page. However, you cannot add multiple navigation menu groups to a single Summary page.

Group

Id

String

Enter a unique group ID for each group in the file. Text within the group is formatted based on the setting for this ID in the Template xml file. Do not change this value for the More group.

Group

Caption

String

Enter the text that will appear as the caption in the group header.

Group

IsCollection

Boolean

Enter True if the group is a collection object in OEAS. Enter False if the group is not a collection object in OEAS. If you have entered True, do not add fields to the group or move fields from such a group to a group where this value is False.

Group

IsPrimaryGroup

Boolean

Do not change this value.

Group

IsCount

Boolean

This attribute is available only for the More group. Enter True to enable users to view the number of records for each menu item in the group. Enter False to hide the record count for all menu items.

Rows

    

This element marks the details to be included for each record within a specific Do More menu item.

Row

-

-

This element marks a row on the Summary page. All fields within a single Row element appear in a single row on the Summary page.

Row

FieldSpace

  

This attribute is used to configure the separator for the Individual first name and last name values. The default value for this attribute is ',' (comma).

You can also use this attribute to separate multiple fields displayed in a single row, for example City, State, Country, and Postal Code.

Enter 'blank' as the value here to maintain a single space between two fields.

Row

EntityType

  

Enter the entity name for which data is to be retrieved. Currently, this attribute is used to retrieve the entity for a Do More item on the Incident Summary page.

Field

-

-

This element marks a field that is displayed on the Customer Summary page.

Field

FieldName

String

Enter the name of the LBO column from which to retrieve data.

Field

DisplayName

String

Enter the caption for the field that will be displayed on the page.

Field

Visible

Boolean

Enter True to enable users to view this field. Enter False to hide this field on the page.

Field

Parent

 -

If the field is a child field, then enter the LBO Column name of the parent field.

Field

MaskFormat

String

Enter a value to specify if the field validates a mask format for the value typed by the user.

Field

LboName

-

Enter the name of the LBO object to which the property belongs.

Field

PropertyPath

-

Enter the XML node path for the field within the LBO.

Field

Value

-

Do not edit this field.

Field

DataType

String

Enter the data type that is displayed on the screen for this field. The possible values are String, Int, GUIData, and DateTime.

Field

LinkColumnName

  

If the field can be tapped to display an OEP record page, enter the FieldName value to which data in this field is linked. This field is not displayed on the screen.

Field

Hyperlink

Boolean

Enter true if the field is hyperlinked to an OEP page, or to an external link. Enter false if there is no link from the field.

Field

DomainData

-

Enter a value for the domain data. The possible values are referenceLookup, country, region, contactType, and user.

Field

IsPrimary

Boolean

Enter 1 for the primary email and phone number field. Enter 0 if the email or phone number is not a primary field. This attribute can be configured only for the email and phone number fields.

MenuItem

MenuItemId

Integer

Enter a unique value for each item in the navigation menu.

MenuItem

LBOName

  

Enter the LBO name from which to retrieve data for the list.

MenuItem

LBOMethodName

  

Displays the name of the LBO method that is used to retrieve data for the list. You can enter the name of the method that you want to use.

MenuItem

DisplayName

String

Enter the text that will appear as the caption for the menu item on the screen.

MenuItem

ResourceId

-

Enter the UI resource ID that is associated with the menu item in OES.

Filter

Property

-

Enter the name of the LBO column by which to filter data for the menu item. You can enter multiple properties to filter by.

Filter

Value

-

Enter the value that corresponds to the parameter that you want to filter the list by.

Filter

contextVariable

-

Enter a contextual value for the filter property, if it is dependent on the record displayed on the screen. This value is returned to OGS when a page is loaded. Based on this value received, OGS sends the relevant data for the filter to OEP. For a list of contextual values currently used, see Context Variable values.

Field

IsExpand

Boolean

This attribute is available only for the More group. By default, True enables users to expand and view the More section. Enter False to collapse the More section on this page.
