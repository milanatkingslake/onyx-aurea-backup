Step 1: Clean the migrated Onyx CTI data for upgrade

If you are upgrading Onyx CTI, you must clean the data in the Onyx CTI tables on the 7.6.1 OEDB before installing Onyx CTI files. Cleaning ensures that data conforms with the 7.6.1 OEDB’s more extensive referential integrity constraints and that the data is thus upgraded.

The script used in this step identifies referential integrity issues of data in the Onyx CTI tables. You can then correct the issues and rerun the script until issues are no longer identified, thus ensuring that the data is clean. The script also automatically fixes issues related to the use of the invalid user ID OnyxEASSQL during installation of Onyx CTI 7.6.1.

Only the clean data will be upgraded to Onyx CTI 7.6.1. Upgrading of clean data occurs automatically during installation of Onyx CTI files to the OEDB. For more information about this installation step, see Step2: Install Onyx CTI to the OEDB.

To clean the migrated Onyx CTI data for upgrade:

  1. Copy the pre-upgrade script from the Onyx CTI product CD to a local machine (the 7.6.1 OEDB server or a separate machine).
    Full path: <CD directory>\Onyx CTI\Setup\Database\source\UpgradeScripts\cti_pre-upgrade.sql
  2. Run the upgrade validation script.
  3. If the output contains errors, update (correct) the related data and rerun the script. Continue until the output contains no errors.
