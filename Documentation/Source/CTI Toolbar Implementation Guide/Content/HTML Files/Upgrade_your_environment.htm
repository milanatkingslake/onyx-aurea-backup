Upgrade your Onyx environment to 7.6.1

If you are upgrading Onyx CTI, you must first upgrade OEAS (including the OEDB) and OEP, which requires data migration. Follow all of the instructions in the Data Migration Toolkit Guide, with special attention to the following step occurring before migration:

  • Correct all referential integrity issues identified by the pre-migration scripts. These scripts identify referential integrity issues in the core platform tables, some of which contain Onyx CTI data. (Referential integrity issues in the Onyx CTI tables are identified after migration in Step 1: Clean the migrated data for upgrade.) Issues specific to Onyx CTI 7.6.1 data include invalid user IDs in the UI_Resource table (chInsertBy) for the UI:OEP.CTI.Toolbar resource, which also affect dependent records, such as resource associations in the userResource object. You can correct identified instances of invalid user IDs by either adding the user ID to the Users table or updating the record’s insert_by value to a valid user ID. Only tables and data that meet referential integrity requirements will be migrated.

For more information on the data migration process, see the Data Migration Toolkit Guide.
