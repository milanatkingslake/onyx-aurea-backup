Step 3: Install Onyx CTI to OEAS

In this step, you run a Setup file that adds CTI business objects to the Onyx Enterprise Dictionary (OED) on the OEAS server and then you validate and publish the updated OED.

To install Onyx CTI to the OEAS server:

  1. Ensure that the existing OED is validated, published, and backed up. (Default location: C:\Program Files\Onyx\AppServer\Applications\Onyx\OnyxEnterpriseDictionaryWC.xml)
  2. If you are upgrading Onyx CTI, use OES Object Designer to remove the existing Onyx CTI business objects from the OED.
  • ctiAgentAssociation
  • ctiConnector
  • ctiFrequentPhones

For information about using OES Object Designer, see the OES Help.

  1. Using the Windows Component Services administration tool on the OEAS server, shut down the OnyxEASApp application (located under Console Root\Component Services\Computers\My Computer\COM+ Applications).
  2. Copy the \Setup\Platform folder from the Onyx CTI product CD to the OEAS server.
  3. Within the copied Platform folder, run the Setup.hta file, bypassing any security warnings that might appear.
  4. In the Onyx CTI OED Setup window, click Next.
  5. Click Browse to find and enter the location of the OED file, or click Install Now to use the default location (listed in the Select OED file box).
  6. Click Finish.
  7. Using Onyx Enterprise Studio (OES) Object Designer, confirm that the following CTI business objects have been added to the OED (if not, rerun the Setup.hta file):
  • ctiAgentAssociation
  • ctiConnector
  • ctiFrequentPhones

For information about using OES Object Designer, see the OES Help.

  1. Click the Validate button.
  2. Click the Publish button and then click OK when prompted to overwrite the OED.
  3. If you receive a message indicating that the publish script failed, you may need to manually start the

    OnyxEASApp application using the Windows Component Services administration tool.
