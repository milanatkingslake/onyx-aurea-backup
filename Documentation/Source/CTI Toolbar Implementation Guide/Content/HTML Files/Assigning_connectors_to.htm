Assigning connectors to agents

In environments that use multiple Onyx CTI connectors to access multiple CTI systems, you must assign a connector to each agent who is not using the default connector. We recommend assigning connectors to all agents. (Agents are Onyx users who use third-party CTI systems to respond to incoming calls.)

Each association record you create in the cti_agent_association table maps an Onyx user ID to an Onyx CTI connector ID. Onyx CTI checks this table every time an agent loads OEP. If no association record exists for the agent’s Onyx user ID, Onyx CTI uses the default connector listed in the main configuration file (full path: YourOEPwebsite\cti\cti_config.xml, where YourOEPwebsite is your OEP installation directory).

Because the cti_agent_association table is not configured by default for display in OES Table Administration, you must manage the table directly in the database.

For information about the ctiAgentAssociation business object and the main configuration file, see the Onyx Computer Telephony Integration Developer’s Reference.

To assign a connector to an agent:

Add a ctiAgentAssociation record by directly accessing the database.
