Step 4: Install Onyx CTI to OEP

In this step, you run a Setup file that adds website files to the OEP server for the Onyx CTI toolbar and overwrites the OEP website files named in the procedure.

This procedure must be completed in the OEP development installation (development environment) for correct compilation of UCW changes affecting the website files. For more information on compiling UCW changes, see the OEP Technical Guide.

To install Onyx CTI to the OEP server:

  1. On the OEP development installation (development environment), ensure that the following website files are backed up:
  • YourOEPwebsite\powerpage\application_main.js
  • YourOEPwebsite\powerpage\oep_automate.asp
  • YourOEPwebsite\powerpage\ucf_application_main.asp
  • YourOEPwebsite\QuickSearch\search-client.js

where YourOEPwebsite is the OEP installation directory (default: C:\Program Files\Onyx\Employee Portal).

  1. Copy the \Setup\Website folder from the Onyx CTI product CD to the OEP server.
  2. Within the copied Website folder, run the Setup.hta file, bypassing any security warnings that might appear.
  3. In the Onyx CTI OEP Setup window, click Next.
  4. Type the full path of the OEP installation folder, or click Install Now to use the default location (listed in the OEP Install Directory box).
  5. Click Finish.
  6. Confirm that the Onyx CTI toolbar appears:
  1. Stop and restart your OEP Web server, such as by using the Internet Information Services (IIS) Manager tool or by executing the IISReset command (via Command Prompt).
  2. On the client machine used to access the toolbar, delete the files in Internet Explorer's Temporary Internet Files folder.
  3. Log on to OEP with the user name and password of a user that has been granted access to the toolbar. (The system administrator is granted toolbar access by default).
  1. Compile and publish all UCW changes to finalize the website files.

For instructions on compiling and publishing UCW configurations, see the OEP Technical Guide.

  1. If you upgraded to OEP and your website files (listed in Step 1: Clean the migrated data for upgrade of this procedure) previously included customizations, reapply those customizations.
  2. Manually copy the compiled website files from the OEP development installation to the production environment OEP server. (For the full paths of the website files, Step 1: Clean the migrated data for upgradeof this procedure).
