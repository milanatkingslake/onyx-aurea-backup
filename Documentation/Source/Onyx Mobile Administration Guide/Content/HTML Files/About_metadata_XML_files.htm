Understanding XML Files

Onyx Mobile App/Onyx Mobile Web uses meta data stored in XML files to control the information displayed on your mobile device. You can modify these XML files to add new elements, and to change attributes for the existing elements. To add new elements or change attributes for Onyx in Japanese language use the xml section that comes after <LangNode languageCode="JPN">. For the English language, use the xml section that follows <LangNode languageCode="ENG">. The xml file is found in this location: ..\Onyx\AppServer\Applications\Onyx\OnyxGatewayService\App_Data.

You can only add LBO properties to your Onyx Mobile App/Onyx Mobile Web if those properties already exist or were previously created in your Onyx application.

For more information about adding LBO properties to your Onyx application, refer to the Onyx Technical Guide.

The following tables list the XML files used to configure Onyx Mobile App/Onyx Mobile Web, along with a description of what each file is used for. All the XML files used by Onyx Mobile App/Onyx Mobile Web are located on the OEAS server, within your Onyx Gateway Service installation folder.

The default file path is: C:\Program Files\Onyx\AppServer\Applications\Onyx\OnyxGatewayService\App_Data\Profiles.

Do not change the name of an XML file even if you have customized the name of the entity.
