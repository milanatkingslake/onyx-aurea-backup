Configuring Add/Edit Screens

The fields and tasks available on each Add/Edit screen depend on the configuration made within the XML file corresponding to the screen. Use this information to configure XML files for Add/Edit screens.

Values you enter in the XML file are case-sensitive.

To configure Add/Edit screens:

  1. Open the XML file corresponding to the Add/Edit screen you plan to modify. For a list of XML files and the screens that they correspond to, see Understanding XML files.
  2. Configure attributes for each element within the XML file.
  3. Configure attributes for each group within the XML file. You cannot add a new group to this screen.
  4. Configure attributes for each field within each group.

In the WorkNoteMetaData.XML file, do not remove the logMark and publishBitMark fields as these are essential to successfully save work notes.

  1. Save and close the XML file.
  2. To format the text displayed on the screen, modify the Templates file for the screen. For more details about modifying the Templates file, see Formatting Screen Text.
  3. Log on to Onyx Mobile App/Onyx Mobile Web and verify your changes.
Elements on an Add/Edit screen

An administrator can now create a custom entity list by using the Add/Edit<entityname>MetaData.xml format. Ensure that you follow the element structure as the OOTB Add/Edit XML files.

Use the following table to configure properties for each group and field displayed on an Add/Edit screen:

Element Attribute Data Type Description
EntityId - Numeric Do not change this value. This node displays the number corresponding to the entity for which the list is displayed.
EntityName - String Do not change this value. This node displays the name of LBO that corresponds to the entity.
PrimaryId -   Placeholder for the primary ID of LBO that corresponds to the Edit screen. This value is retrieved when the Edit screen is loaded. Do not edit this value.
AddPageTitle - String Type the text that will appear as the title for the screen when users open it to add a new record.
EditPageTitle - String Type the text that will appear as the title for the screen when users open it to edit an existing record.
EditPermission - -  
ResourceId EditPermission - The UI Resource permission for editing the particular entity.
Value EditPermission - By default, there should be no value. This contains the edit permissions of the particular entity.
LBOMethodName - - Displays the name of the LBO method used to save data for the screen. You can type the name of the method that you want to use.
Group -   This element marks a section on the Edit screen. Each group has a header and a detail section.
Group Id String Type a unique group ID for each group in the file. Text within the group is formatted based on the setting for this ID in the Template XML file.
Group Caption String Type the text that will appear as the caption in the group header.
Group IsCollection Boolean
  • Type True if the group is a collection object in OEAS.
  • Type False if the group is not a collection object in OEAS.
You must add fields for a collection object group that users can update. If you have typed True, do not add fields to the group or move fields from such a group to a group where this value is False.
Group IsPrimaryGroup Boolean Do not change this value.
Field - - This element marks a field that is displayed on the Onyx Mobile App/Onyx Mobile Web screen.
Field ResourceId - Type the UI resource permission value for the field.
Field FieldName String Type the name of the LBO column from which to retrieve data.
Field DisplayName String Type the caption for the field that will be displayed on the screen. To avoid cropping, we recommend that captions do not exceed 10 characters, including spaces. Do not enter a value if the default value for this attribute is blank.
Field Visible Boolean
  • Type True to enable users to view this field.
  • Type False to hide this field on the screen.
Do not type a value if the default value for this attribute is blank.
Field Type String Type the data type that is displayed on the screen for this field.
Field Parent String If the field is a child field, type the LBO column name of the parent field. For example, Country is a parent field for State.
Field ControlType String Type the control type to use for the field. The possible values are TextBox, ComboBox, DropDown, CheckBox, URL, Link, RadioButton, and Scan.
Field DisableLinkColumn String This attribute is used only for the Primary Contact firstname field. Type the dependent fieldname that will become uneditable when this field is populated by linking it to another record. For example, when a company is linked to a primary contact, the primary contact's last name field is populated automatically based on the linked contact's details. At this time, users should not be able to type in a different last name. The fieldname you enter should also be listed in the DependentFieldName attribute for the field.
Field MaskFormat String Type a value to specify if the field validates a mask format for the value typed by the user. Currently this attribute is used for the postal code and phone fields.
Field IsRequired Boolean
  • Type True if a value is needed in the field to save the record.
  • Type False if the field can be left empty.
Field IsEditable Boolean
  • Type True to enable users to enter values in the field.
  • Type False to make the field read-only.
Field CacheName String Type the name of the OEAS Reference Lookup table.
Field CacheParentId - Do not change this value. In the current implementation, your changes will not be reflected in the application.
Field LboObjectName String Type the name of the LBO object to which the property belongs.
Field PropertyPath String Type the XML node path for the field within LBO.
Field LboProperty String Type the name of the LBO object to which the property belongs.
Field linkPropertyPath String Type the name of the LBO property for the primary contact ID in the CompanyMetaData.XML file or the parent company ID in the IndividualMetaData.XML file.
Field linkEntity Integer
  • Type 1 to indicate a link to an Individual record.
  • Type 2 to indicate a link to a Company record.
Field ReadValue String From the Reference Lookup XML, select the attribute name.
Field LinkFieldReadValue String Contains description for the drop-down.
Field TypeIdProperty String Type the LBO property name for the primary value selected for a collection field.
Field TypeIdValue Integer Type the ID that corresponds to the collection field's lookup value. To get this value, in OES Reference Table Administration, click the Lookup Value field, and copy the value from the ID of Lookup Value field. This attribute is mandatory to ensure that collection fields display accurate values.
Field LinkFieldName String Type the LBO field name for the primary contact or parent company.
Field DependentFiedName String Type the field name for the primary contact's details from the Company LBO and associate it with the field name for the Individual's details from the Individual LBO, in the format contactFirstName;firstName. You must enter this data for the following fields:
  • First Name
  • Last Name
  • Department
  • Title
  • e-mail
All dependent fields should also be present in IndividualDetail.xml.
Field OnyxTimeStamp - Do not change this value. This value is dynamically populated with the user's Onyx session ID when a call is made to OGS.
Field MaxLength Integer Type the maximum field length.
Field Value - Do not type a value for this field. The value is retrieved when loading the screen.
Strings - - This element is not used in the current implementation.
