Prompted Action

The Prompted Action feature allows you to quickly update certain fields of the entities through the DoMore section on the Detail screen of the Onyx Mobile without going into the Edit screen. The Prompted Action uses XML data to display the edit interface.

It is possible to implement the Prompted Action for the entities with the following properties:
• The LBO properties which are direct descendent of the respective object
• The properties other than type collection

Control Types

Following are the supported control types:

  • Assigned to
  • Check box
  • Date field
  • Drop-down list box
  • Dependency Drop-down list box
  • Text box
  • Text Area

These fields can be set as mandatory.

Configuring Prompted Action

To configure the Prompted Action, perform the following steps:

  1. Open the required XML file(s) in edit mode from the \Onyx\AppServer\Applications\Onyx\OnyxGatewayService\App_Data\Profiles folder. For the list of screens and their XML file names, see Understanding XML files.
  2. Add the required element nodes in the DoMore section of the required XML file(s).
  3. Save and close the XML file.
  4. Log on to Onyx Mobile Native App or Onyx Mobile Web and verify the changes.

Example:

Following example shows the configuration of Prompted Action in the CompanyDetails.xml file:

  1. Add the following element node in the DoMore section of the CompanyDetails.xml file.

<MenuItem MenuItemId="20" DisplayName="Update Company Name" ResourceId="" PromptAction="true" LogicalObjectName="company" LogicalMethodName="update">

<Filter Property="companyName" ControlType="textbox" IsRequired="true" DisplayName="Company Name" Parent="" CacheName="" />

</MenuItem>

  1. The Prompted Action is configured for Update Company Name, appears as shown in the following image.

Use the OOTB elements and attributes from the following table to configure the Prompted Action.

Element Attribute Data Type Description

MenuItem

MenuItemId

Integer

Enter a unique value for each item in the Navigation menu.

DisplayName

String

Enter the text that you use as a caption for the item. To avoid cropping, limit the number of caption characters to 10 including spaces.

ResourceId

-

Enter the UI resource permission value for the field.

PromptAction

Boolean

Do not change this value.

LogicalObjectName

String

Enter the logical object name.

LogicalMethodName

String

Enter the logical method name.

customPromptActionCall

String

Enter the logical method name.

ContextVariableNodeName

String

Use semi-colons (;) to separate the node names for the context variables.

Filter

Property

String

Enter the name of the LBO column by which you filter data for the menu item.

ControlType

String

Enter the control type to use for the field.

The possible values are Text box, Drop down list, Check box, DateTime, and Tree.

IsRequired

Boolean
  • Enter True if a value is required in the field to save the record.
  • Enter False if the field can be left empty.

DisplayName

String

Enter the text that you use as a caption for the item. To avoid cropping, limit the number of caption characters to 10 including spaces.

Parent

-

Enter the property value of the parent field if the field is a child.

CacheName

String

Enter the name of the OEAS Reference Lookup table for reference fields, if applicable.

parameter

XML

Enter the LBO XML for which you need to perform when the customPromptActionCall is used

CacheParentId

-

Do not change this value. In the current implementation, your changes do not reflect on the application.
