Configuring Details Screens

The fields and tasks available on each Details screen depend on the configuration within the XML file that corresponds to the screen. Use this information to configure XML files for Details screens.

Values you enter in the XML file are case-sensitive.

To configure Details screens

  1. Open the XML corresponding to the Details screen you wish to modify. For a list of XML files and the screens that they correspond to, see Understanding XML files.
  2. Configure attributes for each element within the XML file.
  3. Configure attributes for each group within the XML file. You can add or remove groups on this screen.
  4. Configure attributes for each field within each group.
  5. Configure attributes for each item in the <!--Do More> section. This group contains navigation menu items that open other Onyx Mobile App/Onyx Mobile Web screens.  

Custom web pages can be launched from the Testing Scripting option in the More section of an entity details screen, though a single click. To do this, add the following code in the <!--Do More> section in the <entity>Details.xml file:

<MenuItem MenuItemId="16" DisplayName="Testing Scripting" ResourceId="" Action="scriptingUrl" Url="www.avolin.com" PageTitle="Scripting" ContextVariableList="siteId;">

</MenuItem>

For the above menu item the context variables are applicable.

  1. Save and close the XML file.
  2. To format text displayed on the screen, modify the Templates file for the screen. For details about modifying procedures. Formatting Screen Text .
  3. Log on to Onyx Mobile App/Onyx Mobile Web and verify your changes.

Linking a primary contact to a company or incident record:

To be able to link a primary contact to a company record, ensure that the following fields are present in the IndividualDetails.XML file:

  • firstName
  • lastName
  • departmentDesc
  • titleDescription
  • e-mailAddress

If you do not want these fields to be displayed on the Details screen, you can set the attribute Visible=False for these fields.

You cannot add or remove fields from the PrimaryContact group in the CompanyDetails.xml and the IncidentDetails.XML files.

Elements on a Details screen:

An administrator can now create a custom entity list by using the <entityname>Details.xml format. Ensure that you follow the element structure as the OOTB Details XML files.

Use the following table to configure properties for each group and field displayed on a Details screen:

Element Attribute Data Type Description
EntityId - Numeric Do not change this value.This node displays the number corresponding to the entity for which the details are displayed.
EntityName - String Do not change this value.This node displays the name of LBO that corresponds to the entity.
SubEntityName - String Do not change this value.This node displays the name of LBO that corresponds to the sub entity.
PrimaryId - - Do not change this value.This is a placeholder for the primary ID of LBO that corresponds to the Details screen. This value is retrieved when the Details screen is loaded.
PageTitle - String Type the text that will appear as the title for the Details screen.
LBOMethodName - String Displays the name of the LBO method used to retrieve data for the list.You can type the name of the method that you want to use.
EditPermission - -  
DeletePermission - -  
ResourceId EditPermission/DeletePermission - The UI Resource permission for editing or deleting the particular entity.
Value EditPermission/DeletePermission - By default, there should be no value. This contains the edit or delete permissions of the particular entity.
LBOMethodName - - Displays the name of the LBO method used to retrieve data for the Details screen.You can type the name of the method that you want to use.
Group - - This element marks a section on the Details screen. Each group has a header and a detail section. You can add or remove groups on the Details screen. However, you cannot add multiple navigation menu groups to a single Details screen.
Group Id String Type a unique group ID for each group in the file. Text within the group is formatted based on the setting for this ID in the Template XML file.Do not change this value for the Do More group.
Group Caption String Type the text that will appear as the caption in the group header.
Group IsCollection Boolean
  • Type True if the group is a collection object in OEAS.
  • Type False if the group is not a collection object in OEAS.
If you have entered True, do not add fields to the group or move fields from such a group to a group where this value is False.
Group IsPrimaryGroup Boolean Do not change this value.
Group IsCount Boolean This attribute is available only for the Do More group.
  • Type True to enable users to view the number of records for each menu item in the group.
  • Type False to hide the record count for all menu items.
Row - - This element marks a row on the Onyx Mobile App/Onyx Mobile Web screen. All fields within a single row element appear in a single row on the Details screen.
Field - - This element marks a field that is displayed on the Onyx Mobile App/Onyx Mobile Web screen.
Field ResourceId - Type the UI resource permission value for the field.
Field FieldName String Type the name of the LBO column from which to retrieve data.
Field DisplayName String Type the caption for the field that will be displayed on the screen. To avoid cropping, we recommend that captions do not exceed 10 characters, including spaces.
Field FontWeight String Type one of the following font weights:
  • bold
  • normal
Field FontSize String Type the font size.
Field Visible Boolean
  • Type True to enable users to view this field.
  • Type False to hide this field on the screen.
Field Parent - If the field is a child field, type the LBO column name of the parent field.
Field MaskFormat String Type a value to specify if the field validates a mask format for the value typed by the user.
Field LboName - Type the name of the LBO object to which the property belongs.
Field PropertyPath - Type the XML node path for the field within LBO.
Field Value - Do not edit this field.
Field DataType String Type the data type that is displayed on the screen for this field. The possible values are String, Int, GUIData, and DateTime.
Field TapType String Type a value for the action to be performed when a user taps this field.For a list of tap type values currently available, see Specifying Tap Type Values.If a row contains multiple fields, the TapType attribute can be enabled for only one of those fields.
Field TapValue - Do not type a value for this field.This value is retrieved when loading the screen.
Field LinkColumnName - If the field can be tapped to display another Details screen, type the FieldName value to which data in this field is linked.This field is not displayed on the screen.
Field DomainData - Type a value for the domain data.The possible values are referenceLookup, country, region, contactType, and user.
MenuItem MenuItemId Integer Type a unique value for each item in the navigation menu.
MenuItem DisplayName String Type the text that will appear as the caption for the menu item on the screen.
MenuItem ResourceId - Type the UI resource ID that is associated with the menu item in OES.
MenuItem Action String Type a value for the action to be performed when a user taps the menu item.For a list of action values currently available, see Specifying Action Values.
MenuItem PageTitle String Type a value for the title of the screen that opens on tapping the menu item.
MenuItem ContactTypeDid Numeric This attribute is used only for the Add to my favorites menu item.Type the Contact Type ID value that corresponds to the contact type Favorite in OES Contact Type Administration under Internal\Company and Internal\Individual.
Filter Property - Type the name of the LBO column by which to filter data for the menu item.You can enter multiple properties to filter by.
Filter Value - Type the value that corresponds to the parameter that you want to filter the list by.
Filter contextVariable - Type a contextual value for the filter property, if it is dependent on the record displayed on the screen.This value is returned to OGS when a screen is loaded. Based on this value received, OGS sends the relevant data for the filter to Onyx Mobile App/Onyx Mobile Web.For a list of contextual values currently used, see Specifying Context Variable Values.
MenuItem contextVariableNodeName String Type the semi-colon (;) separated node names for the context variables, if the action is custom.The node name values have two types - fixed name or context.
MenuItem LogicalObjectName String Type the logical object name if the action is custom.
MenuItem LogcialMethodName String Type the logical method name if the action is custom.
MenuItem Web String Type Web="false" to disable an action on Onyx Mobile Web.
MenuItem Icon String

Type the icon name. For the list of available icons, see Icons.
Parameter Null XML Type the LBO XML for which you need to perform when the action is custom.
