Scan Utility

The Scan Utility feature allows you to retrieve the following information by scanning the bar code of a product:

  • On the Home screen
  • List of customer records.
  • List of sales / service / support incident records.
  • List of work ticket records.
  • On the Add or Edit screen
  • Product description.

The Scan Utility is supported only on the Onyx Mobile App.

Activating Scan Utility

To activate the Scan Utility, add the scan registration key in the value attribute of the following node in the OnyxGatewayWindowsService.exe.config file from the \Onyx\AppServer\Applications\Onyx\OnyxGatewayService folder:

  • Android

<add key="androidScanKey" value="Enter your scan registration key here" />

  • iOS

<add key="iOSScanKey" value="Enter your scan registration key here" />

It requires a scan registration key to activate the Scan Utility. Contact your Accounts Manager for the scan registration key.

Configuring Scan Utility

To configure the Scan Utility, perform the following steps:

  1. Open the required XML(s) file in edit mode from the \Onyx\AppServer\Applications\Onyx\OnyxGatewayService\App_Data\Profiles folder. For the list of screens and their XML file names, see Understanding XML files.
  2. Add the following element node in the required XML file(s).
  • Home Screen

<MenuItem Web ="False" Id="14" Name="ScanWorkticket" DisplayName="Scan for Workticket By products2" Icon="Scan.png" Url="../customer/scan_search.js" Scan ="true" SequenceIndex="1" Visible="true" LogicalObjectName="workTicket" LogicalMethodName= "csretrieveWorlticketByPro" XMLName="HomeWorkTicketList.xml" />

Where,

  • Name is the name of the node that you want to assign.
  • DisplayName is the name that appears on the screen.
  • Web="False" disables the scan on the Home Screen for Onyx Mobile Web.
  • LogicalObjectName is the name of the LBO.
  • LogicalMethodName is the method name to fetch the data from the scan. This method should have a ProductCode parameter.
  • XMLName is the name of the *List.xml file which fetches the result list on the user interface.
  • Add or Edit Screen

<Field FieldName="productId" DisplayName="Product" Visible="True" Type="CHAR255" Parent="" ControlType="scan" MaskFormat="" IsRequired="True" IsEditable="True" CacheName="incident.product" CacheParentId="3" LboObjectName="incident" PropertyPath="incident/" LboProperty="productId" MaxLength="22" Value=""/>

  1. Save and close the XML file.
  2. Log on to Onyx Mobile Native App and verify the changes.

When the ControlType scan for the entity is configured, it appears as on the Onyx Mobile Native App and appears as drop-down on the Onyx Mobile Web.
