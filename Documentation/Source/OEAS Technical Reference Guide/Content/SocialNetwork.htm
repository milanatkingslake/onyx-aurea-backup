SocialNetwork Object

The socialNetwork object is used to associate social media information entered by a user with the appropriate customer record in the OEDB. The socialNetwork object has one key property, PrimaryId, that uniquely identifies the social media integration in the database. Once the object is created, the value of this property cannot be changed.

Methods

Method

Description
insert Persists a new object instance to the database
update Persists modifications to an existing object instance
retrieveCollection Returns a collection of objects based on the search criteria supplied in the parameters XML
saveCollection Performs multiple actions at once: inserting, updating, and deleting objects, as appropriate

Properties

Property

Description

Caption

Contains the tool tip that is displayed for the social media

identifier

Contains the account or profile ID in the case of link integrations, and the custom URL in case of feed integrations

insertBy

Contains the user ID of the Onyx user who initially saves the object

insertDate

Contains the time and date on which the object is initially saved

onyxTimestamp

Facilitates concurrency control with the Onyx Enterprise Database

ownerId

Identifies the parent object of the child object

ownerType

Contains a numerical value that identifies the type of parent object that owns the child object

primaryId

Contains a unique ID assigned to the object when it is first saved to the database

sequence

Defines the sequence in which link icons and feeds are displayed on the Social Media tab

socialNetworkId

Specifies the social network type that is used when rendering the social media integration

updateBy

Contains the user ID of the Onyx user who last saved modifications to the object

updateDate

Indicates the time and date on which modifications to the object were last saved
