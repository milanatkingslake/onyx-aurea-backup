Welcome to Security Administration

Help for Security Administration contains procedures for accomplishing tasks in the course of managing access permissions.

For detailed instructions on using this tool, click to access the Help system.

Quick overview

The Security Administration interface has four main areas:

  • The context area, which lists the tool you're using (Security Administration) and the application you're connected to, on which site.
  • The toolbar, which contains command buttons you can use to perform common tasks quickly.
  • The navigation area (the left pane), which contains a tree of the users, groups, roles, and resources that are available for managing.
  • The edit area (the right pane), where you manage permissions and change the properties of users, groups, roles, and resources.

Following are the buttons on the Security Administration toolbar and their uses:

  • Click to add a new group, user, or role.
  • Click to delete a role or resource. Note that the deletion does not actually occur until you click You can only delete roles or resources. You cannot physically delete a user or group; you can only deactivate it.
  • Click to create a user with the same properties, including group and role membership but not including access permissions, as the user that is selected when you create the new user.
  • Click to change preferences for Security Administration. Currently, the only preference that can be changed is the default Onyx password for new users.
  • Click to save the changes that you have made to groups, users, roles, and resources since you last opened Security Administration or last saved changes. You can save several changes with one click. However, Onyx recommends that you save changes as frequently as is practically possible.
