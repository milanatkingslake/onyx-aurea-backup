OEAS Configuration Utility

The OEAS Configuration Utility is a program with which you can change the following settings without having to run the Onyx Configure Enterprise Application Server Setup:

  • OES SQL account information (SQL account name and password)
  • SQL Server name
  • OEDB name
  • Connection timeout
  • Query timeout
  • Maximum number of rows returned by most queries
  • Event log settings specifying the level of detail logged to the Windows Event Viewer
  • SMTP email protocol settings

The Onyx Configure Enterprise Application Server Setup program overwrites these values with default values or with values that you supply as you run the program. After running the Onyx Configure Enterprise Application Server Setup, you can change the values by running the OEAS Configuration Utility.

The following procedures explain how to use the OEAS Configuration Utility.

To install and use the OEAS Configuration Utility, you must be logged on to the OEAS server with an account that has local administrative privileges.

Several of the following procedures instruct you to stop the System Application COM+ application, if the server™s operating system is a member of the Windows 2000 family of servers. Stopping System Application interrupts any work that is occurring within other COM+ applications. Stopping OnyxEASApp stops any work that is occurring in the OEAS components and flushes the caches.

To install the OEAS Configuration Utility:

  1. On the OEAS product CD, open the Utilities folder.
  2. Copy the OEASConfig folder to a suitable location on the OEAS server.

The OEAS Configuration Utility is ready to use.

Application data

To reconfigure an OEAS logical application with the OEAS Configuration Utility:

  1. In Windows Explorer, navigate to the directory where you copied the OEAS Configuration Utility files and double-click OEASConfigTool.exe. The OEAS Configuration Utility window appears.
  2. In the Application Name box, click the down arrow and then the name of the OEAS logical application for which you want to change configuration data.

The current application configuration information appears in the rest of the boxes.

  1. Change the application configuration as desired, and then click Save Changes:
  • The Database Server Name, Database Name, Database Server User ID, and Database Server Password boxes contain database-connection information that you specified in the setup in which you created this OEAS logical application. The database server user ID is the SQL Server SA user; the database server password is the SQL Server SA user password.
  • The Connection Timeout value specifies how many seconds can pass without OEDB server access before the connection is dropped.
  • The Query Timeout value specifies how many seconds can pass without a query being completed before the query is dropped.
  • The Max Rows value specifies the maximum number of rows that can be returned on a query.
  • The OBPM Organization ID identifes an organization available on an Onyx Process Manager system.
  1. To save your changes, click Save Changes.
  2. To close the utility, click Exit.
  3. To make your changes take effect, do one of the following, depending on whether the OEAS server™s operating system is Windows 2000 or Windows 2003:
  • If Windows 2000: Shut down and then restart the COM+ applications OnyxEASApp and System Application.
  • If Windows 2003: Shut down and then restart the COM+ applications OnyxEASApp, and then restart the server.

Event-logging

Using the OEAS Configuration Utility, you can specify event-logging settings for the logical application.

There are four levels of logging detail that you can specify for any Onyx user that has been created using Security Administration:

  • 1: run-time errors
  • 3: run-time errors and warnings
  • 5: run-time errors, warnings, and tracing statements
  • 7: all logging messages

Click the Logging Data tab, the Onyx user is represented by a key, which is the ID of an Onyx user that you have created in Security Administration. The setting assigned to the GlobalLogSettings key, which appears by default in the KeyList box, applies to all users including those for whom you create a key a higher logging preference.

For additional details regarding the four levels of logging detail, see What is Onyx logging?.

To add an individual logging preference for a user:

  1. To open the OEAS Configuration Utility, double-click OEASConfigTool.exe.
  2. Click the Logging Data tab.
  3. In the Key box, type the ID of an Onyx user.
  4. In the Value box, type either 1, 3, or 5, or 7 to set the logging preference.
  5. Click Add Key.
  6. To save your changes, click Save Changes.
  7. To close the utility, click Exit.
  8. To make your changes take effect, do one of the following, depending on whether the OEAS server™s operating system is Windows 2000 or Windows 2003:
  • If Windows 2000: Shut down and then restart the COM+ applications OnyxEASApp and System Application.
  • If Windows 2003: Shut down and then restart the COM+ applications OnyxEASApp, and then restart the server.

To change the logging preference for a user:

  1. To open the OEAS Configuration Utility, double-click OEASConfigTool.exe.
  2. Click the Logging Data tab.
  3. In the KeyList box, double-click the user ID of the user that requires a different event log setting.
  4. In the Value box, change the logging preference setting.
  5. To save your changes, click Save Changes.
  6. To close the utility, click Exit.
  7. To make your changes take effect, do one of the following, depending on whether the OEAS server™s operating system is Windows 2000 or Windows 2003:
  • If Windows 2000: Shut down and then restart the COM+ applications OnyxEASApp and System Application.
  • If Windows 2003: Shut down and then restart the COM+ applications OnyxEASApp, and then restart the server.

To remove individual logging preferences for a user:

  1. To open the OEAS Configuration Utility, double-click OEASConfigTool.exe.
  2. Click the Logging Data tab.
  3. In the KeyList box, double-click the user ID of the user that no longer requires an individual logging preference.
  4. Click Remove Key.
  5. Click Save Changes and then click Exit.

Email send data

Using the OEAS Configuration Utility, you can specify Email Send authentication settings for the logical application.

To change the Email Send authentication method:

  1. To open the OEAS Configuration Utility, double-click OEASConfigTool.exe.
  2. Click the Email Send Data tab.
  3. From the Authentication Protocol drop-down list, select an authentication method.

The configuration utility supports three protocols, Anonymous, Basic, and NTLM. As the name suggests, Anonymous attempts to log into the SMTP service without any credentials, as an anonymous user. Basic authentication sends a user name and password in clear text to the SMTP service. NTLM attempts to authenticate the COM+ identity that is running OnyxEASApp, the OEAS COM+ application.

When using Basic authentication, the utility presents Send User Name and Send User Password text boxes. Enter the authentication credentials for the user that you configured on the SMTP server for sending emails from the Email Send step component. Onyx provides OPSEncrypt utility to encrypt the user's password before entering in the password box. This ensures that the password is not stored in clear text anywhere.

  1. In the SMTP Port box, type the port number on which the SMTP service listens. Typically this is port 25.
  2. If your business requires it and the SMTP service supports it, select the Use SSL checkbox to ensure privacy of the authentication with cryptography.

Onyx does not support Transport Layer Security (TLS).

  1. If you do not want to keep changes you have made in this session, click the Restore Values button to revert to previously saved settings.
  2. If you want to keep the changes you have made in this session, click the Save Changes button.
  3. To close the utility, click Exit.
  4. To make your changes take effect, do one of the following, depending on whether the OEAS server™s operating system is Windows 2000 or Windows 2003:
  • If Windows 2000: Shut down and then restart the COM+ applications OnyxEASApp and System Application.
  • If Windows 2003: Shut down and then restart the COM+ applications OnyxEASApp, and then restart the server.
