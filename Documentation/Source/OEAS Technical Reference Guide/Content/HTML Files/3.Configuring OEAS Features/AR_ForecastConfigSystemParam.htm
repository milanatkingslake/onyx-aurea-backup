Setting system parameters

There are several system parameters that you can use to modify the default behavior of the OEDB and OEP for the Forecasts and Quotes feature. The following table briefly describes these system parameters.

System Parameter

Description

DefaultCurrencyCode

The default currency code assigned to new products entered into the database through Product Administration.

EnableConversionFromTotal

When this parameter is enabled, the total value in a secondary currency for a list of line items in a quote is determined by converting the total value of the quote in the primary currency. If this parameter is not enabled, the total value for the quote in the secondary currency is calculated by summing each of the line items from the individual prices in the secondary currency. Values can be rounded when the system converts a currency, so these methods might produce different results.

EnableCurrencyConversion

Enables users to perform currency conversions for forecasts and quotes on the Forecast and Quotes tab. With this system parameter enabled, when a user selects a currency code that is different than the base currency of a line item product, the system converts the base currency to determine the new price and totals based on the new currency chosen. The CurrencyConversion table supports the conversion calculation by storing the conversion rate between the two currencies. If EnableSecondaryCurrency is also enabled, conversions are performed from the primary currency to the secondary currency.

EnableForecastQuoteSales

Determines whether the Forecast and Quotes tab appears on the Sales Incident page.

EnableForecastQuoteService

Determines whether the Forecast and Quotes tab appears on the Service Incident page.

EnableForecastQuoteSupport

Determines whether the Forecast and Quotes tab appears on the Support Incident page.

EnableSecondaryCurrency

Enables the system to display forecasts and quotes in primary and secondary currencies. Additionally, the summary data displayed in the bottom right-hand corner of the Forecast and Quotes tab is displayed for both currencies. For secondary currency values to be visible to the end-user, you must have both EnableSecondaryCurrency and EnableCurrencyConversion set to (Y).

You administer system parameters using System Parameter Administration. Using this tool, you can set the parameter value that controls each system parameter. For details regarding System Parameter Administration and specific details regarding each system parameter, see System Parameter Administration Help.

Before you configure a forecast or quote, Onyx recommends that you specify a default currency code in Product Administration for the forecast/quote (it is optional). Otherwise, no currency code will be assigned when you save the forecast or quote.
