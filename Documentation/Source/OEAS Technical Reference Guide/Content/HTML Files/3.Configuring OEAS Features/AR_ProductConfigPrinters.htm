Configuring printers

A printer is an attribute for a literature product. When you have created a printer item, it appears as a selection in the Printer list box within the Literature Information edit area when you have selected a product from the navigation area. You associate a printer with a product that you are configuring as a literature product, so that when an end-user creates a literature order using that literature product, OEAS can send the order to the appropriate printer to be printed.

Within the Literature Orders feature, printer data is an attribute of the underlying product that a user associates to the literature order. Printer data is not visible to, nor can it be modified by, the user creating or modifying the literature order. For details regarding literature orders, see Literature Orders.

Using the ServicePrinterInfo filter in Table Administration, you can:

  • Create a new printer path
  • Modify an existing printer path
  • Set a printer path to active or inactive
  • Delete a printer path

For further details on using this tool, see Table Administration Help.

The following table lists printer elements displayed by Table Administration.

Element Description

primaryId

Not editable as it is a primary key field. An application assigned unique identification number that is generated for each printer. It is called throughout the application to process each literature order.

literatureTypeId

The type of literature that is to be printed on this printer. These values are determined through the LiteratureType table, which assigns an ID for each literature type.

printerShare

The share path to the printer (sometimes the same as the printer name).

printerName

The value displayed in Product Administration for the selected printer.

active

Specifies whether the data is active or inactive: 1 is active and 0 is inactive.
