Standard configurable fields and user-defined fields

Whether a field is a standard configurable field or a user-defined field determines the extent to which you can configure it.

For information on how to retrieve national language strings for modifying captions not supported by the reference data, see the OEAS Developer’s Reference.

Using Reference Table Administration, you can do the following:

  • Create, delete, and modify lookup values for the scriptsession.status configurable field
  • Modify user-defined fields

Configuring the scriptsession.status configurable field

Onyx Employee Portal (OEP) uses the scriptsession.status configurable field to display the status of the script session. To do so, OEP matches the extraData property to a set of constants. Changing the following existing status values therefore results in OEP’s displaying either an inaccurate status description or no status description at all:

extraData value scriptsession.status description

1

New

2

In progress

3

Suspended

4

Completed

If you create additional extraData status values, you must also customize OEP in order for the additional status descriptions to appear. However, OEAS does not recognize additional status values. When a script session is resumed and OEAS encounters an unexpected status value, it returns the script to the first step. Therefore, it is recommended that you neither change existing scriptsession.status values nor create additional ones.

However, you can safely modify the field caption for the scriptsession.status configurable field.

The scriptsession.status configurable field is supported by the Reference Data feature. For information on the Reference Data feature and on administering configurable fields, see Reference Data.
