Creating additional task categories

By default, OEAS supports the task categories, sales, service, and support, as described in Tasks. These task categories are installed by default when you install the OEDB and are differentiated by the ID values listed in the following table.

Default task categories Default TASK_CATEGORY_DID

Support

11

Service

12

Sales

13

Do not change the default category IDs for the support, service, and sales tasks. These default categories are necessary for several other OEAS features.

In addition to these default task categories, however, the OEAS can support additional task categories that you configure as your organization requires. The additional task categories are supported by the same business objects as the default categories. As with the default categories, they are differentiated within the task object by the categoryId property.

Creating additional task categories in your application requires several steps, using different tools of the Onyx Enterprise Studio (OES). In addition, the user interface must be customized to use any new categories. This section outlines the middle tier implementation steps for creating additional incident and task types.

Tools you need to be familiar with include:

  • Table Administration, detailed in "Table Administration Help."
  • Reference Table Administration, detailed in "Reference Table Administration Help."

You also need to understand the following:

  • Only one task category is allowed for each incident category by setting a unique index on (task_category_did, site_id) on the task_category table.
  • Values of incident_category_did in the range of 1 to 99 are reserved for Onyx use.
  • Onyx recommends you assign your first user-defined incident category a value of 100 and its corresponding task category a value of 200, and increment subsequent user-defined categories from this starting point.

To create a user-defined task category:

  1. Create a new row in the Task_Category table for your new task category.

Use Table Administration to insert the new row. Assign the incidentCategoryId field the value you entered for the Incident Category primaryId for the corresponding new incident category you previously created.

The table lists Task Category fields displayed in Table Administration.

Field Description

categoryCode

Non-localizable name for the category. This is referenced by application code.

categoryName

Localizable name for the category.

incidentCategory

primaryId of the incident category.

catptionId

ID for caption from the NationalLanguage table.

toolTipId

ID for tool tip from the NationalLanguage table.

gridHeadingStart

ID for grid header from the NationalLanguage table.

description

Description of the category.

active

Specifies whether the field is active or inactive. 1 is active and 0 is inactive.

  • Optionally, you can insert new resolution code domain data into the ReferenceFields table for the new task category.
