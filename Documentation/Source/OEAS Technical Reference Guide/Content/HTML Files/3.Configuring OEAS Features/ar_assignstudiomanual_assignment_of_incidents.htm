Configuring the manual assignment of incidents

In the Object Designer, you can change the default behavior of the incident update process through the step additional data. The XPath statement outlined in red in the following graphic shows the executeCondition element. This element enables you to change the criteria used by the incident update method to call the OPM process for the manual assignment of incidents. For overview information, see Manual assignment of incidents.

The following example demonstrates how to change this behavior by configuring OEAS to call OPM when the status of an incident is set to "closed.” For this example “closed” is assumed to have a reference value of 151.

To change the criteria on which the incident update method calls OPM:

  1. Open the Object Designer and expand the incident business object.
  2. Under the incident update method, select the territoryAssociation step.

The step's properties appear in the edit area. In the Additional Data box, the executeCondition element determines how OPM is called during the incident update process. Changing this data involves writing any valid XPath statement. If the XPath statement returns a node, then the step executes and calls OPM.

In this example, the XPath statement is changed to call the OPM process when the status of an incident is "closed" (reference value of 151 in this example).

  1. Change the existing XPath statement to: executeCondition=parameters/incident[statusId=’151’]
  2. Click Save, Validate, and then Publish.
