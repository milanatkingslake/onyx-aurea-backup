Setting end-user permissions

For users of the Onyx Employee Portal (OEP), functionality of the Cognos Integration Module is governed by permissions that administrators set using Security Administration, an administration tool of Onyx Enterprise Studio (OES). The following table lists the module permissions administrators can set using Security Administration.

Security resource Description

Access to Cognos with Integrated Onyx Security

When using integrated Onyx authentication with Cognos, this resource enables users to access Cognos Connection directly, Cognos Framework Manager directly, or Cognos functionality within OEP. This is a gateway setting to the following UI resources. Security roles must also be enabled for the following UI-specific resources.

UI.OEP.reportsAnchorBar.view

Enable access to Onyx Reports (OEP integration of Cognos Connection portal).

UI.OEP.reportsHomePage.view

Enable access to the Reports section in Home Page.

UI.OEP.reportsPowerPage.view

Enable access to the Reports tab on the PowerPage.

UI:OEP.reportControl.saveSettings

Enable access to save-last-report-run and open-in-new-window OEP preferences.

UI:OEP.reports.runAuto

Enable users to run reports when an OEP page loads. If the report has a saved output, OEP loads the saved output instead. Otherwise, the user must select the report and choose to run it. Administrators should use this setting with discretion, as it has performance implications.

For performance reasons, no roles are granted access to UI:OEP.reports.runAuto during OEDB setup. With the exception of UI:OEP.reports.runAuto, OEDB setup populates Cognos Integration Module security resources with the following security roles. Members of these roles have access to the corresponding functionality.

  • Administrator
  • OEP.administrator
  • OEP.poweruser
  • OEP.user
