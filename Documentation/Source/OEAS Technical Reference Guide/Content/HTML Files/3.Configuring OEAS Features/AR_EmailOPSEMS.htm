Configuring email monitor subsystems to process inbound mail

Inbound email messages are sent to configured mailboxes on designated network mail servers where they are monitored (at predefined intervals) by email monitor subsystems (EMSs). EMSs run within the Onyx Processing Server (OPS) framework.

EMSs process and assimilate email into OEAS. You can configure them to monitor the following:

  • A corporate email account such as a generic sales, service, or support inbox.
  • An email account set up in conjunction with the Onyx Email for Outlook inbox. Onyx Email for Outlook is a plug-in for Microsoft Outlook with which users can save messages into the database from their personal Outlook inboxes and track new messages sent from their personal account in OEP. For more information, see the Email feature.

The EMS does not support Exchange Server users with identical aliases  If Active Directory has two or more users with an identical Exchange Server alias, OEP-Offline synchronization for the alias fails. However, EMS supports external aliases (contact aliases) that are identical to OEP-Offline user aliases.

OPS Administration is an administration tool that interacts with OPS and OEAS. You use this tool to configure and monitor the OPS service and EMSs. OPS Administration enables you to:

  • Add, configure, and delete EMSs and monitored mailboxes.
  • Start, stop, pause, or continue the OPS service and EMSs.
  • Activate and deactivate EMSs and monitored mailboxes.
