Configuring search tables

The search tables represent the database views against which the search criteria are evaluated. These views, and the columns they expose, correspond to the search tables/result set groups and search items/result set columns that exist in search view details. Search tables also include an alias for each view, which OEAS uses to create the WHERE clause that is appended to the underling database query associated with the search.

For instance, when a user creates a List Manager search for a Companies search view that includes search items from the search table Product, the search table items that OEAS uses to build the database query include the views company_lm_view and product_lm_view, using the aliases of “B” and “C” respectively for referencing columns in the query. This demonstrated in the following code sample.

SELECT DISTINCT

 '101'=B.COMPANY_ID,

 '102'=B.SECONDARY_ID,

 '116'=B.COMPANY_NAME,

 '105'=B.ADDRESS1,

 '106'=B.ADDRESS2,

 '107'=B.ADDRESS3,

 '108'=B.LOCALITY1,

 '109'=B.REGION_CODE,

 '110'=B.COUNTRY_CODE,

 '111'=B.POST_CODE,

 '103'=B.CONTACT_FIRST_NAME,

 '104'=B.CONTACT_LAST_NAME,

 '112'=B.PHONE_NUMBER,

 '113'=B.EMAIL_ADDRESS

FROM

 COMPANY_LM_VIEW B,

 PRODUCT_LM_VIEW C

WHERE

 (B.SITE_ID = 1) AND

 (B.company_id = C.owner_id)  AND

 (C.PRODUCT_CODE IN (N'newProduct2'))

Several search table items (views) are installed by default during installation of the OEDB, and you can administer them using Table Administration. Using this tool you can:

  • Add new search table items
  • Modify existing search table items
  • Set search table items to active or inactive
  • Delete existing search table items

Configuring items in search tables identifies the view name and alias to be used by List Manager; it does not configure the actual view. To configure views, see Configuring search view details.

 The following table describes the editable fields in Table Administration.

Element Description

primaryId

This is a unique system generated ID. Once created, it is not editable as it is a primary key field.

name (required)

This is the name of the database object (table or view) against which the query evaluates search criteria.

description

This is a description of the table or view.

alias (required)

This is the alias for the table or view. It is used to build the query.

active

Specifies whether the search table item is active or inactive: 1 is active, 0 is inactive.

For more information on using Table Administration, see Table Administration Help.
