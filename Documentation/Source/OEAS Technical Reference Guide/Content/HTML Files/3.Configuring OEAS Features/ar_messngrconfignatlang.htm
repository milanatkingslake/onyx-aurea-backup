Configuring end-user error messages and column headings

For the Messengers and Reminders features, you can configure end-user error messages and some column headings for data that is retrieved from the database. This can be useful in the following situations:

  • You customize an OEAS step in order to perform additional validation and require a message in case the validation fails.
  • A customization returns data that is not consistent with a default column heading.

These end-user error messages and column headings are maintained in the national language tables, and you can administer them using the Table Administration tool in Onyx Enterprise Studio.

For additional information on configuring error messages and column headings, see National Language.
