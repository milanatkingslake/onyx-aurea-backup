Configuring country data

Country data is used by the Customers feature for both the address object and the region object. Countries appear to end users in a list on the Company and Individual forms, and when the end users select a country to associate with a customer, the country they select determines the list of regions that they can also associate with the customer.

Country domain data is also used by the Literature Orders feature. For more information regarding this feature, see Literature Orders.

The country domain data is installed by default during installation of the OEDB, and includes the following information for each country:

  • The country name
  • A country code that is used as part of the unique identification for each country
  • A phone mask for each country
  • A zip (post) code map for each country

Using the Country table in Table Administration, you can administer the country domain data by:

  • Creating new countries
  • Modifying existing countries
  • Setting a country to active or inactive
  • Deleting a country

See Table Administration Help for further details.

The following table lists the editable fields displayed in Table Administration.

Element Description

countryCode (required)

Not editable, once it has been created, as it is a primary key field. A unique code assigned to each country.

countryName (required)

The name of each country that corresponds to the two-letter country code. All names entered at installation are standard but you can make changes to these names as necessary.

phoneMask

The phone mask used for each country. When you insert the country’s telephone code here it then appears in the phone mask for all customers from that country.

postCodeMask

The postcode mask used for each country, which also appears in the Post Code list on the customer page whenever a particular country is selected.

addressFormat

The specific address format required for mailings sent to the country. If no address format is known, you can leave this property NULL.

active (required)

Specifies whether the country data is active or inactive: 1 is active, 0 is inactive.

In the Country table, you can create field masks for phone_mask and post_code_mask. See Masking for further information.

There is no support for the Region table in Table Administration. This table must be accessed and maintained through a query tool such as Microsoft SQL Query Analyzer.
