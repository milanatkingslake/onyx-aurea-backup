Work Notes

For the Work Notes feature, you can configure end-user error messages and some column headings for data that is retrieved from the database. Doing so can be useful when you customize an OEAS step to perform additional validation and require a message when the validation fails, or when your customization returns data that is not consistent with a default column heading.

  • You customize an OEAS step in order to perform additional validation and require a message in case the validation fails.
  • A customization returns data that is not consistent with a default column heading.

These end-user error messages and column headings are maintained in the national language tables, and you can administer them using Table Administration.
