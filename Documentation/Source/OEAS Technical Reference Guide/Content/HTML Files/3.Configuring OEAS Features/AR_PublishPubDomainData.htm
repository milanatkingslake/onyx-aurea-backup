Publishing domain data

To publish an initial set of domain data after running the OEP-Offline client Deployment Server Setup program, see the OEP-Offline Implementation Guide. Once you have published that initial set of domain data, you can subsequently publish domain data on an as-needed basis, publishing only that domain data which has been updated. You can select which objects you want to publish using Subscription Designer, as described in "Publishing domain data."

Publishing details

When you modify domain data, you must re-publish it in order for OEP-Offline users to get the updated data. OEP-Offline users receive published domain data updates when they start OEP-Offline while connected to the network.

The process for publishing domain data includes running the CreateFileList.exe file which updates the OEPOfflineFileList.xml file.

Best practices for updating domain data include:

  • Making all necessary changes and then running CreateFileList.exe once to gather the changes into a single update, rather than running it for each change.
  • Not publishing domain data directly to the client files share. Instead, publishing the files to a temporary or staging directory, examining or testing the data, and then copying the files to the client files share.
  • Publishing domain data on a scheduled basis.

To publish domain data:

  1. Log on to the client deployment server with an account that has local administrative rights.
  2. In OES, open Subscription Designer.
  3. In the navigation area, click Publishable Objects.
  4. In the edit area, in the Publication Directory box, type the UNC or path of the directory in which to write the domain data files. The account that you specified in the OEAS Setup program’s Local Account dialog box must have read/write permissions for this directory.
  5. Click Publish Selected Data Now.

This process can take a little while.

  1. If the domain data files that you created are satisfactory, copy them to the client files share. The following path is the default location of the share:

C:\Program Files\Onyx\server_share_name\ClientFiles\Website\Application\domaindata

where server_share_name is the name of the share that you created in the procedure, “To install OEP-Offline client installation files on the client deployment server:” in the OEP-Offline Implementation Guide.

  1. Double-click the CreateFileList.exe shortcut to run the executable file.

If you have not previously created the CreateFileList.exe shortcut, see “Publishing domain data” in the OEP-Offline Implementation Guide.

The domain data is now published and ready for updating to OEP-Offline clients. When the client files directory is shared, users receive the new domain data the next time they start OEP-Offline while connected to the network.
