Configuring search views

Search views provide the starting point for building a List Manager search and control what type of objects are returned by the search. By default, search views include Companies and Individuals. Searches that users create using the Companies search view return company object instances and searches that users create using the Individuals search view return individual object instances.

Search view items of Companies and Individuals are installed by default during installation of the OEDB and are maintained in the search_view and search_view_ml tables. You can administer this data using Table Administration. Using this tool you can:

  • Add new search view items
  • Modify existing search view items
  • Set search view items to active or inactive
  • Delete existing search view items

The following table lists the editable fields for search views displayed in Table Administration.

Element Description

primaryId

This is a unique ID that you provide for the search view item. Once created, it is not editable as it is a primary key field.

name

The name of the search view as it appears to end users.

description

A description of the search view and its purpose.

objectType

This is the object type enum (object ID) that corresponds with the OEAS object type for the view. For a list of valid object type IDs, see “Object and table IDs” in the OEAS Developer’s Reference.

active

1 (active) or 0 (inactive). Inactive search views are not available in the UI.

For more information on using Table Administration, see Table Administration Help.
