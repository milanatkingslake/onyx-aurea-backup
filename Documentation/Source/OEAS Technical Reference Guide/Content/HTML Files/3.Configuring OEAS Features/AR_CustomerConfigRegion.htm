Configuring region data

Region data is used by the Customers feature for the region object. Regions appear to end users in a list on the Company and Individual forms, as determined by the country that the end user has selected.

Region domain data is also used by the Literature Orders feature. For more information regarding this feature, see Literature Orders.

The region domain data is installed by default during installation of the OEDB, and includes the following region specific information for each country:

  • The country code of the country to which the region belongs
  • The region code that is used as part of the unique identification for each region
  • The region name

Using the Region table in Table Administration, you can administer the region domain data by:

  • Creating a new region
  • Modifying existing regions
  • Setting a region to active or inactive
  • Deleting a region

See Table Administration Help for further details.

The following table lists the editable fields displayed in Table Administration.

Element Description

regionCode (required)

Not editable, once it has been created, as it is part of a primary key. A two to four-character code assigned to each region.

countryCode (required)

Not editable, once it has been created, as it is part of a primary key. A unique code assigned to each country.

regionName (required)

The name of each region that corresponds to the region code. All names entered at installation are standard but you can make changes to these names as necessary.

active (required)

Specifies whether the data is active or inactive: 1 is active and 0 is inactive.
