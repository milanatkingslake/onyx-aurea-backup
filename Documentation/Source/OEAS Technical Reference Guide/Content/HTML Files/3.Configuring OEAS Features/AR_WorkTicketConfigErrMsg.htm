Configuring error messages and column headings

For the Work Tickets feature, you can configure end-user error messages and some column headings for data that is retrieved from the database. This can be useful in the following situations:

  • You customize an OEAS step or a stored procedure in order to perform additional validation and require a message in case the validation fails.
  • A customization returns data that is not consistent with a default column heading.
