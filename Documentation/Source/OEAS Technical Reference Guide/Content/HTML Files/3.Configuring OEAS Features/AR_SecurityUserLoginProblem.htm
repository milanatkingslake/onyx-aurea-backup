Security troubleshooting

The following sections describe problems that might occur for users, and how you can solve them.

Membership in Windows roles

If you have problems with membership of roles after installation, correct the membership using Component Services in Windows 2000.

OEAS logical application cannot connect to OEDB

If at any point you restore the database from a backup, OEAS logical applications might not be able to connect to the OEDB. The cause is a known SQL Server problem, in which user logon information is not preserved when a database is restored from its backup. For prevention of or a solution to the problem, see article Q168001 (“PRB: User Logon and/or Permission Errors After Restoring Dump”) in the Microsoft Developers Network (MSDN) Knowledge Base.

User logon problems

Some causes of user logon problems are specific either to integrated Windows authentication or to Onyx authentication. Others are independent of the kind of user authentication being employed.

In addition to checking the items that are specific to either kind of user authentication, check the following:

  • If the AppName parameter or the SiteID parameter is specified in the URL on which the user logs on, make sure that they match the OEAS logical application name and the site ID in the Onyx Enterprise Studio Web site on which you have added the user. The system does not check these parameters.
  • If two or more Onyx Enterprise Application Server (OEAS) logical applications have been configured on one OEAS server, confirm that the OnyxEASApp component was shut down after the last configuration. If multiple OEAS logical applications are installed on one OEAS server, the OnyxEASApp component must be shut down before anyone can logon to the OEAS logical application.

If the user is trying to log on with integrated Windows authentication and gets the log on error message “Access Denied. Please contact your System Administrator,” then check the following:

  • Make sure the user is has been created correctly in Security Administration:
  • Verify that the Windows Integrated Authentication box is checked.
  • Verify that the domain account typed in the Domain Account box is a valid, existing account.
  • Check that the Windows account with which the user is logged in to his or her computer is the same as domain account that is typed in the Domain Account box in the edit area for the user.

If the user is trying to log on with Onyx authentication and gets the logon error message “Invalid User ID and/or password,” check the following:

  • Make sure the user is has been created correctly in Security Administration. In this case, doing so means verifying that the Onyx Authentication box is checked.
  • Verify the user is using the correct User ID and password to log on.
