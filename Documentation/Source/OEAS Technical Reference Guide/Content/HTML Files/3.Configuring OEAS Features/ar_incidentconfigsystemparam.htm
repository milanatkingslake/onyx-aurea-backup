Setting system parameters

There are several system parameters that you can use to modify the default behavior of the OEDB and OEP for the Incidents feature.

You can administer these system parameters using System Parameter Administration. Using this tool, you can set the parameter value that controls each system parameter. For details regarding System Parameter Administration and the system parameters of the Incidents feature, see System Parameter Administration Help.

The following table briefly describes these system parameters.

System Parameter Description

BlankSourceOnSalesEdit

Used to blank out the Source value on the Sales Incident window when a user is editing a sales incident. This can be used as a means of tracking the communication type for each incident update as it forces the user to enter an appropriate Source value before they can save the incident.

BlankSourceOnServiceEdit

Used to blank out the Source value on the Service Incident window when a user is editing a service incident. This can be used as a means of tracking the communication type for each incident update as it forces the user to enter an appropriate Source value before they can save the incident.

BlankSourceOnSupportEdit

Used to blank out the Source value on the Support Incident window when a user is editing a support incident. This can be used as a means of tracking the communication type for each incident update as it forces the user to enter an appropriate Source value before they can save the incident.

EnableSpellCheck

Enables spell checking for all features that support spell checking, including incident work notes.

OEPHistoryDuration

Specifies the number of days in the past, from the current date, for which OEP displays the history list.

TrackSalesTime

Determines whether or not time-tracking is enabled for sales Incidents. With time-tracking enabled, the system displays the Time Adjustment window every time the user saves an open sales Incident.

TrackServiceTime

Determines whether or not time-tracking is enabled for service Incidents. With time-tracking enabled, the system displays the Time Adjustment window every time the user saves an open service Incident.

TrackSupportTime

Determines whether or not time-tracking is enabled for support Incidents. With time-tracking enabled, the system displays the Time Adjustment window every time the user saves an open support Incident.
