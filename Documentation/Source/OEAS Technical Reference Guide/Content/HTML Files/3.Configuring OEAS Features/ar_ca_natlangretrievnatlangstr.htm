Retrieving National Language strings from a local resource DLL

Some Onyx components are implemented to retrieve strings from a local resource DLL if the attempts to retrieve the National Language strings from the OnyxNatLangCache fail. Each Onyx component references a unique resource DLL for this purpose. You can identify a resource DLL by its name. These resource DLLs have the characters Loc¬ù appended to the name of the supported Onyx component.

For example, foo.dll has a resource DLL called fooLoc.dll. These DLLs provide a default set of strings to a component when a call to the cache fails. If you create a custom component that uses INatLangCache, you might want to provide a default string resource in case the cache is unavailable.
