Configuring currency codes

Currency codes are used by the Forecasts and Quotes feature to establish forecasts and quotes in currencies other than the base currency of the products selected for the forecast or quote. End users can select the currency code for a quote or forecast from a list box in the user interface. Once selected, the currency code controls the display of international currencies, such as the pound or the yen. OEAS uses the selected currency code to determine the currency conversion values to use to convert the currency.

Within Product Administration, the currency types are also displayed in a drop-down list so that you can assign a base currency type to a product.

Currency code data is installed by default with installation of the Onyx Enterprise Database (OEDB), and includes the following information for each currency type:

  • The currency name
  • A currency code that is used as part of the unique identification for each currency
  • The locale ID, which is the ANSI standard locale ID for the currency type (the locale ID is used to format the currency type in association with the monetary value)

Within OEP, the currencies are displayed in a drop-down list on the properties page of a forecast or quote. If you have implemented secondary currencies for the Forecasts and Quotes feature, two drop-down lists are provided, one for the primary currency of forecast or quote, and one for the secondary currency.

Using the Currency table in Table Administration, you can administer currency codes by:

  • Creating new currency codes
  • Modifying existing currency codes
  • Setting a currency code to active or inactive
  • Deleting existing currency codes

See "Table Administration Help" for further details.

If you implement multiple currency support, and if you modify any of the currency codes, you must also modify the currency conversion data to reflect those changes. Specifically, the currency_code_from and currency_code_to fields in the currency_conversion table must match a value in the currency_code field in the currency table. Otherwise, certain currency conversions that you perform may not work properly. For more information about currency conversion data, see Configuring currency conversion data.

The Currency fields displayed by Table Administration are listed in the following table.

Field Description

currencyCode

Not editable (once it is created), because it is a primary key field. currencyCode is the ANSI standard currency code abbreviation for a particular currency. When adding a new currency, a unique currencyCode must be provided. This field’s data values are referenced in many other places in the application.

description

A currency description is provided for each currency type that is visible in areas of the application where end-users can select a currency type. The default data lists the chCurerncyCode abbreviation in parentheses with the description, but this is optional. This column is editable.

localeId

The value stored in chLocaleId is the ANSI standard code that represents the currency formatting that is applied to the monetary values of a particular currency type. This column is editable.

active

Indicates if a record is active or not active. If active is 1, the record is active and available throughout the application. If active is 0, the record remains in the table but is not available throughout the application.
