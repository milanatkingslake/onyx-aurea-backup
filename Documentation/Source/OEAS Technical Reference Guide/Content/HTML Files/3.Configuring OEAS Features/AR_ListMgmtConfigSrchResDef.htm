Configuring search result defaults

Search result defaults are the default columns returned in the result set for each search view. When users want to modify a result set, the default result set is their starting point.

The search result defaults for the default views of Company and Individual are provided with installation of OEDB and are maintained in the search_result_default table. You can administer search result defaults using Table Administration. Using this tool you can:

  • Add new search result defaults
  • Modify existing search result defaults
  • Set search result defaults to active or inactive
  • Delete existing search result defaults

The following table lists search result default elements displayed in Table Administration.

Element Description

primaryId

This is a unique system generated ID. Once created, it is not editable as it is a primary key field.

searchViewDetailId (required)

This ID value must correspond to a primaryId value that exists for a search view detail item.

searchViewId

This ID value is the denormalized searchViewID from searchViewDetails that corresponds to the selected searchViewDetailId. It is not editable.

searchTableId

This ID value is the denormalized searchTableId from searchViewDetails that corresponds to the selected searchViewDetailId. It is not editable.

searchViewDetailFieldName

This value is the denormalized fieldName from searchViewDetails that corresponds to the selected searchViewDetailId. It is not editable.

required (required)

Indicates whether or not the field is required in the result set.

0 - not required

1 - required

sequenceNumber (required)

Indicates the order of the column in the default result set.

active (required)

Specifies whether the search result default is active or inactive.

1 - active

0 - inactive.
