Configuring National Language data

The National Language table of the OEDB stores all of the captions and message strings that are provided at install or through upgrades for OEAS. This table is administered in Table Administration and provides a central location for you to perform data string maintenance and localization for the end-user interface. Following are the types of modifications you can make in the National Language table.

  • Create new error messages and column headings
  • Modify existing error messages and column headings
  • Set error messages and column headings to active or inactive
  • Delete error messages and column headings

Always use Table Administration to modify national language strings in the database. If you make any direct updates to the National Language table through a Query Analyzer tool, all modifications will be lost during an upgrade and the OnyxNatLang cache will not load properly. See Refreshing the cache for further details.

Do not customize ID generation for national language data strings. IDs created with a value less than 1,000,000 are lost during system upgrades.

Where are custom National Language strings stored?

When you customize (add, modify, or delete) strings in the National Language table for your particular installation of OEAS, your customized strings are stored in the nat_lang_config table. Having two tables eases software upgrades because the message strings in the National Language table are updated with the new software and your customized strings remain untouched in the nat_lang_config table.
