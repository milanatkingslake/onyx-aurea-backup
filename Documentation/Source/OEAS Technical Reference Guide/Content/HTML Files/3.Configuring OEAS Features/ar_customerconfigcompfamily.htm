Creating company families

You can create company families so that end users can logically group companies. For example, if companies B and C are actually subsidiaries of company A, you can create a company family record that represents company A, and end users can then associate companies B and C to that company family.

Using the CompanyFamily table in Table Administration, you can create and administer company family data by:

  • Creating new company families
  • Modifying existing company families
  • Setting a company family to active or inactive
  • Deleting a company family

See Table Administration Help for further details.

The following table lists editable CompanyFamily fields displayed by Table Administration.

Element Description

familyCode (required)

A 1-10 character code must be created for each company family. Once this code is generated, you can edit it but you must then update all records in the application that use this code.

familyName (required)

The full name of the company family. This is the value that appears in user interface.

familyDetail

Text description of this company family.

onyxTimestamp

This is a system-generated property that is updated automatically when new data is created and when modifications are saved. The onyxTimestamp property enables optimistic concurrency control within the Onyx Enterprise Database. See the onyxTimestamp property for further details.

active

Specifies whether the data is active or inactive: 1 is active, 0 is inactive.
