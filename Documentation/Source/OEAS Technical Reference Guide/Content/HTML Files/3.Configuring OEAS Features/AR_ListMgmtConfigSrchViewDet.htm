Configuring search view details

Search view details represent the search criteria/result set groups and search items/result set columns that users can select to create their searches and to customize the default result set of a search view. Search view details also define the result set column headings.

Search view detail data are installed by default during installation of the OEDB and are maintained in the search_view_detail table. You can administer this data using Table Administration. Using this tool you can:

  • Add new search view details data
  • Modify existing search view details data
  • Set search view details data to active or inactive
  • Delete existing search view details data

The following table lists the editable search_view_detail fields that are displayed in Table Administration.

Element

Description

primaryId

This is a unique system generated ID. Once created, it is not editable as it is a primary key field.

searchViewId (required)

This ID value must correspond to a primaryId value that exists for a search views item.

searchTableId (required)

This ID value must correspond to a primaryId value that exists for a search table item.

searchTableName

Indicates the view against which the search table or search item is evaluated. This value is not editable. It is dependent on the value of searchTableId.

searchTableAlias

Indicates the alias for the view against which the search table or search item is evaluated. This value is not editable. It is dependent on the value of searchTableId.

name

The name of the search criteria/result set group or search item/result set column as it appears in the user interface.

description

Description of the search criteria/result set group or search item/result set column.

level (required)

Determines the nested level of a search view details item.

1 - a search criteria/result set group

2 - a search item/result set column

sequenceNumber (required)

Indicates the order in which the search view details items appear in the user interface. The sequence number is global for all search view details items, so you must assign search items/result set columns a sequence number that places them sequentially after the sequence number for the appropriate search criteria/result set group.

nodeType (required)

Designates a search view detail item as a search criteria/result set group or a search item/result set column.

TABLE - search criteria/result set group

FIELD - search item/result set column

fieldName

For search view details items with a nodeType of “FIELD”, this is the database field name associated with the search item/result set column. For search view details items with a nodeType of “TABLE”, this is the table or view name as defined in search tables.

fieldType

Controls which SQL operators are available to end users when they select a particular search item. For a list of the valid values and which SQL operators are available for a particular fieldType, see Valid field types for search view detail items.

joinCondition

The join string that enables a user to query on search tables (or views) that are related to the “anchor” view. This can be “(empty)” or it can store an SQL join. A “~~” denotes the “anchor” view. The double tilde passes the local iSiteId as a starting point for the join. A join string is added to the query to join additional views.

searchConditionFunctionCall

The database function applied to the value that is being returned from the database. The format of searchConditionFunctionCall is function(~0). The UI replaces “~0” with the fieldName value in the “column alias “.” Field name” format when constructing the query.

userInputFunctionCall

This defines any function that must be applied to user-entered values to ensure accurate comparisons of those values against data in the database. For instance, dates entered by a user must be converted from text to a date value, so that the date can be accurately compared to date values in the database. For this reason, “onyx_utility.text_to_date(~0) is supplied for the userInputFunctionCall element of any search view details item that represents a date field. In this format, “~0” is replaced with the user-entered value.

searchAllowed (required)

Indicates whether or not the field is included in the search criteria tree as a search table or item.

0 - not included

1 - included

indexed (required)

Indicates whether or not a search item is indexed.

0 - not indexed

1 - indexed.

Search view details items with a nodeType of “TABLE” should have a indexed value of 0.

resultAllowed (required)

Indicates whether or not the field is included in the result set tree as a result set group or column.

0 - not included

1 - included

searchResultDefaultRequired

Indicates whether or not the field is required in the result set.

0 - not required

1 - required

searchResultDefaultRequired is not editable in the Table Administration Search View Details filter. It is editable in the Table Administration Search Result Default filter.

resultVisible

Indicates whether or not the field will be visible in the result set.

0 - not visible

1 - visible

resultFunctionCall

The function applied to the field when it is being returned in a result set. The format of resultFunctionCall is function(~0). The UI replaces “~0” with the fieldName value in the “column alias “.” Field name” format when constructing the query.

resultType

resultType defines the field type. Default valid values for resultType include:

  • companyPrimaryId
  • individualPrimaryId

The UI uses this value to enable a link to another UI page from a column result value. For instance, a search for companies returns a list of companies with a column for the primary ID, where the UI renders each primary ID as a hyperlink to the company detail page. When the user clicks the hyperlink, the UI recognizes the companyPrimaryId result type and returns the company information with the corresponding primary ID. To customize this field, you must configure the UI to recognize the new result type and define a function in the UI to handle that result type.

enumerationObject

For search view details items that have a fieldType of ENUM_I or ENUM_S, an OTM call is required to return the valid values for a search item. The enumerationObject, enumerationMethod, and enumerationParametersXML are used to create this OTM call.

enumerationObject: This is the object name for the OTM call.

enumerationMethod: This is the method name for the OTM call. It must be a valid method of the object named for the enumerationObject.

enumerationParametersXML: This is the parameters XML that is passed for the OTM call.

enumerationMethod

enumerationParametersXML

enumerationItemPath

For search view details items that have a fieldType of ENUM_I or ENUM_S, an OTM call is made and returns XML containing the valid values for a search item. The enumerationItemPath, enumerationIDPath, and enumerationDescriptionPath are used to extract from the return XML the values to display in the user interface.

enumerationItemPath: This is the path in the returned XML that leads down to the level where the search item values are located. For example: //parametersReturn/literatureOrderLineStatuses/literatureOrderLineStatus.

enumerationIDPath: This is the path for the search item value under each returned search item. Using the same example: literatureOrderLineStatusCode.

enumerationDescriptionPath: This is the path for the search item description under each returned search item. Using the same example: description.

enumerationIDPath

enumerationDescriptionPath

active

Specifies whether the search view details item is active or inactive.

1 - active

0 - inactive.
