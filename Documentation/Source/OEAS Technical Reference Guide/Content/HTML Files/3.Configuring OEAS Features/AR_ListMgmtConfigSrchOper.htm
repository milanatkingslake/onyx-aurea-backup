Configuring search operators

When users select a search item to include in their List Manager query, a list of SQL operators that they can apply to that search item are displayed. Default search operators are installed with OEDB and include:

  • operator symbols, such as:
  • = (equals)
  • <> (not equal to)
  • > (greater than)
  • <= (less than or equal to)
  • string operators, such as BETWEEN, LIKE, IN, NOT IN, IS NULL and IS NOT NULL.

The search view detail controls which search items support which SQL operators. For additional information, see Configuring search view details.

The search operators are maintained in the search_operator and search_operator_ml tables. You can configure search operators using Table Administration. With this tool, you can:

  • Modify the description of a search operator
  • Set a search operator to active or inactive
  • Delete existing search operators

Support for configuring search operators includes deleting, activating, and deactivating the default search operators, and modifying the description element of a search operator. Configuring new search operator values, or modifying the default search operator values, is not supported.

The following table lists search operator elements displayed by Table Administration.

Element Description

primaryId

This is a unique system generated ID. Once created, it is not editable as it is a primary key field.

value (required)

This is the actual SQL operator that is used in the query code. For example, “>”, “<>”, “in”, and “between”.

description

This is the description of the SQL operator. For instance, for the SQL operator “<>”, the description is “not equal to”. The description appears to end users with the SQL operator in the following manner: not equal to (<>).

includeNulls

Specifies whether the search operator will include NULL values in comparisons: 1 is true, 0 is false.

active

Specifies whether the search operator is active or inactive: 1 is active, 0 is inactive.

For more information about using Table Administration, see Table Administration Help.
