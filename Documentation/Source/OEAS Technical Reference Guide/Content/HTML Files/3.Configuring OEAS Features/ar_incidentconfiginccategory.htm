Creating additional incident categories

By default, OEAS supports incident categories for work ticket, sales, service, and support, as described in Incidents. These incident categories are installed by default when you install OEDB and are differentiated by the ID values listed in the following table.

Default incident categories Default incident_category_did

support

1

service

2

sales

3

workTicket

4

Do not change these default category IDs. They are necessary for several other OEAS features.

In addition to the default incident categories, OEAS can support additional incident categories that you configure as your organization requires. The additional incident categories are supported by the same business objects as the default categories. As with the default categories, they are differentiated within the incident object by the categoryId property.

Creating additional incident categories in your application requires several steps, using different tools of the Onyx Enterprise Studio (OES). This section outlines the implementation steps for creating additional incident categories in the middle tier business logic. You must also customize the presentation tier (OEP) to display the new functionality. For more information on OEP customization, see the OEP Technical Guide.

Tools you need to be familiar with include:

  • Table Administration, detailed in "Table Administration Help"
  • Reference Table Administration, detailed in "Reference Table Administration Help"

To create a new incident category:

  1. Using Table Administration, add a row to the incident_category table for the new incident category.

The following table lists the editable fields displayed in Table Administration.

Field Description

categoryCode

Non-localizable name in a single string with no spaces. This is referenced by application code.

categoryName

Localizable name displayed in the UI

captionId

ID for caption from the NationalLanguage table

toolTipId

ID for tool tip from the NationalLanguage table

gridHeadingStart

ID for grid header from the NationalLanguage table

description

Description of the category

active

Specifies whether the field is active or inactive. 1 is active and 0 is inactive
  • Using Reference Table Administration, create new reference fields for the incident category resolution codes. In order for this category to work the Incident feature, the Field Name must use the following format:

incident.<category>.code1

You replace <category> in the previous syntax with the categoryCode you configured in the previous step. Create resolution codes in the same manner as the other incident categories, in order that the Incident feature can recognize them. To create the hierarchical relationship, set the Field Type to Combo Box and specify the Parent Field according to the following table.

You can assign any Caption according to your business requirements. Set Sequence # and Active to 1.

  • Using the Lookup Value by Field option, specify lookup values for incident.resolutiongroup under the new the new category.
  • Using Reference Table Administration, add values to the following for the new incident category:
  • incident.priority
