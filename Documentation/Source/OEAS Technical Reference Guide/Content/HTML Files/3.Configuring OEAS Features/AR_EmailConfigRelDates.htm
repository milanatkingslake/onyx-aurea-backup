Configuring relative date ranges for the Email feature

In Task Manager, relative date ranges enable users to filter searches for emails by a number of days, weeks, or months before the current date. You can configure relative dates to calculate the date ranges used to filter searches for emails in the end-user interface. For more information about filtering by relative dates, see the Task Manager topics in OEP Help.

Relative dates are stored in the relative_date table and are administered in Table Administration. For procedural information, see Table Administration Help.

The following table lists the editable fields displayed in Table Administration.

Field Description

description

Type a description of the relative date. For example:

Today

Yesterday

Last 7 days

Last Month

Last 3 Months

Last Year

duration

Type an integer value to represent time. The duration unit (see next row of this table) defines the unit of time, as shown in the following example.

Today - duration 0

Yesterday - duration 1

Last 7 days - duration 7

Last Month - duration 1

Last 3 Months - duration 3

Last Year - duration 1

durationUnit

Set to one of the following values:

Y- year

M - month

D - day

sequence

Use to sort a list of relative dates.

truncateToSingleDay

Set to one of the following values:

1 - single day search (yesterday or today)

0 - not a single day search

objectType

Type the object type for which this row should be used.

active

Set to 1.
