Modifying message strings in the National Language table

Before you can view or alter the data in the National Language table, you must select a filter from the Filter list. The extensive number of National Language strings in OEAS requires you to specify the area of the application affected using the Filter list. For example, if you want to modify an error message for the Forecasts & Quotes feature, you must select the appropriate filter, EAS - Forecast/Quote.

If your customization returns additional columns, thus requiring additional column headings, ensure that the column headings you configure in Table Administration match (in order) the column names indicated in the OED for the rowSet parameter of the affected method. For additional information about customizing the OED, see Understanding the OED.

The National Language Category table controls what filters you can use within Table Administration for the National Language domain data. In the National Language Category table, the values you create for the description property become the filters available in the Filters list for National Language domain data.

The following table describes National Language data displayed in Table Administration.

Property Description

messageId

Not editable as it is a primary key field. The unique identification number that identifies each string of national language domain data.

languageCode

The language code for the NationalLanguage record.

When you add custom strings to the National Language table, be sure to include the language code representing the language of your custom strings. If the language code does not exist in the database, your custom strings will not be supported in the cache. See Refreshing the cache for further details.

categoryId

Correlates to the primaryId of the national language category. It is automatically set to the category for the current filter. It is not editable.

categoryDescription

Correlates to the description of the national language category. It is not editable.

message

Contains the text for the message.

configured

Indicates whether the string is an Onyx default string or a user-configured string: 1 is configured and 0 is default. It is not editable.

active

Specifies whether the data is active or inactive: 1 is active and 0 is inactive.

The following table describes National Language category displayed in Table Administration.

Property Description

primaryId

Not editable as it is a primary key field. The unique identification number that identifies each string of national language category domain data.

description

Brief description of the national language category. This is the value that appears in the Filters list for National Language domain data in the Table Administration.

active

Specifies whether the data is active or inactive: 1 is active and 0 is inactive.
