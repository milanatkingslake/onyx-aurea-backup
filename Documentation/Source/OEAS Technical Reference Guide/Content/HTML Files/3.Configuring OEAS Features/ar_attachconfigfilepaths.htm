Configuring file paths for attachments

The file_location table stores information about the file shares that are responsible for storing saved files. Every record in this table represents a path to a defined file share. Using Table Administration, you can configure (add, update, or delete) file path data in the file_location table. See Table Administration Help.

In Table Administration, you can configure additional file paths, however, in this version of OEAS only the first active path entered will be used to determine the file™s storage location. Additional paths can be made active only through customization.

The following table lists the editable fields displayed in Table Administration.

Field Description

description

User friendly name for the file share

pathName

Fully qualified UNC file path

The first active file path entered in the table is used to determine the file storage location. Additional paths can be made active only through customization.

active

Specifies whether the record is active or inactive: 1 is active, 0 is inactive.
