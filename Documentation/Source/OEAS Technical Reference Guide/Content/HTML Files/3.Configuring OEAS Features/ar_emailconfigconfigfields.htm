Configuring user-defined fields for the Email feature

You can configure a number of user-defined fields of the Email feature to meet the specific requirements of your business.

There are no standard configurable fields for the Email feature.

To administer user-defined fields for the Email feature, you use Reference Table Administration in OES. Using this tool, you can:

  • Define the field type as text box, check box, or combo box.
  • Modify the field caption that appears on the user interface.
  • Create, delete, and modify lookup values for the field types that you have defined as combo boxes.

For procedural information, see Reference Table Administration Help.

The objects and associated elements that support user-defined fields for the Email feature are listed in the following tables. By default, OEAS includes only the logical properties that support user-defined fields. To use these fields, a developer must customize the physical model, create the logical to physical mappings in the middle tier, and customize the presentation layer. For more information on customizing OEAS, see Customizing OEAS. For information on customizing the presentation layer, see the OEP Technical Guide.
