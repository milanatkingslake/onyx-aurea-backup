Setting system parameters for the Email feature

You can use system parameters to modify the default behavior of the database and the end-user interface. Following are descriptions of the system parameters that you can configure for the Email feature in System Parameter Administration. For more information, see System Parameter Administration Help.

System Parameter Description

EmailSendStepSmtpMode

This system parameter specifies whether the SMTP service used by the OnyxEmailSendStep is co-located on the OEAS server or on a remote server. Valid values are: local or remote.

EmailSendStepSmtpServer

When the OEAS server is not local to the SMTP server, this system parameter specifies the DNS name or IP address of the SMTP server.

EmailCreateFileLinkforLit

This system parameter determines whether to create a fileLink record when an OEP user attaches a literature item to an email message.

For security reasons, after installing OEAS you need to manually configure it if you want to use the OnxyEmailSendStep (also known as OEP Email). You should either:

  • Reconfigure the Email feature system parameters to use a remote SMTP server (EmailSendStepSmtpMode and EmailSendStepSmtpServer)

or

  • Run a Microsoft script to grant the OnyxEASUser access to read the IIS Metabase so that OEP Email can use the local SMTP server.

    The command to do this is:

    cscript adadd.vbs computerName\OnyxEASUser

You can obtain the "adadd.vbs" script by downloading it from http://support.microsoft.com/kb/816789
