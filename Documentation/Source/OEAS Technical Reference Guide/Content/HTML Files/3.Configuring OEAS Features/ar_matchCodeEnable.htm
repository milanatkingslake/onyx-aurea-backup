Enabling match code functionality

Enabling match code functionality involves changing the Disabled status of post procedures for relevant logical methods in OED. The match code functionality includes a set of database stored procedures and functions that produce match code values for companies and individuals. These database procedures are included as disabled post procedures on the core step of relevant logical methods in OED. To enable the match code feature, clear the disabled checkbox for each of these post procedures and publish the OED through Object Designer. For more information on using Object Designer and publishing OED, see Using the Object Designer. Note that enabling match code functionality only creates the match codes for customer records. To programmatically prevent insertion of duplicate records or programmatically merge duplicate records requires further customization of the database layer. For information on creating custom database procedures, see Creating and customizing procedures.

When the match code functionality is enabled, a row is added to the match_code_value table for every call to the methods listed in the previous table. If you want to create match code values for all customers, where none existed before, or if you want to re-create match codes based on a new match code formula, you can run the MatchCodeCreation utility script provided by Onyx.
