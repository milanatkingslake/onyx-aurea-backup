Setting system parameters for the Cognos Integration Module

You can use system parameters to modify the behavior of the OEP integration with Cognos functionality. Following are descriptions of the system parameters that you can configure for the Cognos Integration Module in System Parameter Administration.

System parameter Description

EnableAnalytics

Controls whether OEP exposes any ReportNet functionality. The default value is N.

ReportNetHomePageFolderName

Determines the name of the ReportNet folder created by OEP for each user. The default value is blank. If this is left blank, OEP assumes the private Home Page folder functionality is disabled. OEP still renders administrator-configured reports.

Example: “Home Page reports”

ReportNetHomePageFolderPath

Controls the ReportNet content store location for the OEP Home Page folder. The default value is blank. If this is left blank, OEP assumes the private Home Page folder functionality is disabled. OEP still renders administrator-configured reports.

Example: CAMID("OnyxAuthentication:?userId?")/folder[@name='My Folders']

When specifying the path, you must include the ‘?userId?’ syntax. OEP replaces this string with the currently logged-on user’s ID at run-time.

ReportNetOnyxNamespace

Controls which security namespace OEP uses to access ReportNet resources on behalf of an OEP user. This contains the name of the Onyx authentication namespace created during implementation of the ReportNet Integration Module.

The default value is ‘OnyxAuthentication.’ If EnableAnalytics=’Y’ and this is blank, the user receives a warning that reports cannot be retrieved when attempting to access reports from OEP Home Page and PowerPage.

ReportNetURL

Controls which ReportNet application OEP uses for accessing reporting functionality.

The default value is ‘http://serverName/crn.’

If EnableAnalytics=’Y’ and ‘serverName’ does not reference a valid ReportNet server, the user sees a Page Not Found error when attempting to access ReportNet from OEP.
