Configuring configurable fields

Configuring fields

You can configure a number of fields of the Tasks feature to meet the specific requirements of your organization. Configuring these fields can include modifying field captions; setting field types as a text box, check box, or combo box; and creating lookup values for fields. For instance, in the Tasks feature, you can define the lookup values associated with the Status field as it appears in Onyx Employee Portal (OEP), or you can configure a field for the task.user1 property for sales tasks, defining it as combo box, check box, or text box.

Configurable fields for tasks are supported by the Reference Data feature, as described in Reference Data, and are defined as either standard configurable fields or user-defined fields. Whether a field is a standard configurable field or user-defined field determines the extent to which you can configure the field.

You can administer configurable fields with Reference Table Administration. Using this tool, you can do the following.

For standard configurable fields

  • Create, delete, and modify lookup values.

For user-defined fields

  • Define the field type as text box, check box, or combo box.
  • Modify the field caption that appears on the user-interface.
  • Create, delete, and modify lookup values for the field types that you have defined as combo boxes.

For additional information about administering configurable fields, see Reference Table Administration Help.

The default values for “category” are sales, service, and support .
