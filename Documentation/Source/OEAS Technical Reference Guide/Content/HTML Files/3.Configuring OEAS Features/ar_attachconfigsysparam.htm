Setting system parameters for attachments

The following system parameters modify the default behavior of OEDB and OEP for the Attachments feature. The following table briefly describes these system parameters. For more information, see System Parameter Administration Help.

System Parameter Description

FileAgentProgID

COM+ program ID for the OnyxFileAgent component. OnyxFileAgent (OFA) setup configures this value.

OEPAltUploadServer_Onyx

The name of the alternate server that has been set up to handle uploading/downloading of attachment files when Onyx authentication is in use. The parameter value must be a complete URL to the virtual directory for Onyx authentication (for example, http://servername/oep-dir). Leave the parameter value empty if you are not using an alternate upload/download server for Onyx authentication.

OEPAltUploadServer_Windows

The name of the alternate server that has been set up to handle uploading/downloading of attachment files when Windows authentication is in use. The parameter value must be a complete URL to the virtual directory for Windows authentication (for example, http://servername/oep-dir). Leave the parameter value empty if you are not using an alternate upload/download server for Windows authentication.

OEPMaximumDocumentSizeKB

Limits the size of files that can be uploaded to the file share. This is the system-wide maximum size (in kilobytes) of a file that can be uploaded.

EmailCreateFileLinkforLit

This system parameter is used by the Email feature to determine whether or not to create a fileLink record when an OEP user attaches a literature item to an email message.
