Creating template groups for templates

In Template Maintenance in the Onyx Employee Portal (OEP), templates are organized in a hierarchy of template groups to provide logical groupings of templates to end users.

Template groups are stored in the merge_template_group table of the Onyx Enterprise Database (OEDB). Using Table Administration, an Onyx Enterprise Studio (OES) administration tool, you can administer template groups by:

  • Creating template groups and specifying group hierarchies.
  • Modifying template groups.
  • Deleting template groups.

For procedural information on using Table Administration, see Table Administration Help.

The merge_template_group fields displayed by Table Administration are listed in the following table.

Field Description

merge_template_group_did

Not editable, because it is a primary key field. This property is a unique, application-defined identification number given to each template group.

merge_template_group_name

The name you provide for the template group.

merge_template_group_desc

The description you provide for the template group.

prnt_merge_template_group_did

The ID of the template group that is the parent of this group. If you do not want to specify a parent template group, leave the field empty.

record_status

Whether the template group is active or inactive: 1 is active, 0 is inactive.

Once a template group has children (you have made the template group a parent to another template group or, using Template Maintenance in OEP, you have added templates to the group), you cannot delete it but only make it inactive. When you make a template group inactive, that template group, any child groups, and any templates that are part of those groups are no longer displayed in Template Maintenance.
