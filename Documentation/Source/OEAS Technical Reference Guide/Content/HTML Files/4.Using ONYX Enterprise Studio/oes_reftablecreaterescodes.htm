Creating resolution codes

Resolution code fields are hierarchically-related fields that capture information about how incidents are resolved (closed) and enable you to measure, report on, and analyze the effectiveness of your sales, service and support processes.

The four resolution code fields are combo box fields that are hierarchically related to one another. The fields use the naming convention incident.category.coden, where n represents a numeric value 1, 2, 3, or 4. For example, the first resolution code field for a sales incident is incident.sales.code1.

The four resolution code fields are grouped together under a parent field called the resolution group field (incident.resolutiongroup). The resolution group field only displays in Reference Table Administration and Product Administration. It does not appear in the user interface. It is used for grouping the four hierarchically related resolution code combo box fields and for associating resolution groups with incident products.

Before you can define lookup values for the resolution code fields, you must first create the resolution group name. Then you can associate the resolution code field lookup values with the resolution group name.

It is important to note that resolution codes are associated with specific incident products, and incident products in turn are associated with specific incident categories. For instance, for the sales incident category, you might have specific sales incident products defined. For a specific sales incident product, you can associate a specific resolution group. This enables you to track how incidents involving specific incident products are resolved.

For example, using resolution codes, you can track the number of sales incidents that your sales team has closed by incident product. You can also track how many of those sales incidents resulted in a win, how many resulted in a loss. Of the wins, how many were due to price, how many to features, how many to reputation, etc. Conversely, you can track and report on how many sales incidents for a specific product resulted in a loss. Why did you lose (price, features, reputation, etc.)?

To create resolution codes:

  1. In the navigation area, select the Lookup Value by Fields mode option to display the available nodes.
  2. Click , or double-click the incident node, to display the incident.category field.
  3. Click in front of the incident.category field, or double-click the incident.category field to display the incident category fields (Sales, Product Support, Customer Service, and Work Ticket).
  4. Click in front of an incident category (such as Sales) or double-click the field in order to view the standard sales incident fields.
  5. Select the incident.resolutiongroup field.
  6. Click and type the new incident resolution group name in the Lookup Value box.
  7. Click to save the resolution group. The new incident resolution group appears in the navigation area with a in front of it.
  8. Click in front of the new incident resolution group to view the resolution code fields for that group.
  9. Select the incident.<category>.code1 field where <category> is sales, service, support, or WorkTicket.
  10. Click and type the new incident.<category>.code1 field lookup value in the Lookup Value box.
  11. Repeat this procedure and add all of the lookup values for the first resolution code field.
  12. Click to save your lookup values.
  13. In the navigation area, under the same incident.<category>.code1 field, click in front of one of the new lookup values to display the new incident.<category>.code2 field.
  14. Select the incident.category.code2 field.
  15. Click and type the new incident.<category>.code2 field lookup value in the Lookup Value box. Repeat this procedure and add all of the lookup values for the second resolution code field.
  16. Click to save your lookup values.
  17. Repeat steps 8-13 to add the lookup values for the two remaining resolution code fields.

Once you have defined your resolution group and the associated resolution code values, you must associate the resolution group with an incident product. See Product Administration Help for instructions on associating resolution groups with incident products.
