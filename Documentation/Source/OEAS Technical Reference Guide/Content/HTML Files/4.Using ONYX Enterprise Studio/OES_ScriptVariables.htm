Working with variables

You must define a variable before it appears in the Edit Step area, although not before using it in a rule or in text merging. You can use either of two Web Page Dialogs to define a variable:

  • Variables Web Page Dialog
  • Method Call Binding Builder Web Page Dialog

The Method Call Binding Builder is recommended for creating variables that you will use with method calls. For information on using the Method Call Binding Builder, see Working with method calls.

Part of the definition of a variable is its scope. For script variables, “scope” refers to whether it is available to a single step (“step-local” scope) within a script or to all the steps in the script (“global” scope).

No two variables in the same script can have the same name, even if the scope of one is step-local and of the other is global.

If you rename a variable, you must manually change each occurrence of it in the script, whether in a rule, header, footer, prompt, or method call binding.

Context variables are created automatically, and cannot be defined, renamed, or deleted. For more information on context variables, including their names, see Variables.
