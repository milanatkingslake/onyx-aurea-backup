Making the script accessible to users

Users can only access and start a script session as long as the following conditions are satisfied:

  • If you have defined dates of availability, the current date falls between the script’s effective date and the expiration date.
  • The script is published and active.

If an active script is not published, only users who have access to the UI:OEP.scripting.designer resource can start a script session with it.

If a script is not active, no one can start a script session with it.

When you modify a script, you must refresh the script in OEAS to make the changes available to the end-user application’s script cache.

To make a script available to users:

  1. In the Edit Script area:
  • To limit the time during which the script is available to users, type dates in the Effective Date and Expiration Date boxes. A script session that is begun within the range of availability remains available after the expiration date passes, until the session is completed.
  • Select the Published and Active check boxes.
  • Under Owner Types, select at least one of Company, Individual, and Incident, depending on which objects you want to be able to own the script session.
  1. Click to save your changes to the database.
  2. On the toolbar, click to refresh the OEAS cache of scripts.

OR

To refresh just the current script in the OEAS cache, click on the toolbar.
