Naming user-defined fields

When you define a user-defined field that does not already exist in Reference Table Administration, the name that you provide for that field must follow a specific naming convention if that field is a field supported by OEP by default. The naming convention followed is <element>.<fieldname><n>. The following descriptions explain the components of the naming convention syntax:

  • element represents the logical grouping in which the field appears in Reference Table Administration.
  • fieldname represents the field name equivalent for the user-defined property for which you are creating the field.
  • n represents a numerical value corresponding to the user-defined property for which you are creating the field.

Do not edit the names of user-defined fields that are defined by default in Reference Table Administration. Editing these field names can break compatibility with OEAS and OEP.

The number of user-defined fields is fixed based on whether or not the user-defined property exists on an object. For a list of the user-defined properties available in the Onyx Enterprise Dictionary (OED), see User-defined fields. Specific naming conventions are detailed in , within each “feature” section.
