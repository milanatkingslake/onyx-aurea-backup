Quick overview of the Security Administration tool

The user interface of Security Administration is divided into four areas:

  • Context area
  • Toolbar
  • Navigation area
  • Edit area

Context area - The context area lists the name of the application and the site being maintained.

Toolbar - The toolbar appears under the context area and contains the toolbar buttons that perform the tasks related to security.

Icon Tool tip Function Context

Add

Used to add a new group, user, subscription profile, role, or resource depending on which is selected in the navigation area. This button is disabled when the Business Objects node under Resources is selected.

Enabled only when a Group, User (or its Subscription Profiles node), Role, Other Resource, or UI Resource is selected.

Delete

Used to delete a subscription profile, role, Other Resource, UI Resource, and users and groups that have not yet been saved. The item deleted depends on which item you selected in the navigation area.

Enabled when you:

  • Select a newly created user or group, role, Other or UI resource that has not yet been saved.
  • Select an existing role or subscription profile or a resource that is listed under UI Resources, or Other Resources.

Clone

Used to create a user with the same group and role memberships as the user that is selected when you create the new user.

Present on the toolbar (and enabled) only when you select a user. Groups, roles, and resources cannot be cloned.

Preferences

Used to change preferences for Security Administration. Currently, the only preference that you can change is the default password.

Always enabled.

Save

Used to save to the database all the changes you have made to groups, users, roles, and resources since you opened Security Administration or last saved changes.

Enabled after you make a change in Security Administration.

Help

Launches Help for Security Administration.

Always enabled.

Navigation area - The navigation area presents a hierarchical structure of groups, roles, and resources. Users reside within groups. Each user contains a Subscription Profiles node.

Edit area - The edit area enables you to change the attributes for groups, users and their subscription profiles, roles, and resources. The fields that appear in the edit area change according to whether a user, group, role, or resource has been selected in the navigation area.
