Configuring a region prompt

For region prompts, you can specify the following:

  • Whether the prompt appears as a drop-down list box or a plain list box.
  • Minimum and maximum number of responses that the user can select when the prompt is a list box.
  • The number of lines to display in the list box.
  • The country whose regions (states, provinces, counties, etc.) appear in the prompt.

In the following procedure, all the steps except the last (in which you save changes) take place on the step’s Prompts tab.

To specify attributes for a region prompt:

  1. In the Min box, set the minimum required number of responses.

If you specify 0 in Min, the prompt is optional. If, however, you specify 0 and also select Required, selecting Required overrides Min, so that the minimum number of responses is effectively 1, even though you specified 0.

  1. In the Max box, set the maximum allowed number of responses.

By default, the value in the box is 1. If you specify 1 in Max, the prompt is a single-select prompt, and you can also specify an output variable. For information on specifying an output variable, see the procedure To add a prompt to a step:.

To specify an unlimited number of responses, delete the value in the box.

  1. To make the prompt a drop-down list box, in the Extra Data box, specify “dropdown=1” (the default value).

OR

To make the prompt a list box, specify “dropdown=0”.

If you specify a value greater than 1 in the Max box, specifying that the prompt is a drop-down list box overrides the Max value, and the prompt is a single-select prompt by virtue of being a drop-down list box. If you specify a Min value greater than one and specify that the prompt is a drop-down list box, then in a script session, the step returns an error and the session does not continue beyond the step.

  1. To specify the country with whose regions the prompt is populated, in the Extra Data box, append the country code to the string “countryCode=”.
  2. In the Size box, type the number of lines to display in the list box.
  3. On the toolbar, click to save your changes to the database.
