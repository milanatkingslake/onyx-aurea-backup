Defining a variable

You define variables on the Variables Web Page Dialog or on the Method Call Bindings Web Page Dialog.

To open the Variables Web Page Dialog from the Edit Script area:

  • In the navigation area, highlight the script and then, in the Edit Script area, click Variables.

To open the Variables Web Page Dialog from within a step:

  1. In a script step, on the Prompts tab, create or select a prompt of any control type except Divider or Label.
  2. Under Default & Output Options, beside either the Default Variable or the Output Variable box, click the ... button.

The Variables Web Page Dialog appears as follows:

If you open the Variables Web Page Dialog from within a step, then by default, it shows only the variables that are available to that step.

To define a variable:

  1. Open the Variables Web Page Dialog.
  2. To add a variable, click .
  3. Specify attributes in the following columns:
  • Name - The name of the new variable. Do not include either the braces that identify the variable elsewhere in the edit area or the dollar sign that identifies the text portion of the variable. The variable name must be a valid Extensible Style Sheet (XSL) variable name and be unique within the script. Variable names can not include the left brace, double quotation marks (“ or ”), a trailing dollar sign, or a leading underscore. Processing removes leading and trailing spaces, tab, linefeeds and carriage returns.

Variable names are not case sensitive. Within a script, you can not define two variables whose names differ only by case.

  • Description - A description of the variable.
  • Type - Either of two available types, List (a variable used to populate a prompt control type that displays lists whose contents are generated by a method call) and Scalar (a variable whose value is set by user response to a prompt or by a method call). For more information on these two types of variables, see Variables.

You do not create context variables. OES creates them automatically. For more information on context variables, see Variables.

  • Scope (Step Name) - Whether the scope of the variable is global or step-local (that is, specific to a given step). By default, the scope of a new variable is global. To make a variable step-local, choose an existing step from the drop-down list in this column.

Specifying step-local scope for a list variable that holds many list items improves performance because the items in the list are not transported between client and server after every subsequent step.

  • Active - Whether the variable is available for use.
  1. Click OK.

The Web Page Dialog closes.

  1. On the toolbar, click to save your changes to the database.
