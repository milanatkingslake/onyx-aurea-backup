Configuring a masked edit prompt

For masked edit prompts, you can specify the following:

  • Mask for the user response
  • Prompt character (the character that leads the cursor in spaces the user needs to fill) and whether the prompt character appears during a script session
  • Mask characters (characters that occupy the spaces that the user will fill in and restrict the user’s response)
  • Mask literals (characters that appear unchanged in the completed field, such as parentheses that can appear around the area code in an American telephone number)
  • Minimum and maximum number of characters that the user can enter in the control, including the mask
  • Number of lines to display in the masked edit prompt

In the following procedure, all the steps except the last (in which you save changes) take place on the step’s Prompts tab.

To specify attributes for a masked edit prompt:

  1. In the Min box, set the minimum number of characters in the user’s response, including the mask characters.
  2. In the Max box, set the maximum number of characters in the user’s response, including the mask characters.
  3. In the Size box, type the number of lines to display.
  4. In the Extra Data box:
  • To include prompt characters when the data is saved to the database, specify “promptInclude=1”. To omit prompt characters when the data is saved to the database, specify “promptInclude=0”. The default value is 0.
  • To specify the prompt character, append the desired prompt character to “promptChar=”.
  • To define the mask: In the ExtraData box, append the masking characters and literals to “mask=”. For instance, the masking characters and literals “(999)999-9999” define the mask for a telephone number. The masking character “9” restricts the user’s response to a digit (0-9) or a decimal point.
  1. On the toolbar, click to save your changes to the database.
