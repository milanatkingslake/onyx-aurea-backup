Configuring a reference lookup prompt

For reference lookup prompts, you can specify the following:

  • The name of the reference field that is the source of the data to be listed in the prompt.

A name value is required.

  • If the reference field that is the source of the data to be listed has a parent reference field, the parentId of the parent reference field.

A parentId value is optional.

  • Whether the prompt appears as a list box or as a drop-down list box.
  • Minimum and maximum number of responses that the user can select when the prompt is a list box.
  • The number of items to display in the list box.

In the following procedure, all the steps except the last (in which you save changes) take place on the step’s Prompts tab.

To specify attributes for a reference lookup prompt:

  1. In the Min box, set the minimum required number of responses.

If you specify 0 in Min, the prompt is optional. If, however, you specify 0 and also select Required, selecting Required overrides Min, so that the minimum number of responses is effectively 1, even though you specified 0.

  1. In the Max box, set the maximum allowed number of responses.

By default, the value in the box is 1. If you specify 1 in Max, the prompt is a single-select prompt, and you can also specify an output variable. For information on specifying an output variable, see Adding a prompt.

To specify an unlimited number of responses, delete the value in the box.

  1. In the Size box, type the number of lines to display for the prompt.
  2. To make the prompt a drop-down list box, in the Extra Data box, specify “dropdown=1” (the default value).

If you specify a value greater than 1 in Max, specifying that the prompt is a drop-down list box overrides the Max value, and the prompt is a single-select prompt by virtue of being a drop-down list box. If you specify a Min value greater than one and specify that the prompt is a drop-down list box, then in an end-user interface session using this script, the step will return an error and the session will not continue beyond the step.

  1. To specify the parentId of the data to show in the prompt, in the Extra Data box, append the parentId value to “parentId=”.

To specify a parentId dynamically, you can append a variable rather than a specific value. For example, if the parentId is stored in the code portion of a variable called myVariable, you could use the string “parentId={myVariable}”.

  1. To name the reference field whose data the prompt presents to the user, in the Extra Data box, append the reference field name to “name=”.

The name is required for this prompt.

  1. On the toolbar, click to save your changes to the database.
