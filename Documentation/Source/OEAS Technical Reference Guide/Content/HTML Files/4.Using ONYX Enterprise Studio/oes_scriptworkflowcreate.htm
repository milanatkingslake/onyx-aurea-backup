Creating a script

When you create a script, among the attributes that you specify in the edit area’s Edit Script area are those defining the following:

  • The availability of the script
  • Some aspects of the script’s appearance
  • The script group that the script is a member of
  • Whether the script can be associated with company, individual, or incident objects as owners of the script
  • Whether the script allows the user who returns to a previously completed step to change the data previously entered in that step without re-evaluation of the rules

To create a script:

  1. In the navigation area, highlight the Scripts node within a script group.
  2. On the toolbar, click .

In the Edit Layout area, you can create script steps, view the entire script, and move steps in relation to each other. For more information on the Edit Layout area, see Viewing and changing the script in the Edit Layout area.

  1. In the Edit Script area, specify the following attributes:
  • Name - The name of the new script.
  • Description - A description of the new script. The description should be meaningful to anyone who works with the script.
  • Initial Step - A box in which you designate the step that will start the script session. By default, a new script includes one step (newStep1), which is designated as the initial step. To change the initial step, select it from the drop-down list.
  • Parent Group - A box in which you designate a group to contain the script.
  • Allow Quick Edit - Whether Quick Edit is available to users of the new script. Quick Edit allows a user to return to a step that he or she has already completed, change the response to that step, and then immediately return to the last step of the script session, without re-evaluation of rules.
  • Under Availability, you can set the following:
  • Effective Date - The date and time that the script, once published, becomes available to users. The script becomes available at the same time in each client machine’s local time zone.
  • Expiration Date - The latest date and time that the published script is available to users. The availability of the script expires at the same time in each client machine’s local time zone. Any script sessions that are in progress at that moment can continue, but no new script session can begin.
  • Published - Whether the script is available to normal users in OEP. If this check box is not selected, the script is available in OEP only to users who have access to the UI:OEP.scripting.designer resource. By default, members of the Administrator and OEP.administrator roles have access to UI:OEP.scripting.designer. For information on providing users with access to resources, see Security Administration Help.
  • Active - Whether the script is available for any users. If the check box is selected, the script is available. When the script is inactive, it appears in italics in the navigation area.

Use the Active attribute sparingly. Among other unintended consequences, you can lose the results of script sessions that are paused when the script is made inactive.

Under Display, you can set the following:

  • Step History - Whether a list of completed steps appears during a script session. By default, this check box is selected.
  • Banner/Toolbar - Whether the following appear: 1) a banner at the top of the script Web page, containing information about the script, and 2) the script session toolbar. By default, this box is selected.
  • Banner Context - Whether context information about the script, such as the name of the owner, appears in the script’s banner. If the Banner/Toolbar check box is not selected, this check box is not available. By default, this check box is selected. Context information is information such as the script owner’s ID.

The script owner is the company, individual, or incident with which the script is associated. Under Owner Types, you can select one or more of the Company, Individual, or Incident check boxes. These check boxes are unchecked by default.

Under Variable Options, you can:

  • Select the Use Context Variables check box. Context variables are predefined scalar variables that store information about the owner of the script session and can be used for the same purposes as other scalar variables.
  • Click Variables to open the Variables Web Page Dialog, in which you create the script’s variables. For more information on creating variables, see Working with variables.
  • OEAS automatically generates the ID attribute the first time you save changes to a script.
