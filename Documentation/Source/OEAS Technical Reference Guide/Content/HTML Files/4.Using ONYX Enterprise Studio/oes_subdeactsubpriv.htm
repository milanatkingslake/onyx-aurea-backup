Deactivating and re-activating subscription profiles

You might choose to deactivate a subscription profile for users that need to have their data synchronized, but do not need to explicitly subscribe to data. For example, you might want to prevent a user from subscribing to data, but you want to be sure that their changes are upsynced. Or, custom logic might have been implemented that automatically subscribes items to users, but the users themselves are not permitted to explicitly subscribe to records.

When you deactivate a subscription profile:

  • The user no longer has access to subscription functionality within OEP.
  • Advanced subscribers can no longer subscribe or unsubscribe on behalf of the user whose profile was deactivated.

Clearing the selection of the Profile Active attribute of the subscription profile has no effect on the user’s ability to synchronize records to which they have previously subscribed. They can continue to work with these records in the offline data store but they are no longer able to explicitly subscribe to additional records.

Implicit subscriptions can still occur when:

  • The user creates a new record while working while disconnected and synchronizes that record to OEP.
  • A newly created record is a parent or child of a record to which the user is already subscribed and an existing cascading relationship automatically subscribes the user to the new record.

To deactivate subscription for the subscription profile:

  1. In the Security Administration navigation area, select the subscription profile.
  2. In the edit area, uncheck the Profile Active box.
  3. Click on the toolbar to save your change.
  4. Stop and restart the OEP Web site.

When the user next logs on, the subscription controls in OEP are no longer available.

To re-activate subscription for the subscription profile:

  1. In the Security Administration navigation area, select the subscription profile.
  2. In the edit area, make sure that the Profile Active box is selected.
  3. Click on the toolbar to save your changes.
  4. Stop and restart the OEP Web site.

When the user next logs on, the subscription controls in OEP are no longer available.
