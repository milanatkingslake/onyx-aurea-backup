Activating or deactivating a method call detail

Scripting method call details and merge template method call details that you deactivate behave similarly to each other:

  • Existing scripts or merge templates that use a deactivated method call detail continue to function normally.
  • A deactivated method call does not subsequently appear in the Script Designer for use in a new script or in Template Maintenance in OEP for use in a merge template.
  • A deactivated method call does not subsequently appear in the Script Designer even when the script designer edits an existing script that uses that method call. To save the script, the script designer must select an active method call to assign to it.
  • In Template Maintenance, a warning to the template designer results when a template has previously added a merge field based on a deactivated method call detail. The deactivated field is not merged. Additionally, the deactivated field does not appear in the available method call details even if the field exists in the template text. When saving the merge template, the template designer must select an active method call detail to assign to the merge field.

To activate or deactivate a method call detail:

  1. In the navigation area, select the method call detail you want to activate or deactivate.

The attributes of this method call detail appear in the edit area.

  1. Check or uncheck the Active check box to activate or deactivate the method call detail.
  2. Click to save your changes.
