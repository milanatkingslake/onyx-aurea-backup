Common terms

The following table defines basic vocabulary for Assignment Administration.

Term Description

Expressions

A rule that evaluates to true or false based on its combination of a property, an operator, and a value. Any property of the customer, individual, and incident objects can be used. Example expression: countryCode = US.

Conditions

A set of expressions (logical statements) that are processed by rules to associate customers or incidents with territories and decide which, if any, assignments are appropriate. A condition cannot evaluate to true unless all of its expressions also evaluate to true.

Territories

An entity that contains zero to many conditions. Companies and individuals are associated with a territory if they meet at least one of its conditions. For organizational flexibility, a territory can contain other territories or conditions.

Assignment sets

A pool of users that can be linked to conditions. Users are created as internal contacts for companies, individuals, or incidents, and they are optionally assigned to incidents.
