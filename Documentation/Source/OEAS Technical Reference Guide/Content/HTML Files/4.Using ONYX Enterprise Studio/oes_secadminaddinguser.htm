Adding a user

Within Security Administration, users appear within the context of a group. Use the following procedure to add a new user.

For information on adding a subscription profile to a user, see Adding and deleting a user’s subscription profile.

To add a new user:

  1. In the navigation area, open Groups and then the specific group to which you want to add the user.
  2. Within the group, select either the Users node or a single user, and then click on the toolbar.

A new user titled newUserName appears at the bottom of the list in the navigation area, followed by an asterisk (*).  

  1. In the edit area, specify values for the new user as follows:
  • User ID - Enter a unique string for the user ID. You can only change the user's ID in this required field when you create the user, and you must do so in order to successfully create and save the new user.
  • When creating a user account, do not use the following characters in the userId. They are not supported.
  • Less than (‹)
  • Greater than (›)
  • Double quotes (")
  • Language - Select a language for the user.
  • Group - Select a group for the user to belong to. Because group names do not have to be unique, the groups are identified in the box by the group name followed, in parentheses, by the group ID. For example, if the group name is “Sales” and the group ID is “Sales-USA,” then the group is identified in this box as “Sales (Sales-USA).” A second group named “Sales” whose group ID is “Sales-Europe” would be identified in this box as “Sales (Sales-Europe).” The default value for this list box is the name of the group in which you created the user. When you select another group from the list and then save changes with the user selected, in the navigation area, the user’s node moves to that group. If the user does not belong to a group, then the Group box can be left blank. Users saved with no group are listed in the “Users [No Group]” group node.
  • Group - Select a group for the user to belong to. Because group names do not have to be unique, the groups are identified in the box by the group name followed, in parentheses, by the group ID. For example, if the group name is “Sales” and the group ID is “Sales-USA,” then the group is identified in this box as “Sales (Sales-USA).” A second group named “Sales” whose group ID is “Sales-Europe” would be identified in this box as “Sales (Sales-Europe).” The default value for this list box is the name of the group in which you created the user. When you select another group from the list and then save changes with the user selected, in the navigation area, the user’s node moves to that group. If the user does not belong to a group, then the Group box can be left blank. Users saved with no group are listed in the “Users [No Group]” group node.
  • Allow Impersonation of Other Users - (OPS only) This box must be selected for the OPS Service user account. This option is used by OPS for login and session management and OEP email processing. Do not select this box when configuring other user accounts.
  • Active - Select this box to activate the user. Only activated users can log in. By default, the box is selected. If the user is not active, then in the navigation area, the user name either is present but unavailable (grayed out and in italic) or, if you have selected Show Only Active Users and Groups, not present.
  • User Name - Enter the user’s name. This can be the user’s friendly name, or the personal portion of his or her email alias. Special characters such as Less than (‹), Greater than (›)  or Double quotes (")  are not supported. This field is required.
  • Email - Enter the user’s email address. Clicking the email envelope icon to the right of the box starts an email message to the user. This field is blank by default; filling it is optional.
  • Manager ID - Enter the Onyx user ID of the user’s manager. This field is blank by default; filling it is optional.

To remove a user, you deactivate it rather than delete it. There is no functionality to delete a user.

  • Onyx Authentication - Select this box to enable the user to log in with his or her Onyx user name and password. By default, this box is not selected.
  • Password - Enter a password for the user. The password you specify is required for login only if Onyx Authentication is selected. If Onyx Authentication is not selected, this box is not available. The password that appears in this box by default when you create a user is the password you have specified as the default password in Security Administration Preferences (to open Security Administration Preferences, click on the toolbar). User passwords in Onyx Enterprise Application Server are case-sensitive.
  • Windows Integrated Authentication - Select this box to enable the user to log on with integrated Windows authentication. Logging on with integrated Windows authentication enables the user to access the Web site without entering the Onyx user name and password. By default, this box is not selected.
  • Domain Account - Enter the user’s domain account, including the domain name. The syntax is <domain_name>\<user_name>. If Windows Integrated Authentication is not selected, this box is not available. If Windows Integrated Authentication is selected, this field is required.
  • Role Membership - Contains the roles to which the user has been added. You cannot add a user to a role in this list; it reflects the roles to which you have added the user. Note, however, that changes in role membership are not immediately reflected in this list.
  • Insert By, Insert Date (GMT)- These fields reflect the name of the person who added the user and the date that the user was added. They are automatically updated when you restart Security Administration.
  • Insert Date (GMT)- This system-generated field reflects the date the user was created.
  • Update Date (GMT)- This system-generated field reflects the date when data for the user was last updated. When you have modified and saved this user, this attribute becomes Previous Update Date (GMT).

The Insert Date (GMT) and Update Date (GMT) fields are automatically updated when you restart Security Administration.

  1. Click on the toolbar to save your changes.
