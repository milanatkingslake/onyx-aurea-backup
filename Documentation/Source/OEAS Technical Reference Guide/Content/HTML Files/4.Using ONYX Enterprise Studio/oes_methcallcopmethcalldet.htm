Copying a method call detail

Remember the following when copying method call details:

  • You cannot copy method call details between method call groups (Scripting and Merge Template).
  • You cannot copy method call details between method call detail types (Inputs and Outputs).
  • You can copy only one method call detail at a time.
  • When you select an existing method call detail and click , the copied method call detail is pasted into a position immediately following the existing detail that you had selected. When you select a method call detail group node (Inputs or Outputs) and click , the copied method call detail is pasted at the bottom of the list within the selected detail type node.
  • You can copy and paste across browser instances. These browser instances can even be pointed at different databases.

To copy a method call detail:

  1. In the navigation area, select the method call detail that you want to copy, and then click .
  2. Within the same method call group, expand the method call to which you want to copy the method call detail.
  3. Select the appropriate method call details node or an existing method call detail from within that node, and then click to paste the copied method call detail.

The type of node—input or output—that you paste the method call into must be the same as that from which you copied the method call detail.

A copy of the original method call detail named <originalName><n> appears in the navigation area, followed by an asterisk (*). The edit area shows the Edit Method Call Detail area for the copied method call detail.

  1. In the Edit Method Call Detail area, make any necessary changes to the method call detail’s attributes.

The method call detail attributes are described in Method call detail attributes.

  1. Click to save your changes.

Method call details that sequentially follow the newly inserted method call detail also appear in the navigation area with an asterisk. When a new method call detail is inserted in a list, the sequence number for each of those following the new method call detail in the list is updated. There is otherwise no change to the existing method call details.
