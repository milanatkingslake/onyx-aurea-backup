Configuring fields

The two categories of configurable fields are briefly described below. You can configure all three categories in the Fields mode of Reference Table Administration. For additional details regarding these configurable fields, see Reference Data.

  • Configurable standard fields - Configurable standard fields are combo box fields (drop-down lists) that users select from in the user interface. They have a fixed field type (combo box) and fixed hierarchical relationships. The only attributes of configurable standard fields that you can configure are the lookup values that appear in the lists.
  • User-defined fields - User-defined fields enable you to configure the application to capture additional data specific to your business needs.

Before you create new field names in Reference Table Administration, verify whether or not field names in your end-user interface (UI) are case sensitive. If they are, you must enter new field names in the same case in which they exist in the UI, so they can be recognized by the application. For example, in a case sensitive language, “abc” is considered different data than “ABC.”
