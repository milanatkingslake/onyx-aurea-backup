Getting started

Following are the general steps to create a assignment territory structure in Assignment Administration:

  1. Create parent/child territories in the Assignment Tree.

Territories can contain other territories to provide further organization of the conditions. A geographic example is country, region, state: United States (highest territory), Pacific Northwest (second highest territory), and Washington (third highest territory). The name "territory" implies a geographic assignment; however, you can organize territories by other classifications, such as market sector, named accounts, and much more.

  1. Create conditions for companies, individuals, and/or incidents according to your plan for territory and incident assignment.
  2. Create some expressions using properties of the customer, individual, and incident objects combined with operators and values.
  3. Create a customer assignment set (either company or individual) and associate users by entering their user IDs and internal contact types as follows.
  4. Create an incident assignment set and associate users with the assignment set by entering their user IDs and internal contact types. For the routing of leads, incident assignment set users are optionally assigned to incidents based on the following assignment set types:
  • Normal

Assigns incidents to a single user within the assignment set.

  • Round Robin

Assigns incidents to the next user in a pool of users (in turn).  This option keeps track of who was assigned last and determines who should be assigned next.

  • Least Busy

Assigns incidents to the user with the lightest workload (least amount of open incidents).

  1. Link the assignment set to the appropriate conditions. Assignment sets are associated with conditions of the same object type and they are reusable throughout the Assignment Tree.

When complete, a set of territories and their corresponding conditions are created. You can verify the territory association of customers in List Manager. For instructions, see the FAQ.
