Defining lookup value attributes

When you select a lookup value in the Lookup Value by Field mode, the Table Administration edit area displays the attributes for that lookup value. Use the following descriptions to assist you in defining lookup value attributes when you are:

  • Adding lookup values to a combo box field
  • Modifying a lookup value for a combo box field.

When you select a lookup value in the Lookup Value — All mode, the only attribute you can configure is Lookup Value, and Active.

Lookup Value - The Lookup Value attribute is the lookup value that appears in the user interface. To create or modify this value, you can type a new value in the Lookup Value box or you can select an existing value from the Existing Lookup Value box.

Existing Lookup Values - When you type a character in the Lookup Value box, the Existing Lookup Values box displays lookup values that have been previously defined in the database, sorted alphanumerically, that begin with the same character that you entered. Using the Existing Lookup Values box, you can select existing lookup values for the selected field. The lookup data value list does not allow duplicate values.

Extra Data - This box is optional and enables you to link additional information to a lookup value that is associated with a combo box field. You can use this to attach contextual descriptions to short or abbreviated lookup values or to group similar data values for tracking purposes. For example, a field pertaining to sales status can capture varying degrees of the values Open and Closed such as active, waiting, dormant, and finished. Open is specified as an extra data value to active and waiting, whereas Closed is associated with dormant and finished. Group association makes it easier to run concise queries. Querying for all open Sales incidents results in a list of all active or waiting incidents.

To ensure application functionality, two extra data values are called by stored procedures within the application and must be implemented correctly. Fax is a standard extra data value associated with the lookup data value Facsimile or Fax. The lookup value describing the fax number can be whatever you choose. However, since procedural coding is linked to the extra data value Fax, it must not be changed. The extra data value acts as an unchanging value to which stored procedures can reference. The other standard extra data value is Closed, which is associated with all incident status drop-down values which signify Closed.

Do not modify the extra data values Fax and Closed. Changing one or both of these values cause certain stored procedures to fail.

Char Code - This attribute is similar to the Extra Data attribute and is also optional. It enables you to assign a code or abbreviation to lookup values, such as “Pres” to President. If you import a file which has the code value associated with it, the system can read that file and its attributes correctly. Only one code can be assigned for each lookup value.

The CharCode for company SICCode (Standard Industry Code) is not optional; the character Char Code is used for the lookup data value. If an SIC Code is created without a Char Code, or with a Char Code value of anything other than an integer, the SIC Code does not save with the customer record.

Active - This check box enables you to set a lookup value to active or inactive. A checked box indicates an active lookup value that is visible in the user interface. An unchecked box indicates an inactive lookup value that is not visible in the user interface.

ID of Lookup Value - Each lookup value is assigned a unique, system-generated identification number called the iParameterId. Once you have saved a lookup value, this identification number appears in the edit area. The ID of Lookup Value attribute is read-only.

ID of Combo Box Field - This field displays the identifier value, called Reference ID, for the selected field to which the lookup value belongs. The Reference ID is also a system-generated identifier and is read-only.

The Add Lookup Value button enables you to quickly save a lookup value and begin defining another one. When selected, the lookup value you defined is saved and associated to the field, and the edit area is refreshed, allowing you to define a new lookup value.
