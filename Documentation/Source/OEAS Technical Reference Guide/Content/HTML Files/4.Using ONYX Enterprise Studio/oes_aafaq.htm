Basic FAQ

How do operators used by the expressions work?

The operators used in expressions are capable of both alphabetic and numeric comparisons. Assignment Studio processes use OED definitions to determine the type of comparison to perform. For example, if a property of the individual, company, or incident is a number in the OED, then a numeric comparison is used.

Unexpected results may occur when an alphabetic comparison is occurring but a numeric one is expected. This is particularly true for the following operators: Is Less Than, Is Greater Than, and Is Between.

 

Maintaining territories FAQ

How do I verify the territory association of customers?

You can use List Manager to search for customers who are assigned to a selected territory.

Before you use these instructions, update all associations by restarting Internet Information Services (IIS) on the OEP server. For more information, refer to Microsoft documentation.

To search for customers assigned to a territory:

  1. Open List Manager and select either Companies or Individuals from the - Select a view - drop-down.
  2. Select the Territory Assignment folder.
  1. Select the Territory Name option.

The Input Criteria dialog box displays the list of territories created in Assignment Administration.

  1. Select the appropriate territory and click Save.
  2. To refine your search, select additional criteria.

For example, you could search for all companies within a city through the Territory Name column.

  1. In the List Manager window, click Search.

All associated companies or individuals are displayed.

How do I verify the assignment of incidents?

Using a reporting tool, you can create a report that queries the assignment_association table for a particular incident.

Incident assignments cannot be verified in Assignment Studio or through List Manager.

How do I assign customers to a new territory?

Using List Manager, you can either delete the customer data or move the data to another territory.

Before following these instructions, update all associations by restarting Internet Information Services (IIS) on the OEP server.

To assign customers to another territory:

  1. Open List Manager and select either Companies or Individuals from the - Select a view - drop-down.
  2. Select the Territory Assignment folder.
  3. Select the Territory Name option.

The Input Criteria dialog box displays the list of territories created in Assignment Administration.

  1. Select the appropriate territory and then click .
  2. Click Search.

All associated companies or individuals are displayed.

  1. From the results list, select the customers or individuals you want to re-assign.
  2. Select the Reassign Territory bulk action and then click Execute Selected Bulk Action.
  3. Run the search query again.

The selected records are no longer displayed in the results list.

How do I remove a territory with associated records?

In Assignment Administration, you cannot delete a territory with associated company, individual, or incident records. However, you can run a SQL database script to remove the associations from the assignment_associations table. After the associations are removed, you can delete the territory in Assignment Administration.

Onyx recommends that you deactivate items with associated records instead of deleting them. When you deactivate items, they no longer appear for active use, but remain in the application for records that reference that item.
