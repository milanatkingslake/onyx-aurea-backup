Renaming an object or method

In Object Designer, it is possible to change an object or method name; however, doing so has the following consequences related to Security Administration:

Consequences of renaming an object:

  • The old object name no longer appears in the Business Objects resource list; the new object name now appears representing the renamed object.
  • The methods on the renamed object become unsecured.
  • The object method’s new <object.method> name is the basis for the resource's resource ID, when the method is again secured.

Consequences of renaming a method:

  • The renamed method becomes an unsecured method.
  • If the method was previously secured, an entry with the old resource ID remains in the OEDB resource table, but is no longer valid.

Onyx recommends that you not rename any Onyx-supplied objects or methods, because doing so breaks any user interface that uses OEAS, such as Onyx Employee Portal. More consequences of changing object.method names in Object Designer follow this warning.

Similarly, in Object Designer, it is possible to delete an Onyx-supplied object or method, but Onyx also recommends that you not do this, because of the following results:

  • The deletion of an Onyx-supplied object or method breaks any user interface that uses the method.
  • If the resource that represented the deleted object.method was secured, an entry with the old resource ID remains in the OEDB resource table, but is no longer valid.
