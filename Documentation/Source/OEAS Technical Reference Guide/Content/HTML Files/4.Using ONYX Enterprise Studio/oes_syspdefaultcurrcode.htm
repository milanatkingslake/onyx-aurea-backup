DefaultCurrencyCode

The default currency code assigned to new products entered into the database through Product Administration.

For example:

  • USD, for US Dollar
  • ESP, for Spanish Peseta
  • DEM, for German Mark

Before you configure a forecast or quote, Onyx recommends that you specify a default currency code (DefaultCurrencyCode) in System Parameter Administration for the forecast/quote (it is optional). Otherwise, no currency code will be assigned when you save the forecast or quote.

Selecting a default currency code in this system parameter enables that currency code to be automatically selected and saved when you update or create a new product in Product Administration. Setting this system parameter is useful if you enter many of your products through Product Administration, and generally enter them with one currency type.

The currency code that you enter for the default value must be a valid currency code, as listed in the currency table. It must also be entered using the same case, upper or lower, as it has been saved in the currency table.

  • Allowed Parameter Values - The parameter value stores a string value of no more than four characters.
  • Default Parameter Value - DefaultCurrencyCode is installed without a default value.
