Quick overview of the Campaign Administration tool

The user interface of Campaign Administration is divided into four areas:

  • Context area
  • Toolbar
  • Navigation area
  • Edit area

Context area - Lists the name of the application and the site being maintained.

Toolbar - Appears under the context area and contains the toolbar buttons that perform the tasks related to campaigns.

Icon Tool tip Function Context

Add

Adds a new campaign, tracking code type, tracking code, milestone category, or milestone action. The item added depends on which item you selected in the navigation area.

Enabled when an item is selected in the navigation area.

You cannot add new tracking code types, milestone actions or milestone categories from within the Campaigns node. These items are administered through the Tracking Code Types node, Milestone Actions node, and Milestone Categories node.

Delete

Marks an item for deletion. The item marked for deletion depends on which item you selected in the navigation area. Clicking this icon again undeletes an item marked for deletion.

Enabled when a single item is selected from within a list in the navigation area. (All deletes are physical deletes; a warning appears when an item is deleted.)

Deactivate items instead of deleting them. When you deactivate an item, it no longer appears to end-users for active use, but remains for records previously associated with it.

Move up

Moves a tracking code, tracking code milestone up one position in the navigation area to change the order in which the tracking codes appear in both the end user UI and in the Campaign Administration UI.

Enabled only when a tracking code or tracking code milestone is selected.

Move down

Moves a tracking code, tracking code milestone down one position in the navigation area to change the order in which the tracking codes appear in both the end user UI and in the Campaign Administration UI.

Enabled only when a tracking code or tracking code milestone is selected.

Save

Save all modifications to the Onyx Enterprise Database.

Enabled after any changes have been made.

Help

Launches Help.

Enabled at all times.

Navigation area - You define and administer campaigns, tracking code types, tracking codes, tracking code milestones, milestone categories, and milestone actions by accessing them through the appropriate node in the navigation area and then modifying their attributes.

There are four first-level nodes in the navigation area:

  • Campaigns node - The Campaigns node enables you to create and administer campaigns and their associated tracking codes and tracking code milestones.

The following example shows how an expanded Campaigns node appears for a campaign called “Advertising.” This example includes a tracking code type called “Newspapers” and its associated tracking codes and tracking code milestones.

  • Tracking Code Types node - To administer tracking code types, you must access them through the first-level Tracking Code Types node.
  • Milestone Actions node - Enables you to define individual actions required to meet tracking code milestones. All milestone actions are administered through this node.
  • Milestone Categories node - Enables you to define categories for tracking code milestones. All milestone category administration occurs through this node.
  • Edit area - In the edit area, you view and modify the attributes of a campaign or other item. What appears in the edit area changes according to what is selected in the navigation area.
