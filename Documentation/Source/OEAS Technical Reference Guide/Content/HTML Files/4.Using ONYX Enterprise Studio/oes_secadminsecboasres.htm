Securing a business object method as a resource

To secure a method as a resource:

  1. From within the Resources node in the navigation area, expand the Business Objects node, and select the method you want to secure as resource.  
  2. In the edit area, click Secure Resource.

    3. Clicking this button secures the method that is selected in the navigation area and denies access permission to all users and roles except the sa user (the system administrator), which has access to all resources. You must grant access permission to users and roles individually. Note that this button appears only when an unsecured method is selected in the navigation area. The entities listed under UI Resources and Other Resources are considered resources by default and are therefore secured by default.  

  3. Following are descriptions of the attributes in the edit area.
  • Resource ID - The resourceâ€™s unique ID string.
  • Description - A description of the resource.
  • Unsecure Resource - Click this button to unsecure a business object resource that is selected in the navigation area. Unsecuring the resource makes the method accessible to all users. Because it is no longer a resource, it appears unitalicized in the navigation area.

Important: The Unsecure Resource button appears only when you have selected a business object resource. These are the only resources that you can unsecure.

  • Manage permissions - Click this button to change the edit area so that you can grant or deny access permission to individual users and roles. See Managing permissions for resources for further instructions.
  • Access permission lists - Lists of the users and roles that have been either granted or denied access permission for the resource. As you add users and roles, their IDs and description or name appear in the relevant access permission list.
  1. Click on the toolbar to save your changes.

    2. After you have secured the methods as resources, you must set user-access permissions on these resources. Use the procedures in Managing permissions for resources to define these permissions.
