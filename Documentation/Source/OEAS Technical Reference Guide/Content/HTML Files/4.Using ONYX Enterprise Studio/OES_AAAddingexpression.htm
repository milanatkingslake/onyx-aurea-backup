Adding expressions

To add an expression:

  1. In the navigation area, expand an individual, company, or incident condition node.

When you expand the condition node, the nodes for child expressions and linked assignment sets appear.

  1. Either select the Expressions node or select an existing expression and then click .

A new expression named Expression: newExpression<n> appears in the navigation area, followed by an asterisk (*).

  1. In the edit area, define the attributes for this expression.

The edit area changes depending on the lookup values selected for the property name and operator.

  • Property Name - Select a property for this expression from the drop-down list. Expressions are filtered depending on the object type of the condition. For example, if the condition object type is company, then company properties appear in this list.
  • Operator - Select an operator for this expression from the drop-down list.
  • Value - Type a value or select one or more values by clicking to display the Select a value window. Values are case-sensitive and must be entered accurately.

The list that appears in the Select a value window corresponds with the data selected in the edit area for the property name and operator. For example, if you selected "countryCode" for the property name, "Is Equal To" for the operator, this list would reflect the values for all countries around the world. Select a value and then click OK.

  • Click to save your changes.

The new expression is created.
