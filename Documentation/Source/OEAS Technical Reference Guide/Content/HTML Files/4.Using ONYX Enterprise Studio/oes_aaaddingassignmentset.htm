Adding assignment sets

To add an assignment set:

  1. In the navigation area, either select the Assignment Sets node or select an existing assignment set, and then click .

A new assignment set named newAssignmentSet<n> appears in the navigation area under the appropriate individual, company, or incident object type, followed by an asterisk (*).  When you expand the newAssignment set<n> node, nodes for child users appear.

  1. In the Edit Set area, define the attributes for this assignment set.
  • Name - Type the name you want to assign to this assignment set. You can edit this name after a save.
  • Description - Type a description of this assignment set. You can edit this description after a save.
  • Active - Clear this check box if you want to deactivate this assignment set. The default is active (selected).

If you are creating an incident assignment set, the Assignment Type attribute displays. For the routing of leads, you can optionally assign users to incidents by selecting one of the following assignment types:

  • Normal

Assigns incidents to a single user within the assignment set.

  • Round Robin

Assigns incidents to the next user in a pool of users (in turn).  This option keeps track of who was assigned last and determines who should be assigned next.

  • Least Busy

Assigns incidents to the user with the lightest workload (least amount of open incidents).

  • Click to save your changes.

The new assignment set is created.

  • Next, associate users with the assignment set.
