Quick overview of the Table Administration tool

Overview of the Table Administration tool

The user interface of the Table Administration tool is divided into four areas:

  • Context area
  • Toolbar
  • Navigation area
  • Edit area

Context area - Lists the name of the administration tool, the application, and the site being maintained.

Toolbar - Appears under the context area and contains the toolbar buttons that control the tasks related to Table Administration.

Icon Tool tip Function Context

Add

Used to add a new data to a domain data area.

Enabled at all times.

Delete

Used to delete existing domain data.

Enabled after you have selected a row within a table to delete.

Save

Used to save all modified domain data to the Onyx Enterprise Database.

Enabled after any changes have been made.

Help

Launches Online Help.

Enabled at all times.

Navigation area - Appears under the toolbar area and contains the following controls.

  • Table list - Contains all the domain data areas that have been configured to be administered through Table Administration. Select the domain data you want to edit in the Table list.
  • Description field - Displays information about the domain data selected in the Table list.
  • Filter list - Appears when the domain data selected in the Table list has attached filters. For very large areas of domain data, a filter enables you to use additional criteria to narrow your search. For example, the National language domain data area requires you to select a filter from the Filter list, thereby decreasing the number of values returned and reducing the size of the generated data grid.

Edit area - Appears under the navigation area and contains a grid that enables you to modify domain data.

  • Table Administration grid - The lower part of the Table Administration page contains the Table Administration grid. When you select an area of domain data from the Table list, this grid displays the data rows for the domain data of that area. If the domain data area has associated filters in the Filter list, only the rows that fit the filter’s parameters display. At the top of each grid column is the field name. Procedures for viewing, editing, adding, or deleting records in a table are located in the Table Administration Help.
  • Non-editable fields - There are certain fields in each table which cannot be edited:
  • User-defined ID fields (primary key fields) are entered by the user and cannot be updated once saved to the database.
  • System generated ID fields are generated on insert and cannot be updated by either the system or the user.
  • Date fields are automatically generated on insert and are automatically updated by the system. Date fields cannot be updated by the user unless you have configured the date fields to be editable strings. This involves adding a conversion process to the SQL scripts provided in Adding new tables to Table Administration.
