Configuring a list box prompt

For list box prompts, you can specify the following:

  • Whether available responses are fixed or are determined by a list variable and method call.
  • A fixed set of possible responses.
  • Whether to have responses alphabetically sorted in the client user interface. When you localize a script, the sorting in certain Unicode language sets that takes place in the database can produce incorrect sorting in the client user interface.
  • Minimum and maximum number of responses that the user can select.
  • The number of items to display in the list box.

In the following procedure, all the steps except the last (in which you save changes) take place on the step’s Prompts tab.

To specify attributes for list box prompts:

  1. Under Response Options:
  • To create a fixed set of responses:
  • Select Fixed.
  • Open the Prompt Responses Web Page Dialog and add responses.
  • For procedures for adding responses, see Adding, deleting, and changing the order of fixed responses.
  • To determine the set of responses using a variable that a pre method call populates:
  • Select Variable.
  • Select a variable either from the box beside Variable or, to select the variable from the Variables Web Page Dialog, click ....
  • The list variable that you use to populate a set of responses must be populated by a method call. For information on working with method calls, see Method calls and Working with method calls.
  1. To sort the responses alphabetically in the user’s interface, select Sort In UI.
  2. In the Size box, type the number of lines to display in the list box.
  3. In the Min box, set the minimum required number of responses.

If you specify 0 in Min, the prompt is optional. If, however, you specify 0 and also select Required, selecting Required box overrides Min, so that the minimum number of responses is effectively 1, even though you specified 0.

  1. In the Max box, set the maximum allowed number of responses.

By default, the value in the box is 1.

If you specify 1, the prompt is a single-select prompt, and you can also specify an output variable. For information on specifying an output variable, see the procedure To add a prompt to a step:.

If you specify 1 but do not select Required, selecting Required overrides Max, so that the prompt is not required.

To specify an unlimited number of responses, delete the value in the box.

  1. On the toolbar, click to save your changes to the database.
