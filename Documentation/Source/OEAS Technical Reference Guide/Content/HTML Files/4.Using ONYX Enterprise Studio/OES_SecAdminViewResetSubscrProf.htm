Viewing and refreshing a subscription profile’s subscription rules

The subscription rules table in the edit area for a subscription profile summarizes the relationships between subscribable objects and the rules regarding automatic subscription. Click View Subscription Rules to view subscription rules for a particular profile.

The Relationship column contains the description of the subscription relationship between two subscribable objects, such as “Company ID of the individual” or “Company owner of an incident.” A subscription relationship between a primary object and a secondary object creates a subscription to one of the objects for a user when he or she subscribes to the other. Whether the subscription is for the primary object as a consequence of the user’s subscribing to the secondary object or vice versa depends on the following rules.

Under the Primary Object heading for the relationship:

  • The Required? column indicates whether, in Subscription Designer, the Always subscribe the primary object (and unsubscribe the secondary object when the primary object is unsubscribed) rule is specified for the primary object in the relationship. If set by the administrator, this subscription-relationship rule specifies that:
  • When a user subscribes to the secondary object, the user is also subscribed to the primary object. If there is another required relationship defined for the primary object, the user is also subscribed to the object in that required relationship and so on, until the end of the chain of required relationships is reached.
  • Unsubscribing from the primary object also unsubscribes the user from the secondary object.
  • The Cascade? column indicates whether the Subscribe primary object and cascade its subscription relationships rule is specified in Subscription Designer for the primary object in the relationship. This subscription-relationship rule specifies that when a user subscribes to the secondary object in the relationship:
  • The user is automatically subscribed to the primary object in the relationship.
  • If the primary object has subscription-relationship rules that cascade subscription to other objects, those rules are carried out as a result of this initial subscription.

Under the Secondary Object heading for the relationship:

  • The Cascade? column indicates whether the Subscribe secondary object and cascade its subscription relationships rule is specified in Subscription Designer for the primary object in the relationship. This subscription-relationship rule specifies that when a subscriber subscribes to the primary object in the relationship, automatic cascaded subscriptions are created for that subscriber for the secondary objects in the relationship. If the secondary object has subscription-relationship rules that cascade subscription to other objects, those rules are carried out as a result of this initial subscription. This chain continues until the end of the chain of relationships is reached.
  • The Filter column displays any filter that has been created in Subscription Designer for limiting the secondary objects with which a cascading relationship to the primary object in the relationship is created.

To view or refresh a profile’s subscription rules:

  1. In the edit area for the user’s subscription profile, click View Subscription Rules.

A table showing the subscription rules for the profile appears.

  1. To refresh the view, click Refresh Subscription Rules.
