Adding a group

Use the following procedure to add a new group.

To add a group:

  1. In the navigation area, select either the Groups node within which you want to add the new group or another group at the level at which you want to add the new group.
  2. Click on the toolbar. A new group titled newGroupDescription appears at the bottom of the list in the navigation area, followed by an asterisk (*).  
  3. In the edit area, specify values for the new group's attributes.
  • Group ID - Enter a unique string for the group ID. You can only change the group ID in this required field when you create the group, and you must do so in order to successfully create and save the new group.
  • Group Description- Enter the group’s name or description. This field is required.
  • E-Mail - Enter the group’s email address, if there is one. Clicking the email envelope icon to the right of the box starts an email message to the alias in the box.
  • Manager ID - Enter the Onyx user ID of the group’s manager. This field is blank by default; filling it is optional.
  • Escalation - Enter the Onyx user ID of the group’s primary contact. This field is blank by default; filling it is optional.
  • Parent Group - In this box, if you want to create a hierarchical relationship between groups, select the group that you want this group to be subordinate to. Because group names do not have to be unique, the groups are identified in the box by group name and group ID. When you change the parent group for this group and then save changes with this group selected, the group’s name moves in the navigation area, to the Groups list contained by the new parent group. Leave this box blank if the group has no parent group.
  • Active - Select this check box in order to activate the group. If the group is not active, then in the navigation area, the group name either is present but unavailable (grayed out and in italic) or, if you have selected Show Only Active Users and Groups, not present. Whether the group is active does not affect whether the users who are members of the group are active.

To remove a group, you deactivate it rather than delete it. There is no functionality to delete a group.

  • Insert By, Update By - These fields reflect the name of the person who added the group and the date that the group was added. They are automatically updated when you restart Security Administration.
  • Insert Date (GMT)- This system-generated field reflects the date the group was created.
  • Update Date (GMT)- This system-generated field reflects the date when data for the group was last updated. When you have modified and saved this group, this attribute becomes Previous Update Date (GMT).

The Insert Date (GMT) and Update Date (GMT) fields are automatically updated when you restart Security Administration.

  1. Click on the toolbar to save your changes.
