Adding and deleting a user’s subscription profile

Adding or deleting a subscription profile follows the model for adding or deleting most other objects in OES administration and design tools.

After you have added or deleted a subscription profile, you must restart the OEP Web site before the user’s ability to use the subscription-related functionality in OEP is changed.

To add a subscription profile to a user:

  1. In the navigation area, expand the user, and then select Subscription Profiles.

If you change sync subsystem, staging server, or subscription service data you might have to refresh your browser to view the Subscription Profiles node.

  1. On the toolbar, click .
  2. Specify values for the following attributes:
  • Staging Server - The name of the staging server through which synchronization of subscribable objects takes place. Click the down arrow and select a staging server from the list.
  • Staging Server User ID - The user’s alias (the ID by which he or she is recognized) on the staging server. When a mailbox is created in the staging server’s Active Directory, the alias is by default the mailbox name, although it can be changed to any name.
  • Subsystem - The name of the Onyx Processing Server (OPS) subsystem to be used with this subscription profile. Click the down arrow to view a list of all available synchronization subsystems.
  1. Select or remove the selection from the following attributes:
  • Sync Active - Enables synchronization of Onyx data with the offline data store for the subscription profile.
  • Reset User at Next Sync - Specifies that in the next server synchronization cycle, all the user’s data in the staging server data store is deleted. Then, new copies of the user’s data are created based on the current subscription information in the database. During the next client synchronization, all data in the offline data store is deleted and replaced by the newly created data. The flag is cleared after the reset occurs.

Resetting a user’s subscription profile causes all of their unsynchronized changes to be discarded. To avoid data loss, coordination with the user is recommended to ensure that all data is upsynced prior to the reset. For additional information, see Resetting a user’s subscribed data.

  • Profile Active - Activates the Subscription Feature in the OEP user interface (UI), revealing the UI elements that enable users to subscribe to records.
  1. On the toolbar, click to save your changes.

When you have saved the subscription profile, the Show Subscription Rules button appears in the edit area. For more information on working with subscription rules, see Viewing and refreshing a subscription profile’s subscription rules.

To delete a subscription profile:

  1. In the navigation area, select the subscription profile.
  2. On the toolbar, click to mark the subscription profile for deletion.
  3. Click on the toolbar to complete the deletion and save changes.

A profile with existing subscription items cannot be deleted.
