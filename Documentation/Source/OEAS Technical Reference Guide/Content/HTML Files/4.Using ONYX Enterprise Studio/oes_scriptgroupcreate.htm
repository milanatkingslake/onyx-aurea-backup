Creating a script group

The procedure for creating a new script group is similar to creating objects in other OES Administration and Design Workbench tools.

To create a new script group:

  1. To create a first-level script group, in the navigation area, either select a top-level group or select nothing.

OR

To create a script group within an existing group, select Groups within that group.

  1. On the toolbar, click .

A new script group named newGroup<n> (in which “n” is an integer) appears in the navigation area, followed by an asterisk (*). The asterisk indicates that the changes to this workgroup have not been saved yet. The edit area, now labeled Edit Group: newGroup<n>, displays controls for changing various attributes of the script group.

  1. In the edit area, specify the following:
  • Name - The name of the new group.
  • Description - A description of the new group. This attribute is optional.
  • Parent Group - The name of the group that contains the new group. This box is a drop-down list box that contains the names only of existing groups. If you change the parent group in this box, OES immediately moves the group to the new parent group in the navigation area.
  • Active - Whether the scripts in this group are available to any users in a end-user application such as Onyx Employee Portal (OEP). When the script group is inactive, it appears in italics in the navigation area.

The first time you save changes to a group, the ID property (which appears in the edit area beside ID) is automatically generated.

  1. Click to save your changes to the database.
