Planning the script

Although the Edit Layout area acts as a storyboard, planning your script before creating it rewards you with a speedier process and fewer problems when you create it. When you plan a script, consider the following:

  • The script end-user’s starting point with the customer
  • Possible end points of the script
  • Rules that define the user’s possible paths on the way to those end points
  • Conditions (logical expressions that are evaluated to true or false) that those rules use to decide between branches
  • Prompts and responses for each step, including whether each prompt should be required
  • Variables needed for use in condition statements, to merge text, to pass values to method calls, and to store values returned by method calls
  • Desired scope of variables: whether the variable should be available to more than one step in the script and whether the user will want its value to be preserved when using Quick Edit
  • Method calls needed for the script to gather, modify, and/or disseminate data
  • The points in the script that require data obtained from method calls, that use method calls to update the database, or that use method calls to perform other actions, such as creating and sending an email message
  • Which data needs to be passed to method calls
  • Which data the method calls should insert into, update, or retrieve from the database
  • Quick Edit—whether the end-user should be able to return to a step and change the data in the responses without re-evaluation of the rules.
