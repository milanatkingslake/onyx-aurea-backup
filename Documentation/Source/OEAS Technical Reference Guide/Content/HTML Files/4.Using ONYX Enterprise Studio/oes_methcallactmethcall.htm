Activating or deactivating a method call

Deactivated method calls behave similarly for both the Scripting feature and the Merge Template feature:

  • Existing scripts or merge templates that use a deactivated method call continue to function normally.
  • Once a method call is deactivated, it is not available for use in a new script or merge template.
  • Once a method call is deactivated, it is not available to the script designer even when he or she is editing an existing script that uses the method call. It is also not available to the user editing an existing merge template that uses the method call. If the script designer or merge template designer chooses to save the edited method call or template, he or she must first select a new (active) method call to assign to it.

To activate or deactivate a method call:

  1. In the navigation area, select the method call you want to activate or deactivate.

The attributes of this method call appear in the edit area.

  1. Check or uncheck the Active check box to activate or deactivate the method call.
  2. Click to save your changes.
