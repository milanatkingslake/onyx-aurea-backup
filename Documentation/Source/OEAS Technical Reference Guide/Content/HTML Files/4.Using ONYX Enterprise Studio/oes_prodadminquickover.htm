Quick overview of the Product Administration tool

The user interface of Product Administration is divided into four areas:

  • Context area
  • Toolbar
  • Navigation area
  • Edit area

Context area - The context area lists the name of the application and the site being maintained.

Toolbar - The toolbar appears under the context area and contains the toolbar buttons that perform the tasks related to products.

The following toolbar buttons are available:

Icon Tool tip Function Context

Add

Adds a new product group, product, or incident product, depending on the current selection within the navigation area.

Enabled only when:

  • The top-level Products and Literature node or an existing static Groups node is selected.
  • A Products node under a product group is selected, or an existing product is selected.
  • An incident category node or incident product is selected.

Delete

Marks for deletion the currently selected product group node, product, or incident product within the navigation area.

The delete button is also used to undelete a previously deleted item, if used prior to saving the deletion.

Enabled only when:

  • A product group is selected.
  • A product is selected.
  • An incident product is selected.

Save

Saves all modifications made since the previous save.

Enabled after any changes have been made.

Help

Launches Help.

Enabled at all times.

Navigation area - The three different types of products that are administered with Product Administration are displayed in the navigation area within a navigation tree comprised of two main nodes:

  • Products and Literature
  • Incident Products

Literature products are simply products to which you have added literature information.

Products and Literature node

Within the Products and Literature node, the navigation area enables you to view and expand groups of products, and to view the products within those groups.

You can create additional sub-product groups within each product group, and products within these sub-product groups, creating a hierarchy of product groups and products. There is no limit to the number of nested sub-product groups you can create.

Incident Products node

Within the Incident Products node, the navigation area enables you to view and expand incident product categories, and to view the incident products within those categories.

When you select a product group or an individual product from within the Products and Literature node, or an incident product from within the Incident Products node, the attributes for that item display in the edit area, as described in the following section.

Edit area - The edit area displays the attributes for the item that you select in the navigation area, and you edit these attributes to modify the item.

Within Product Administration, you can modify the following items:

  • Product groups, within the Products and Literature node.
  • Products (including products and literature products), within the Products and Literature node.
  • Incident products, within the Incident Products node.

The attributes differ from item to item, depending on whether you select a product group, a product, or an incident product.

Data cells in the product edit area

Much of the product information is available in five expandable data cells within the product edit area. These five cells are:

  • Literature Information
  • Inventory Information
  • Warranty Information
  • User-Defined Information
  • Insert/Update Information

Clicking anywhere in the header of any of these data cells will expand the data cell to reveal the product information within it. Clicking an expanded data cell's header will contract it.
