Deleting a method call detail

After you mark a method call for deletion and save your changes, you cannot undo the deletion.

You cannot delete a method call detail if that method call detail is being referenced by a script. You must remove the association between the method call detail and the script before you can delete the method call detail. To remove the association, either delete the script item that references the method call detail, or reference a new method call detail to that item.

You can delete a method call detail belonging to a method call that is being used by a merge template, but you must then also update the merge template to use another method call or method call detail. Until you update the merge template:

  • If the deleted method call detail was an output, a raw output tag appears in the merge template text, because no field exists to merge.
  • If the deleted method call detail was an input, an error occurs when a user tries to execute the merge template.

You can also choose to deactivate the method call detail rather than deleting it, as described in Activating or deactivating a method call detail.

To delete a method call detail:

  1. In the navigation area, select the method call detail that you want to delete.
  2. Click to mark the method call detail for deletion.

    3. The method call detail name appears struck through in the navigation area (for example: newMethodCallDetail).

  3. Click to save your changes and delete the method call detail.

If you have marked the method call detail for deletion and have not yet saved the changes, you can still remove the mark for deletion.

To remove a mark for deletion from a method call detail:

  1. In the navigation area, select the method call detail that you previously marked for deletion.

Method call details that are marked for deletion appear struck through in the navigation area.

  1. Click to remove the mark for deletion from the method call detail.
  2. Click to save your changes.
