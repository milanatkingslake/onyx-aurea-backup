Disabling the subscription profile

Disable the subscription profiles of all users who no longer require offline permissions. You disable a user’s subscription profile by deactivating the subscription and synchronization settings for that profile.

  • Removing the selection of the Profile Active attribute prevents the user from explicitly subscribing to OEP records.
  • Removing the selection of the Sync Active attribute prevents the user from synchronizing data between the OEP-Offline client and the OEDB.

For additional information about the effects of removing the selection of the Profile Active and Sync Active attributes, see Deactivating and re- activating subscription profiles, and Deactivating and re-activating synchronization .

After you complete the following procedure and restart the OEP Web site, he or she can no longer subscribe to records for the disabled subscription profile or synchronize the client’s Outlook data store with the staging server data store.

To disable a user’s subscription profile:

  1. In the Security Administration navigation area, select the subscription profile for the user.
  2. To empty the user’s synchronization folders and messages in the staging server data store, select the Reset User at Next Sync attribute, and then click on the toolbar to save the change.

After the next server synchronization is complete, all the user’s folders are removed from the staging server data store.

  1. After the user’s folders are removed from the staging server data store, return to Security Administration.
  2. In the edit area for the user’s subscription profile, clear the check boxes for the Profile Active and Sync Active attributes of the user’s subscription profile.
  3. Click on the toolbar to save changes.
  4. Stop and restart the OEP Web site.

When the user next logs on, the subscription controls in OEP are no longer available.
