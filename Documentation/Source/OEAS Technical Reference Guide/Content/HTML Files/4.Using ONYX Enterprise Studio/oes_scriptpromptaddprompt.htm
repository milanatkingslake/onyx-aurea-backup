Adding a prompt

When you add a prompt, you also perform some of the following tasks. For more information on prompt control types, see Prompts. For the design options available for each prompt type, see within this section the section on the prompt type.

  • Specify the prompt’s control type.
  • If the prompt is a text box control, specify the data type.
  • Type the prompt’s text, including HTML tags and variables, if you wish.

OEP does not check any HTML or script in the text for validity or safety.

  • Add possible responses.

For procedures for working with fixed responses, see Adding, deleting, and changing the order of fixed responses.

For information on creating variable responses, see the section on the prompt type that you create.

  • Specify an input value for the prompt, if desired, and input and output variables.
  • Specify attributes such as whether the user is required to respond to the prompt.

The following procedure describes how to add a prompt to a step and specify the attributes that are common to all prompt types. To add a prompt to a step:

  1. In the Edit Layout area or in the navigation area, select the step to which you want to add a prompt.
  2. In the Edit Layout area toolbar, click .

OR

In the Edit Step area, click the Prompts tab, and then, in the toolbar above the Prompts box, click .

You can also right-click the step in the Edit Layout area, and then select Add Prompt from the menu that appears.

A new prompt, named newPrompt<n> (where “n” is an integer), appears and is selected in the Prompts box of the Prompts tab.

  1. In the Name box, rename the prompt.
  2. In the Control Type box, select a control type.

For a divider prompt, the only available attribute is the Active box. Skip to step 11.

  1. In the Text box, type the text that the user should see.

You can include HTML tags and/or scalar variables in the prompt text. For information on using the HTML Editor Web Page Dialog to format text in the prompt text, see Working with HTML tags in the header, footer, or prompts.

For a label prompt, the Default & Output Options area discussed in the following steps is not available.

  1. To specify a literal as the default input value for the prompt, under Default & Output Options, specify the value in the Default Value box.
  2. To specify a variable that provides the default input value, under Default & Output Options, specify the variable in the Default Variable box, do one of the following:
  • Select the variable from the drop-down list box.
  • Click the ... button and in the Variables Web Page dialog box either define or select the variable.

If you specify both a literal value and a variable and the variable contains a value, the variable provides the default value, overriding the value in the Default Value box.

If you use a list variable to populate a list of possible responses to a prompt and specify a default variable whose value is not included in the list provided by the list variable, then in the script session there is no default selection of a response to the prompt. The same is true of country and region prompts: if the value contained in the default variable does not match one of the countries or regions listed, then in the script session, there is no default selection of the country or region.

  1. If the prompt should populate a variable with the user’s response, then under Default & Output Options, select Output Variable, and then do one of the following:
  • Select the variable from the drop-down list box.
  • Click the ... button and in the Variables Web Page dialog box either define or select the variable.

If the prompt type is one of country, region, incident product, reference lookup, list box, or check box set and you specify that the user can select multiple responses, the Output Variable box is unavailable. For information on specifying the maximum number of responses, see the appropriate subsection below.

  1. If the user must respond to the prompt, select Required.

For a label or divider prompt, Required is not available.

  1. If the user should be able to see but not respond to the prompt, select Read Only.

For a label or divider prompt, Read Only is not available.

  1. If the prompt should be visible in the step, select Active.
  2. Set additional attributes for the prompt according to the following sections, which describe options to set for each prompt type.

For check box, divider, label, tracking code, and user prompts, there are no more settings to specify. For more information on these prompts, see Scripting.

  1. On the toolbar, click to save your changes to the database.
