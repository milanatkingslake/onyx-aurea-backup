Implementing a user-defined field

Using Reference Table Administration, you can configure user-defined fields by:

  • Defining the field type as text box, check box, or combo box.
  • Modifying the field caption that appears on the user-interface.
  • Creating, deleting, and modifying lookup values for the fields that you have defined as combo box field types.

However, there might be additional customization tasks that are required to implement the field. The following steps describe how to implement a user-defined configurable field.

To implement a user-defined field:

  1. Identify the user-defined property for which you want to configure a user-defined field.
  2. Review , for the OEAS feature associated with the user-defined property for which you are configuring a user-defined field. Reviewing the configuration options for that feature, determine if the corresponding user-defined field is defined by default in Reference Table Administration and if the user-defined field is supported by OEP by default (if the field is not supported by OEP by default, this is noted).
  • If the user-defined field is not defined by default in Reference Table Administration, follow the naming conventions described in Naming user-defined fields and use Reference Table Administration to add the user-defined field, as described in Adding new fields.
  • If the user-defined field is by default defined in Reference Table Administration, use Reference Table Administration to configure the user-defined field attributes, as described in Modifying a field’s attributes.
  1. If the user-defined field is not supported by OEP by default, customize the presentation tier to use the new field.

For information on customizing the presentation tier, see Customizing OEP.
