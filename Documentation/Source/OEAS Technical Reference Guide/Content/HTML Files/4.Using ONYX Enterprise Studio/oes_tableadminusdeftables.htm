Using the default tables

Default tables described

The following table provides descriptions of the default tables that you can administer in Table Administration. They are listed by the domain data that you can administer, and describe the OEDB tables that support the data.

The scope of Table Administration is not confined to these objects. You can add custom objects to Table Administration using the procedure "Adding a new object to Table Administration."

Data Table Descriptions Field Descriptions

Action Batch Status

The action_batch_status table is used to track the status of an action batch. Warning: Do not modify the fields New, In Process, Completed, or Errored for the action batch status domain data and the literature order line status domain data. This is OPS-specific domain data, and any changes to these fields can result in OPS failure.

See Setting up the OPS service and subsystems

Company Family

The company_family table stores records for grouping companies as families so that different companies can be related through one family record. The nature and definition of these families is not predefined and must be input.

See Creating company families

Country

The country and country_ml tables store all of the values designating the countries that are available to your users for different areas of your application. By default, the Onyx Enterprise Database provides a list of all countries with a corresponding abbreviation. If you must remove or edit countries, you can do this in Table Administration.

There is a hierarchical relationship in the OEDB between the country field (parent) in the Country table and the region field (child) in the region table. In the end-user interface, the regions displayed are dependent on the country selected. The relationships between the parent and child records is defined from the child region record which specifies the parent record in the country_code field in the region table. The region table can also be administered in Table Administration.

See Configuring country data

Currency

The currency and currency_ml tables store information related to different currency types, including a currency description, currency code, and locale ID.

See Configuring currency codes

Custom Templates

Custom Templates enables you to maintain a list of the various documents that are used as the templates for letters that you create for forecasting and quoting purposes.

 

Email Address Book Item

Email address queues are used to facilitate the creation and management of emails in the end user interface. They are stored in the EmailAddressBookItem table. Use this table to configure email address queues.

See Configuring email address book items (queues)

File Location

The file_location and file_location_ml tables store information (file paths and descriptions) about the file shares that are responsible for storing saved files. Every record in this table represents a path to a defined file share. The first active path entered in the table is used to determine the file’s storage location.

See Configuring file paths for attachments

Incident Category

The incident_category and incident_category_ml tables contain a row for each incident type (Sales, Service, or Support) as well as any additional types you configure.

Configuring additional incident categories is currently not supported.

Literature - Terms

The literature_term and literature_term_ml tables store literature payment methods.

See Configuring terms

Literature Order Line Status

The order_line_status table is used to track the status of literature orders. Warning: Do not modify the fields New, In Process, Completed, or Errored for the literature order line status domain data or the action batch status domain data. This domain data is OPS-specific, and any changes to these fields can result in OPS failure.

See Setting up the OPS service and subsystems

Merge Template Group

Use the MergeTemplateGroup table to configure template groups for email templates and to specify the parent (first) email template group for the tree hierarchy that appears in Template Maintenance in OEP.

See Creating template groups for templates

National Language

The national_language table stores all of the default message strings that ship with OEAS. It is used for customizing captions, labels, column headings, and messages that appear in the end-user interface. When you modify a string in the national_language table, your changes are stored in the nat_lang_config table of the OEDB. During software upgrades, having two tables ensures that customized strings remain untouched in the nat_lang_config table while new message strings for the application are added to the national_language table.

See Modifying message strings in the National Language table

National Language Category

The nat_lang_category and nat_lang_category_ml tables drive the list of categories that are used to filter National Language strings.

See Modifying message strings in the National Language table

Region

The region and region_ml tables store the information for country codes, region codes, and region names.

See Configuring region data

Relative Date

Relative dates can be configured to calculate the date ranges used to filter searches for emails in Task Manager. The calculation is based on the current date. Use the RelativeDates table for configuring relative dates.

See Configuring relative date ranges for the Email feature

Search Operator

The search_operator and search_operator_ml tables store List Manager domain data.

See Configuring search operators

Search Result Default

The search_result_default table specifies the default columns returned in the result set for each search view. This table is part of the List Manager feature.

See Configuring search result defaults

Search Table

The search_table and search_table_ml tables store List Manager domain data.

See Configuring search tables

Search View Details

The search_view_detail and search_view_detail_ml tables store SELECT clause and join information, which is used to build queries in List Manager.

See Configuring search view details

Search Views

The search_view and search_view_ml tables are used to specify sections of the database to be searched when creating a list in the List Management window. By default, the Onyx Enterprise Database searches Company and Individual views, which means that only the selected customer type is searched when a list is created.

See Configuring search views

Service Printer Info

The literature_service_printer and literature_service_printer_ml tables store information about the printers used for literature orders.

See Configuring printers

Ship Via

The literature_ship_via and literature_ship_via_ml tables store information listing the shipping methods. Users select from these methods to indicate the shipping method for a literature order.

See Configuring ship methods

Staging Server

The Staging Server table stores the name of the staging server and the user ID that is used for the synchronization of subscribed data.

 

Table Administration Item

The table_administration_item and table_administration_item_ml tables contain records which correspond to the default domain data tables listed in Table Administration. To use Table Administration to administer a table other than those installed by default, you must add it as Table Administration Item domain data.

See Configuring table administration items

Task Category

The task_category and task_category_ml tables contain a row for each task type (Sales, Service, or Support) as well as any additional types you configure. The task_category table uses a foreign key (incident_category_did) to associate a task type to its corresponding incident type. Only one task type is allowed for each incident type.

Configuring additional task categories is currently not supported.

Template

The Template table enables you to maintain a list of the various documents that are used as the templates for letters that you create for forecasting and quoting purposes.

See Configuring quote templates
