Adding a publishable object

Domain data is made available for publishing by creating a publishable object of the business object or view that identifies the data. Once you create the publishable object, it is added to the list of publishable objects that you can select for publishing.

For assistance determining an appropriate retrieval object and retrieval method when creating a new publishable object, see “Adding an object to domain data publishing” in the OEP-Offline Developer’s Reference.

To add a publishable object:

  1. In the navigation area, select the Publishable Objects node or an existing publishable object and click .

A new publishable object named newPublishableObject<n> appears in the navigation area followed by an asterisk (*).

  1. In the edit area, define the following attributes for this publishable object:

Name - This attribute is the user-friendly name you provide for the publishable object. It is meant to be understood by end-users who will be publishing domain data from Subscription Designer. There is no requirement for the name to have any particular relation to the underlying OEAS object or method. The name can be a maximum of 127 characters.

Description - The Description attribute is the brief description that you provide to explain the use of the publishable object to the intended end-users of Subscription Designer. The description can be a maximum of 255 characters.

Retrieval Object - This is the business object that you want to use to retrieve the domain data you want to publish. In many cases, this is the domain object for the particular domain data you want to publish. You can also choose the publishableObject object for its extract method, as described in Retrieving and filtering publishable data.

If you fail to provide values for the Name and Description attributes, values are provided when you select a retrieval object.

Retrieval Method - This is the method that you select to retrieve the domain data. In Subscription Designer, you select this value from a list of all available methods for the object that you specified for the Retrieval Object attribute.

Retrieval Method XML - This is the input XML used when executing the retrieval method. The input XML is derived from the OED based on the retrieval method you specify. The Retrieval Method XML can be a maximum of 1000 characters.

Publication Filename - This is the filename for the input XML created to retrieve the domain data you want to publish. It represents the file for this particular domain data that is published to the OEP-Offline deployment server. This filename is created automatically in Subscription Designer using the name you provide for the Name attribute, appending .xml to the name provided. The Publication Filename can be a maximum of 255 characters.

The Onyx naming standard for the publication filename is <LBO>.xml.

Do not change the publication filename for the default publishable objects installed with OEAS. These filenames are required by the OEP-Offline client. For a list of the default publishable objects and their corresponding publication filenames, see Default publishable objects.

Active - The Active attribute controls whether or not the publishable object is selected by default when you access the Publish Data screen. For more information about the Active attribute, see Activating or deactivating a publishable object.

  • Click to save your changes.
