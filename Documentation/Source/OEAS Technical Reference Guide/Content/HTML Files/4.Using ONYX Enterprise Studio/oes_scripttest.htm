Testing scripts

Before you publish the script, be sure to test it. Because copying scripts between servers is not supported, you must test the script in the production environment.

To test a script:

  1. In the Script Designer, in the Edit Script area, make sure the Active check box is selected and the Published check box is not selected.
  2. On the toolbar, click to save your changes to the database.
  3. To refresh just the selected script in OEAS, click in the toolbar, above the navigation area.

OR

To refresh all scripts in OEAS, click in the toolbar, above the right side of the edit area.

  1. Log in to an end-user application such as Onyx Employee Portal with a user account that has access to the UI:OEP:scripting.designer resource.
  2. In an end-user application such as OEP, create or identify a test individual, company or incident whose records you can insert, update, or delete, as the script requires.
  3. Using the end-user application run the script to confirm the following:
  • All prompts and rules perform correctly.
  • Method calls associated with the script’s steps correctly insert, modify, or delete any data they modify.
  1. Using the Script Designer, correct any errors in the script.
  2. Repeat the procedure until the script runs as desired.
