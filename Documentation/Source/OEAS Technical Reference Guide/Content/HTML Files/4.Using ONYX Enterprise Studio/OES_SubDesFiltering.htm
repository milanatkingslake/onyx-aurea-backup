Filtering secondary object cascaded subscriptions

You can set the Cascade only the secondary objects that meet this filter expression rule for a subscription relationship at both the relationship level and the role level, as described in Adding a subscription relationship, and Adding a subscription role. To set this rule, you can either manually type the filter expression into the provided text field, or use the filter helper menu to select an appropriate filter expression.

Adding a filter criteria manually

The text you provide for the Cascade only the secondary objects that meet this filter expression field is the expression that is used to filter cascaded subscriptions for the secondary object in the relationship. The expression must be in the following format: child.<field><SQL operator><value>, where:

  • child. represents the secondary object in the relationship and includes the literal text "child."; must be placed at the beginning of your expression.
  • <field> represents the column of the database table (which contains the secondary object) on which you want to filter the cascaded subscriptions.
  • <SQL operator> represents a valid SQL comparison or character operator (for example, =, <, <>, <=, etc., or BETWEEN, IN, LIKE, NOT IN, etc.).
  • <value> represents the value you want to filter on.

For instance, suppose you provide the expression child.incident_category_did=1 for a relationship where company is the primary object and incident is the secondary object. When a company object is subscribed to, cascaded subscriptions are created only for those incidents where incident_category_did=1 (support incidents).

There is no validation performed on filter expressions. If you provide an expression that is not valid SQL, or that references a field that does not exist in the secondary object’s underlying database table, the filter expression could be disregarded, or cascading subscriptions for that relationship could fail altogether.

Adding a filter criteria using the filter helper menu

A filter helper menu button, , appears to the right of the text field for the Cascade only the secondary objects that meet this filter expression attribute. When you click this button, a filter helper menu appears from which you can select an appropriate filter for cascading secondary object subscriptions.

The filter helper menu is enabled only for those relationships for which a default filter expression exists. To add a default filter expression to the filter helper menu, see Configuring filters for Subscription Designer’s filter helper menu.
