Adding conditions

A territory can have any number of conditions , which are typically used for:

  • Associating companies, individuals, and incidents with territories.
  • Creating internal contacts for individuals, companies, and incidents.
  • Assigning incidents to OEP users.

To add a new condition:

  1. In the navigation area, expand the appropriate territory.
  2. Select an individual, company, or incident condition node, and then click .

A new condition named newCondition<n> appears in the navigation area, followed by an asterisk (*). When you expand the newCondition<n> node, the nodes for child expressions and linked assignment sets appear.

  1. In the Edit Condition area, define the attributes for this condition.
  • Name - Type the name you want to assign to this condition. You can edit this name after a save.
  • Description - Type a description of this condition. You can edit this description after a save.
  • Active - Clear this check box if you want to deactivate this condition. The default is active (selected).
  • Click to save your changes.
  • The new condition is displayed under the appropriate company, individual, or incident condition in the Assignment Tree.
