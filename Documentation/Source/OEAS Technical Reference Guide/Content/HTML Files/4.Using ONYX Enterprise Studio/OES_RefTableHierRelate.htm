Hierarchically relating combo box fields

The Parent Field attribute of a configurable field enables you to hierarchically relate two combo box fields to one another (two user-defined combo box fields). This option is enabled only when you select Combo Box as the field type for a configurable field. You select the parent field name (for example, company.user1) from the Parent Field drop-down box.

Hierarchical relationships are defined from the child field to the parent field. For any fields to be hierarchically related, you must define this parent/child relationship in the properties of the child field. If a field is a combo box field and is not hierarchically related to another field, choose None from the Parent Field box.

To hierarchically relate two fields:

  1. In the navigation area, select the Fields mode option.
  2. Click to display the fields associated with a particular node.
  3. Select the field that you want to assign as the child field in a hierarchical relationship. That field’s attributes display in the edit area.
  4. In the Parent Field box, select the field that you want to set as the parent of the current field. Notice that as you modify the field attributes, the field name in the navigation area displays an asterisk (*) and the field moves below the assigned parent field within the navigation tree.
  5. Click on the toolbar to save your changes.

To remove a hierarchical relationship between fields:

  1. In the navigation area, select the Fields mode option.
  2. Click to display the fields associated with a particular node.
  3. Select the child field that is currently in a hierarchical relationship. That field’s attributes display in the edit area.
  4. Select None from the Parent Field box.
  5. Click on the toolbar to save your changes.
