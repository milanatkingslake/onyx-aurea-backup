Managing groups

In Security Administration, groups are an organizationally based collection of users. Groups can be nested within groups. However, a user can be a member of only one group.  

There is no functionality to delete a group, you must deactivate it. See Activating or deactivating a group for further details.

The Users node that appears within the Groups node is created by default and is actually not a group, it is a container area for users (Onyx user accounts) who are members of this group. In order to make changes to a user, you must open the group that the user is a member of and select the user.

You cannot add groups to resources or grant or deny them access permissions to resources, and you cannot add groups to roles.

Managing groups involves the following tasks:

  • Adding a group
  • Activating or deactivating a group
  • Modifying a group
  • Moving a group
