Working with method calls

After using the Method Call Designer to create a method call, you can associate the method call with a script step. Doing so involves binding script variables to inputs and outputs for the method call. Therefore, the script design should specify which variables will be bound to the method call inputs and outputs.

For more information on method calls and on planning considerations related to using them in scripts, see Method Calls. For information on using the Method Call Designer, see Method Call Designer Help.

You can create method call bindings one of two ways:

  • Manually, using only the Method Call Bindings Web Page Dialog.
  • Automatically, using both the Method Call Bindings and the Method Call Binding Builder Web Page Dialogs. Using the Method Call Binding Builder Web Page Dialog to create bindings automatically also creates the variables that store the method call input and output values.

To associate a method call with a step using the Method Call Bindings Web Page Dialog:

  1. With the step selected in the navigation area, click the Method Calls tab in the Edit Step area.
  2. On the Method Calls tab, under Method Calls, click .

By default, (blank method call) appears in the Before Load box.

  1. In the Method Call box, click the down arrow and select the appropriate method call.

In the Before Load box, (blank method call) changes to the name of the method call that you selected.

  1. Beside Call Type, select either Before Load (for a pre step method call) or After Save (for a post step method call).

If you select After Save, the method call moves to the After Save box.

  1. To specify the input and output data for the method call, click Bindings.

The Method Call Bindings Web Page Dialog appears.

  1. In either the Input Bindings or the Output Bindings area, according to whether you are specifying bindings for input or output data, click .
  • Input bindings hold data that the script passes to the method call.
  • Output bindings hold data that the method call returns to the script.
  1. If you add an input binding, specify the following:
  • Name - The name of the input parameter or method call detail, as defined with the Method Call Designer.
  • Binding Type - How the data is passed to the method call: empty, literal, null, or a variable code or variable text. For more information on the binding types, see Method Calls.

If you leave the binding type blank, no value is bound to the method call detail. If you leave the binding type blank and there is a default value defined by the method call, the method call uses that value.

  1. Variable Name - The name of the variable that stores the data. You can either select the variable name from the list or click the ... button and select the variable from the list in the Variables Web Page Dialog, which provides more information about the available variables. This column is available only if the binding type is variable code or variable text. You can differentiate the names of variables that you use to store the data by giving all the inputs and outputs for each method call a unique prefix.
  2. The variable for the input cannot be a list variable.
  • Literal - The literal string. This column is available only if the binding type is literal.
  • Active - Whether the method call binding is active.
  1. If you add an output binding, specify the following:
  • Name - The name of the input parameter or method call detail, as defined with the Method Call Designer.
  • Variable Name - The name of the output variable that the method call populates. You can either select the variable name from the list or click the ... button and select the variable from the list in the Variables Web Page Dialog, which provides more information about the available variables. Within a script, each variable used to store an output binding must have a unique name. You can differentiate the names by giving all the outputs for each method call a unique prefix.
  • Active - Whether the method call detail is active.
  1. Both input and output bindings also have an ID column. When you save the changes, OEAS provides the ID for the detail.
  2. In the Method Call Bindings Web Page Dialog, click OK.
  3. Click to save your changes to the database.

To create method call bindings automatically:

  1. With a method call selected (see steps 1 through 5 of the preceding procedure), click Bindings to open the Method Call Bindings Web Page Dialog.
  2. Depending on whether you want to build method call input bindings or output bindings, click on the Input Bindings or Output Bindings area toolbar.

The Method Call Binding Builder Web Page Dialog appears.

The following screenshot is for the inputs version of the Method Call Binding Builder Web Page Dialog. Apart from the input- or output-specific labels, the Web Page Dialog and the procedure for using it are the same for input bindings and output bindings.

  1. From the Unused Method Call Inputs or Unused Method Call Outputs list, select those which you want the method call to use.
  2. To attach a prefix to the method call’s input or output variables, type the prefix in the Prefix box.

Do not start the prefix with an underscore. Leading underscores are reserved for context variables.

  1. To set the scope of the input or output variable, make the appropriate selection by Scope.

To get best performance in script sessions, specify step-local scope for list variables that store many list elements.

  1. Click Build.

The Method Call Bindings Web Page Dialog appears, displaying any input or output bindings that you built.

  1. When you have finished creating bindings for the method call, click OK.
  2. Click to save your changes to the database.

To change an attribute of an input or output binding:

  1. Open the Method Call Bindings Web Page Dialog.
  2. Select the input or output binding whose attribute you want to change and make the desired change in the appropriate column.

The Method Call Binding Builder Web Page Dialog is for adding new inputs or outputs. You can not use it to change input or output attributes.

  1. Click OK.
  2. Click to save your changes to the database.
