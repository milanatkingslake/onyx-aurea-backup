Managing user access with Security Administration

Security Administration enables you manage user access to the following resources: OEAS business object methods, user interfaces of OES and OEP, and any other user interface or entity that you create as a resource in Security Administration.

In discussion of Security Administration, the sa user refers to the Onyx sa user, which is the Onyx system administrator.

Although security services in OEAS include user authentication and licensing as well as user access permissions, Security Administration is not the tool for all matters of security. You do not manage licensing with this tool. You use Security Administration to specify which kind of authentication (Onyx authentication or integrated Windows authentication) users log in with, and each user’s login name and password.

Managing User Access involves the following tasks:

  • Securing business objects methods
  • Adding users to the Administrator role
  • Changing the default password for Security Administration
  • Work flow for managing user access
