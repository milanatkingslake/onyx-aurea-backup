Working with rules

The main parts of creating a rule are:

  • Building a condition statement, if any.
  • Specifying the destination step for the rule.

To add a rule to a step:

  1. In the Edit Layout area, select the step to which you want to add a rule.
  2. In the Edit Layout area toolbar, click .

OR

In the Edit Step area, click the Rules tab, and then, in the toolbar above the Rules box, click .

You can also right-click the step in the Edit Layout area and then select Add Rule from the menu that appears.

On the Rules tab in the Edit Step area, a new rule, named “newRulen,” appears and is selected in the Rules box.

  1. In the Name box, rename the rule.
  2. In the Condition box, type the rule’s condition statement.

When you type the condition statement, you can type either the code or the text form of the variable. However, the code form (for instance, {variablename} ) is recommended, because the text form of the variable can change.

OR

Create the condition statement using the Condition Builder area on the Rules tab.

In condition statements, nonnumeric values must have single quotation marks around them.

  1. In the Destination Step box, click the down arrow and select the step that should follow if the condition is true.

Each rule must have a destination step specified.

  1. Click to save your changes to the database.

You can change the destination step of a rule or access the rule’s attributes in the Edit Layout area. To change the destination step of a rule, click the rule and drag it to the new destination step. To access the rules tab for a specific rule, click the rule.

To use the Condition Builder to create a condition statement that evaluates a user-entered value:

  1. On the Rules tab, under Condition Builder, select By Variable.

When you select By Variable, the drop-down list boxes below are labeled Available Variables and Operators for building boolean statements.  

  1. Select one or more variables in the <select a variable> box.

You cannot use a list variable in the condition statement.

  1. Select one or more operators in the <select an operator> box.

The condition statement as you have built it so far appears in the Condition box.

You can build statements that describe multiple conditions. Each variable or operator that you select is inserted at the current cursor position.

  1. In the Condition box, make any necessary adjustments to complete the condition statement correctly, such as:
  • Moving operators or variables.
  • Typing values.
  • Supplying any necessary sets of parentheses.
  1. Click to save your changes to the database.
  2. Finish creating the rule as described in the previous procedure, see To add a rule to a step.

To use the Condition Builder to create a condition statement that evaluates the user’s choice from a fixed list of responses:

  1. On the Rules tab, under Condition Builder, select By Response.

When you select By Response, the drop-down list boxes below are labeled Prompts and Responses.  

  1. Select a prompt from the Prompts list.
  2. Select a response from the Responses list.

To complete the condition statement, the prompt must have an output variable that stores the user’s response. If it does not, you must include the variable in the condition statement for the rule.

The output variable for the that you choose appears in the Condition box. The response that you choose appears as a value, the response’s response code. The operator is equivalence (=). The condition is built as ({variablename} = ‘responseCode’).

You can make multiple choices of prompts and responses as you build statements that describe multiple conditions. Each variable or operator that you select is inserted at the cursor position or is appended to the end of the statement, so you must supply any necessary AND and OR operators and sets of parentheses.

  1. Click to save your changes to the database.
  2. Finish creating the rule as described in the previous procedure, see To add a rule to a step.
