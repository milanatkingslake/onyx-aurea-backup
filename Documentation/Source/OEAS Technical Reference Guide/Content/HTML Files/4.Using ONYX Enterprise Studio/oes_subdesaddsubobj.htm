Adding a subscribable object

Use the following procedure to add a subscribable object.

All objects that are supported in OEP-Offline are created as subscribable when OEAS is installed, however when you add a new subscribable object with Subscription Designer, you are completing only part of the process of adding a new object to OEP-Offline.

To add a subscribable object:

  1. In the navigation area, within the appropriate subscription service node to which you want to add a subscribable object, select the Subscribable Objects node.

The Subscribable Objects view appears in the edit area.

While this tool enables you to make reminder, workNoteHeader, and workNoteDetail objects subscribable objects, Onyx does not recommend doing so. These objects, along with the workNote object, contain hard-coded relationships that already provide correct cascading subscription behavior. In addition, Onyx does not recommend modifying the default subscription relationships provided for forecast and quote objects. Failure to heed these recommendations can cause duplicate data in OEDB.

  1. In the edit area, select the OEAS business object that you want to add as a subscribable object.

The list of available objects contains only those transactional objects in the OED that have not yet been made subscribable.

  1. Click Make Subscribable.

A new subscribable object (with the same name as the original OEAS business object) appears in the navigation area followed by an asterisk (*).

  1. In the edit area, define the attributes for the subscribable object.

Object - This is the name of the object that you selected to make subscribable. You can change your selection from the provided list of available objects, which again only includes those objects that exist in the OED that have not yet been made subscribable.

Object Type - The object type is not editable as it reflects the object type value for the underlying OEAS business object, as defined within the OED.

Top-Level Object - This attribute identifies a top-level object. An object identified as a top-level object is recognized by the cascade process as an object that requires no further upward cascaded subscriptions as it is not considered a secondary object to any other objects.

Collision Handler ProgID - The Collision Handler ProgID identifies the collision handler you want to use for the subscribable object. OnyxCollisionHandler.CollisionHandler is the Prog ID of the collision handler that is provided with OEP-Offline. It is a registered COM component. The Collision Handler ProgID can be a maximum of 255 characters. For information about customizing the collision handler, see Add custom collision handling. If a Collision Handler ProgID is not specified, no collision handling is performed (no collisions are automatically resolved) and all collisions are reported to the offline user.

Extra Data XML - The Extra Data XML attribute supports the subscribableObject metadata. It contains well-formed XML that includes the data passed into the extraData node of the CollisionHandler and the data passed into the extraData node of the sync subsystem. The Extra Data XML can be an unlimited number of characters. For more information about this attribute, see Understanding Extra Data XML.

Active - The Active attribute controls whether or not the subscribable object is available for subscription.

  1. Click to save your changes.

After you have added a new subscribable object, you can add subscription relationships to enforce appropriate cascaded subscriptions of related objects. For more information, see Defining subscription relationships .
