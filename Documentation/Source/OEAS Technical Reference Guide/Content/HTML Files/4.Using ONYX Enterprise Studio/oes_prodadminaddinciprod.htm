Creating an incident product

You can also create an incident product within a single incident category and then copy that incident product to other incident categories. See Copying an incident product for details. This is possible because the product number for an incident product must only be unique within a particular incident category.

To create a new incident product:

  1. Expand the Incident Products node.
  2. Select the Incident Category node of the incident category in which you want to add the new incident product, and click .

OR

Expand the Incident Category node of the incident category in which you want to add the new incident product. Select an existing incident product within the expanded Incident Category node and click .

  1. In the edit area, define the attributes for this incident product.
  • Product Number - Type the product number for this incident product. This is a required field and accepts a maximum of 20 characters. The product number must be unique within the incident category. You cannot edit the Product Number attribute once the new Incident Product is saved.
  • Description - Type the incident product description (or name). This is a required field and accepts a maximum of 50 characters.
  • Resolution Group - From the dropdown list select a resolution group. For further information, see Reference Table Administration Help.
  1. Click on the toolbar to save your changes.
