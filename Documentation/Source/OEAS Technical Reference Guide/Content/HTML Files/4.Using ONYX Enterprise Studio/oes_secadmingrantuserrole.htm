Granting a user or role access to a resource

There are two distinct actions or situations that give users or roles access to a resource.

  • You add the user or role to the access-permissions list of a UI resource, Other resource, or a secured Business Object resource.
  • You grant access to a user or role that is already on the access-permission list for a resource but has been denied access.

To add a user or role to a resource's access-permissions list:

  1. In the navigation area, select the resource.
  2. Click Manage Permissions.
  3. In the navigation area, click the user or role.
  4. Either drag the user or role to the heading area of the resource's access-permissions list or, in the resource's edit area, click the Add button.
  5. When you have finished managing permissions, click Done.
  6. Click on the toolbar to save your changes.

To grant access to a user that is currently denied access:

This procedure assumes that the user to which you want to grant access is currently a member of the access-permissions list and has been denied access.

  1. In the navigation area, select the resource.
  2. Click Manage Permissions.
  3. In the resource's edit area, click the user in the access-permissions list and then click Grant/Deny.

Use the Grant/Deny button to toggle between granting and denying permission to a user that is selected in the access-permission list. When you add a role to the list, it is automatically granted access; remove it from the list if you do not want it to have access permissions. The purpose of the Grant/Deny button is to control access for individual users. For example, if you have added a role to the access-permission list for a resource, you can use the Grant/Deny button to deny access to any users within that role. Conversely, you can grant access to individual users regardless of their role membership.

  1. When you have finished managing permissions, click Done.
  2. Click on the toolbar to save your changes.
