Reset data flow

The following steps represent the data flow when a user’s subscribed data is reset:

  1. Using Security Administration, the administrator checks the Reset User at Next Sync check box, and clicks Save.
  2. During the upsync of the next synchronization cycle:
  • The sync subsystem deletes all of the user’s folders in Exchange. Any unsynchronized changes (including items not yet inserted into the OEDB) are lost.
  • The sync subsystem calls the subscriptionItem.reset method in OEAS.
  • The reset method marks all subscribed items for the user to be synchronized to OEP-Offline by changing their sync_action to “1”, and deletes the subscription_item record for any items that are no longer subscribed online.
  • The reset method clears the reset flag on the user’s subscription profile.
  • The sync subsystem creates a new set of folders in Exchange for the user.
  1. During the downsync portion of the synchronization cycle, the subscription items that were reset begin downsyncing to the newly created folders.
