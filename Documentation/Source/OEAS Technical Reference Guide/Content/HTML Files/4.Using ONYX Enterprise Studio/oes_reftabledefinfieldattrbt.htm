Defining field attributes

When you have selected a configurable field in the Fields mode, the Table Administration edit area displays the attributes for that field. Use the following descriptions to assist you in defining field attributes when you are “Adding new fields” or “Modifying a field’s attributes.”

Field Name - Field names are case-insensitive and require unique names that are a maximum of 30 characters. Case-insensitive means that you cannot create duplicate field names that differ only by case. For example, you cannot create company.user1 (lower case) and then company.USER1(upper case); the application will not save the duplicate name.

The unique field names you create reference certain fields in the database that are used by the application objects, database stored procedures, and user interface. The naming convention used in a field name is <element>.<fieldname><n>. This standard naming convention helps identify the location of each field and is essential for OEAS and OEP support of user-defined fields. For specific details, see Naming user-defined fields.

Do not alter field names of the configurable standard fields as the system objects depend on these specific field names to function properly.

Caption - Type the caption you want to appear in the end-user interface. It enables the update of captions for user-defined fields only, up to a maximum of 30 characters.

Captions for all fields, except user-defined configurable fields, are modified in resource files.

Sequence # - The sequence number is used to control the relative position of user-defined fields as they appear in application windows.

Field Type - Reference Table Administration supports three field types: Text box, Check box, and Combo box.

If you change a field's Field Type property from a combo box to a check box or text box, you will lose any lookup values associated with that field. Subsequently, if you change the field's Field Type property back to a combo box, the lookup values that were previously associated with that field are no longer associated with it. To return the combo box to its previous state, you must manually re-add the lookup values to the field.

  • Text Box - Text boxes are editable fields in which users can enter and modify text. Data entered in a text box is stored in the database as unicode data.
  • Size - Size limits the number of alphanumeric characters a user can enter into a text field (0 - 255 characters). By entering 0 here, users can enter the maximum number of characters (255) into a text field.
  • Mask - Field masks control the format of data entered into user-defined text fields. Masks applied with Reference Table Administration are supported by user-defined fields only. Field masks can support up to 255 characters. For additional details on configuring a field mask for a text field, see Configuring a field mask.
  • Combo Box - Combo box fields are fields that have drop-down lists of text values. A combo box field presents the user with a drop-down list of values from which to choose, but does not enable users to enter new values or text. You can use Reference Table Administration to define the list of values that appear in a combo box field, as described in Adding lookup values to a combo box field. User-defined combo box fields can be hierarchically related to one another, and can also share the same lookup values.
  • Parent Field - You can use this attribute to hierarchically relate one combo box field to another combo box field. This control is enabled only when you select combo box as the field type. It is important to note that hierarchical relationships are defined from the child field to the parent field. You select the parent field name, for example, company.user1, in the Parent Field drop-down box. If you do not want the combo box field to be hierarchically related to another field, choose None from the drop-down list.
  • Related to Existing Field - This field relates one combo box field to another, enabling two fields to share the same lookup values. Subsequently, you can modify the lookup values for either field, and the changes are applied for both fields.
  • Check Box - Check box fields have boxes which users can check or uncheck to activate or deactivate an option. The check box appears directly beside the caption of the field. Check box data is stored in the database as a variable character with a value of 1 (checked) or 0 (not checked).
