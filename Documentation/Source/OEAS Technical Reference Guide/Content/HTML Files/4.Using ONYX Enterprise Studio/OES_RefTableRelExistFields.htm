Relating existing fields

The Related to Existing Fields attribute of a configurable field enables you to relate two combo box fields, allowing those two fields to share the same lookup values. When two fields are linked, any modifications to the lookup values of one field are automatically applied to the related field.

Although this option enables the same values to appear in different combo box fields, the drop-down values saved by the user against customer records for each field are saved and stored individually and are not related to each other. In other words, the data entered and saved from the user interface in one field does not update the other field.

The Related to Existing Field attribute is enabled only when you select Combo Box as the field type for a configurable field. You select the related field (for example, individual.user1) from this drop-down box.

If a field is a simple combo box field (not related to another field), choose None from the Related to Existing Field drop-down box.

To relate one field to another:

  1. In the navigation area, select the Fields mode option.
  2. Click to display the fields associated with a particular node.
  3. Select the field that you want to relate to another field. That field’s attributes display in the edit area.
  4. In the Related to Existing Field box, select the name of the field to which you want to relate the current field. Notice that as you modify the field attributes, the field name in the navigation area displays an asterisk (*).
  5. Click on the toolbar to save your changes.

To remove a relationship from one field to another:

  1. In the navigation area, select the Fields mode option.
  2. Click to display the fields associated with a particular node.
  3. Select the field for which you want to remove a field relationship. That field’s attributes display in the edit area.
  4. In the Related to Existing Field box, select None.
  5. Click on the toolbar to save your changes.
