Managing roles

You can grant users the permission to use certain OEP features either by assigning them to one or more roles, or by granting them access to specific resources, or both.

Roles provide a means to grant (or deny) a broad set of users access to a common set of functionality (resources). Implementing role-based security for a collection of users is much more efficient than specifying access permissions to each user individually.  

When you assign a role to a resource, all users who belong to that role have the permission to use that resource. You can still, however, deny specific users from accessing a resource. Denying users from accessing a resource overrides permissions that are set via role membership.

Managing roles involves the following tasks:

  • Adding a role
  • Adding a user to a role
  • Removing a user from a role
  • Deleting a role
