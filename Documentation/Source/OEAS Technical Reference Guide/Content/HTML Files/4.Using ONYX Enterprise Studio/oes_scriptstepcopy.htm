Copying a script step

Making a copy of a script step can save you work, but remember to do the following in the copy, after pasting the copy:

  • Check each rule in the copy of the step to make sure that its condition statement and destination step are correct.

Copying a step also copies its rules. When you paste the copy, the copies of the rules originate in the new step but have the original destination step.

  • If you change variables in the new step, change the variable name wherever it’s used.
  • Check usage of method calls to make sure that they’re still appropriate.

To copy a script step:

  1. In the navigation area or Edit Layout area, select the step that you want to copy.
  2. To copy the step to the Clipboard, on the toolbar, click .
  3. To paste the step, on the toolbar, click .

The new step appears in both the navigation area and the Edit Layout area. The new step is named <originalStepName><n> (where “n” is an integer).

You can paste the step into a different script by selecting, in the navigation area, the steps node or a single step in the target script, and then pasting the step. The step’s rules are included in the new step. Each rule has its original condition statement, but the destination step is blank. No variables are copied to the new script.
