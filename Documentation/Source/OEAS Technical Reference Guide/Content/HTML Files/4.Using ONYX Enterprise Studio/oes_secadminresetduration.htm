Reset duration considerations

There are several factors that can affect the duration of a reset that you should take in to consideration before resetting a user. These include:

  • The number of items a user has subscribed to (i.e., the number of items in the subscription_item table for the user).
  • The maximum number of items the syncServer downsyncs per cycle for a given user (Maximum Items Downsynced per Cycle). This value is set to 1000 by default, but you can modify it using OPS Administration. For more information about changing this setting, see Adding and configuring a synchronization subsystem.
  • The number of subsystems and threads per subsystem.

Based on these factors, it might take multiple synchronization cycles to downsync all of the items.

Example

Assume the following configuration:

  • 1 subsystem with 4 threads.
  • 400 users to synchronize.
  • Run interval set to 3 minutes.
  • A typical cycle with little or no data to synchronize takes about 3 minutes.
  • Downsyncing 1 item takes about 300 ms.

If you reset a user who has 10,000 items:

  • The next 10 cycles will each downsync 1,000 items for this user.
  • This adds 300 seconds (5 minutes) for 1,000 items on one thread.
  • The 10 cycles will then last 5 to 8 minutes each, depending on when this user is handled (at the beginning or at the end of the cycle).

If you reset 12 users at the same time, then the next 10 cycles will last 18 minutes each (based on the formula of 12/4 threads * 5 minutes + 3 minutes = 18 minutes).
