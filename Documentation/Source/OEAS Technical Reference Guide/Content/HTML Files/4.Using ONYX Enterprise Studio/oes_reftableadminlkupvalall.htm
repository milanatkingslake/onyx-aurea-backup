Configuring lookup values system-wide

The Lookup Value - All mode enables you to administer lookup values system-wide. Changes made in this mode effect all instances of the lookup value.

For example, if you delete a lookup value in this mode, all instances of that lookup value are logically deleted from all combo boxes in the database. If you change the description of a lookup value, all instances in all combo boxes are changed.

Consider the example that was described in Modifying a lookup value for a combo box field. In that example, where the lookup value was modified from within the Lookup Value by Field mode, the lookup value was only modified for field a and field b (field b because it was related to field a). Had the lookup value been modified from within the Lookup Value - All mode, all instances of the lookup value would have been modified, for fields a, b, and c.

In Lookup Value — All mode, you can create duplicate field names that differ only by case (ABC = abc). However, you cannot do this in either Lookup Values by Fields mode or Fields mode, because field names in these modes are case-insensitive.
