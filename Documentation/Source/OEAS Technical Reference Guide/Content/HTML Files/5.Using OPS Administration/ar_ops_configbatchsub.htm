Configuring batch subsystems

Working with batch subsystems

After creating OPS connections, you are ready to work with batch subsystems. You can have multiple subsystems for each connection. The batch subsystems, which process action batches, are configured and controlled independently of each other. Using OPS Administration, you can start, pause, continue, or stop a subsystem. For more information about batch subsystems, see Batch subsystem processing.
