Adding and configuring monitored mailboxes

Before configuring mailboxes in the Edit Mailbox area, ensure that you have completed the following tasks:

  • For each monitored mailbox that you configure in OPS Administration, set up a corresponding email account and password on the network mail server.
  • For each email account on the network mail server, created mail folders for storing processed emails and error messages sent by the EMSs during post-processing.
  • Created an Onyx internal user (OPS user) in Security Administration that can be specified for each monitored mailbox in OPS Administration.

The user ID that is created for the OPS user in Security Administration must be specified in the Impersonated User ID field in OPS Administration. The user ID will appear in the insertBy property of the record created by the EMS. Impersonation is required for OEP email processing.

A monitored mailbox must receive only one type of email: either standard email messages or Email for Outlook messages. A mailbox that receives Email for Outlook messages requires a special setting in the Edit Mailbox area. For more information, see Adding and configuring a mailbox for Onyx Email for Outlook.

Use the following procedure to add a new mailbox in OPS Administration.
