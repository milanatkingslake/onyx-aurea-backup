Step 4: Create templates

After you decide what types of literature products you need, you can create and/or customize the templates you will use. The OPS Setup program installed sample templates.

There are two types of templates:

  • Open XML Word document with Mail Merge fields, used for the Print document fulfillment type
  • XSL template, used for email fulfillment types

For details on creating and customizing templates, see Creating templates in the OEAS Developer’s Reference section.

Be sure to save templates where they are accessible by OPSSvcUser on the OPS server. A standard approach for this is to create a file share for literature products and specify the file paths in UNC notation in Product Administration. You should grant read access to the share to the OPSSvcUser (for fulfillment) and to the OnyxEASUser (for emailing literature as attachments).
