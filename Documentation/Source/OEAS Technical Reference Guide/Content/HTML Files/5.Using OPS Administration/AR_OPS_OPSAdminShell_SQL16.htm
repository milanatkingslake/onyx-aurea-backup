Mailbox settings

Mailbox settings

The important fields for configuration are shown in the following tables.

Setting or option Description

Name

Type a unique name for the mailbox. This is a required field with a maximum of 50 characters.

Description

Type a description for this mailbox with a maximum of 255 characters. This field is optional.

Message Type

This is a required field that determines whether a specified format will be used to create the email message that is interpreted by the inbound email service.

  • Select Standard Format as the message type for standard email processing.
  • Select Save in Onyx Format as the message type for Email for Outlook email processing.

Impersonated User ID

Type the User ID for the Onyx user to impersonate when making the OTM calls to persist the email message and related objects in the database during email processing. This is a required field.

Mark for follow-up

This checkbox specifies whether a flag is applied to messages created from the mailbox.

Post-processing Action

This required field determines the action to take after a message is successfully processed by the subsystem. You can choose to archive messages, to permanently delete them, or to leave them in the inbox.

Maximum Processing Time (minutes)

Type the maximum number of minutes that this mailbox (inbox) is monitored by the inbound email service (EMS) before it moves on to the next mailbox. If this field is blank, the inbound email service processes every message in the inbox before moving on to the next.

Attachment file location

Select the file location path that specifies where attachments are to be stored on the file share.

Active

This checkbox activates or deactivates a mailbox. By default, it is selected, which makes the mailbox active.

The following table shows important settings for configuring mail server settings:

Setting or option Description

Server

Type the name of the mail server where the mailbox resides.

Logon

Type the logon name that is used for the mailbox that will be monitored.

Password

Type the password that is used to log on to the mailbox.

Monitor Folder

Type the name of the folder to be monitored. This includes the relative path to the root directory. For example, Inbox/Q1 Promotions.

Archive Folder

If you specified “Move message to archive” as the post-processing action, then type the name of the folder where successfully processed mail will be archived. This includes the relative path to the root directory. For example, Inbox/Archived Message Folder.

Error Folder

Type the name of the folder that will store messages that are not processed successfully. This includes the relative path to the root directory. For example, Inbox/Failed Message Folder. When an inbound email processing error occurs, the EMS logs the error in the event log and moves the email to the error folder.
