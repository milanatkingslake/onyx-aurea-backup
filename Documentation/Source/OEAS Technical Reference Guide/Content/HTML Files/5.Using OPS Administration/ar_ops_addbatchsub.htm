Adding a batch subsystem

Batch subsystems process List Manager actions that are grouped into action batches. After you add a subsystem, you must reload any running instances of OPS Administration for the newly added subsystem to appear in the navigation area. For more information on the settings, see Batch subsystem settings.

To add a batch subsystem:

  1. Navigate to the Batch Subsystem collection to which you want to add a batch subsystem.
  2. Click on the toolbar.

A new subsystem node titled newSubsystem1 (added) appears in the navigation tree.

  1. In edit area, specify the following attributes:
  • Name — A unique name for the subsystem. This is a required field with a maximum of 50 characters. When you add a new subsystem, the default name is newSubsystem <n>, where n is an integer.
  • Description — A description for this subsystem. This is an optional field with a maximum of 255 characters.
  • Additional Data — A semi-colon delimited list of name-value pairs representing data required for sending SMTP messages. These messages contain error notifications. View valid values
  • Active — When this box is checked, the subsystem is active. The box is checked by default.
  • Start Time (00:00 to 23:59) — The start of the subsystem’s operation window for processing. The time must be entered in 24-hour format with a valid range from 00:00 to 23:59. The default is 00:00 (midnight). To configure a subsystem to process 24 hours a day, set both the Start Time and Stop Time to 00:00.
  • Stop Time (00:00 to 23:59) — The end of the subsystem’s operation window for processing. The time must be entered in 24-hour format with a valid range from 00:00 to 23:59. The default is 00:00 (midnight).
  • Run Interval (minutes) — The number of minutes that the subsystem waits between checks of the queue for an action batch. The valid range is 1 - 10,080 (one week).
  • Governor Interval (milliseconds) — The number of milliseconds between Onyx Transaction Manager (OTM) calls. The valid range is 0 - 60,000 (one minute).
  • Mail Server — The IP Address or name of the server that sends error messages. If the SMTP server is local, enter 127.0.0.1 to optimize performance.
  • Administrator Email Address — The complete email address to which error notifications for this subsystem are sent.
  • From Email Address — The complete email address from where the error notifications for this subsystem are sent.
  • From Email Display Name — The name that is displayed in the From field of the error notifications. Including the subsystem name tells the administrator which subsystem was the source of the error notification.
  1. To save your changes, click .

The subsystem is ready to start.
