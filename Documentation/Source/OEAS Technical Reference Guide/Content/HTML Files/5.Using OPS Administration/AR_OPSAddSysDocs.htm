Step 8: Add system documents

In Step 2, Determine your literature products and subsystems, you decided if you needed systems documents and, if so, what type you needed. In this step, you will create and enable the documents.

To create and enable a system document:

  1. Create a system document template that you can use to generate the system document.

To create an order system document, use either OPSOrderReport.doc (for a printed document) or OPSHTMLEmailOrderSysDoc.xsl (for an email document).

To create a batch system document, use either OPSBatchReport.doc (for a printed document) or OPSHTMLEmailBatchSysDoc.xsl (for an email document).

For details on creating and customizing OPS templates, see Creating templates in the OEAS Developer’s Reference section.

  1. In Product Administration, add the template as a literature product to the System Document group, or the group configured as the system document group, making the following changes in the edit area:
  • Active Literature: Check this box to indicate that the literature product is active.
  • File Path: Enter the file path to the location where the system document template is located.
  • Printer: If you are creating a printed system document, select the printer where you want the document to print. If creating an email system document, leave empty.
  • Is a Label: Leave empty.
  • Label Product: Leave empty.
  1. If you are creating a printed system document, under Fulfillment Types, select Send as Printed Document.

OR

If you are creating an email system document, select one or more of the following:

  • Send as Email Body (supports merging)
  • Send as Email Attachment (supports merging)
  1. In OPS Administration, add the system document to a subsystem.

For information on adding a system document to a subsystem, see Adding a system document.
