Adding a system document

The following procedure assumes that you have already used Product Administration to create a literature product for a system document. For more information on system documents, see System documents or Add system documents.

To add a system document to a subsystem:

  1. In the navigation area, expand the subsystem that to which you want to add a system document.
  2. Select the System Documents node.
  3. Click on the toolbar.

A new node named (undefined)* appears in the navigation tree.

In the edit area, the ID is the primary Id of the system document. The ID is automatically generated when the system document is saved. This attribute is read-only.

  1. In the edit area, you can modify the following attributes for the subsystem:
  • In the Product box, select the literature product that represents the system document that you want to associate with the subsystem. This drop-down list includes all active literature products in the System Documents group or the group configured as the system documents group in Product Administration. This is a required field.
  • In the Fulfillment Type box, select a fulfillment type. This list includes only the fulfillment types that are associated with the literature product in Product Administration. This is a required field.
  • In the Generate Document With box, select Batch to generate one system document with each batch of orders, or select Order to generate one system document with each order. This is a required field.
