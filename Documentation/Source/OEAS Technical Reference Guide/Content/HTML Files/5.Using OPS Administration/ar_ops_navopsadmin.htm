Quick Overview of OPS Administration

The OPS Administration window is divided into four areas:

  • Toolbar
  • Navigation area
  • Edit area
  • Context area

Toolbar

Appears under the context area and contains the toolbar buttons for performing common tasks. Tool tips are provided for each button. The following table describes the buttons available in OPS Administration.

Icon Tool tip Function Context

Refresh

Refresh the status of the OPS service and subsystem in the navigation area.

This refreshes only the status, not the entire navigation tree.

Always enabled.

Add

Add a new subsystem or a system document.

Enabled when a subsystem collection or system document collection node is selected in the navigation area.

Delete

Delete a subsystem or a system document. Note that the deletion does not occur until you save your changes.

Enabled when a subsystem or system document is selected in the navigation area.

Start / Continue

Start the OPS service or subsystem. Continues OPS service or subsystem processing that has been paused.

Enabled when the OPS service node or a subsystem node is selected in the navigation area.

Pause

Pause the OPS service or a subsystem.

Enabled when the OPS service node or a subsystem node is selected in the navigation area.

Stop

Stop the OPS service or a subsystem.

Enabled when the OPS service node or a subsystem node is selected in the navigation area.

 

 

or

Expand/Contract Edit Area

Expand or contract the edit area to increase the viewing area.

Always enabled.

Queue Preferences

Set the number of lines returned in the new and completed queues for fulfillment and batch subsystems.

Always enabled.

Save

Save all the changes to the database that you have made to the service or subsystem since you opened OPS Administration or last saved changes. Onyx recommends saving changes frequently.

Enabled after you make a change in OPS Administration.

Help

Open Help for OPS Administration.

Always enabled.

Navigation area

The navigation area presents a hierarchical structure of OPS connections, subsystems, and queues.

Edit area

In the edit area, you can:

  • Change the properties of subsystems and system documents.
  • View queues.

The fields that appear in the edit area change according to what is selected in the navigation area.

Until you select an item in the navigation area, the edit area displays Help text for introducing you to OPS Administration.

Context area

The context area lists the name of the administration tool, the master application, and the master database site. This tool can monitor multiple connections at the same time.
