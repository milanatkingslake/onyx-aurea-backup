Specifying subsystem error notifications

When you configure a subsystem to provide error notification, you must stop and restart the subsystem for the changes to take effect.

For email subsystems, the same information is used for the email profile for outgoing email order lines and subsystem error notification.

To specify error notifications:

  1. Select the subsystem that you want to provide error notifications.
  2. In the edit area, specify the following attributes:
  • Mail Server — The server’s IP address or name. If the SMTP server is local, enter 127.0.0.1 to optimize performance.
  • Administrator Email Address — The destination email address to which error notifications are to be sent. Email system documents are also sent to this address.
  • From Email Address — The complete email address from which the error notification is sent.
  • From Email Display Name — The name to be displayed in the From field of the error notification. Including the subsystem name tells the administrator which subsystem was the source of the error notification.
  1. Click to save your changes.
  2. Restart the system to activate the subsystem modifications.
