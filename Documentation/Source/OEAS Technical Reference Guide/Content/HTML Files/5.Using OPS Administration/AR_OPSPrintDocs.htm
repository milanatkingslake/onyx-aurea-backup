Print literature products

Print literature products are literature products that have a fulfillment type of Send as Printed Document. When the literature product is created, you specify a printer for the document.

You can modify printer settings only in Product Administration or OPS Administration. End users cannot choose where documents print. However, system administrators can use OPS Administration to change the printer for an order line.

To create and enable a print document:

  1. Create a Open XML Word with Mail Merge template from a sample OPS template.

Use this template to generate the literature product. For details on creating and customizing templates, see Print Templates in the OEAS Developer’s Reference.

  1. In Product Administration, create a literature product with a file path to the template.
  2. In Product Administration, add the template as a literature product, making the following changes in the edit area:
  • Active Literature: Check this box to indicate that the literature product is active.
  • File Path: Enter the file path to the location where the Open XML Word with Mail Merge template is located.
  • Printer: Select the printer that will print the print literature products.
  • Is a Label: Leave this box empty.
  • Label Product: To generate a mailing label with the print document, select a mailing label. You must create a label literature product before you can associate it with another literature product.
  • Fulfillment Types: Select Send as Printed Document.
