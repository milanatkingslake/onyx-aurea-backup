Inputs

According to the requirements of this method call, you must create input method call details for the elements described in the following tables.

Elements

This list includes elements representing input parameters of the individual.insert, address.saveCollection, and phone.saveCollection OEAS methods.

Many of the listed elements are necessary not because they are required by the method call itself, but because a method call detail is required for all required input parameters of the underlying OEAS methods. These method call details are noted in the following table.

Element

Details

individual

A method call detail representing the parent individual node is required by the method call in order to properly construct the XML.

primaryId...salutation

Method call details representing the input parameters in the range from primaryId through salutation are required by individual.insert; however, no value needs to be passed to these parameters.

firstName

This method call requires input method call details that are based on these input parameters of individual.insert so that the method call can insert the customer’s first, middle, and last names.

middleName

lastName

suffix...onyxTimestamp

Method call details representing the input parameters in the range from suffix through onyxTimestamp are required by individual.insert; however, no value needs to be passed to these parameters.

phones

A method call detail representing the parent phones collection node is required by the method call in order to properly construct the XML.

phone

A method call detail representing the parent phone node is required by the method call in order to properly construct the XML.

(phone) primaryId

A method call detail representing the input parameter primaryId is required by the insert action of phone.saveCollection; however, no value needs to be passed to this parameter.

(phone) primary

A method call detail representing the primary input parameter for phone.saveCollection is required so that the method call can insert the customer’s phone number as the primary phone number.

(phone) ownerId

A method call detail representing the input parameter ownerId is required by the insert action of phone.saveCollection; however, no value needs to be passed to this parameter.

(phone) ownerType

A method call detail representing the ownerType input parameter for phone.saveCollection is required so that the method call can insert the customer’s phone number with an owner type of “customer.”

(phone) phoneTypeId

A method call detail representing the input parameter phoneTypeId is required so that the method call can insert the phone number as the business type phone number.

(phone) phoneNumber

This method call requires an input method call detail based on the phoneNumber input parameter for phone.saveCollection so that the method call can insert the customer’s primary phone number.

(phone) readOnlyAccess...onyxTimestamp

Method call details representing the input parameters in the range from readOnlyAccess through onyxTimestamp are required by the insert action of phone.saveCollection; however, no value needs to be passed to these parameters.

addresses

A method call detail representing the parent addresses collection node is required by the method call in order to properly construct the XML.

address

A method call detail representing the parent address node is required by the method call in order to properly construct the XML.

(address) primaryId

A method call detail representing the input parameter primaryId is required by the insert action of address.saveCollection; however, no value needs to be passed to this parameter.

(address) primary

A method call detail representing the primary input parameter for address.saveCollection is required so that the method call can insert the customer’s address as the primary address.

(address) ownerId

A method call detail representing the input parameter ownerId is required by the insert action of address.saveCollection; however, no value needs to be passed to this parameter.

(address) ownerType

A method call detail representing the ownerType input parameter for address.saveCollection is required so that the method call can insert the customer’s address with an owner type of “customer.”

(address) addressTypeId

A method call detail representing the input parameter addressTypeId is required so that the method call can insert the address as the business type address.

(address) salutation...companyName

Method call details representing the input parameters in the range from salutation through companyName are required by the insert action of address.saveCollection; however, no value needs to be passed to these parameters.

(address) address1

This method call requires input method call details that are based on these input parameters of address.saveCollection so that the method call can insert two lines of the customer’s street address.

(address) address2

(address) address3...address5

Method call details representing the input parameters in the range from address3 through address5 are required by the insert action of address.saveCollection; however, no value needs to be passed to these parameters.

(address) city

This method call requires input method call details that are based on these input parameters of address.saveCollection so that the method call can insert the city, region, country, and postal code of the customer’s address.

(address) regionCode

(address) countryCode

(address) postCode

(address) validAddress...onyxTimestamp

Method call details representing the input parameters in the range from validAddress through onyxTimestamp are required by the insert action of address.saveCollection; however, no value needs to be passed to these parameters.
Attributes

The following table describes the values to define for the attributes of the input method call details.

For additional information regarding the method call detail attributes, see Method call detail attributes.

Attribute Value/Description

Name

Provide a logical name for each method call detail.

Description

Provide a logical description for each method call detail.

Type

The default and only valid selection for an input method call detail type is scalar.

Hidden

For the method call details representing the following elements, you can select this check box as it is not necessary that this input be visible to the Scripting users.

individual primaryId...salutation suffix...onyxTimestamp phones & phone (phone) primaryId (phone) primary (a default value is passed) (phone) ownerId (phone) ownerType (a default value is passed) (phone) phoneTypeId (a default value is passed) (phone) readOnlyAccess...onyxTimestamp addresses & address (address) primaryId (address) primary (a default value is passed) (address) ownerId (address) ownerType (a default value is passed) (address) addressTypeId (a default value is passed) (address) salutation...companyName (address) address3...address5 (address) validAddress...onyxTimestamp

For all other elements, do not select this check box, because the input detail must be available to the Scripting users.

Active

Select this check box for each method call detail to activate it within the method call.

Default to Null

For the method call details representing the following elements, do not select this check box. A null value is not appropriate for these method call details.

individual phones addresses phone address (phone) primary (address) primary (phone) ownerType (address) ownerType  

For the method call details representing the remaining elements, review the OEAS Developer’s Reference and your own business rules to determine whether a null or an empty element is a valid value. When no value is provided and you have not defined the method call detail to default to null, an empty element is created.

Default Value

(phone) primary - For the method call detail representing the (phone) primary element, type 1. This value ensures that the inserted phone number is the customer’s primary phone number.

(phone) ownerType - For the method call detail representing the (phone) ownerType element, type 9. This value ensures that the inserted phone number has an owner type of “customer.”

(phone) phoneTypeId - For the method call detail representing the (phone) phoneTypeId element, type the ID of the lookup value for a business phone for an individual. This value can be found in Reference Table Administration and ensures that the inserted phone is a phone type of “business.”

(address) primary - For the method call detail representing the (address) primary element, type 1. This value ensures that the inserted address is the customer’s primary address.

(address) ownerType - For the method call detail representing the (address) ownerType element, type 9. This value ensures that the inserted address has an owner type of “customer.”

(address) addressTypeId - For the method call detail representing the (address) addressTypeId element, type the ID of the lookup value for a business address for an individual. This value can be found in Reference Table Administration and ensures that the inserted address is an address type of “business.”

For all other method call details, this attribute is disabled if you previously selected the Default to Null check box. If you did not select the Default to Null check box, your business rules determine whether you must provide a default value.

Element Path

See Element Path List.

Attribute List

phone - For the method call detail representing the phone element, type action=insert. This specifies that this is an insert for phone.saveCollection.

address - For the method call detail representing the address element, type action=insert. This specifies that this is an insert for address.saveCollection.

For all other method calls, leave this attribute empty.
