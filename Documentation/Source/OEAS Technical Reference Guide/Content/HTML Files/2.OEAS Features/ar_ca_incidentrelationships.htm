Relationships

When reviewing the object and table relationships, note that an incident:

  • Is owned by a customer (company or individual).
  • Must have an incident category.
  • Can have any number of audited entries and work notes.
  • Can have any number of tasks.
  • Can have any number of associated work tickets.
  • Can have only one reminder.
  • Can have only one alert.
  • Can be associated with only one incident product.
  • Can be assigned to only one user.
  • Can be associated with any number of users as internal contacts.
  • Can be associated with any number of external contacts.
  • Can have a single primary contact, which is either an individual or a company.
  • Can be associated with any number of tags.
  • Can have any number of file attachments.
