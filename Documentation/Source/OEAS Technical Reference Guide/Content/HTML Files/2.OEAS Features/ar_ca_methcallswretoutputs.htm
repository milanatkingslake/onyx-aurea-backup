The following table describes the values to define for the attributes of the output method call details.

For additional information regarding the method call detail attributes, see Method call detail attributes.

Attribute Value/Description

Name

Provide a logical name for each method call detail.

Description

Provide a logical description for each method call detail.

Type

As each of these method call details represent a single value, for each one select scalar.

Active

Select this check box for each method call detail to activate it within the method call.

Element Path

For each of the following elements, specify the given path, typing it or selecting it from the Element Path Helper:

companyName: company/companyName contactFirstName: company/contactFirstName contactLastName: company/contactLastName phoneNumber (primary): company/phones/phone[primary=1]/phoneNumber phoneNumber (fax): company/phones/phone[phoneTypeId=56]/phoneNumber address1: company/addresses/address[primary=1]/address1 address2: company/addresses/address[primary=1]/address2 address3: company/addresses/address[primary=1]/address3 city: company/addresses/address[primary=1]/city regionCode: company/addresses/address[primary=1]/regionCode postCode: company/addresses/address[primary=1]/postCode

In the Method Call Designer, you can use the Element Path Helper to create this path, although the path is created automatically when you use the Method Call Detail Builder to create the method call detail. Observe the following exceptions:

  • You must add the filter information for the primary phone number.
  • You must supply the filter information for the phoneTypeId based on the value for a fax number. You can identify this value using Reference Table Administration. This example assumes a value of 56.
  • You must add the filter information for the primary address.

In this case, if the “fax” phone type is the primary phone number, no value is returned by this output detail.
