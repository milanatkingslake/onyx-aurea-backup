The following temporary tables are created to support the cascade process outlined in the following flow charts:

  • #process_item - This temporary table holds the values (child object type and key information) that are returned when evaluating the subscription relationships.
  • #process_item_unique - This temporary table contains the unique values from #process_item.
  • #subsc_relation - This temporary table is used to store the subscription relationships and their cascade settings and the SQL statements required to select the primary and secondary objects of a subscription relationship.
  • #subscription_item - This temporary table is used to store the subscription_item records.
