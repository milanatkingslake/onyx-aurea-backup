Elements of security

Security begins with users. In addition to users, the primary components of security are roles and resources. The following sections discuss each of these components.

  • Users
  • Roles
  • Resources

Users

Each request to OEAS is submitted in the context of an Onyx user.

An Onyx user is network-independent, which means that you can log on as a given Onyx user from anywhere on the network, either with or without a Windowsdomain account. Independence from the network allows flexibility in accessing OEAS. For example, users can access OEAS from outside the network through the Web while maintaining the same access privileges as when accessing from inside the corporate network.

Individuals are logged in as Onyx user accounts either directly or indirectly, depending on which kind of authentication they log in with:

  • Integrated Windows authentication

The user must have a Windows domain account. Once authenticated, the domain account is mapped to an Onyx user account.

  • Onyx authentication

The user logs in directly using an Onyx user account.

Roles

Roles are collections of users formed in Security Administration for the purpose of granting (or explicitly denying) access permissions to multiple users at once.

You cannot add groups to roles; you can only add individual users to roles.

Resources

Resources represent the entities to which you can administer access permissions. These entities can include the OEAS object methods, user interfaces, or any other item that you create as a resource.

You control access to these entities by granting and denying access permissions for the resources that represent them in Security Administration. With Security Administration, you can specify user permissions very precisely, without having to program customizations.
