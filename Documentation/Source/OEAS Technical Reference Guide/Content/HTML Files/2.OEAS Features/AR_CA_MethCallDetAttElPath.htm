Element Path

The method call detail Element Path attribute specifies the XML node that the method call detail references. If used for a collection, it specifies a collection of nodes. The element path can be a relative path or an absolute path. Absolute paths begin with a slash (/) character.

The behaviors and requirements of the element path depend on whether you are creating an input or output method call detail.

Input method call details

For input method call details, the path specifies an element that can be created in the XML that is sent to the underlying OEAS method (paramXml or customDataXml). The element so specified is called the input element. An input element path cannot refer to an attribute or any other type of node other than an element.

The path can use only a simple location path (containing element names and slashes). It cannot contain filters, an axis specifier, node tests, or any XPath syntax other than element names and slashes.

If the path is relative, no leading slash (/) is required since it is relative to the parameters node in paramXml.

If the path is absolute, it must start with one of the following strings:

  • /customData - The input element is created in the customDataXml that is sent to the Transaction Manager. The text after /customData is treated as a relative path specifying an element under the customData node in customDataXml.
  • /parameters - The input element is created in the paramXml that is sent to the Transaction Manager. The text after /parameters is treated as a relative path specifying an element under the parameters node in paramXml.

After the initial string, there must be a valid relative XPath (i.e., it is not valid to use /parameters by itself).

It is necessary to create a method call detail for each element in the element path. For instance, if the elementPath is company/phones/phone/primary, you must create a method call detail for the company element, the phones child element, the phone child element, and the final child primary element. You must also include any attributes or text when they are needed on an intermediate (parent) node.

Examples of valid input Element Path values:

  • company/secondaryId
  • /parameters/primaryId
  • literatureOrder/literatureOrderLines/literatureOrderLine/productNumber
  • /customData/myCustomParameter

Output method call details

The path locates a node or set of nodes within the XML returned by the OEAS method (returnXml). Each node so located is called an output node.

The path can fully utilize any XPath syntax supported by the underlying implementation (often MSXML), including filtering, functions, expressions, etc. In addition, the path can refer to attribute nodes or any other type of node accessible via XPath syntax.

If the path is relative, it is relative to the parametersReturn node in the return XML and no leading slash (/) is required.

If the path is absolute, it must start with one of the following strings:

  • /returnXml - The output nodes are located in the return XML created by the Transaction Manager. The text after /returnXml is treated as a relative path specifying a node or set of nodes under the returnXml node.
  • /returnXml/customData - The customData node is already a part of the return XML, so the valid path starts with /returnXml/customData.

After the initial string, there must be a valid relative Xpath. It is not valid to use /returnXml by itself.

In the case of a scalar type method call detail, the Element Path attribute is expected to identify a single node within the return XML. If the XPath identifies multiple nodes, only the first is considered the output node.

In the case of a list type method call detail, the Element Path attribute is expected to identify a set of nodes within the return XML. Each node within the set is an output node.
