Delete

In the event that an object is physically deleted from the Onyx Enterprise Database (OEDB), delete change-capture triggers ensure that these physical deletes are cascaded appropriately by setting the physically deleted object’s sync_action value to 7. This ensures that the deleted object and it's secondary objects are subsequently marked as deleted.

As noted in Update, most transactional objects deleted through OEP and OEP-Offline are logically deleted in the database rather than physically deleted, and are caught by the update change-capture trigger.
