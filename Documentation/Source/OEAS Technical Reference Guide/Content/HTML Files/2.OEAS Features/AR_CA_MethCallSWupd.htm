update

The following example uses the incident.update OEAS method. This method call is designed to:

  • Accept the primaryId of the previously retrieved incident.
  • Accept the onyxTimestamp of the previously retrieved incident.
  • Accept a value to be updated for the statusId of the incident.

Using Method Call Designer, you can create this method call by defining the method call and method call detail attributes based on incident.update.

Define the method call attributes

The following table describes the values to define for the method call attributes.

For additional information regarding the method call attributes, see Method call attributes.

Attribute Value/Description

Name

Provide a logical name for the method call.

Description

Provide a logical description for the method call.

Object

Select incident.

Method

Select update.

Active

To activate the method call so that it is available for use, select this check box.

Define the method call detail attributes

You create input method call details for each element to be passed to the underlying OEAS method, as well as for each parent element that is also included in the element path. No output details are required for this method call.
