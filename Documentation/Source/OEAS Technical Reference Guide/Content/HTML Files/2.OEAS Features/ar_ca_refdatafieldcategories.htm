Field Categories

OEAS supports the following categories of fields:

  • Configurable standard fields
  • User-defined fields
  • Custom reference fields

Two of the categories for reference data are supported for configuration by Onyx Employee Portal (OEP). The category determines the level of configuration that is possible for a field; based on its category, a field can support from one to three field types.

The two configurable categories are:

  • Configurable standard fields
  • User-defined fields

OEP respects the supported modifications to these fields. If you modify these fields in a manner that is not supported, you must make the appropriate changes to the presentation tier.

You can create a custom reference field for an object, but to use it requires customization of the presentation layer (UI). For instance, you create a custom object called myObject in OEAS, and you create new reference fields for that object, so that you can maintain reference data related to it. You might create custom reference fields myObject.status field, a myObject.priority field, and so on. If you want to use these in OEP, you must write custom OEP code. For more information on customizing OEP, see the OEP Technical Guide.
