Subscribable objects

Subscribable data is the transactional data that is available for OEP end-users to subscribe to so that they can synchronize it to OEP-Offline. You create subscribable data by making the business object that identifies that data subscribable, which you can do using Subscription Designer.

The following OEAS transactional business objects have been created as subscribable objects in the default installation of OEAS.

Object Notes

company

Because the phones and addresses collections are tightly coupled to the company object, all phone and address information is included when a company is subscribed to.

forecast

The forecast object is automatically subscribed with incident objects based on relationships and rules installed with OEAS. These rules should not be modified when you want your end-users to be able to work with forecasts offline, as there is no means, in either OEP or OEP-Offline, to explicitly subscribe to or unsubscribe from a forecast.

 

Because the forecastLine object is tightly coupled to the forecast object, all related forecast line information is included when a forecast is subscribed to. The forecastLine object does not exist separately as subscribable object.

individual

Because the phones and addresses collections are tightly coupled to the individual object, all phone and address information is included when an individual is subscribed to.

incident

The workNote and reminder objects are automatically subscribed with the incident object. The reminder object does not exist separately as subscribable object.

quote

Because the quoteLine object is tightly coupled to the quote object, all related quote line information is included when a quote is subscribed to. The quoteLine object does not exist separately as subscribable object.

task

The workNote and reminder objects are automatically subscribed with the task object. The reminder object does not exist separately as subscribable object.

workNote

The workNote object is always automatically subscribed with task and incident objects. Cascaded subscription, unsubscriptions, and deletions cannot be modified. If desired, you can configure the subscribable object’s extra data xml.

To create a subscribable object for an object that is not by default supported for subscription by OEAS or by OEP-Offline, you must perform additional system customization as described in “Adding a subscribable custom object” in the OEP-Offline Developer’s Reference.

For more information about creating subscribable objects, see “Subscription Designer” in the OEAS Administrator’s Reference.
