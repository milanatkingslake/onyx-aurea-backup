Inputs

This method call requires you to create input method call details for the elements described in the following table.

Separate internalContact nodes are required for each internalContact for which an internalContact.saveCollection action is required. For additional details regarding the behavior of saveCollection method calls, see saveCollection methods.

Element Details

internalContacts

The method call requires an input method call detail representing the internalContacts collection node in order to properly construct the XML.

internalContact(1)

The method call requires an input method call detail representing the first internalContact parent node in order to properly construct the XML that can delete an internal contact from the internalContacts collection.

primaryId(1)

This method call requires an input method call detail that is based on the internalContact object’s primaryId property in order to accept the primaryId of an internalContact object and delete that object.

internalContact(2)

The method call requires an input method call detail representing the second internalContact parent node in order to properly construct the XML that can insert an internal contact into the internalContacts collection.

primaryId(2)

This method call requires input method call details that are based on all the required input parameters of the insert action of the internalContact.saveCollection OEAS method. As such, a method call detail representing the primaryId input parameter is required. The primaryId is an empty node for the input XML, and it reflects the unique identifier that OEAS assigns to the internalContact record in the return XML. Therefore, no value needs to be passed to this parameter.

ownerId(2)

An input method call detail representing the internalContact object’s ownerId property is required so that you can pass to the OEAS method the primaryId for the company to which you are adding the internal contact.

ownerType(2)

An input method call detail representing the internalContact object’s ownerType property is required so that you can specify the owner type of the object to which you are adding the internal contact.

contactTypeId(2)

An input method call detail representing the internalContact object’s contactTypeId property is required so that you can specify the type of internal contact you are inserting.

contactUserId(2)

An input method call detail representing the internalContact object’s contactUserId property is required. This method call detail identifies the individual within your own company and maps to the user table.

contactUserName(2)

This method call requires input method call details that are based on all of the required input parameters of the insert action of the internalContact.saveCollection. For this reason, method call details representing the contactUserName and comment input parameters are required. However, no value needs to be passed to these parameters.

comment(2)

The following table describes the values to define for the attributes of the input method call details.

For additional information regarding the method call detail attributes, see Method call detail attributes.

Attribute Value/Description

Name

Provide a logical name for each method call detail.

Description

Provide a logical description for each method call detail.

Type

The default and only valid selection for an input method call detail type is scalar.

Hidden

Do not select this check box for the method call details representing the following elements. These inputs must be visible to the script designer.

  • primaryId(1)
  • ownerId(2)
  • contactUserId(2)

For all other elements, select this check box so that the input detail is not available to the script designer. These elements require no input.

Active

To activate each method call detail within the method call, select this check box for each detail.

Default to Null

Do not select this check box. A null value is not appropriate for these method call details.

For the method call details representing the contactUserName(2) and comment(2) elements, review the OEAS Developer’s Reference to determine whether the property accepts null or an empty element as a valid value. When no value is provided and you have not defined the method call detail to default to null, an empty element is created.

Default Value

ownerType(2) - For the method call detail representing the ownerType element, type the value 2. Valid values for the internalContact ownerType property include 1 (individual), 2 (company), 3 (customerProduct), 6 (incident), 8 (task), and 10 (workTicket). In this case, because this method call always inserts an internal contact for a company object, the default value is 2.

contactTypeId(2) - For the method call detail representing the contactTypeId element, type the value 25. This method call inserts an Established Business Account Manager internal contact type for the company object. Because this is always true, you can set a default owner type of 25. This example assumes that you have previously defined a “company” internal contact type of Established Business Account Manager. It also assumes that the contact type ID for that internal contact type is 25.

Regarding the method call details representing the contactUserName(2) and comment(2) elements, selecting the Default to Null check box disables this attribute. If you do not select the Default to Null check box, your business rules determine whether you provide a default value.

For all other elements, keep the default setting of no default value. A default value is not appropriate for those method call details.

Element Path

For each of the following elements, specify the given path, typing it or selecting it from the Element Path Helper:

internalContacts: internalContacts internalContact(1): internalContacts/internalContact primaryId(1): internalContacts/internalContact/primaryId internalContact(2): internalContacts/internalContact primaryId(2): internalContacts/internalContact/primaryId ownerId(2): internalContacts/internalContact/ownerId ownerType(2): internalContacts/internalContact/ownerType contactTypeId(2): internalContacts/internalContact/contactTypeId contactUserId(2): internalContacts/internalContact/contactUserId contactUserName(2): internalContacts/internalContact/contactUserName comment(2): internalContacts/internalContact/comment

In the Method Call Designer, you can use the Element Path Helper to create this path, although the path is created automatically when you use the Method Call Detail Builder to create the method call detail.

Attribute List

internalContact(1) - For the method call detail representing the internalContact(1) element, type action=delete for the attribute.

internalContact(2) - For the method call detail representing the internalContact(2) element, type action=insert for the attribute.

For all other elements, leave this attribute empty.
