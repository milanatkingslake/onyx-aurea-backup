Email

The Email feature is the collection of Onyx Enterprise Application Server (OEAS), Onyx Enterprise Studio (OES), and Onyx Processing Server (OPS) features with which you can manage the creation and processing of email messages (emails). This email management system enables end users to read and compose emails that are associated with customer and incident records. All emails sent through the user interface are recorded and tracked in the Onyx Enterprise Database (OEDB).

The Email feature enables the tracking of email threads, including the following:

  • Attachments sent with an email.
  • Internal users, individuals, and/or companies that have sent or received an email.
  • Contact associations (links) to individual, company, and incident records related to an email.
  • Non-contact associations (links) to emails associated with individual, company, and incident records unrelated to the sender or receiver of the email.

Emails are automatically assigned to threads when created, but users can manually attach emails to different threads if necessary. An email can be assigned to only one thread at a time.

  • Email messages can be associated with individuals, companies, internal users, or incidents.
  • Email messages can have many file or literature item attachments.
  • A contact is created for each recipient and sender of an email.
  • An association can be related to a contact.
