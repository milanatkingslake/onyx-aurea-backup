Object to table mappings

The keyWord business object maps to the following Onyx Enterprise Database (OEDB) tables.

Business object OEDB table

keyWord

key_word

keyWordAssociation

key_word_association

The keyWordAssociation object has no methods, only properties, and defines the relationships of the tags. The keyWord object does not save tags to the database. It has only a retrieveList method which retrieves tags from the key_word table based on the associations defined in the key_word_assocation table. Tags are persisted to the key_word and key_word_association table by the incident, task, and workTicket objects.
