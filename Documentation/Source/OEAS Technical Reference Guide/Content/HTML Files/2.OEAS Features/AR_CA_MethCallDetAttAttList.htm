Attribute List

You can define the method call detail Attribute List attribute when you create:

  • An input method call detail for both scripting and merge template method calls.
  • An output method call detail for merge template method calls.

Input method call details

When you create input method call details, you can use the Attribute List attribute to define a semicolon-delimited list of name=value pairs that are added to the created element for the input method call detail. The names are attributes to create on the underlying XML element and the values are the attribute text. Attributes are created on the underlying element in the order in which they appear in the attribute list. The following points are relevant for the Attribute List attribute:

  • Quotation marks are not required and should not be used for wrapping attribute names or values. If present, they are treated as part of the name or value.
  • For attribute values, all characters are valid except the semicolon (;) and the equals sign (=).
  • No characters within the attribute list text can be escaped, including those that would normally need to be escaped to be part of an attribute value. For instance, the following is a valid attribute list that would generate an attribute “a” with the value “‘b’ < 5”, and an attribute “b” with the value “Bob’s Cars”:
  • a=‘b’< 5; b=Bob’s Cars

Output method call details

When you create a merge template output method call detail, you can use the Attribute List attribute to control presentation of output values. Adding attributes to method call detail outputs allows the UI to display the correct format for various data types and to transform the output data from IDs to descriptions. For example, dates can be displayed correctly and user names can be displayed from the OEP cache instead of displaying the user IDs or retrieving the names from the database. The Attribute List can also enable other custom processing for a customized UI.

Supported dataType attributes include dateTime, domainData, and userId. For each output method call detail for which you want to control the output presentation, include one of the following strings in the Attribute List:

  • dataType=dateTime
  • dataType=domainData
  • dataType=userId

When these dataType attributes are used, the method call detail returns the date in local format, the domain data description rather than the ID, and the user name rather than the user ID.

Within the Templates feature, the method call detail supports only one data type. So, unlike the attribute list for input method call details, the attribute list for output method call details supports only one name=value pair.
