Creating assignment conditions

Following are the general steps to create an assignment territory condition in Assignment Administration:

  1. Create parent/child territories in the Assignment Tree.  

Territories can also contain other territories to provide further organization of the conditions. A geographic example is country, region, state: United States (highest territory), Pacific Northwest (second highest territory), and Washington (third highest territory). The name "territory" implies a geographic assignment; however, you can organize territories by other classifications, such as market sector, named accounts, and much more.

  1.  Create conditions for companies, individuals, and/or incidents according to your plan for creating internal contacts and assigning incidents to users.

  1. Create some expressions using any property of the customer, individual, or incident objects combined with operators and values.

  1. Create a customer assignment set (either company or individual) and associate users by entering their user IDs and internal contact types as follows.

  1. Create an incident assignment set and associate users with the assignment set by entering their user IDs and internal contact types. For the routing of leads, incident assignment set users are optionally assigned to incidents based on the following assignment set types:

Normal

Assigns incidents to a single user within the assignment set.

Round Robin

Assigns incidents to the next user in a pool of users (in turn).  This option keeps track of who was assigned last and determines who should be assigned next.

Lease Busy

Assigns incidents to the user with the lightest workload (least amount of open incidents).

  1. Add the assignment set to the appropriate conditions. Assignment sets are associated with conditions of the same object type and they are reusable throughout the Assignment Tree.

When complete, a set of territories and their corresponding conditions are created. You can verify the territory association of customers in List Manager. For more information, see Maintaining territories.
