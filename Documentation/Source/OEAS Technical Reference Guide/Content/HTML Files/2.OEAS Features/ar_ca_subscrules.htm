Subscription rules

There are four rules you can use to control cascading behaviors between related subscribable objects. You can set all four of these rules at the relationship level, meaning that they apply to all subscribers. Three of the rules can also be set at the role level, meaning they are specific to a particular group of subscribers, based on existing Security Administration roles. When you assign a subscription rule to a role, you are granting a specific cascade rule to all subscribers who are members of that role.

For more information about subscription roles, see Subscription roles.
