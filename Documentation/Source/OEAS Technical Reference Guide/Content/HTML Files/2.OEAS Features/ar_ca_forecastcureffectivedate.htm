Role of the currencyConversion effectiveDate Property

The effectiveDate property in currencyConversion object enables you to store historical and pending conversion rates for the same currency combinations that are currently being used by the system. For example, you can download new conversion rates on Friday that you don’t want to take effect until Monday. To implement this, insert them into the CurrencyConversion table with an effective date of midnight Sunday, and at midnight on Sunday, the system begins using the updated conversion rates.

The currencyConversion object retrieves the rows of data that have the most recent effective date that is not greater than the present date.

If your organization updates currency conversion rates on a frequent basis, monitor the size of the CurrencyConversion table on a regular basis to ensure its size does not negatively impact performance.
