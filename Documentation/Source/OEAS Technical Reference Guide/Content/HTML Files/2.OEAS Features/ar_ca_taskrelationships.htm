Relationships

When reviewing the object and table relationships, note that a task:

  • Cannot exist independently; it is always owned by an incident, and the task category must be related to the incident category.
  • Can have any number of work note entries.
  • Can have only one reminder.
  • Can be assigned to only one user.
  • Can have only one primary contact.
  • Can have any number of tags.
  • Can have any number of file attachments.
