Sequence of operation in script steps

Scripting user workflows

For each script step, the following actions take place in the order presented here:

  1. Any pre method calls are carried out, and variables are updated from the results of the method calls.

The pre method calls are carried out in the order in which they reside in the Method Calls box on the Method Calls tab in the Edit Step area for a step in the Script Designer

  1. The step is loaded in the end-user interface.
  2. The end-user responds to the prompts.
  3. The prompt responses are saved, and the variables are updated from the responses.
  4. Any post method calls are carried out, and the variables are updated from the results of the method calls.
  5. Any rule condition statements are evaluated.

If a post method call failed, the rules are not evaluated.

  1. The session is saved, including the state of all global variables.

The script session is saved even if the post method calls failed.

  1. All errors and messages are returned to the end-user interface, including pre and post method call errors and script engine errors.
