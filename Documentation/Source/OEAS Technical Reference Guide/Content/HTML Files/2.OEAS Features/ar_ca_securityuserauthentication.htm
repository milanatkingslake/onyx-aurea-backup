User authentication: Login and session management

User authentication

Authentication is the process in which a user is identified by his or her account information and password in order to start a session.

  • Login scenarios
  • Session management
  • Login scenarios

There are two scenarios under which users log in and are authenticated:

Only users whom the Onyx administrator has added as Onyx users in Security Administration can be authenticated to use OEAS.

  • Integrated Windows authentication

The user has an intranet domain account for which integrated Windows authentication is available.

In this scenario, the user logs in with a domain account name and password. The user is authenticated by Windows domain security, the domain account is mapped to an Onyx user account, and a session is established.

Windows authentication is only supported when OEAS and OES are installed on the same machine. Onyx authentication supports physically separate OEAS and OES servers.

  • Onyx authentication

These users typically log in through a Web page with a user ID and password. The user ID and password, which is case-sensitive, constitute the user’s credentials for authentication, and the session is established.

Session management

In order for user information to be passed between Web sites located on separate physical Web servers, all session information must match. Because the domain account that the IIS server on the calling Web server is passed as part of the authentication process, the authentication fails when the calling domain account doesn't match the domain account that initiated the session. For this reason any Web site that will pass information and user authentication to another Web server before making a call to the middle tier must use the same anonymous user domain account.

For more information on session management, see the OEAS Developer’s Reference.
