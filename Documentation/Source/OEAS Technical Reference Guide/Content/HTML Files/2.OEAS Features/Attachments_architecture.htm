Architectural elements for Attachments

Following are the main architectural elements for the Attachments feature:

The standard installation of Onyx Employee Portal (OEP) sets up the upload/download file server on the same machine as the OEP Web Server. However, you can optionally deploy the file storage to a separate physical machine if desired.

Element Description Comments

Client Web browser (running Internet Explorer)

OEP - Web client

Displays the file links to the end user and accepts streamed files from the file share.

Upload/download file server

Server with ASPUpload software installed which provides OEP access to the user's file system for uploading and downloading files

If this is a different physical machine from the OEP web server, you must install a duplicate copy, including any customizations, of OEP on the machine.

OEP Web Server

Accepts requests from the client, processes ASP pages, and returns HTML to the client

Normally, ASPUpload software resides on the same machine as the OEP Web Server.

OnyxFileAgent

COM+ application used by OEP to access the file share

This application must be installed on the OEAS server, or it must have a COM+ proxy to the OEAS server.

File share

The file system location where the file attachments are saved

The file share can physically reside on any server accessible to the COM+ identity user for OnyxFileAgent component.

Client-side applications

Required for viewing downloaded files

If a user tries to view a file whose required application does not exist on the client machine, the standard Windows “Open With...” dialog box appears.
