Combo box field type

When you update an existing Combo box configurable field (referenceParameter object), or when you create a new one, you typically create or specify the lookup values that can be selected by a user for that combo box. Then, you associate the referenceField object to the desired referenceParameter objects using the referenceLookup object. As the following diagram illustrates, the referenceId and parameterId properties are essential for creating this relationship.

In the diagram below, notice that the referenceField objects named incident.status and task.status share the same referenceParameter object (“Active”), just as the referenceField objects named task.status and forecast.status share the same referenceParameter object (“Inactive”). Therefore, the referenceLookup objects that correspond to these referenceFields share the same parameterId property (10135 and 11345, respectively).

Also notice that each referenceLookup object has a different referenceId property—except for the two referenceLookup objects that are associated with the referenceField object named task.status. In this example, only the referenceField object named task.status gives end-users the opportunity to select more than one possible value.
