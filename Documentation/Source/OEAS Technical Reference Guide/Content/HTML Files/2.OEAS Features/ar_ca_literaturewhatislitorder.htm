What are literature orders?

Literature orders are transactional records, created by employees, that associate literature products with customers. You can use them to track what literature products a customer has been sent or has requested, and you can use them to facilitate delivery of literature products to a customer when you implement a literature fulfillment system, such as with Onyx Processing Server.

Employees can create any number of literature orders for a customer, associating any number and combination of literature products with these orders. They can select the shipping method, payment terms, and literature type. They can also associate the literature order with a tracking code, and indicate if the order should be completed immediately or in a regularly scheduled batch job.

Using the Onyx handlers or a custom application of your own, you can configure your application to process literature orders immediately or in a batch. If your customer requires the order as soon as possible, the employee can mark it for immediate real-time processing. Otherwise, the order is processed in a batch with other orders, at a time that you configured the application to process the batch.

The main elements of the Literature Orders feature include the following:

  • Literature order header
  • Literature order details
  • Literature printer
  • Literature products
  • Literature type
  • Ship method

Literature order header - The literature order header is the part of the literature order that contains information about the entire order, regardless of the number of literature products included in the order. This information includes such details as the customer name, billing and shipping addresses, order priority, order date, ship method, and payment terms.

Literature order details - Each order detail for a literature order contains the reference to a specific literature product and information about that literature product including price, quantity, and shipping status.

Literature printer - The literature printer is a networked printer that you have associated with those literature products that can be printed prior to delivery.

Literature product - A literature product is any type of electronic or physical item that can be delivered as mail, and/or email, and that you have configured as a literature product using Product Administration. Literature products can include marketing collateral, such as letters, brochures, pamphlets, posters, press materials, white papers, direct mail items, and photographs; saleable items, such as mailing lists, software programs, or printed documentation; or informational packets or promotional pamphlets that you send to customers to promote and supplement your standard products. Compared to a standard product, literature products can have additional attributes, such as a file path location and a printer path location.

Additionally, you can create literature items that end users can attach to their email messages through the Add Literature Attachments page. See configuring Email in the OEAS Administrator’s Reference for complete instructions.

Fulfillment types - The fulfillment types are attributes that you associate with a literature product when you create the product in Product Administration. These attributes support the Onyx Processing Server (OPS) and impact how the literature product is processed. You can associate more than one fulfillment type with a literature product, but employees can select only one of those fulfillment types when they add that literature product to a literature order. For additional information regarding the available fulfillment types and the purpose of each, see What is the Onyx Processing Server? in the OEAS Administrator’s Reference.

In the OEP Web client, fulfillment types are listed as literature types.

Ship method - Ship method is a literature order attribute that employees select when the literature order needs to be mailed to the recipient. The different means for shipping can include First Class, Second Class, UPS, Federal Express, and so on.
