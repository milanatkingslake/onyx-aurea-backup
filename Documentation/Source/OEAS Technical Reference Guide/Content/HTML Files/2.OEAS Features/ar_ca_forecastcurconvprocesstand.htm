Understanding conversions

OEAS can convert both forwards and backwards between two currencies, so there is no need for inversely paired combinations of currency types in the currency conversion domain data (the CurrencyConversion table). OEAS identifies the conversion requested between the two currencies and either multiplies or divides by the conversion rate to calculate the new currency value.

Example

The two conversions are stored in the CurrencyConversion table as one row of data. In the case of a conversion from AUD to ARP, the AUD value is multiplied by the value in the rate column, whereas in a conversion from ARP to AUD, ARP is divided by the rate value.
