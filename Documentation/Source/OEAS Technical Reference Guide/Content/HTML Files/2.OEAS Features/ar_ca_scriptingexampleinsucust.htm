CustomerSearch step

This is the first step of the script. It obtains customer data with which to either obtain and confirm more customer data or to establish that the customer is new.

In the following table, the rules appear in the order in which they occur in the Rules box, which is also the order in which they are evaluated.

To search for an existing customer, the user can enter data in any one of the three prompts.

Item Type Name Notes

Prompt

Text box (string)

CustomerNumber

Not required.

User types in customer’s ID. If customer is new, user leaves text box empty.

Output variable: {findCustomerId}

Prompt

Text box (string)

FirstName

Not required.

User types in customer’s first name. If customer is new, user leaves text box empty.

Output variable: {findFirstName}

Prompt

Text box (string)

LastName

Not required.

User types in customer’s last name. If customer is new, user leaves text box empty.

Output variable: {findLastName}

Variable

Global scalar

{findCustomerId}

Output value, set to one of following:

  • For existing customer, customer ID entered in CustomerNumber prompt
  • For new customer, NULL

Variable

Global scalar

{findFirstName}

Output value, set to one of following:

  • For existing customer, customer first name set in FirstName prompt
  • For new customer, NULL

Variable

Global scalar

{findLastName}

Output value, set to one of following:

  • For existing customer, customer last name set in LastName prompt
  • For new customer, NULL

Rule

 

NoSearchCriteria

Condition statement: {findCustomerId} = "" and {findFirstName} = "" and {findLastName} = ""

Destination step: CustomerInfo

Rule

 

SearchCriteria

No condition statement.

Destination step: CustomersFound
