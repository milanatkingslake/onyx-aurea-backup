Prompts

A prompt is a control that facilitates a user response within a step. Most prompts elicit user responses directly, but some prompts provide such graphical elements as labels or rules. Prompts usually include text, which can include HTML tags and, for text merging, scalar variables.

OEAS includes the following standard control types for prompts. Creating these standard prompt types involves a common procedure. Many of them require that you specify some additional attributes. For the procedure common to creating all standard prompt types, see the Help for Script Designer. Where appropriate, the following descriptions of standard prompt types include cross-references to the sections that describe how to specify any necessary additional attributes.

  • Check box

A check box prompt presents a standard check box. The end-user cannot provide additional text.

  • Check box set

A check box set prompt provides a list of check boxes, one for each possible response.

  • Country

In OEP, a country prompt displays a list of the countries in the OEP cache.

  • Divider

A divider prompt displays a dividing line across the prompt area. It is useful for visually separating groups of prompts within a step, but does not accept any end-user input.

  • Drop-down list box

A drop-down list box prompt displays a standard drop-down list box. The end-user can only select a single item from the list.

  • Incident product

In OEP, an incident product prompt displays a list of incident products.

  • Label

A label prompt contains any text or HTML that you specify in the Script Designer. The label control does not accept any end-user input.

  • List box

A list box prompt presents a standard list box.

  • Masked edit

A masked edit prompt displays a standard masked edit control, such as those used for telephone numbers or postal zip codes.

For more information about masks and masking characters in OEP, see the OEAS Developer’s Reference.

  • Multiline text box

A multiline text box prompt presents a text box in which the end-user can enter multiple lines of text. A vertical scroll bar appears as needed.

  • Product

In OEP, a product prompt displays the standard OEP product tree, from which the end-user selects a response. In the Script Designer, you can specify whether the product tree is a customer, literature, or forecast-and-quote product tree.

For information on specifying attributes for a product prompt, see the Help for Script Designer.

  • Radio button set

A radio button set prompt presents a standard set of radio buttons. The end-user can only select a single radio button.

  • Reference lookup

A reference lookup prompt shows lists of reference lookup data, as identified by the reference field that contains the data. The data contained in the reference field is defined in Reference Table Administration.

  • Region

In OEP, a region prompt displays a list of the regions for a particular country in the OEP cache.

  • Text box

A text box prompt presents a standard text box.

  • Tracking code

In OEP, a tracking code prompt displays the standard OEP tracking code tree (the same as the one that is used in the OEP Products feature).

  • User

In OEP, a user prompt displays a standard OEP user-tree control. The end-user can select only one user from the tree, and cannot provide additional text.

In addition to the above standard control types, you can create a custom control type for prompts. For information on doing so, see the OEAS Developer’s Reference.
