Method call details

A method call detail provides an input value or an output value that is passed between the method call and the script step:

  • A method call input is a value that a script step passes to the method named by the method call.
  • A method call output is a value that the method passes to the script step.

When you define the method call inputs, you also define the binding type: how the value is passed to the step. The binding types all override the default value for the method call input, if one has been defined in Method Call Designer.

The possible binding types are:

  • Empty

Does not bind a value from the script to the method call.

  • Literal

Binds the specified value from the script to the method call.

  • Null

Binds NULL (the NULL=1 attribute) from the script to the method call.

  • Variable code

Binds the value represented by the specified variable code from the script to the method call.

  • Variable text

Binds the value represented by the specified variable text from the script to the method call.

Method call outputs are passed only as variables.

The status of method calls is not tracked within a script and cannot be used to conditionally call additional method calls on the same step. If one method call on a step fails, none of the following method calls for that step are executed. If one method call fails, another that follows it will not be carried out.

For more information:

  • On creating method calls, see the Help for Method Call Designer.
  • On associating a method call with a script step, see the Help for Script Designer.
