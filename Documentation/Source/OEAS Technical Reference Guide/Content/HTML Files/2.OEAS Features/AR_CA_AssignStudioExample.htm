Example: Geographic territory assignment

The following diagram demonstrates the relationships and types of conditions, expressions, and assignments for a geographic territory.

In this example, the United States is the parent territory of the Pacific Northwest and Southwest (these are child territories). Washington, Oregon, and Idaho are conditions for the Pacific Northwest, and California is a condition for the Southwest. Assignment sets are associated with these conditions (Sales = Bob, Service = Sue is the first assignment set).

Let's assume a company called MBI is created with a primary address that contains a country code of US and a region code of WA. When MBI is created, Bob is made an internal contact for the Sales team, and Sue is made the internal contact for the Service team. Both teams are associated with MBI by using the ownerName, ownerId, and ownerType properties on the incident.

  • If MBI’s primary address contained "OR" (Oregon) in its regionCode, then its sales incidents would include Bob as the contact and the incident would be assigned to Mac.
  • If MBI’s primary address contained "ID" (Idaho) in its regionCode, the rules would cause an internal contact to be created using George. The internal contact would be associated to MBI by the ownerId and ownerType properties.

The final assignment set  (Sales = Bob or Mac or George) illustrates the Round Robin assignment type, which assigns incidents to the next user in a pool of users (in turn). The system keeps track of who was assigned last and determines who should be assigned next. You can also assign incidents to the user with the lightest workload through the Least Busy assignment type.
