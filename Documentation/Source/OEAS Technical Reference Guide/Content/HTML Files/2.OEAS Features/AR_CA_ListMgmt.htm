List Manager

The List Manager feature enables users to run database searches that return a list of records based on specific criteria. By default, these lists of records can include individuals or companies. However, List Manager can be customized to return any type of record.

List Manager is restricted to permissions granted to the OnyxLMViews role. By default, List Manager does not have access to any Onyx tables. List Manager actions are stored in the action_batch table and are processed by OPS Batch Subsystem.

Using List Manager, users can:

  • Create and run database searches based on specific search criteria
  • Save the created search as a public search so that all users can access and run it, or save it for personal use
  • Run a previously saved public or personal search
  • Modify or delete a previously saved search
  • Select additional columns to include in the search result list
  • Open a record listed in the result list
  • Perform bulk actions on selected records in the result list, such as exporting to Microsoft Excel, adding internal and external contacts, and adding incidents.

The following examples illustrate two of the many uses of List Manager:

  • Users can build a list of all individuals who purchased a particular product from your organization and subsequently create sales incidents (sales opportunities) for the entire or partial result set as a bulk action.
  • Users can build a list of all individuals employed by a particular company who have a Title Code of Sales Engineer and, using the bulk action of creating an external contact, assign them a manager.
