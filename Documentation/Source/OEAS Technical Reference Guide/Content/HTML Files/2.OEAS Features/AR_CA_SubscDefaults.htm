Default subscription relationships and rules

The following subscription relationships are created with installation of OEAS to support the subscribable objects provided with that installation. Rules that are set to true by default for each relationship are listed. No role-based rules are set to true by default.

The Always subscribe the primary object rules that are provided by default are required rules. Changing this rule can adversely affect the display of records in OEP-Offline. It can also cause incomplete unsubscription cascades resulting in unnecessary synchronization of records and orphaned offline records (records that the end-user cannot access). For more details about required relationships, see Required subscription relationships.
