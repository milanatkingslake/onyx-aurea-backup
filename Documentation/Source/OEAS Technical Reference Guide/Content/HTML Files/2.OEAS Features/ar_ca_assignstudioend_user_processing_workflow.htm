End user workflow

The following end user workflow describes how conditions are evaluated and processed through OEAS and OPM:

  • The OEP user creates a company, individual, or incident record and saves it to the Onyx database.
  • At run time, the OPM process retrieves an ordered, filtered list of expressions (grouped by condition) from OEAS.
  • The OPM process finds the first condition that is true for the record and stops evaluating the expressions. (A condition is true when all of its expressions are true.)
  • OPM associates new and updated records with the territory that owns the condition.
  • OPM assigns the incident and creates the correct internal contact records.

Condition processing runs in the background and is not visible to end users.
