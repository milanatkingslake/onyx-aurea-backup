Subscription

The Subscription feature is part of the OEAS functionality that enables OEP end-users to copy transactional data to OEP-Offline so that they can access that data while they are disconnected from the network. To take existing data offline, end-users must first subscribe to the data in OEP. Then, after they have synchronized the data with OEP-Offline, they can work with the data in OEP-Offline while disconnected from the network.

The Subscription feature enables:

  • Administrators to define OEP end-users as subscribers, who can then subscribe to OEP data and work with it in OEP-Offline.
  • Administrators to define OEP end-users as advanced subscribers, who can then subscribe to OEP data on behalf of one or more other offline users.
  • Administrators to define the subscribable objects that control which types of OEP transactional data the subscribers can take offline.

The main elements of the Subscription feature are described in the following sections and include:

  • Subscribers
  • Subscribable objects
  • Subscription relationships, rules, and roles

The OEAS offline functionality is supported by the synchronization subsystem of the Onyx Processing Server (OPS). For an overview of the offline functionality, see the OEP-Offline Implementation Guide.
