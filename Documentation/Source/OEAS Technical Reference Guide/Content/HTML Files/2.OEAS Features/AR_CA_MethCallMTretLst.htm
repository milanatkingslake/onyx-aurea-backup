retrieveList

Although retrieveList OEAS methods are supported method types for merge template method calls, the Templates feature itself supports returning only one object. As such, a typical use of a retrieveList method call is to return a single object, but specifically to return a value that is not available from a standard retrieve method.

For instance, incident.retrieveListQuickSearch returns the following values that incident.retrieve does not return:

  • Assigned To (the name rather than the ID value)
  • Owner ID (the secondaryId of the owner of the incident object)
  • Contact First Name
  • Contact Last Name
  • Company Name
  • Phone Number (the primary phone number of the contact)
  • Source (the description rather than the ID value)
  • Priority (the description rather than the ID value)
  • Status (the description rather than the ID value)

The following example uses the incident.retrieveListQuickSearch OEAS method. This method call is designed to:

  • From an OEP incident details window, accept the incident’s primary ID as the primary ID of the incident for which you are gathering the information.
  • Return the contact name for the incident, based on the rules of the retrieveListQuickSearch method.
  • Return the primary phone number for the incident’s contact.

The retrieveListQuickSearch rules for returning contact information are as follows:

When a company owns the incident When an individual owns the incident
  • If the incident has a primary contact, return that primary contact’s name and phone number.
  • If the incident does not have a primary contact, return the name and phone number of the company’s primary contact.
  • If the company has no primary contact, then only the phone number is returned.
  • Return the individual’s name and primary phone number, regardless of whether or not the incident has a different primary contact.
