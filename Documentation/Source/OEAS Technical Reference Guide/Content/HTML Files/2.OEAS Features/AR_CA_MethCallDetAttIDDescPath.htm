ID Path and Description Path

You can define the method call detail ID Path and Description Path attributes only for a script output method call detail of the list type. List type output method call details support an output having both an ID component and a description (text) component, although neither component is required. The ID component is typically an identifier for the item returned (such as country.countryCode or company.primaryId), whereas the text component usually represents a user-friendly display string for the item returned (such as country.description).

The ID Path and Description Path attributes are always relative to the output nodes, which are located when you specify the Element Path attribute. For a description of the Element Path attribute, see Element Path.

The following additional details are relevant to the ID Path and Description Path attributes.

ID Path

The ID Path attribute specifies an XPath that locates the outputâ€™s ID component. The ID Path is optional. If it is null, the returned output does not contain an ID component.

The path is always relative to the output nodes, which are located by the Element Path attribute. The path should lead to a single element that is a descendant of an output node. The path should not include traversal operations that try to access elements above the output node. If the path does not result in a selected node when the method call is executed, the returned output contains an empty ID component.

Description Path

The Description Path attribute specifies an XPath that locates the value of the text component. The Description Path attribute is optional. If it is null, the returned output does not contain a text component. However, it can still contain a text component and/or an ID component.

The path is always relative to the output nodes, which are located when you specify the Element Path attribute. The path should lead to a single element that is a descendant of an output node. Like the ID path, the description path should not include traversal operations that try to access elements above the output node. If the path does not result in a selected node when the method call is executed, the returned output contains an empty description component.
