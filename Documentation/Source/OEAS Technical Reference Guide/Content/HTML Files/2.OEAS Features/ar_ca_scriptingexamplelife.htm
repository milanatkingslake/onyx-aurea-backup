LifeInsuranceQualifications step

This step obtains information that qualifies the customer for one of several life-insurance policies.

In the following table, the rules appear in the order in which they occur in the Rules box, which is also the order in which they are evaluated.

Note the following:

  • Arithmetic operators - The “<” operator is used only in the condition statement that performs a numeric evaluation. In rules, the “<” and “>” operators cannot perform a string evaluation.
  • Order of rules evaluation - Rules are evaluated in the order in which they are listed in the Rules box on the Rules tab for a step. Evaluation of rules for a step stops as soon as a rule evaluates to true. As a result of these two facts, the order of the rules in the list can affect which step the script session goes to next. Because the last four rules are the four possible combinations of being either single or married and either a smoker or a nonsmoker, the Single-Smoker rule does not need a condition statement, because it is only evaluated when the previous three condition statements have evaluated to false. With all the other possibilities eliminated, only single smokers are left.
