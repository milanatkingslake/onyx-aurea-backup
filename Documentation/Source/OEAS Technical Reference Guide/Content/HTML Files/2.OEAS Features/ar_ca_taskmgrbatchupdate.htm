Batch updates

Of the records retrieved in Task Manager, users can specify the following:

  • Certain records to update with a single action
  • Which fields within those records to update
  • The values with which to update the fields

Each save of a record in a batch update is a separate transaction, so that even if one fails, those which precede and follow it still succeed.

There is no concurrency control for batch updates. That is, if a userâ€™s batch update updates a record that another user is modifying, the save action of the batch update is successful. When the second user tries to save the record, he or she sees a message that someone else has modified the record. This user must re-open the record and repeat the changes.
