Tasks

You use a task to identify and track one or more assignments that need to be completed in order to address issues that arise from a single incident. The Task feature enables users to create tasks, to assign tasks to other users, and to update tasks either to track progress or to change the scope of the task. Users can also specify when tasks are due and set reminders for them, as well as create work notes with each update to provide details of their progress. Users can also relate tags and internal and external contacts to a task.

In OEAS, the three default task categories are Sales tasks, Service tasks, and Support tasks. You can configure additional task categories to meet other functional requirements of your organization.

You can add any number of tasks to an incident. A task:

  • Cannot exist independently; it is always owned by an incident, and the task category must be associated with the incident category.
  • Can have any number of work note entries.
  • Can have only one reminder.
  • Can be assigned to only one user.
  • Can have only one primary contact.
  • Can have any number of tags.
  • Can have any number of file attachments.
