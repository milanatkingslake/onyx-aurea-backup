Object relationships

The object relationships that support the Security feature are illustrated in the following graphic.

Note these items concerning security objects:

  • The Onyx Enterprise Database (OEDB) contains tables that associate user objects to resource and role objects, and that associate resource objects to role objects. These tables are not represented in the OEAS object model. OEAS uses these associations to create a resource cache for permissions checking.
  • The role and group objects serve different purposes. The role object helps administrators efficiently assign resource access permissions to users. The group object lets administrators arrange users in the Security Administration tool according to a business organization’s structure.
  • The group object supports nesting. The parentGroupId property identifies the owning group.
