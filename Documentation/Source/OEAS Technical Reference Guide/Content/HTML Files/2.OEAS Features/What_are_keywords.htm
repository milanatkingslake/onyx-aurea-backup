What are Keywords?

A keyword is a word or phrase that an end user associates with an incident, task, or work ticket when they are creating or updating the record. The keyword is used to provide a logical criterion on which users can subsequently search to locate the record.

Keywords data is not case sensitive. The keywords “procedure” and “Procedure” are saved as a single instance of a keyword.

Keywords are saved to the database associated to the specific record with which they were created. In addition, after end users have associated a particular keyword to records of the same type a specified number of times, that keyword becomes a keyword lookup value for all records of that type. This means that subsequently, when a user creates or updates a different record of that same type, that keyword value appears as a lookup value so that users can select it to be associated with the record, rather than having to manually type the keyword.

The default records that support keywords are:

  • Work ticket
  • Incident
  • Task

Keywords for incident and task records are further defined by incident and task categories.

The number of times a keyword must be used before it becomes a keyword lookup value is determined by a system parameter. For keywords for incidents and tasks, the system parameter is KeywordIncludeCount. For keywords for work tickets, the system parameter is WorkTicketKeyWordIncludeCount.
