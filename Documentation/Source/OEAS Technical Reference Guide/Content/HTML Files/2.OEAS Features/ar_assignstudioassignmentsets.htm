Assignment sets

An assignment set is a pool of users that can be linked to conditions and assigned to incidents, companies, and individuals.

Assignment sets enable you to:

  • Create internal contacts to associate with company, individual, and incident conditions
  • Assign incidents to users

Creating internal contacts for conditions

Users in assignment sets are the internal users responsible for contacting customers and resolving incidents within OEP. These users are created as internal contacts through their user IDs and internal contact type IDs. Only users of the same object type as the active assignment set are available. You can type IDs or select them from a list. IDs are case-sensitive and must be entered accurately.

Assigning incidents to users

Users in incident assignment sets are optionally assigned to incidents based on the following assignment types:

  • Normal

Assigns incidents to a single user within the assignment set.

  • Round Robin

Assigns incidents to the next user in a pool of users (in turn).  This option keeps track of who was assigned last and determines who should be assigned next.

  • Least Busy

Assigns incidents to the user with the lightest workload (least amount of assigned incidents).

Manual assignment of incidents

Unlike the company and individual update process, the incident update process is not configured to run on every incident update. For incidents, the update process only runs when an OEP user manually re-assigns an incident to a particular user called the Auto Assign user. For more information, see Manual assignment of incidents.

Defining assignment sets

To define an assignment set, you must add an assignment set to an assignment set node, associate users with the assignment set, and link the assignment set with the appropriate conditions (of the same object type). You can reuse assignment sets throughout the Assignment Tree.
