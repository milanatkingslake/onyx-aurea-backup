Revising a script that has been published

There are several considerations to keep in mind when revising a script that has been published:

  • If you deactivate a script while a script session based on the script is suspended, the end-user who tries to resume the session receives an error. Also, end-users cannot view sessions for inactive scripts.

When you re-activate the script, both of these issues are resolved, but might have already caused problems.

  • If you delete a step, prompt, or rule from a script, then when the script is again available, end-users who review the script’s sessions will encounter an error when they arrive at the deleted or affected step.
  • When a prompt is inactive, its responses are not displayed in the end-user interface.

Therefore, to make a script unavailable, do not make it inactive. Instead, in the Script Edit area, under Availability, do one of the following:

  • Uncheck the Published box.
  • Specify a date in the Effective Date or Expiration Date box that excludes the time for which you do not want the script to be available.

In addition, the following are best practices for revising scripts that have been published:

  • To change text in the script, unselect the Published box and then reselect it after completing the text changes.
  • To delete a step, prompt, or rule from the script:
  • If you need to save the previous version of the script and its sessions, clone the script, then perform the deletions in the clone. That is to say, create a new script.
  • If you do not need to save the previous version of the script and its sessions, make the changes in the current script.
