What are OEAS Step Components?

Step components are compiled COM components that are responsible for performing a predefined set of actions. OEAS step components differ from OEAS business objects in several significant ways. In general, step components:

  • Are pre-compiled, binary COM+ components that run within a COM+ library application.
  • Are instantiated using the step data stored in the OED.
  • Implement special logic or system integration functionality beyond the basic delete, insert, retrieve, and update methods of a typical business object.

OEAS includes one step component, Onyx Data Manager, that implements every method that uses SQLGen. For more information, see Onyx Step components.

During OEAS setup, the OnyxEASUser COM+ role is not set up as a Power User account by default and may therefore not have permissions to access every directory on the machine. Therefore, do not put step components in directories such as the desktop or directories under \WINNT\Profiles. Ensure that the OnyxEASUser has directory access to all step components.

All step components implement the IStep interface.

The OTM instantiates step components from their corresponding methods/steps whenever a client calls a method. The step component receives step package XML from the OTM and performs its processing. If necessary, it modifies the step package and then returns it to the OTM. Step components can also leverage the IInternalTransactionManager interface to make calls to logical business objects.
