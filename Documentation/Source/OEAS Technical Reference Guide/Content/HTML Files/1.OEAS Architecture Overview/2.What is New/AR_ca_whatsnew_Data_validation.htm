Data Validation

In OEAS, all methods begin with a data validation step "OnyxDataValidationStep". All methods in OEAS must use the OnyxDataValidationStep component as the first step. Omitting OnyxDataValidationStep from a method is not supported and might cause the method to behave in non-standard ways. The only exception to this requirement is a custom method which does not follow any of the OEAS method standards.

Custom methods that are intended to conform to OEAS standards must begin with this step. The data validation step improves consistency, accuracy, and integrity in the database.

This step calls a COM component that validates the input XML for the method against logical property metadata in the Onyx Enterprise Dictionary (OED).

Data validation includes such things as:

  • Default property values
  • Property constraints
  • Trimming of white space (which could break other methods)
  • Partial updates

For more information on data validation, see "OnyxDataValidationStep" in the Customizing OEAS.
