Referential Integrity

The new Onyx Enterprise Database (OEDB) supports referential integrity by implementing foreign keys on many tables. The referential integrity built into OEDB guards against data corruption by preventing an application or user from entering invalid data in a column that has a foreign key relationship with a column in another table. Because the database implements referential integrity, client applications should confirm the validity of input data to prevent database generated errors. For more information, see "Referential integrity" in Customizing OEDB.
