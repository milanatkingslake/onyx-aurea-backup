Customizing Objects and Tables

The new OEAS supports customizing objects and tables without writing any code.

To add custom data to an existing object:

  • Add a custom column to the appropriate table in the database
  • Add the column's physical description to the Onyx Enterprise Dictionary (OED)
  • Add a logical property description to the OED
  • Create a map between the logical property and the physical description in the OED

By default, SQLGen automatically provides CRUD support for custom data (columns) implemented with the previous procedure. The procedure for adding custom objects is similar but a little more involved, as the customer must create a new database table and all of its logical and physical descriptions and mappings in the OED, however no coding is required for CRUD functionality.

To make this procedure faster and easier, Object Designer supports copy and paste of most nodes with the notable exception of logical-physical maps. You can copy an object and paste a duplicate within Object Designer. By renaming, adding, and deleting nodes (object, method, property, and so on) in the duplicate, you can quickly build custom objects. Nodes can be copied and pasted across owner objects as in copying a property from one object to another.

For more information, see Customizing OEAS and Customizing OEDB.

UDF Support

Although UDF (User Defined Field) properties are still supported, Onyx recommends using the previously described approach instead, to ensure meaningful names and strongly typed data. The latter is important especially in the light of new support for data validation. Using Reference Table Administration, UDFs are a quick way to implement hierarchical relationships between UI drop-down list controls.
