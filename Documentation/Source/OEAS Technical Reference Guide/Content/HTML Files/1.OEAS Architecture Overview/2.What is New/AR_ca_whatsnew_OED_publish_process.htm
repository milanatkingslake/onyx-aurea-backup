OED Publish Process

To put customization into production, a developer or administrator must publish the OED using Object Designer. By default, Object Designer implements the following publish process.

  1. Validate OED content (XML)
  2. Create runtime copy of OED from working copy
  3. Generate abridged OED (AOED - optimized for SQLGen)
  4. Generate XSL templates used for run-time SQLGen
  5. Generate SQL stored procedures
  6. Compile generated SQL stored procedures into database
  7. Insert data into OEDB tables (oed_object_map, oed_relationship_map)
  8. Stop OnyxEASApp COM+ application to refresh OED Cache and Method Cache when the application restarts.

The publish vbs script is called during the publish process. Your OEAS CD includes the script that implements the publish process as open source code. You can modify this script to meet your business requirements for deploying OED customizations.
