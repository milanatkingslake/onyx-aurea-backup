OED Structure

For the new OEAS, the OED (Onyx Enterprise Dictionary) structure includes many new elements. In addition to the Business Objects structure in previous versions, the new OED includes a Physical Model structure. The Physical Model describes the tables, procedures, and functions in OEDB. The Logical Business Objects structure includes new child elements required for SQLGen. These new child elements enable functionality such as cascading deletes, data validation, and physical persistence. This structure enables customization of OEAS without writing code. By describing both the physical and logical models, and by creating a mapping between the two, customers get create, retrieve, update, and delete (CRUD) functionality automatically through SQLGen. For non-standard functionality, business logic can still be mapped directly to database code or custom COM components.
