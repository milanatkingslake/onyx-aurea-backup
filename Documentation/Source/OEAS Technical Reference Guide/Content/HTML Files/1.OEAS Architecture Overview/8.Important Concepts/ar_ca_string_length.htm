String Length

When you map properties with a string data type to the database schema, it is important to understand how the string length relates to the database column length and how a maximum constraint affects string data handling in the middle tier.

OEDB uses nchar, nvarchar and nvarchar(max) data types for strings. The column length value should always be supplied for columns defined by these types. The nvarchar(max) datatype is specified by selecting “NVARCHAR” for the column type and “MAX” for the length. Onyx uses the nvarchar(max) data type for large strings.

String properties can have maximum constraints applied in the Logical Business Objects section of the Object Designer. For insert and update method calls, the data validation component enforces the constraint if it exists.

The following table presents three cases where a string value exceeds the database column length and the resulting behavior of the Onyx platform.

String data case Method, action, or procedure type Result

String property value with maximum constraint exceeds specified length

insert or update methods and actions

The data validation component returns a failure.

String property value without maximum constraint exceeds database column length

insert or update methods and actions

No data validation check; OEDB truncates the data to the length of the underlying column in the table.
