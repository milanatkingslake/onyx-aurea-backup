Concurrency Control within a Transaction

The optimistic concurrency control of the Onyx Enterprise Database (OEDB) supports the following:

  • No records are locked outside of a database or COM transaction.
  • Records within database or COM transactions are locked, preventing users from overwriting each otherâ€™s data.
  • Locked records are released at the completion of a transaction.

An attempt is made to lock a record within a transaction. If the record is locked, the database will pause until the other transaction has been completed and the new lock can be acquired. The value retrieved for the onyxTimestamp property is checked after the lock has been acquired.
