includeMask parameter

When you retrieve data from the database, you can choose to append the mask to the data, to apply the mask to the data, or to disregard the mask altogether. OEAS uses an input parameter, called "includeMask" to indicate how the mask should affect the data upon retrieval, if at all. The following table explains these options.

In OEAS, all includeMask parameters default to “append” if no value is passed.

Mask option Input parameter Description Sample result

none

<includeMask>none</includeMask>

Returns the original data string.

5555555555

append

<includeMask>append</includeMask>

Returns the original data string, and appends the mask to the end of the string. The code **MASK: indicates that a mask is appended (in the sample result, the mask is 99.99).

5555555555**MASK:(999)999-9999

apply

<includeMask>apply</includeMask>

Returns the data string formatted according to the rules of the mask.

(555)555-5555

The following input XML for the customer retrieveList method shows the includeMask parameter with a value of 'apply'.

<parameters> 

<type>9</type> 

<primaryId></primaryId> 

<secondaryId></secondaryId>
