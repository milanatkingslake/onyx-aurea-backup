Date and Time Data

To maintain consistent behavior in an environment that might contain client applications from different time zones, the OEAS processes all date and time information using Coordinated Universal Time (UTC). Custom client applications must implement a scheme to convert this data on input and output to suit end-user requirements. UTC, like Greenwich Mean Time, is set at 0 degrees longitude on the prime meridian. All date and time data are stored in UTC in the Onyx Enterprise Database. The OEAS application assumes all date and time data are in UTC. Custom step components that generate date/time values must do so using UTC. Client applications that save date and time data to the Onyx Enterprise Database without using the Onyx Transaction Manager must do so using UTC. Failure to follow either of these practices might produce data corruption and unexpected results.
