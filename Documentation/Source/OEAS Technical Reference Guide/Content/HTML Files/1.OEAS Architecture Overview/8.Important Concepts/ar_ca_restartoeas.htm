Restarting OEAS

OEAS must be restarted when you want Onyx Enterprise Dictionary (OED) and National Language customizations to take effect. You customize OED using Object Designer. Changes to National Language strings can be made through Table Administration. These customizations appear in each tool as users make them. To improve application performance, OEAS caches OED and National Language data in memory at run time. When you customize either OED or the National Language tables, your changes are stored to disk. However, they are not available to OEAS until they have been published and the system restarted.

If users are on the system, pending COM+ transactions might not complete as expected when you shut down OEAS. For this reason, restarting OEAS should be performed when no users are on the system.

Restarting OEAS involves stopping the Microsoft COM+ application package that contains OEAS.

To restart OEAS:

  1. From the Start menu, choose Programs\Administrative Tools\Component Services.
  2. Under Console Root, expand Component Services\Computers\My Computer (or target OEAS machine)\COM+ Applications.
  3. Right click on the OnyxEASApp COM+ application package and choose Shut down.

The next time your client application calls OEAS, the COM+ application will restart automatically.
