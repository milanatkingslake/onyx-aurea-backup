Using UserID Preferences with GlobalLogSettings Preference

The GlobalLogSettings data value and the UserID data value settings are complementary. This means that the individuals to whom you assign UserID log preferences can receive more log messages than those whose log settings are derived only from the GlobalLogSettings value, but they can never receive fewer log messages.

As the following table demonstrates, if you set the value of GlobalLogSettings to 3, error messages and warning messages are logged for all users, even for those whose UserID (JDOE) is set to 1 (error messages only).

As demonstrated in the following table, if the value of someone’s UserID (BROSS in this case) is set to 5 (error and debug), that person receives messages for error and debug message types as well as warning message types. This is because the value of the UserID setting (5) indicates that this person receives error and debug messages, and the GlobalLogSettings value (3) indicates that all users receive error and warning messages. Since the results of the data value settings are complementary, this person receives both what is logged globally (error and warning) and their customized settings (error and debug), resulting in logging messages of all types.
