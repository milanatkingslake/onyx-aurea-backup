What is Onyx Logging?

Onyx Enterprise Application Server (OEAS) uses an Onyx logging component to write a log event to your OEAS server(s). Depending on how you set your logging preferences (see Setting logging preferences), the logging component can provide message types as demonstrated in the following table.

Message type Description

Error messages

Error messages are logged when a critical failure occurs. An example is “The Session ID is invalid.”

Warning messages

Warning messages are logged for informational purposes for an unexpected and non-catastrophic result, or for validation and business logic failures. An example is “The attempt to load the AppData Settings at ‘MyKey’ failed. Defaulting to the Predefined value ‘myValue’.”

Debug messages

Debug messages include trace enter messages, trace exit messages, and progress messages. Use trace enter and trace exit messages to track the execution path for a method, and use progress messages to log any other debug message that you want. An example of a trace enter message is “OnyxDataManager.DataManager.Execute.Enter.” You typically write trace messages to understand the execution order for a method, and particularly for logging the start and finish of a method.
