Setting Logging Preferences

You can choose to log any combination of error, warning, and debug messages with the exception that regardless of the settings that you choose, error messages are always logged and you cannot turn them off. A system administrator manages these settings using the OEAS Configuration utility. For more information, see OEAS Configuration utility.

You can set your logging preferences by:

  • Modifying the event log settings for GlobalLogSettings.
  • Adding an event log setting for an individual user (based on their User ID configured in Security Administration) who requires a different set of log messages than those provided by the GlobalLogSettings value,.

Using the following table, you can determine the value you must set for the type of event log setting that you want to provide. For example, to provide only error and debug messages, assign the GlobalLogSettings or UserID with a value of 5. Or, to provide error, warning, and debug messages, assign a value of 7.

The default logging preference is for a GlobalLogSettings value of 1.

If you have created custom Onyx Steps (using the Object Designer) that pass sensitive data in XML format, do not enable debug messages unless you also intend to implement encryption algorithms such as public-domain RSA. Otherwise, the sensitive data can be obtained unencrypted from your Onyx log.

Data value Description

1

Logs error messages only. This is the minimum type of logging allowed and is the default.

3

Logs error and warning messages.

5

Logs error and debug messages.

7

Logs error, warning, and debug messages.

If the logging behavior does not function as expected, review the acceptable logging preference data values and make sure that you have set the correct value for the logging messages that you require.
