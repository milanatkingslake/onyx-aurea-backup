Step 3

The calling client sends the task data, in XML, to the OTM. Since the OTM does not inherently know how to process the request (the task.insert method), it queries the Onyx Enterprise Dictionary (OED) in-memory cache for the steps that define the insert method for the task object. If the OED cache was not loaded previously, the cache will load itself upon the OTM request. The OTM sends the application name, the object name (task), and the method name (insert) to the OED cache to obtain the instructions for the task.insert method.

The OED returns to the OTM the steps that are required to complete the request. In this example, these steps provide the information that is required for inserting a task record to the Onyx Enterprise Database.

A separate OTM is instantiated for each call to the Onyx Enterprise Application Server, and not necessarily always on login. The first time that the OTM is instantiated for any reason, the caches are loaded and retained in memory until the OnyxEASApp COM+ application is restarted or the server is shut down.

When the National Language cache is needed, such as to report a message to the calling client, the OTM, the ODM, or other step components can reference it as well.
