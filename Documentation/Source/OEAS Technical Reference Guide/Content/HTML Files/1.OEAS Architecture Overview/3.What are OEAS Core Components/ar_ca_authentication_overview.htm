Authentication Overview

OEAS security has two authentication components, AuthenticateNT and AuthenticateOnyx. These authentication components implement two types of user validation schemes. It is the client's responsibility to call an authentication component to verify user access to the Onyx Enterprise Application Server (OEAS). Once the user's identity and access are verified, a session is established.

The authentication portion of the OEAS security is separate from other security functions, primarily to accommodate the many forms of authentication available in the marketplace.

The AuthenticateNT component exposes the IAuthenticateNT interface and is designed for users accessing the system through the local network, mapping Microsoft Windows users to OEAS users. The AuthenticateOnyx component exposes the IAuthenticateOnyx interface and is used when a login and password authentication scheme is desired rather than integrated Windows authentication (single sign-on).
