Literature Order Line Status

The literature order line status is used to track the status of order lines. The status is displayed in the New Order Lines queue and Completed Order Lines in OPS Administration. The following table describes default order line statuses.

Literature order line status code

Status

Description

0

New

(See Warning following this table)

This is a new order line; its processing has not begun.

1

In Process

(See Warning following this table)

The order line is being processed.

2

Completed

(See Warning following this table)

The order line’s processing is complete.

3

Error

(See Warning following this table)

The order line failed during processing.

4

Data Error

There is an error in the order line data.

5

Inventory Hold

Hold the order line; do not process it.

6

Back Order

The order line back-ordered; do not process it.

7

Customer Hold

Hold for customer; do not process the order line.

Do not delete the following domain data for literature order line status: New, In Process, Completed, or Error. If OPS tries to set the status of an order line and the domain data doesn't exist, OPS will fail. You can change the value of the status or description properties on the literatureOrderLineStatus object, for example to localize, but Onyx does not recommend changing their meaning, as it might confuse end users.

You can modify the following statuses:

  • Data Error
  • Inventory Hold
  • Back Order
  • Customer Hold

Using Table Administration, you can administer the data by:

  • Creating a new status
  • Updating an existing status
  • Setting the status to active or inactive
  • Deleting an existing status

To configure the data:

  • In Table Administration, select Literature Order Line Status in the Table drop-down box and make changes.

For more information about using Table Administration to configure this data, see Table Administration Help.
