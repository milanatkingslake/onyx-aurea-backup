Inbound Email Processing Error Handling

When an inbound email processing error occurs, the subsystem logs the error in the event log and moves the email to the error folder. (This folder must be created on the mail server under each mailbox that is monitored.) You can move an email from the error folder back to the inbox to be processed again after the error is resolved.

Using OPS Administration, you can configure an EMS to use emails to send error notifications to an internal user (usually the system administrator).
