Action Batch

An action batch is a specification of any number of actions to perform on one or more records in the database. With List Manager for instance, a user can create and run a database query that returns a list of customers and then perform any number of actions on some or all of the records in the list. Although one action batch can contain unlimited actions on multiple customers, just one row will be added to the action_batch table. This row is referred to as an "action batch."

For information on how how OPS components work together to process action batches, see Action batch process flow.

For example, the Onyx Employee Portal (OEP) List Manager feature uses the action batch subsystem to perform the following:

  • Create sales opportunities for selected items
  • Create service requests for selected items
  • Create support incidents for selected items
  • Add an external contact to selected items
  • Add an internal contact to selected items
  • Create a campaign for selected items
  • Create a literature order for selected items
  • Add a survey to selected items
  • Create script sessions for selected items
  • Reassign territory

The following table describes available action batch attributes.

Attribute How subsystem uses attribute

Action batch XML

The XML that describes the OEAS method calls for the action batch.

Action batch status

Current status of the action batch. For more information, see Action batch status.

Scheduled process date

If this attribute is NULL, then the action batch is processed as soon as possible. If this attribute is NOT NULL, then the action batch is processed on or after the scheduled process date.

Insert by

The user by whom the action batch is executed.

Insert date

The date on which the action batch was created.
