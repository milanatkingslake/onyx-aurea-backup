Mailing-Label Processing

You can configure mailing labels to be printed when users order literature products. The print subsystem is responsible for processing the labels. Labels are a special case in which multiple order lines for the same literature product are printed together. This conserves the label printing media by allowing multiple labels to be printed on a sheet of label stock.

When a user places an order for a literature product that has an associated mailing label, the print subsystem inserts an additional order line into the order for the label product. When a batch has multiple order lines for the same label, all the labels are printed together. The following example illustrates the label processing.

Example: Mailing-label processing

In this example, the following occurs:

  1. You create a label literature product and name it “Mailing Label 5163”.
  2. You create a second literature product, name it “XB Product Brochure”, and associate it with “Mailing Label 5163”.
  3. Six users create literature orders for one “XB Product Brochure” each.

This results in six literature orders with one order line each.

  1. A print subsystem retrieves all the orders in a single batch.
  2. The subsystem inserts one additional order line for “Mailing Label 5163”, which will be inserted into each of these six orders.

Each order now has two order lines for a total of twelve order lines.

  1. The subsystem processes the orders, grouping the six mailing label order lines together and printing the mailing labels at the same time on the same label sheet.
