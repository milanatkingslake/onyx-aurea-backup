Highlights of the example are as follows:

Step 1. The fulfillment subsystem does not check the queue for order lines when outside the operation window.

Step 2. When the operation window begins, the first interval is always a Standard Priority Run Interval. Both standard-priority and high-priority order lines are retrieved during the Standard Priority Run Interval. The high-priority items are processed first.

Step 4. When the scheduled start time for Standard Priority Run Interval and High Priority Run Interval coincide, a Standard Priority Run Interval occurs.

Step 5. During the High Priority Run Interval, only high-priority order lines are processed.

Step 9. The subsystem does not stop processing an existing batch of order lines when the operation window’s Stop Time arrives. The subsystem does not start processing a new batch of order lines when the Stop Time has passed.

Step 12. The subsystem skips an interval if it is still processing a batch.
