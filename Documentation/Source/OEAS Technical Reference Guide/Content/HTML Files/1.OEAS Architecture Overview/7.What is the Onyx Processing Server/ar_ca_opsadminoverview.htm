OPS Administration Tool

The OPS Administration tool enables you to manage the OPS service and its subsystems. Using OPS Administration, you can:

  • Add, configure, and delete subsystems.
  • Start, stop, pause, resume, or restart the OPS service and/or its subsystems.
  • Activate and deactivate subsystems.

In addition, you can start, stop, pause, and restart OPS service from Windows Services Administrative Tool.

Any action that you perform on the OPS service also affects each of its subsystems, as follows.

OPS service - current state Action performed on service Subsystem response

Stopped

Start

All active subsystems are started.

Inactive subsystems remain stopped unless explicitly activated and started.

Started

Stop

All subsystems stop.

Started

Pause

All running subsystems are paused.

Paused

Resume or Restart

All subsystems that were paused/stopped when the service was paused resume processing.

All subsystems that were paused/stopped prior to the service being paused remain paused or stopped.

If a subsystem fails to start, its status is updated to Error. To recover from an error, stop the subsystem and then try again to restart it.

For further details about managing OPS and its subsystems, see Customizing OPS.

Access to OPS Adminstration can be controlled using standard OEAS roles and resources. For more information, see Configuring OPS security.
