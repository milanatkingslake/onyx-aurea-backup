Action Batch Process Flow

This section describes how OPS components work together to process action batches. The batch subsystem processing begins with a request by the OPS subsystems for an action batch. The batch subsystems process action batches, resulting in OEDB updates via method calls to OEAS. The following graphic depicts the OPS action batch process flow:

  1. A user submits a request for actions to be performed for each member of a list of records. This set of actions is an “action batch.” Action batch status is set to New.
  2. The subsystem requests an action batch from OEAS. Status is updated to In Process.
  3. The subsystem begins processing the action batch. The batch subsystem places method calls to OEAS to carry out the actions.
  4. Any errors that occur during the process are logged in the Event Log. You can configure the OPS service to send an error notification via email in addition to logging the event.
  5. The subsystem updates the processed action batches’ status to Completed or Error as appropriate.
