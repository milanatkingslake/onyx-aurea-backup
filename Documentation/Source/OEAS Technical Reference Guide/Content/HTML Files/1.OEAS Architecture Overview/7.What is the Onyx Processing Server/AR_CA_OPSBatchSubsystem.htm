Batch Subsystem Processing

Batch subsystems process batches of actions, such as inserting an incident for multiple customers into the OEDB. (An action batch is defined as a row in the action_batch table.) When the subsystem retrieves an action batch for processing, the action batch XML contains all the necessary instructions and information for the batch subsystem to process the action batch. For more information on action batch XML, see "actionBatchXML schema" in Customizing OEAS.

The subsystemâ€™s processing of action batches takes place according to the subsystem settings that you create and modify using OPS Administration. When the subsystem processes an action batch, the subsystem makes calls to the OEAS server.

When executing OEAS calls, the batch subsystem impersonates the user who inserted the action batch. This ensures that the insertBy and updateBy properties correctly reflect the original user who initiated the actions. To enable the subsystem to successfully impersonate the user and execute the OEAS calls, Allow Impersonation must be selected for the Onyx Processing Service User in Security Administration. For more information on modifying account attributes, see Security Administration Help.
