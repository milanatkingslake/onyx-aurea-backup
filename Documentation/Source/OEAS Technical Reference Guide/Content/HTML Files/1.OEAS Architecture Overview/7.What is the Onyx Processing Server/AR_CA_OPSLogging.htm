Logging

Logging Overview

Up to three log files record events and other data related to the sync subsystem:

  • OPS event log
  • Sync subsystem file log
  • Sync subsystem XML log

OPS always logs error messages to the event log, but you can configure OPS to log warning and information messages, as well. The person who installs OPS configures OPS logging during the OPS Configuration Setup. Because information logging impacts system performance, Onyx recommends that you not enable information logging except to troubleshoot the OPS service or a subsystem.

To enable, disable, or reconfigure these aspects of OPS event logging, you can either run OPS Configuration Setup again or change certain settings in the Registry Editor. For more information on the registry entries, see “Configuring OEP-Offline” in the OEP-Offline Implementation Guide.
