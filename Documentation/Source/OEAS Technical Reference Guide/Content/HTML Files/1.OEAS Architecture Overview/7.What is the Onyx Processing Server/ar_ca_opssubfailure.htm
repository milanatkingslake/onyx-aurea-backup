Subsystem Failure

Subsystem failure can be classified as either non-critical or critical.

  • A non-critical failure, such as an error when inserting email in the OEDB, creates an entry in the event log and moves the email to an error folder. The subsystem continues processing the next inbound email message.
  • A critical failure stops processing for the subsystem, logs the error, and sends an error notification as appropriate. After the error condition is fixed, you must stop and restart the subsystem for it to continue processing. For more information, see OPS Administration tool.
