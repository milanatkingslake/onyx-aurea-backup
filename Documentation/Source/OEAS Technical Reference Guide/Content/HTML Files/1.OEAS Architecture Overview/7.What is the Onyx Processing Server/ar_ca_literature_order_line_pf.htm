Literature Order Line Process Flow

OPS components work together to process literature order lines. The fulfillment subsystem processing begins with a request by the OPS subsystems for literature order lines. The subsystems process print and email order lines, outputting Microsoft Word documents and email messages. The following list describes the OPS process flow:  

  1. A user submits a request for a literature product to be sent via mail or email to a customer. Order line status is set to New.
  2. The subsystem requests a batch of order lines from OEAS. Status is updated to In Process.
  3. The subsystem processes the order lines:

3a. The print subsystem merges the order line data into a print template.

OR

3b. The email subsystem merges the order line data into an email template.

  1. After the subsystem’s processing of the order lines is complete:

4a. The resulting Word document is sent to a print queue for printing.

OR

4b. The resulting email is sent to the recipient.

  1. Any errors that occur during the process are logged in the Event Log. However, only for Print subsystem the errors are logged in the
    ..\<installation folder>\Components\Logs\OPS.log file. You can configure the OPS service to send an error notification via email in addition to logging the event. For more information to configure logging, see Print Subsystem Logging.
  2. The subsystem updates the processed order lines status to Completed or Error as appropriate.
