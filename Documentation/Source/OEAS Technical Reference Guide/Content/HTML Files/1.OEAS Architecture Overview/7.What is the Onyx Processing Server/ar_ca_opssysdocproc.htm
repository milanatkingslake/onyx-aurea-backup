Processing System Documents

When a fulfillment subsystem begins to process a batch of order lines, one or more new order lines are added for each system document associated with the subsystem. For a batch system document, a new order that contains one order line for the document is added to the batch. For an order system document, a new order line is added to each order in the batch. OPS always processes order lines for system documents before processing order lines for literature products.

The following example illustrates the orders and order lines inserted during OPS processing of system documents.

Example: System document processing

Ten literature orders are submitted for one “XB Product Brochure” and one “Welcome letter.” A print subsystem is configured with the system documents as shown in the following table.

Literature product name System document type Fulfillment type

Emailed Batch Report

Batch system document

Email HTML as attachment

Printed Batch Report

Batch system document

Print document

Customer Packing Slip

Order system document

Print document

OPS processes orders these orders as follows:

  1. The fulfillment subsystem retrieves the 10 orders in one batch.

The batch has total of 20 order lines. (10 orders x 2 order lines each = 20 order lines)

  1. One order line is added to each order for the “Customer Packing Slip.”

The batch now has 30 order lines.

  1. One new order is added to the batch with the following two order lines:
  • One order line for the “Emailed Batch Report” (because this order line is for an email fulfillment type, an email subsystem processes it)
  • One order line for the “Printed Batch Report”
  1. The subsystem processes the batch with the following batch system document output:
  • One “Emailed Batch Report” emailed to subsystem’s administrator’s address
  • One “Printed Batch Report” printed
  1. The final order printed output is 10 sets of the following:
  • 1 - “XB Product Brochure”
  • 1 - “Welcome letter”
  • 1 - “Customer Packing Slip”

Note that if you create a system document with an email fulfillment type, the system document is sent to the administrator's email address as defined for the subsystem in OPS Administration. To handle the system document differently, you can create a customized email routing rule to route the system document to the appropriate recipient.

The "Email To" field for order system documents incorrectly lists the Literature order recipient's email address instead of the administrator's email address. However, the email is delivered to the administrator as expected.
