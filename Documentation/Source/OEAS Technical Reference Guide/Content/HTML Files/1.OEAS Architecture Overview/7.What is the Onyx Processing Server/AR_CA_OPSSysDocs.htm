System Documents

System documents are literature products that you configure to generate automatically when a subsystem processes orders. Sample system document templates are installed during OPS installation. The following table describes the two types of available system documents and includes an example of how each might be used.

System document type System document generation Description and example of use

Order system document

One document generated per order

An order system document includes a list of all the order lines that comprise the order. Typically, fulfillment personnel use the document as a packing slip to be included with a customer’s order.

Batch system document

One document generated per batch of order lines

A batch system document includes a summary of all the order lines processed in the batch. Typically, mailroom operators use the document as a batch report when assembling printed documents and physical items for mailing.

You can configure system documents for either print or email subsystems. Each subsystem can have multiple system documents. All system documents used with a print subsystem must use the print fulfillment type. All system documents used with email subsystems must use one of the email fulfillment types. The document is sent to the administrator’s email address as configured in OPS Administration.

To associate the system document with a subsystem, use OPS Administration. For more information on how to create system documents, see Configuring OPS and the OPS Administration Help.
