OPS Service Error Handling

OPS service and subsystem errors are logged to the Windows Event Log. OPS service errors are logged to the Application event log, while OPS subsystem errors are logged to the connection event log that is created via OPS Configuration.

During OPS installation, you can configure the OPS service to send error notifications in an email message to a specific mailbox. Likewise, using OPS Administration, you can configure OPS subsystems to send their errors to specific mailboxes.

Connection-specific event logs are always named with the connection's logical application name and begin with the “OPS_” prefix. For example, running OPS Configuration twice to set up connections to the Main and Secondary logical applications creates the following two event logs:

  • OPS_Main
  • OPS_Secondary

Subsystem errors are logged to their respective connection’s event logs. For more information on sync subsystem event logging, see Logging.

You can also configure OPS to send error notification emails for the OPS service and subsystems to specified mailboxes. Note that an error condition that persists over an extended period of time can generate a large volume of email messages.
