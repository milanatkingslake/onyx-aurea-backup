Default OEAS Security Roles Available for OPS

The following OPS security roles are available in the default implementation of OEAS.

Security role Description Restrictions

OPS.Administrator

This role grants full control over all OPS functionality. OPS Administrators can administer OPS services and add and configure subsystems. This level is intended only for the OPS Administrator.

None

OPS.FulfillmentOperator

This role grants essentially the same access as OPS Administrator except for access to OPS subsystem configuration. Typically, the literature fulfillment or mailroom operator uses this security role.

Cannot configure subsystems

OPS.User

This role grants users the ability to view but not change OPS functionality.

Cannot administer OPS service, configure subsystems, or change order line information

If you have OPS servers in multiple locations, Onyx recommends that you grant OPS Administrator permissions to at least one person in each office where a server resides.
