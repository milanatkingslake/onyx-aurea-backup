Security for Multiple Connections in OPS

If you have multiple connections for OPS, use Security Administration to configure security as follows:

  • Set access permissions in Security Administration for each connection’s OEAS logical application.
  • For each user that accesses OPS Administration, create an Onyx user account in the OEDB that the master application and master site reference.
  • For each connection that a user accesses, create an Onyx user account for the user, in the OEDB that the connection references.
  • Add each user to the appropriate OPS roles for the master application and each connection that he or she accesses.
