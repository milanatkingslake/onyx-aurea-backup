What are OEAS Business Objects?

OEAS includes many features, each represented by one or more business objects defined in the OED. The business objects representing each feature enable you to perform activities that pertain to customer relationship management, such as saving customer data to your database, monitoring your sales cycle and your relationship with your customers, and tracking the incidents and requests generated by interactions with your customers.

The core components use the OED and the OEDB to provide the implementation of the business objects.

The following table explains the relationship between the business objects and the core components.

Core Component Relation to Business Objects

Onyx Transaction Manager (OTM)

OTM receives requests from calling clients to perform methods on business objects. The OTM references the OED to obtain the definitions for these business objects, including their methods and steps. Only after the OTM obtains the definition of a business object can it begin to process the request.

When customizing OEAS, you can create your own step components and business objects to modify existing behavior.  

Onyx Data Manager (ODM)

ODM is the step component required for any method that uses SQLGen to communicate with OEDB. It generates SQL statements based on information passed by the OTM. The ODM sends commands to the Query Engine and receives results in return. It composes results from the Query Engine into XML in the format required by OEAS and passes that XML back to the OTM. As part of its processing the ODM also performs data management functions, for example, generation of unique ID values when required.

Onyx Enterprise Dictionary (OED) and OED cache

OED defines each logical business object and the metadata, a description of data, associated with that object. OED includes physical model information about OEDB, including tables, columns, procedures, and functions in it, as well as mapping information from the logical to the physical model.

When creating a new business object for OEAS, you first need to use the Object Designer to define that object in the OED. You then use Onyx step components or implement step components of your own, that encapsulate the functionality for that business object. 

National Language and National Language cache

Step and core components obtain national language strings, or strings that can be localized from one language to another, from the National Language cache.

Security

Security is responsible for session management in OEAS, and for allowing users access to the system. For more information about securing business objects, see Security and user authentication.

Query Engine (QE)

Components that access the OEDB directly do so using the Query Engine. This type of component is rare, and it is recommended that you use a logical method implemented by the Onyx Data Manager instead of calling the OEDB directly.
