Configuring Domain Data

You can configure and administer OEAS domain data using the Onyx Enterprise Studio (OES) administration tools or another client tool such as LBO Tester. The following table is a partial list of OEAS domain data which lists the type of data and the tool you can use to administer the data.

Typical activities you perform using the administration tools include adding, deleting, and modifying items. For an overview of each administrative tool, see What is the Onyx Enterprise Studio?.

Domain Data Description of the domain data Tool

Company family

Company family data is domain data for the Customer feature. You create company families to logically group companies.

Table Administration

Contact category

Contact category data is domain data for the Contacts feature. Contact categories, together with contact group types, enable users to assign contact types to company, incident, individual, customer product, and task records.

No tool

Contact group type

Contact group type data is domain data for the Contacts feature. Contact group types, together with contact categories, enable users to assign contact types to company, incident, individual, customer product, and task records.

No tool

Contact type

Contact type data is domain data for the Contacts feature. Contact types are displayed to OEP users based on the contact category and contact group type for which the user is adding a contact. Contact types are displayed in a drop-down list box enabling the user to select the appropriate contact type.

Contact Type Administration

Country

Country data is domain data for the Customer and Literature Orders features. The country domain data is displayed to OEP users in the edit and add forms for companies and individuals in a drop-down list box from which the users can select an appropriate country to associate with the customer’s address. Within the Literature Orders feature, country domain data is leveraged from the customer’s address for the countryCode property of the literatureOrder object.

Table Administration

Currency conversion rate

Currency conversion rates are used by the Forecast & Quotes feature to convert monetary values from one currency to another. The domain data for currency conversion rates is not administered using an administration tool, but rather is administered at the database level.

Note: You typically load these rates into the database with an automated process from an exchange rate provider.

No tool

Currency

Currency types are used when creating or updating product domain data, as well as when creating forecast and quote records.

Table Administration

Incident category

Incident category data is domain data for the Incidents feature.

Table Administration

Incident product

Users associate incident products to incident records to facilitate tracking. The incident products that you create are displayed to users in OEP on the Incidents form.

Product Administration

Tags

Tags data is local data for the Incidents and Tasks features. Tags domain data is created when users add tags to incidents and tasks and save these records.

N/A

Literature product

Literature products are used to complete literature order records. In OEP, literature products are displayed to the user in a tree-like structure, so that the user can search for and select the appropriate literature item (product) to associate with the literature order record.

Product Administration

Literature service printer

Literature service printer data is local domain data for the Literature Orders feature. This data specifies the printer to which printable literature items are sent when they are ordered for a customer. This data is configured by the system administrator and is not editable by OEP users.

Table Administration

Literature ship via

Literature ship via data is domain data for the Literature Orders feature. Ship via data displays to OEP users in the Ship Method drop-down box on the Shipping tab of the Literature Order form enabling users to select the shipping method for a literature item.

Table Administration

Literature terms

Literature terms data is local domain data for the Literature Orders feature. Literature terms data displays to OEP users in the Terms drop-down box on the Billing tab of the Literature Order form enabling users to select the payment terms (cash, net 30, net 60) for the literature item they are sending to a customer.

Table Administration

National language

National language data includes captions for the user interface as well as error messages.

Table Administration

Product

Products are used to complete customer product records, as well as forecast and quote records. In OEP, products are displayed to the user in a tree-like structure, so that the user can search for and select the appropriate product to associate with the customer product, forecast, or quote record.

Product Administration

Reference data

Reference data is used throughout the application. With Reference Table Administration, you create and administer reference data to configure the field type and lookup values for fields available in the Onyx Employee Portal. Additionally, you can configure new columns that you add to the OEDB tables to support new properties in the OED.

Reference Table Administration

Region

Region data is domain data to the Customer feature, as well as the Literature Orders feature. The region domain data is displayed to OEP users in the edit and add forms for companies and individuals in a drop-down list box from which the users can select an appropriate region to associate with the customer’s address. Within the Literature Orders feature, region domain data is leveraged from the customer’s address for the regionCode property of the literatureOrder object.

Table Administration

Security

Security domain data includes data related to:

  • users
  • groups
  • resources

Security domain data is used to ensure access is restricted to the Onyx Enterprise Application Server, the Onyx Employee Portal, and the Onyx Enterprise Studio.

Security Administration

Social Network type

Social Network type data is local domain data for the Social Network feature. Social network types are a means of logically organizing the social network for a particular user.

Table Administration

System parameters

System parameter data controls the specific behavior of several OEAS features.

System Parameter Administration

Task category

Task category data is domain data for the Tasks feature.

Table Administration

Tracking code

Tracking code data is local domain data for the Campaigns feature. Tracking codes are the direct link between a campaign and the incidents associated with that campaign. They identify the specific event or activity to be measured or tracked for that campaign.

Campaign Administration

Tracking code type

Tracking code type data is local domain data for the Campaigns feature. Tracking code types are a means of grouping related tracking codes. They provide a means of logically organizing the tracking codes used in a particular campaign.

Campaign Administration

You generally administer domain data with a feature-specific tool of the Onyx Enterprise Studio (OES). For example, tracking codes and tracking code types form the domain data for the Campaigns feature, and are administered with Campaign Administration. Users, groups, and resources form the domain data for the Security feature, and are administered with Security Administration.

The following tools however enable you to administer domain data for a range of features:

Product Administration. Products, literature products, and incident products are domain data for three separate feature areas (Customer Product, Literature Orders, and Incidents). Instead of administering the domain data for each of these within a feature specific administration tool, one tool, Product Administration, is provided to administer the three different kinds of domain data.

Table Administration.Table Administration provides access to several areas of local domain data. Because of the simplicity of the administration tasks associated with these areas of domain data, no specific administration tool is provided. Rather, the functionality for administering the data is provided with this single administration tool.

Reference Table Administration. Reference Table Administration is a tool that enables you to configure reference field data in the Onyx Enterprise Database (OEDB). Depending on the field type selected, you can configure field attributes such as caption, field type, and lookup values for the field.

System Parameter Administration. System Parameter Administration is a tool that enables the editing of system parameters. You use system parameters to control the behavior of several features of the Onyx Enterprise Application Server (OEAS), including how features are presented to users of the Onyx Employee Portal (OEP) Web client.
