Step 1: Develop a Step Component
Develop a step component

For this sample, you create a custom step component that:

  • Receives a company record from a previous step transaction.
  • Locates the stockTickerSymbol and companyName properties in this record.
  • If the stockTickerSymbol property is empty, the step component inserts a symbol into the property. The returned stock ticker data is hard-coded in this sample. However, the sample demonstrates that an external database or data lookup can easily be performed, and the resulting data integrated into a business object method.

All step components invoke the Execute method of the IStep interface. The Execute method performs two main tasks. It:

  • Extracts data from the client-supplied XML using the MicrosoftÂ® XML DOM.
  • Returns data to the client application by inserting additional information into the XML DOM.

To create a custom step component:

Note: Before you begin to develop your custom step component, review building step components.
  1. Set up your development environment.
  2. Implement the IStep interface.
  3. Define the variables.
  4. Handle the step package.
  5. Verify that all required XML nodes are present. Steps six and seven, below, are performed only if the required nodes are returned by the Microsoft XML DOM.
  6. Obtain data from the step package.
  7. Return data to the step package.
