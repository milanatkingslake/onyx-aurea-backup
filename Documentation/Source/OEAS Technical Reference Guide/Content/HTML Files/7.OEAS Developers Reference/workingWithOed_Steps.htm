Steps

You can specify several types of steps on a method. For more information, see Methods and Steps. When the Onyx Transaction Manager (OTM) encounters a "com" step, it calls the COM+ object specified by the progId appearing in the step data. You use "method" steps to call a method on another object. When the OTM encounters a "method" step, it spawns an internal version of the OTM which then executes the specified object method.

When you create a method that uses SQL generation and/or needs to call a stored procedure, you must define at least two standard steps for the method. The first step should call the data validation component. The other step should call the Onyx Data Manager (ODM). The following table supplies information on how to set up these standard steps. You may want to specify additional steps to these two, but these are the minimum required steps. The additional steps can come between data validation and the ODM step, but Onyx recommends that you always call the data validation step first.
