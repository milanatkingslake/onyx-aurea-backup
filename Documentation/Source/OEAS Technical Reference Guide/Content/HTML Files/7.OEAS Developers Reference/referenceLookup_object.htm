referenceLookup object

A referenceLookup object creates a many-to-many relationship between referenceField objects and referenceParameter objects. It can also provide additional data to support external applications (characterCode property) and to facilitate queries for specific reference data (extraData property).

The referenceLookup object is used in conjunction with the referenceField and referenceParameter objects when you want to create referenceField objects whose type property is "combo" or "hiercombo."

The referenceLookup object has three key properties, parameterId, referenceId, and parentId. Once the object is created, the value of these properties cannot be changed.

Methods

Method

Description

retrieveCollection

Returns a collection of objects based on the search criteria supplied in the parameters XML.

retrieveList

Returns a list of referenceLookup objects.

retrieveListEnumeration

Retrieves denormalized reference data.

saveCollection

Performs multiple actions at once: inserting, updating, and deleting objects, as appropriate.

Properties

Property

Description

active

Specifies whether the domain data the object provides is available to the user interface.

characterCode

A code that identifies the field name that can be used to import data from an external application.

description

A text description for a field.

extraData

Either a text description for a lookup data value, or a mechanism used to group similar data values.

parameterId

The unique identifier of a referenceParameter object.

parentId

The identifier of the object that owns the specified referenceLookup object.

referenceId

The identifier of one or more fields in the REFERENCE_FIELDS table.

seqNum

The ordinal position of a referenceParameter lookup value in a combo box.
