logMark (workNoteDetail)

logMark property

The logMark property of the workNoteDetail object indicates whether this particular workNoteDetail starts a new workNoteDetail sequence or is part of a multi-part workNoteDetail.

Data type

boolean

Access

Read/write

Remarks

You use this property to tell the system whether to increment the logMarkSequenceNumber. For the first workNoteDetail on the master object, set the logMark equal to one. For subsequent workNoteDetail objects, set the logMark equal to zero.

You use this property to save a multi-part workNoteDetail object. When a user sends in a publish directive with the workNoteDetail text, you do not need to display an entry user and entry time for each section of the text. You use the logMark property to tell the system where a new detail record begins.
