content Attribute
content attribute

The content attribute defines for OEAS the content of data that it is passing in (within the parameters element), and the content of data that it expects back (within the returnXML element).

The content attribute is not required by OEAS on any input parameters, but OEAS always includes the attribute on parameters returned. A client application can use the content attribute to decide how to handle the returnXML. Because a lot of data passes over the network, it's important to define what the requirements are ahead of time.

The content attribute identifies which properties are present for a given object instance. This attribute has four possible values:

Value Definition

all

Used when all properties of the object are included in the input parameters. This does not mean that every property contains a value, just that all the properties are present.

partial

Used when a subset of properties is necessary to complete a specific task.

any

Used for actions where the data values can vary from one call to the next. Depending on the object, some parameters are usually required on input, while others are optional.

This is used for some search (retrieve) methods, where the search criteria is dictated by the properties that are provided for the call. Thus, the caller can define the boundaries of the search by including or removing properties, as needed.

keysOnly

Used for actions that require the minimum amount of information necessary to identify the object in the database.

You can use the content attribute at design time to identify which object properties are required in the parameters XML to complete a task, and to determine which object properties are included in the returnXML.

Examples

The following examples typify the four content types. The four types are:

Content attribute whose value = all

   <reminder objectType="reminder" content="all">

      <primaryId>1</primaryId>

      <ownerId>21</ownerId>

      <ownerType>6</ownerType>

      <userId>user1</userId>

      <message>Testing Insert</message>

      <dueBy>11/02/2000</dueBy>

      <recurType>0</recurType>

      <recurInterval>0</recurInterval>

      <alarmActive>1</alarmActive>

      <insertBy/>

      <insertDate/>

   </reminder>

Content attribute whose value = keysOnly

   <reminder objectType="reminder" content="keysOnly">

      <primaryId>1</primaryId>

   </reminder>

Content attribute whose value = partial

   <reminder objectType="reminder" content="partial">

     <primaryId>1</primaryId>

     <ownerId>21</ownerId>

     <ownerType>6</ownerType>

   </reminder>

Content attribute whose value = any

<parameters>

   <phones objectType="phone" collection="1">

      <phone content="any">

         <ownerId>3</ownerId>

      </phone>

   </phones>

</parameters>
