Identifying objects and records (key properties)

Keys—identifying objects and records

OEAS uses key properties to identify objects in the database. You can identify key properties using the 'primary' Object Key Type in Object Designer. Use key properties when you want to retrieve, update, and delete records in the database. The following table lists logical business objects and their key properties.

When you pass primaryId and secondaryId as parameters of the insert method, OEAS ignores the values that you supply for primaryId, but supports secondaryId being passed by the calling application. The exception to this are key properties which are natural keys, for example code and userId, and any domain association objects (eg. surveyQuestion) support specifying their primaryId on insert.
