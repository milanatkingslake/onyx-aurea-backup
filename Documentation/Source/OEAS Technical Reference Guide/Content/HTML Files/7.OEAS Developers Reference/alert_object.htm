alert object

The alert object enables users to insert, update, retrieve, and delete alert object instances. A user interface can call methods on the alert object to cause alert messages to display when a user accesses specified company, individual, or incident object instances.

The alert object has one key property, primaryId, that uniquely identifies object instances in the database. Once the object is created, the value of this property cannot be changed.

Methods

Method Description
delete Deletes a specified alert object instance.
insert Saves a new alert object instance.
retrieve Returns an existing alert object instance.
update Updates an alert object instance.

Properties

Property Description
expirationDate The time and date that the alert expires
expired A boolean that indicates whether the alert has expired
insertBy The Onyx user who initially saved the object instance
insertDate The time and date on which the object instance was initially saved
onyxTimestamp Facilitates concurrency control with the Onyx Enterprise Database
ownerId A value that identifies the owner of an alert object instance
ownerType The owner's object type
primaryId A unique numerical value that OEAS assigns to the alert object instance
text The alert message body
updateBy The user ID of the person who last updated the object instance
updateDate The time and date of the last object instance update
