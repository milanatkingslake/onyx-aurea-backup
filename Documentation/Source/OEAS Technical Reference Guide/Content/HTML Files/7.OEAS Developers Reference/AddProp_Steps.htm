Steps (Add Property)
Adding a property steps

This sample demonstrates how to add the stock ticker symbol to the company business object.

Adding a property includes the following general steps.

  • Create a physical column in the database and the OED
  • Add the property to the logical model in the OED
  • In the OED, map the logical property to the database column
  • Validate and publish the customized OED

Creating the physical property

Important: Domain tables with multiple language support have specific requirements. For more information, see Creating and customizing tables.

Using a database administration tool, add a column to the database table representing the company object. Onyx recommends using the ALTER TABLE statement. The following SQL statement adds the stock ticker symbol column to the company table.

ALTER TABLE company

        ADD cc_stock_ticker_symbol  nvarchar(255)  NULL; 

Following implementation of your customization, verify that all other packages and stored procedures remain valid.

Add the column to the Physical Model in the OED.

  1. Open Onyx Enterprise Studio Object Designer.
  2. Navigate to Physical Model\Tables\Company\Columns and select Columns.
  3. Click the Add button to add a new column to the physical model.
  4. In the Name field enter "cc_stock_ticker_symbol" without the quotes.
  5. Enter an optional description in the Description field if you like.
  6. In the Column Type field enter "nvarchar" the database data type you used.
  7. In the Length field enter "255" for the length you assigned to the column in the database.
  8. Click the Save button to save your changes.

Creating the logical property in the OED

  1. Open Onyx Enterprise Studio Object Designer.
  2. Navigate to Logical Business Objects\company\Properties and select Properties.
  3. Click the Add button to add a new property.
  4. In the Name field enter "stockTickerSymbol" without the quotes.
  5. Enter an optional description in the Description field if you like.
  6. In the Data Type field select "string" from the drop-down list of data types.
  7. Click the Save button to save your changes.

Mapping the logical property to the database column

  1. Open Onyx Enterprise Studio Object Designer.
  2. Navigate to Logical Business Objects\Company\Object Physical Maps and select Object Physical Maps.
  3. Click the Add button to add a new map.
  4. In the Property field select "stockTickerSymbol" from the drop-down list of properties.
  5. In the Table field select "COMPANY" from the drop-down list of tables.
  6. In the Column field select "cc_stock_ticker_symbol" for the column to which you want to map the property.
  7. Under Persistence Map, Retrieval Map, and Filter Map, click the Enable option.
  8. In the Type field for each map, select "direct." You do not need to enter anything in the Data field for direct maps.
  9. Click the Save button to save your changes.

Validating and publishing your customizations

  1. Optionally, you can click the Validate button to verify that the modifications you made are acceptable. When you publish, the validation is also performed, so this is an optional step.
  2. Click the Publish button. Answer Yes to the prompt that asks if you want to overwrite the current OED.
  3. Shut down Onyx App COM+ to cause the OED to reload the next time a client calls the OEAS.
