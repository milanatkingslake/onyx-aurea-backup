hidden (methodCallDetail)

hidden property

The hidden property of the methodCallDetail object indicates whether the input parameter is displayed to the method call consumer.

Data type

boolean

Access

Read/write

Remarks

The hidden property is ignored for output method call details. It allows the method call designer to pass values to the encapsulated OEAS method that the method call consumer should not know about or change. For instance, a method call designer can wrap the customer retrieveList method in a way that allows the method call consumer to search only for individuals by hiding the "type" parameter of the method. The method call consumer cannot change the type, but in order to make the encapsulated OEAS method properly search for individuals, a value of "1" must be passed to the method. The designer must provide a default value and set the hasDefault property for this scenario to work.

A hidden input without a default value results in an element being created. This enables the method call designer to build structure in the encapsulated parameters even for elements that contain no data such as onyxTimestamp.

Some methods such as individual insert require all properties of the object be included in the parameters, even though they can be empty. If you know that the method is only going to be used with first name and last name as inputs, it is a good idea to mark the rest of the input details as hidden. The method call consumer must supply a value for any details that are required and not hidden in the input call, or the method fails.
