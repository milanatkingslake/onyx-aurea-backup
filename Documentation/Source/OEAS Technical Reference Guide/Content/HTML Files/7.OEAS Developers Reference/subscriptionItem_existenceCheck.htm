existenceCheck (subscriptionItem)

existenceCheck method

The existenceCheck method of the subscriptionItem object determines whether each specified object instance has a record in the database. For individual and company object instances, this method places a temporary lock on the instances to prevent them from being merged.

Remarks

When a specified object instance is either a company or an individual and a record of the instance does not exist in the database table for the object type, existenceCheck determines whether the instance was merged with another instance. If it was merged, existenceCheck returns the new ID of the instance in the newPrimaryId column. If it was not merged, existenceCheck returns 0 in the objectExists column.

This method supports multiple merges on a single object instance. For example, if instance 1 is merged into instance 2, and instance 2 is subsequently merged into instance 3, an existenceCheck on 1 returns 3 as the new object ID.

When a specified object instance is either a company or an individual, existenceCheck places a temporary merge lock on the instance that prevents the customer from being merged during synchronization processing. This lock is released when the subsequent existenceCheck is called from the sync server or after a time-out period specified by the GlobalMergeLockTimeOut system parameter.

A merge lock is specific to the user/subscription service and the object instance. An object instance can have multiple merge locks when two or more users subscribe to the instance. Or, a user can have multiple merge locks on a single instance when he or she subscribes to it using more than one subscription service.

If any merge locks exist for the user and the value of the releaseExistingLocks parameter is 1, existenceCheck removes the locks.

If a transactional object instance specified in the objectIdTypeList is not an individual or a company, this method checks for the existence of the instance. This method does not check non-customer objects for merges.

Specify the objectIdTypeList parameter as follows:

<objectIdTypeList>D6889156-A2F0-4ABB-A98B-A21FA54A6A1D,2;

                  15071AF8-F4E3-479C-AB62-C41D4DC64341,2</objectIdTypeList>

where GUIDs are object instance primaryIds, and 2 is the object type enumerations of each specified instance.
