Result set interpretation

The format of the result set and the specific contents of the result set are determined by three things:

  • The type of method being called.
  • The return code returned from the database call made by the middle tier.
  • If applicable based on the two points above, the value of the return_code column in the result set.

There are two types of result sets. The first type returns data for successful inserts, updates, deletes, saveCollections, and custom methods. It also returns failures for these methods as well as retrieve, retrieveCollection, and retrieveList method calls. The following table shows the standard result set for the first type.

Column

Data type/nullability

Description

o#_sequence_number

INT

For applicable return code values and method calls, this value corresponds to the position of the record in the input XML. For all other method calls, this value is NULL.

message_code

INT

For applicable return code values and method calls, the message_code column represents a national language ID value corresponding to a message you want to provide the user.

database_code

INT

For applicable return code values and method calls, the database_code column represents a SQL Server database error code.

database_message

NVARCHAR(512)

For applicable return code values and method calls, the database_message column represents a SQL Server database error message corresponding to the value in database_code.

database_id

INT

For applicable return code values and method calls, the database_id column represents the system generated primary ID for domain data records inserted into the database through identity columns.

return_code

INT

The return code value for the row.

The second type of result set returns data for successful retrieve, retrieveCollection, and retrieveList method calls. These result sets are based on the data in the appropriate tables and must change depending on:

  • For retrieve or retrieveCollection methods, the object
  • For each retrieveList methods, the defined rowSet output parameter

For retrieve and retrieveCollection methods, the result set must consist of a column for every property on the object. The columns must be returned in the order in which the properties are listed in the OED. Because the middle tier interprets the columns in the result set based on the OED property order, the column names and aliases are not critical.

When adding or removing properties from an object, the database programmer must also make the corresponding changes to replace procedures for retrieve and retrieveCollection methods on the object, or the methods will fail.
