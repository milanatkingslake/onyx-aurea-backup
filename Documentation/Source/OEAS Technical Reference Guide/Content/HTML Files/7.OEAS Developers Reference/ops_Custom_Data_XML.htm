customData (OPS)

customData

If your customization requires any data that doesn't fit into the literatureOrder or literatureProduct object models, you need to use customData. To use customData in the OPS, you must understand the customData node of the stepPackage XML and the customDataXML parameter of the ITransactionManager Execute method. The customData node of the orderBatch XML contains the XML passed through the customData node of the stepPackage XML. OEAS and OPS reserve this node for your customizations and do not touch its contents. Using customData requires one or both of the following:

  • the user interface inserts data into the customDataXML node before calling the OnyxTransactionManager (OTM)
  • a pre- or post-procedure or a custom step component on one or more of the following methods populates the customDataXML node with data
  • literatureOrder.retrieveCollectionBatch
  • literatureOrderLine.retrieveCollection
  • literatureProduct.retrieveCollectionBatch
  • literatureServicePrinter.retrieveCollection
  • literatureOrderBatch.retrieveListIncomplete
  • literatureOrderbatch.insert

When the subsystem builds the orderBatch XML, it puts the contents of customDataXML from the stepPackage into a customData node of the orderBatch structure. Each customData node contains a source attribute that identifies the method that created the node. For example, the following code snippet shows a customData node created by the literatureOrder retrieveCollection batch method.

<customData source="literatureOrder.retrieveCollectionBatch">

   <resumedBatch>0</resumedBatch>

</customData>

The customData node can contain any valid XML that your customization requires.

Note: For a complete orderBatch example, you can look in the Utilities directory of the product CD for orderBatchEmailSample.xml and orderBatchPrintSample.xml.
