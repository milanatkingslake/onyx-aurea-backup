externalContact object

The externalContact object enables users to manage contact information relating to individuals and companies outside of the user's company or organization.

Note: External contacts are sometimes referred to as customer contacts.

You can associate any company object or individual object (customer objects) with any other company, individual, incident, workTicket, task, or customerProduct object using the externalContact object. The externalContact object identifies a customer (company or individual) that serves as the primary point of contact for internal users (your employees) when interacting with another customer. For example, Mr. Honor is the primary purchasing manager at ABC Widgets company. You can designate the individual object in OEAS representing Mr. Honor as the externalContact object for the ABC Widgets company object in OEAS. That way, internal users of your OEAS system will know to direct their communications to ABC Widgets through Mr. Honor. Perhaps you use another company to channel your products to ABC Widgets. You can designate your partner's company object as the externalContact object for the ABC Widgets company object in OEAS. The externalContact object associates two customer records by containing the primaryId properties of each record in the ownerId and contactId properties of the externalContact object.

The externalContact object has one key property, primaryId, that uniquely identifies the object in the database. Once the object is created, the value of the primaryId property cannot be changed.

Methods

Method Description

retrieveListByOwner Returns a list of externalContact records by ownerId

Properties

Property Description
comment Custom text that describes an externalContact
contactId The company or individual that is designated as the externalContact
contactObjectType The contactObjectType of the contactId
contactTypeId Unique numerical value associated with a specific contactType
contactUserName Name of the external contact
ownerId Uniquely identifies the parent object of the child object
ownerType Contains a numerical value that identifies the type of parent object that owns the child object
primaryId Contains a unique ID assigned to the object when it is first saved to the database
