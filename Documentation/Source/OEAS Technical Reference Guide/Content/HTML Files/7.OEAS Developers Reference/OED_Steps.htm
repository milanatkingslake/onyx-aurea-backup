Steps

The steps collection is a container that holds all of the steps that define individual activities for a method.

Steps, which are owned by a method, describe an individual action or activity within the scope of work performed by the method. For example, a method's first step is often to perform validation tests on parameter data before database actions are performed by subsequent steps.

The following three values are available for the step type attribute:

Step Type

Description

com

A com step causes a specified COM+ object to be called on the business tier.

method

A method step causes a call to a specified method on another object.

transaction

A transaction step signifies either the beginning or end of a COM+ distributed transaction. (Note: Transaction steps are not used in this release of OEAS.)

Steps have specific attributes which include a name that must be unique within the method, a description, and a "step data" attribute that identifies what process should be launched. For a com step, the "step data" value is the ProgId identifying the COM+ object to be executed. COM+ objects that are to be called in a step (otherwise known as step components) must support the IStep interface. For a method step, the "step data" value identifies a method to be called on a business object.

Another important step attribute is "additional data." This is a semi-colon (;) delimited, keyword-value pair string that is used to communicate additional information to the component or method being called (see Onyx Step Components for more information). The step also has an attribute called "override template" that is not used in this release of OEAS and must be left blank.

When calling ODM, the following step additional data are required.

  • stepName = <Name of the step> Ex: core;
  • stepObject = <Name of the step object> Ex: actionBatch;
  • stepMethod = <Name of the step method> Ex: retrieveListByStatusIds;
  • dataPath = <Name of the collection> Ex: addresses;

In addition to its attributes, a step may also own a collection of procedure maps.
