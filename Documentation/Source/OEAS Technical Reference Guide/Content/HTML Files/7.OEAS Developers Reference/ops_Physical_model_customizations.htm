Physical model customizations (OPS)

Physical model customizations

Physical model customizations describe the database schema in the OED. This provides the information for the middle tier to communicate with the database.

Tables

In this section of the Object Designer you must add a new table for the one created in the previous topic. Use the same literal name for the table as you used in the database. For each column in the database, add a column element to the columns collection in the OED. Describe the column attributes in the OED exactly as they are specified in the physical schema. This customization does not require any information in the Database Actions section.

Procedures

In this section of the Object Designer, you must add a new procedure for the one created in the previous topic. Use the same literal name for the procedure as you used in the database. You need to add a parameter to the parameters collection in the OED representing the single input parameter to the procedure. Give it the following attributes.
