Error handling

The ReportNetStep component raises a COM error back to the OTM under the following conditions:

  • An invalid stepPackage DOM is passed in the IStep Execute method.
  • The engine is unable to report an error in the stepPackage XML because it cannot create the methodStatus/statusType node.

The component traps and then reports all other errors under the methodStatus node of the return XML. It also reports them in the Windows event log.
