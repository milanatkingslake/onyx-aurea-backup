Empty Property Values
Empty property values

OEAS components that return empty property values can contain empty property elements. However, these elements should not be considered NULL since XML does not support the concept of NULL.

Note: A missing property does not mean the same as NULL. A missing property indicates that the value of the property is unknown to the sender.

When empty property values are returned, there are two possible situations as outlined in the following table.

Situation

Representation

Missing Property

The property element is omitted from the XML representation of the object.

Empty Property with NULL attribute set to '1'

The property element is present in the XML representation and is encoded with a null="1" attribute.
