Authentication overview

OEAS security has two authentication components, AuthenticateNT and AuthenticateOnyx. These authentication components implement different types of user validation schemes. It is the client's responsibility to call an authentication component to verify user access to the Onyx Enterprise Application Server (OEAS). These must be done before a session can be established.

The authentication portion of the OEAS security is separate from other security functions, primarily to accommodate the many forms of authentication available in the marketplace.

The AuthenticateNT component exposes the IAuthenticateNT interface and is designed for users accessing the system through the local network, mapping Microsoft Windows users to OEAS users. The AuthenticateOnyx component exposes the IAuthenticateOnyx interface and is used when it is not practical to have an associated Windows account for each user.

To implement the OEAS security authentication components, you must be familiar with Microsoft Internet Information Server (IIS) and Microsoft Windows security protocols and procedures.
