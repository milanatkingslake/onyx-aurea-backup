parameters element

The parameters element contains the input data that a calling application sends to the OTM.

OEAS step components use the contents of the parameters element to perform a specified action, or method. Custom step components can modify the contents of this element to filter or validate the input parameters. Be careful, however, when making changes to this data for customization purposes, since the changes you make may affect the behavior of an Onyx step component.

After a step component is finished processing the input data, it assembles its output data into the parametersReturn element, located inside returnXML.

The child nodes of the parameters element might include the following attributes:

  • action
  • content
  • null
  • Required properties

Example

The contents of the parameters element vary from one method to the next. The following example shows a sample parameters element for the retrieve method of the company object:

<parameters>

   <company objectType="company">

      <primaryId>8A768E06-7BD4-4A6D-8F3C-DCF5BE003877</primaryId>

   </company>

</parameters>
