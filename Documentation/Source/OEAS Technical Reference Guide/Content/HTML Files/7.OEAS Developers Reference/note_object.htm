note object

The note business object enables users to save and view notes relating to company and individual object instances.

The note object has one key property, primaryId, that uniquely identifies object instances in the database. Once the object is created, the value of this property cannot be changed.

Methods

Method

Description

delete

Logically deletes an object instance from the database

insert

Creates an object instance

retrieve

Returns an object instance based on the search criteria supplied in the parameters XML

retrieveListByOwner

Returns information about notes owned by a specified object instance

update

Modifies an existing object instance

Properties

Property

Description

description

User-defined text information about the object instance

insertBy

The user ID of the Onyx user who initially saves the object

insertDate

The time and date on which the object is initially saved

onyxTimestamp

Facilitates record locking with the Onyx Enterprise Database

ownerId

Identifies the owning object instance of the note

ownerType

A numerical value that identifies the type of object that owns the note

primaryId

The unique ID assigned to the object instance when it is first saved to the database

privateAccess

Indicates whether the object instance is visible to the public

readOnlyAccess

Indicates that a user has marked the instance as read-only

secondaryId

An alternative identifier assigned to the object instance when it is first saved to the database

text

The body of the note

updateBy

The user ID of the Onyx user who last saved modifications to the object instance

updateDate

The time and date on which modifications to the object instance were last saved
