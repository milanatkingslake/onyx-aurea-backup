stepAdditionalData (data manager)

stepAdditionalData

The Onyx Transaction Manager (OTM) obtains the stepAdditionalData information from the definition of the step in the Onyx Enterprise Dictionary. The OTM then passes this information to the ODM in the stepAdditionalData parameter of the IStep interface Execute method. The stepAdditionalData parameter provides the ODM with information for executing a step according to the requirements of the method that is using the step. This enables a step to be used differently among various methods.

Example

For single-object methods (eg. insert), the stepAdditionalData must contain the object and method where the step is located in stepObject and stepMethod, respectively. For an individual insert method, the stepAdditionalData contains the following:

stepName=core;

stepObject=individual;

stepMethod=insert;

dataPath=

For collection methods (eg. saveCollection), the stepAdditionalData must contain the object and method where the step is located in stepObject and stepMethod, respectively. The dataPath value must contain the collection parameter name. For a group saveCollection method, the stepAdditionalData contains the following:

stepName=core;

stepObject=group;

stepMethod=saveCollection;

dataPath=groups
