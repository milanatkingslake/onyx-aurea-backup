retrieveListByAssociation (workTicket)

retrieveListByAssociation method

The retrieveListByAssociation method of the workTicket object searches the database and returns work tickets based on their association to customer, incident, or other workTicket object instances. The workTicketAssociation object implements these associations.

Remarks

Note that the input for this method is not a "partial" object, but rather two distinct parameters.

When work tickets are associated with other work tickets, the retrieveListByAssociation method returns associations in both directions. For example, when the input XML specifies an objectType of 10, the method returns associations in which the specified work ticket is the associationId in the association, and it returns associations in which the specified work ticket is the workTicketId in the association.

To demonstrate associations in two directions, the following table contains examples of the workTicketAssociation object properties that the retrieveListByAssociation method uses to determine its result set. When retrieveListByAssociation is called with an associationType of 10 and an associationId of 2, work tickets with primaryIds 1, 4, and 3 are returned.

Note: A workTicketId is a global unique identifier (GUID) because it is a primaryId of a workTicket instance. To simplify the above table, workTicketId is depicted as an integer.
