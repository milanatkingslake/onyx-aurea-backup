required (methodCallDetail)

required property

The required property of the methodCallDetail object indicates whether a user must provide a value for an input detail, or whether an error occurs if an output detail does not map to an output node.

Data type

boolean

Access

Read/write

Remarks

Inputs

Conditions

Behavior

required=0

methodCallDetail input value is optional

required=1 and hasDefault=0

methodCall consumer must provide a value for the methodCallDetail to successfully execute the method call

required=1 and hasDefault=1

methodCall consumer can provide a value that overrides the default

Conditions

Behavior

Outputs

A required output can have an empty output node. The required output methodCallDetail only specifies that the node be present.

The methodCall designer must decide which outputs are required based on their understanding of the encapsulated method. If they expect the method to return a node in all cases, they should mark the output methodCallDetail as required.
