Extension table

The extension table (ml table) is named the same as the base table with the suffix of _ml. The base table name may be abbreviated if it is desired. This table contains all of the primary key columns in the base table as well as an additional language_code column as part of the compound primary key. The language_code column has a foreign key relationship to the language table.

Column

Data Type

language_code

NCHAR(4)

The onyx_timestamp column is present in all extension tables, as are all columns that need to be translated. These columns include, but are not limited to, description columns.

Example: country_ml
