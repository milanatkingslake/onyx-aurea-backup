Run-time XML overview

For efficient transfer of heterogeneous data, the scriptSession logical business object encapsulates most of the work performed by the scripting run time components in three logical methods: advanceStep, resumeSession, and loadStep. The script engine logical interface consists of these three methods also known as the script engine methods. They do not follow an OEAS standard method signature, because each of these methods handles different types of data in a single call.

Onyx defines the scripting run-time XML as the XML that is passed to or from the three script engine methods. It also includes the domain data XML that the script cache passes to the script engine. In addition, the OEP UI uses the run-time XML structure, on the client side, to accumulate data about a script session in progress.

Three sections compose the basic structure of the run-time XML:

  • Domain data (everything under the script element)
  • Transactional data (everything under the scriptSession element)
  • Script variables (everything under the variables element)

The following table lists the scripting run time components and their interaction with run-time XML.

Component

Interaction with run-time XML

Script Cache

Produces the domain data section of the run-time XML, based on information it gets from the scripting domain tables in the OEDB.

Script Engine

Produces and consumes all sections of the run-time XML (domain, transactional, variables) through interactions with OEP and OEAS. Uses the script cache to retrieve the domain data section of the run-time XML.

OEP Scripting UI

Consumes all sections of the run-time XML. Produces data for the transactional section of the run-time XML, and small parts of data for the domain and variables sections. The domain data OEP "produces" includes requests for a particular script step ID, script ID, and so on. The data OEP produces in the variables section are the context variables such as _ownerId, _ownerType, etc.

The run-time XML fully supports localization and globalization as per OEAS standards.
