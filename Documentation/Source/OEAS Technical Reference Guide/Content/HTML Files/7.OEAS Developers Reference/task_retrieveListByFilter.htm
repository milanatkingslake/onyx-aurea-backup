retrieveListByFilter (task)

retrieveListByFilter method

The retrieveListByFilter method of the task object searches the database and returns a list of task records that meet the criteria specified by filter values.

Remarks

The retrieveListByFilter search is not case sensitive for the keyWords input parameter.

When the group input parameter is specified and the assignedTo input parameter is not specified, this method returns a list of tasks that includes all tasks assigned to each member of the group or groups. Also, you must specify each group and subgroup that you want to search. Subgroups are not automatically returned with their parent group.

This method retrieves only a limited list of data, and not always the entire list of data available. You can use the OEAS Configuration Utility to configure the maximum rows returned. For more information, see Paging functionality.

When searching the database by group (by passing the group parameter in the input XML), you must specify each group and subgroup that you want to search. Subgroups are not automatically returned with their parent group.

The ownerType parameter is not a valid filter. The step component does not validate the value of this parameter.

The following parameters must be passed in pairs in the input XML:

  • dueByStart / dueByEnd
  • insertDateStart / insertDateEnd
  • updateDateStart / updateDateEnd

If you pass only one parameter (such as "dueByStart") without its corresponding parameter ("dueByEnd"), your results may be unpredictable.

This method supports wild card substitution in search criteria. For more information on wild card substitution, see the retrieveList method (standard behavior) topic.
