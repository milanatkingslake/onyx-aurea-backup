Message and Errors

Messages and errors

Step components can return two types of information to calling client applications, messages and errors. The following information explains how message and error data is added to the methodStatus element of the step package.

Common items

The following two items can appear in both messages and errors. Both are optional.

associationId attribute

This attribute contains a GUID that identifies a piece of data in the parametersReturn returnXml. The same GUID is inserted as an attribute of the associated data. An associationId is included in situations when an error is attributable to a specific object or data element. See associationId Guidelines for more information about adding the attribute to output data.

timestamp element

This element stores the time at which the error or message occurred. Use this value to apply a sequence to the errors and messages in the methodStatus element.

message

A message element contains generic information intended for the caller. By itself, the existence of a message does not imply an error condition, but messages are often paired with errors in the methodStatus element. Each message has at least two and as many as four elements. Use messages for returning statements generated by the SQL Print command from within a stored procedure.

messageId

A numeric identifier for the message. This is typically the national language key value for the text of the message. For messages not taken from the national language table, this element is blank.

messageText

This element contains the text of the message.

A sample message appears below:

<message>

  <messageID>74582</messageID>

  <messageText>The data transfer to the ERP system was completed successfully.</messageText>

  <timestamp>2000-12-01 12:34:56</timestamp>

</message>

error

An error element contains the explanation for a statusType value of failure. The information in an error element is intended to help the client troubleshoot the problem that caused the method to fail.

errorNumber

This element contains a numeric identifier for the error. For errors generated by Onyx components, this number often is tied to a text string in the National Language table.

errorType

The errorType identifies the error as one of four distinct types:

errorType Value

Description

oetHresult

A COM call within the component failed.

oetReturnValue

An internal component failed.

oetBusinessLogic

A component discovered a problem with the data submitted by the client.

oetStoredProcedure

The stored procedure used by the step component returned a failure message.

errorDescription

The text associated with the error message. For errors generated by Onyx components, this text comes from the National Language table.

natLangId

This element contains the National Language ID that is associated with the error number. Onyx components include this element, but custom components may not.

severity

Errors can have one of two severity values: failure or warning. A value of warning indicates a condition that requires further investigation or is not severe enough to warrant failure of the entire method.

sourceType and sourceName

Together, these elements describe the source that detected the error. There are four possible values for sourceType. sourceName identifies the type by name.

sourceType Value

sourceName Description

ostStoredProcedure

A stored procedure name

ostComponent

A string containing object name followed by method name separated by a dot

ostWinAPI

The name of a method from the Windows API

ostMisc

Any value that identifies the source

A sample error element appears below:

<error associationId="7BC0D0BD-BE6A-47C0-9B44-E265D249C8FF">

   <severity>failure</severity>

   <errorNumber>50027</errorNumber>

   <errorDescription>You failed to supply a primary key property: primaryId.  A primary key is required to update, delete or retrieve an object.</errorDescription>

   <errorType>oetBusinessLogic</errorType>

   <natLangId>750027</natLangId>

   <sourceType>ostComponent</sourceType>

   <sourceName>company.retrieve</sourceName>

   <timestamp>2006-03-14 13:37:12</timestamp>

</error>
