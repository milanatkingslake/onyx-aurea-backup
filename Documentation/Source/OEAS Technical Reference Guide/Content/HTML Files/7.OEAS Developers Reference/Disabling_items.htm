Disabling items

When you disable an item in the OED, you remove it from active use, meaning that the item can no longer be accessed at runtime. When you publish a version of the OED that contains disabled items, your users can no longer use the functionality of the disabled item. When you disable an item, you must resolve cross-dependencies so that OEAS continues to function properly. You should also ensure that you did not inadvertently break other functionality, as well.

For example, if you disable an object, you must resolve the dependencies that properties, parameters, and/or steps have on that object. You can locate cross-dependencies after you disable an item by using the validation process. The validation process finds items that are no longer valid as a result of disabling an item, and reports those findings in an error dialog box.

OEAS reacts differently to the disabling of certain items in the OED. In some cases, disabling items affects OEAS drastically, and in other cases has no effect:

Disabling This Item

Has This Effect

Business Object

Makes the business object unavailable to OEAS and all OEAS users. Disabling an object also disables all of its properties, methods, parameters, and steps.

Property

Nothing. OEAS step components continue to use properties that are disabled.

Method

Makes the method unavailable to OEAS and all OEAS users. Disabling a method also disables all of its parameters and steps.

Parameter

Nothing. OEAS step components continue to use parameters that are disabled.

Step

Makes the step unavailable to OEAS and all OEAS users.

Procedure Map

Nothing. OEAS OnyxSGPrep and ODM step component continues to use procedure maps that are disabled. Flag is used only by the Dictionary Installer utility for reporting purposes.
