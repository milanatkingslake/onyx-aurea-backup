returnXml element

returnXml element

Calling clients receive the returnXml element after every call to OEAS. The entire contents of the returnXML element are passed to the client when the method completes.

The returnXml element has the following child elements:

  • otmId
  • context
  • methodStatus
  • parametersReturn
  • customData

The OTM creates each of these elements within the returnXml element before executing the first step in a method, but provides data only for the otmId element and leaves all other elements empty (unless the OTM needs to report an error, in which case it also provides data for the methodStatus element). Step components provide data for all other elements, including the methodStatus element if necessary, as they perform their work.

Example

<returnXml>

   <otmId>047D4B66-FEAC-455E-BFDD-F644DB81FA8D</otmId>

   <context>

      <sessionSecondsRemaining>600</sessionSecondsRemaining>

   </context>

   <methodStatus>

      <statusType>success</statusType>

   </methodStatus>

   <parametersReturn>

      <individual objectType="individual" content="keysOnly">

         <primaryId>9F163440-08D9-4BF5-96B1-F9ED09EB4548</primaryId>

         <phones objectType="phone" collection="1"/>

         <addresses objectType="address" collection="1"/>

      </individual>

   </parametersReturn>

   <customData/>

</returnXml>
