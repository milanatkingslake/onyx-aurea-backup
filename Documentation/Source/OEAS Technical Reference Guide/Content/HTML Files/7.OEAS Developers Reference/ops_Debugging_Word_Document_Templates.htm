Testing print templates

Onyx provides an OPS Print Template Test Utility for the purpose of testing print templates. Onyx also provides a sample orderBatch output along with the sample print templates. The OPS Print Template Test Utility runs the file that is available in the Utilities directory of the product CD.

Note: To change the template, refer to Writing Custom Templates.

Before using the utility, you must make the following modifications to the orderBatch.xml file.

  • modify a literatureProduct filePath property to reference the template you want to test
  • modify a literatureOrderLine to contain the literatureProduct with the template you want to test
  • modify the literatureOrderLine printerName property to reference a physical printer on your network
  • modify the literatureServicePrinter printerShare property to reference a physical printer on your network

To use the utility, click Browse and choose the file containing the orderBatchSamplePrint.xml. When you click Submit, the utility does the following:

  1. Calls the dataset creation method from the same dll’s used by OPS service for printing
  2. Creates dataset, depending upon the type of template mentioned in xml
  3. Dll’s processes the xml perform Mail Merge operation on templates
  4. Sends the document to printer queue specified in the xml
  5. Displays the status of the utility processing under Status label
  6. Logs according to the logging configuration specified in OPSFulfill.exe.config file

For changing the template, refer Creating mail merge field to print with OPS documents.
