retrieveList (customer)

retrieveList method

The retrieveList method of the customer object returns a set of customer (individual and/or company) data that satisfies specific search criteria.

Remarks

This method supports wild card substitution in search criteria. For more information on wild card substitution, see the retrieveList method (standard behavior) topic.

The user can search on any one or more of the parameters listed above. To restrict the search to individuals, set the Type to 1. To restrict the search to companies, set the Type to 2. To search for both individuals and companies, set the Type to 9. When Type is 9 and either companyTypeCode or companySubTypeCode is specified as an input parameter, the method returns only companies that match the search criteria, not individuals.

Information from objects that fulfill the search criteria is returned in the rowSet. The search is not case-sensitive. If no object satisfies the search criteria, an empty rowSet is returned. If no search criteria are provided, the method fails and returns a message requesting that more criteria be provided.

The stored procedure used by this method uses the system parameter MinSearchCriteriaValue. If the value associated with this system parameter is not null, then the stored procedure checks the following input parameters to determine whether the character length of at least one of them is greater than the system parameter's value.

The mask is maintained in the country table. For more information, see "Masking." Valid values for includeMask are:

  • none: phone output might appear as 8185551000
  • append: phone output might appear as 8185551000**MASK:(999) 999-9999 /x 99999
  • apply: phone output might appear as (818) 855-1000

Some restrictions apply to postal code masking. For more information, see Creating Masks.
