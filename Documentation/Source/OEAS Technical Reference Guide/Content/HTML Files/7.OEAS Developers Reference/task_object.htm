task object

task object

The task object enables users to delete, insert, retrieve, and update tasks, which are owned by  objects.

The task object has one key property, primaryId, that uniquely identifies the task product in the database. Once the object is created, the value of this property cannot be changed.

Methods

Method

Description

delete

Either logically or physically deletes an object from the database. Secondary objects, as defined in the relationship section of the Onyx Enterprise Dictionary, may also be logically deleted or updated depending on the logicalCascadeType elements of its relationships.

insert

Persists an object to the database.

retrieve

Returns a collection of objects based on the search criteria supplied in the parameters XML.

retrieveListByFilter

Returns a list of tasks based on filter criteria.

retrieveListByOwner

Returns a list of tasks associated with a specified incident.
retrieveListByCustomer Searches the database by ownerId and ownerType and returns a list of task records.
retrieveListMobileByOwner Searches the database by ownerId and ownerType and returns a list of task records.

saveCollection

Performs multiple actions at once: the insert, update, and delete of objects as appropriate
