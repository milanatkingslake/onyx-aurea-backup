Logical model

The logical model section of the OED contains the metadata that defines the structure and content of all the business objects that the Onyx Enterprise Application Server (OEAS) implements. The following table summarizes the logical model metadata.

Metadata Description

Business objects

Objects that model OEAS functionality

Properties

Items that describe business objects

Methods

Actions that can be performed on or by a business object

Parameters

Additional information that is required by a method to further define its action

Steps

A single unit of instruction performed by a method

Logical business objects model data and actions, that a client application can request and OEAS can provide. The client and OEAS temporarily store and pass object instances in XML documents. In general, OEDB persists instances of business objects in database records.
