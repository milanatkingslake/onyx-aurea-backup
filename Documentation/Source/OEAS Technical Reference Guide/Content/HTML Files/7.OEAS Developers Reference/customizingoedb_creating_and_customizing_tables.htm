Creating and customizing tables

Like objects, OEAS tables make a distinction between domain and transactional use. Each table you create that corresponds to an object should be modeled as domain or transactional based on agreement with the corresponding object type.

When a table corresponds to an object as its principal data repository, this document refers to it as a "base" table.

Important: For domain tables with multiple language support, you must add custom columns to the multiple language view as well as either the base table or the extension table. In addition, you must map the logical property in the OED to retrieve from the multiple language view. Onyx generated procedures retrieve data from the view. If you do not create the column in the view and use it for retrievals, you will get a run-time error.
