elementPath (methodCallDetail)

elementPath property

The elementPath property of the methodCallDetail object contains an XPath expression that locates the parameter element.

Data type

string

Access

Read/write

Remarks

You can use either relative or absolute expressions. Absolute expression must begin with the slash (/) character.

Inputs

Input method call details can only use a simple location path containing element names and slashes. The XPath specifies an element to create in the ParamXML or CustomDataXML of the ITransactionManager.Execute call to the encapsulated OEAS logical method. It can only refer to element nodes. On input, relative XPath expressions relate to the parameters element in the ParamXML. If the elementPath is absolute, it must start with one of the following strings.

  • /customData/ - The input element will be created in the CustomDataXML sent to the Transaction Manager. The text after /customData/ will be treated as a relative path specifying an element under the customData node in CustomDataXML.
  • /parameters/ - The input element will be created in the ParamXML sent to the Transaction Manager. The text after /parameters/ will be treated as a relative path specifying an element under the parameters node in ParamXML.

After the initial string, you must specify a valid relative XPath. You cannot use /parameters/ or /customData/ by themselves.

Every ancestor in the elementPath must have a corresponding methodCallDetail object. OEAS does not create any missing nodes at runtime.

The following XPath expressions show examples of valid input elementPath values.

Absolute

Relative

/parameters/company/secondaryId

company/secondaryId

/parameters/primaryId

primaryId

/parameters/literatureOrder/literatureOrderLines/literatureOrderLine/productNumber

literatureOrder/literatureOrderLines/literatureOrderLine/productNumber

/customData/myCustomParameter

Invalid XPath

Reason

individual/addresses/address[primary="1"]/countryCode

Input XPath cannot contain predicate expressions

/parametersReturn/customData/customElement

Not a valid input element

/parameters

Relative expression must follow the parameters element

incident/child::statusId

Input XPath cannot contain axis identifiers

Outputs

Output method call details support full XPath syntax including filtering, functions, and so on. The XPath specifies an element in the ReturnXML of the ITransactionManager.Execute call to the encapsulated OEAS logical method. Unlike the input elementPath, an output elementPath can refer to any node type accessible with XPath expressions.

If the type is scalar, the elementPath should identify a single node within the ReturnXML. If the XPath identifies multiple nodes, only the first is considered the output node.

If the type is list, the elementPath is expected to identify a set of nodes within the ReturnXML. Each node within the set is an output node.

Relative elementPath expressions refer to the parametersReturn node in the ReturnXML. Absolute elementPath expressions must start with the following string.

  • /returnXml/

After the initial string, you must use a valid relative XPath expression. It is not valid to use /returnXml/ by itself).
