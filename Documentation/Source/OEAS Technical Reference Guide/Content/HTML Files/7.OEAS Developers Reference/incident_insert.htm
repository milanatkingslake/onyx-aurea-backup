insert (incident)

insert method

The insert method of the incident object persists an object to the database.

insert standard method information

Steps

Step data Step type Description

OnyxDataValidationStep.DataValidation

com

Validates input parameters

OnyxGuidGeneratorStep.GuidGenerator

com

Calls a component that generates GUID primary key values used for inserting the tags identified in the incident's keyWords property

OnyxDM.DataManager

com

 Calls a pre-procedure that generates an integer value for the secondaryId when the element is null in the parameters XML

Adds the specified incident object to the database and calls two post-procedures that:

  • Insert or update keyWords
  • Copy the incident instance into a record in the incident audit log table
notificationAdapter.postInsertEvent com This step is added for notification feature. It captures data after saving the object

Example

The following example shows the content of the input parameters and output parametersReturn used to insert an incident object. The following input parameters show the minimum data required for the insert method to succeed. If any of the values shown are missing, the insert fails. Values are the tokens between the node tags. In the following node snippet, the number three is a value: <categoryId>3</categoryId>.
