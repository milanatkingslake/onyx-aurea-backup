Understanding OED copies

Understanding OED Copies

At any one time, four copies of the Onyx Enterprise Dictionary (OED) can exist. These copies include:

  • A production copy, which is the XML file (OnyxEnterpriseDictionary.xml) installed with OEAS. The OED cache is created from this copy.
  • An OED cache copy, which is created from the production copy the first time that a client calls the Onyx Transaction Manager.
  • A working copy, which is created from the production copy when you use the Object Designer. When you make customizations to the OED, the Object Designer creates a copy of the OED (OnyxEnterpriseDictionaryWC.xml) and saves it separately from the production copy of the OED. When you publish this working copy, it replaces the production copy.
  • An abridged OED that the Onyx Data Manager uses to generate SQL at runtime. This copy is created when you publish changes to the OED.

The following diagram depicts how the various OED copies and caches are used:

  1. OEAS setup creates a working copy of the OED production copy so that you can customize it without directly affecting the production copy.
  2. When you save your changes to the OED, the Object Designer saves the changes to this working copy. None of your changes are transferred to the production copy until you publish your changes to the OED.
  3. When you publish your changes, the Object Designer overwrites the production copy (OnyxEnterpriseDictionary.xml) with the contents of the working copy (OnyxEnterpriseDictionaryWC.xml).
  4. After you publish your changes, the working copy and the production copy of the OED are identical. The working copy is not deleted or otherwise removed, and it is used the next time that you make modifications to the OED. Only one working copy exists at a time.

Important: Since the OTM communicates with the OED cache, and not the actual OED source file, the changes that you make to the production copy of the OED (by publishing those changes) are not actually implemented until the OED cache is refreshed. To refresh the OED cache, you must restart OEAS.
