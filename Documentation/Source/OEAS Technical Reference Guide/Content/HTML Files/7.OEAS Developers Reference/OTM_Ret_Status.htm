Returning status

The step package contains a methodStatus element that is used specifically to store status information from a business object method. When the method completes normally, methodStatus contains a single element, statusType, with a value of 'success'. When an error occurs, or if a step needs to return extended information, methodStatus provides a place for step components to assemble detailed messages for interpretation by the client application.

There are three types of elements that normally appear within methodStatus: statusType, message, and error. Each of these has a standard structure. Client applications and custom step components should limit themselves to working within this design for maximum portability between components.

The contents of a sample methodStatus element appear below:

<methodStatus>

   <statusType>failure</statusType>

   <error associationId="7BC0D0BD-BE6A-47C0-9B44-E265D249C8FF">

      <severity>failure</severity>

      <errorNumber>50027</errorNumber>

      <errorDescription>You failed to supply a primary key property: primaryId.  A primary key is required to update, delete or retrieve an object.</errorDescription>

      <errorType>oetBusinessLogic</errorType>

      <natLangId>750027</natLangId>

      <sourceType>ostComponent</sourceType>

      <sourceName>company.retrieve</sourceName>

      <timestamp>2006-03-14 13:37:12</timestamp>

   </error>

</methodStatus>
