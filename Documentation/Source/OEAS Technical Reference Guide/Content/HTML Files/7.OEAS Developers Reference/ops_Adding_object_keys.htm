Adding object keys

Before adding object keys, see adding properties. You must tell the middle tier which property to use to uniquely identify a routingPrinter object. Use the object keys collection to do this. An object can have more than one set of keys, but for this scenario you only specify the primary key.

The following table shows the required attributes for the routingPrinter primary key. The value for the name attribute shows the suggested Onyx convention for naming object keys. Using Object Designer, add an object key with the following attributes.

Value
