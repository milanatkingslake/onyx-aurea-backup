Step 2: Handle the Step Package
Handle the step package

Before your step component attempts to perform a task—such as to receive the data for a stock ticker symbol from a client UI—it should first verify that it is receiving all of the information that it expects to receive. Otherwise, it cannot function properly.

The Onyx Transaction Manager (OTM) passes XML data from one step component to another by referencing an instance of the IXMLDOMDocument interface. Onyx calls the contents of this interface the "step package." Each step component is responsible for reading the contents of this step package (so that it knows what it is receiving) on input, and updating the step package on output.

For this sample, read the step package to:

  1. Verify that all required XML nodes are present.
  2. Obtain data from the step package.
  3. Return data to the step package.
