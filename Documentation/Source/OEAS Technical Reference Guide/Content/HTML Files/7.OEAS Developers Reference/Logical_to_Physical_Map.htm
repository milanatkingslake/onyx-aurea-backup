Logical to physical map

The logical to physical map section maps logical objects to their corresponding physical entities (OEDB schema). The mapping enables the SQL generation engine to generate appropriate SQL to correctly persist and retrieve object instances to and from the database. This section includes mappings for stored procedures that prevent the generation of SQL statements for a given method and replace the functionality of generated SQL statements with a database stored procedure. All retrieveCollection and retrieveList methods replace generated SQL statements with stored procedures. By setting these maps appropriately, developers can use custom procedures instead of generated SQL statements for any method.

The following table summarizes the logical to physical map metadata.

Metadata Description

Object physical map

A mapping between logical objects and properties and corresponding physical tables and columns

Procedure map

A mapping between logical method steps and corresponding procedures or functions
