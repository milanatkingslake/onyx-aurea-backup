OnyxScriptEngineStep

The OnyxScriptEngineStep component enables an OEAS client application to deliver a script. The engine implements the functionality for loading a script, resuming a suspended script, and advancing a script from one step to the next. As required by the respective functions, it can perform any of the following actions.

  • persists Scripting transactional data
  • executes method calls designed to occur after (post) the current script step
  • evaluates branching rules to determine the next script step
  • executes method calls designed to occur before (pre) the next script step
  • updates and persists Scripting variable information as appropriate
  • retrieves domain data for the next step and returns it to the client application

By managing this complex set of tasks, the engine provides efficient and consistent script execution for the client application.

OEAS exposes the script engine functionality through the logical methods of the scriptSession object. The advanceStep, loadStep, and resumeSession methods of the scriptSession logical business object (LBO) each include a com step call to OnyxScriptEngineStep. The OnyxScriptEngineStep determines which method to call from the StepAdditionalData parameter to the IStep Execute method. This step component is not intended for use by other standard OEAS methods or custom methods. If the StepAdditionalData contains anything other than one of these three methods, the engine returns an error.

In addition to validating the StepAdditionalData, the script engine validates the other IStep Execute parameters as outlined in the following list.

  • The application name must be present, or an error is returned.
  • The engine ignores LogicalObjectName.
  • The engine ignore LogicalMethodName.
  • The XMLParser (DOM) must be present. If the parser is missing, the engine raises a COM error. The engine cannot return XML or log errors.
