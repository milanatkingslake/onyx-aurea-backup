Batch Subsystem (OPS)

Batch subsystem

The Batch Subsystem is responsible for processing batches of actions on multiple records in the database. For example, List manager actions are generated by Onyx Employee Portal (OEP) list manager operations. These orders typically consist of applying a single action, such as insert or update, to a list of customer records.

Batch Subsystem Processing

The Batch Subsystem processes an actionBatch record inserted through an OEP List Manager action. The batch action is processed through the following steps:

  1. The Batch subsystem calls the retrieveNextBatch method of the actionBatch business object to retrieve the next batch action for processing.
  2. The sourceList contained in the actionBatch results is processed in a loop. Each action in the actions list is applied to each item in the sourceList.

Each action applied to a sourceList item results in an OTM action call. For example, in the scenario where there are 5 companies in the sourceList and 2 incident insert actions, there are 10 OTM calls.
