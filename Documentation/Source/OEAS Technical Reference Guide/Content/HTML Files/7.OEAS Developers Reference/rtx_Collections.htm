Collections
Collection elements

The run-time XML includes collection elements to organize parent-child related objects. For instance, the scriptPrompt element can contain scriptPromptResponses. Scripting collection elements are not defined in the OED. Instead, Onyx uses them at run time as necessary.

In the interest of economy and performance, Onyx does not pass empty collection elements. In the absence of any child elements, the collection element is not present. All collection elements in the run-time XML should contain at least one child element of the collection object type. This rule applies equally to domain and transactional objects.

Note: Your run-time XML consumer application should be coded defensively to handle the case of an empty collection element but should not generate any empty collection elements.
