Configuring and populating the table

Before configuring and populating the table, make sure you have already installed the customization. The goal here is to use Table Administration to add printers to the ct_custom_routing_printer table. Before you can use Table Administration to add printers, however, you must add your custom table as a new tableAdministrationItem using, you guessed it, Table Administration.

After you add the routingPrinter object as a tableAdministrationItem, you can add routingPrinter objects to the database using Table Administration. Table Administration uses the retrieveCollection and saveCollection methods you defined earlier to manage the data. You may be required to restart IIS for the new table to be displayed in the tool.

You can setup a couple of printers on your local network with fake country codes to test the customization.
