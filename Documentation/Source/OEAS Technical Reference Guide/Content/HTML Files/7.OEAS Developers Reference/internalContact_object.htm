internalContact object

The internalContact object enables you to manage which internal users act as contacts between you and your customers. All active Onyx users can be assigned to customer, incident, and product records as internal contacts. Typically, these are internal employees, although the only requirement is that they have an active Onyx user ID. Users can assign as many internal contacts to a customer as required.

The internalContact object has one key property, primaryId, that uniquely identifies the internalContact product in the database. Once the object is created, the value of this property cannot be changed.
