customerProductDetail object

The customerProductDetail object provides details of customerProduct activity.

The customerProductDetail object has three key properties, primaryId, ownerId, and lineId, that uniquely identifies customer product data in the database. Once the object is created, the values of these properties cannot be changed.

To provide any detail other than lineId, you must modify the database schema and provide the logical to physical mapping in the OED. Onyx provides the behavior, but the customization must provide the logical and physical model of what information goes in a line detail. You can use the user1 through user10 properties for the logical model, but you must create the physical schema and mapping to these properties.

Methods
