Customization Process

The general process associated with nearly every customization is outlined below.

Many customizations require you to modify the contents of the Onyx Enterprise Dictionary (OED).

The customization process can be described in general terms as follows:

  1. Define the business rule that you want to alter/add.

  2. Review this guide to determine what you need to modify or create in order to implement the customization. For instance, in addition to using the Object Designer to modify the OED, you might need to create new database tables or step components for your customization to be successful.

  3. Save the OED source file (OnyxEnterpriseDictionary.xml) to a safe location, implementing standard version control of the file.

  4. Use the Object Designer to modify the appropriate business object or to create a new custom business object.

  5. If necessary, create a custom step component in a language that supports COM. Then, integrate it with OEAS.

Note: When customizing OEAS, remember that the performance of every task is directly related to the amount of time that it takes for the step component to execute.
