Analyzing error messages

If any errors are encountered during the Save, Validate, or Publish processes, a dialog box appears indicating which processes saved successfully and which processes failed. The dialog box also indicates if the process was partially successful. This occurs if you modified several items before invoking the process and some modifications succeeded prior to the failure. It is important to note that no items, including those that passed validation prior to the failure, are saved until successful completion of the process.

Note: If the process succeeds with no errors and no additional data to report, the errors dialog box is not provided - the success of the process is simply noted in the browser status bar.

The list of error items in the dialog box can be expanded for further information regarding any listed item. Click to expand an item within the list. The expanded information is:

Error Item

Description

severity

Indicates the level of the error:

  • Failure: The entire process failed.
  • info: An event occurred that, although not necessarily a failure, should be examined to determine if a problem exists.

errorNumber

A numeric identifier for the error. For DAC-generated errors, this number is a platform specific error number usually associated with a national language ID, although it is not the same as the national language ID. For non-DAC generated errors, this number is the number returned by the component that was called.

errorType

Describes the type of error encountered:

  • oetHresult: Bad HRESULT
  • oetReturnValue: Includes fail results (non-HRESULTS) from the DOM, status values from OEDCache, etc.
  • oetBusinessLogic: DAC generated errors
  • oetStoredProcedure: The return values from stored procedure calls

natLangId

An optional element that contains the national language ID associated with the error number. DAC components should include this element. Components that do not use national language do not include this element with their error.

sourceType

Identifies the source type of the component, or subsystem, that generated the error:

  • ostStoredProcedure
  • ostComponent
  • ostWinAPI
  • ostMisc

sourceName

Identifies the source name of the component, or subsystem, that generated the error, based on sourceType:

  • ostStoredProcedure - Stored procedure name. Example: wbospCustomerInfo
  • ostComponent - Component ProgID. Example: DictionaryCache.Dictionary
  • ostWinAPI - API method name (minus parameters)
  • ostMisc - A value that can be used to identify the source

timeStamp

The time at which the error and or message occurred.
