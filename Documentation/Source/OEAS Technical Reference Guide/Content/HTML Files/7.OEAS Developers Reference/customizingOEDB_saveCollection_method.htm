Additional saveCollection method possibilities

The following table describes the behavior to expect for saveCollection methods when dealing with the collection.  Refer to the appropriate section when dealing with a singular action in a saveCollection method.

Behavior based on the stored procedure return code value defines the entire result set. In this case, the return code column will be listed as not applicable (N/A) because it is not included in the result set.

Behavior only applies to the single row when the return code column is applicable. This is an important distinction, because multiple rows with different meanings might be passed back when methods return a status of success or success with information.

-2 or less

N/A

Return code parameter values of -2 or less are not supported for saveCollection methods. If you return a value of -2 or less a message is returned stating that an invalid return code was returned by the OEDB, but your procedure is otherwise successful.

-1

The row contains the following information:

o#_sequence_number - This value will be null when passing back information on a collection of rows.

status_code - This is a national language message ID, the value of which corresponds to the message you want to provide the user regarding the successful completion of the procedure.

database_code - NULL.

database_message - NULL or the value to use for the "~2" replacement holder in the national language message.

database_id - NULL.

0

N/A

A saveCollection returning a value  of 0 (success) will not contain additional rows in the result set than those created for each action. See the appropriate action for the corresponding rows that may be added to the result set.
