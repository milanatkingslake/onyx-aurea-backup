userInputFunctionCall (searchViewDetail)

userInputFunctionCall property

The userInputFunctionCall property of the searchViewDetail object contains a function call that is applied to the user's criteria upon searching a field.

Data type

string (maximum length)

Access

Read/write

Remarks

This property is used only for items with a nodeType of FIELD.

This property works together with the fieldName and searchConditionFunctionCall properties to drive the construction of the WHERE clause expression for this searchViewDetail.

The userInputFunctionCall defines the right-hand side of the WHERE clause expression. It contains function calls or any other syntax that is valid on the right-hand-side of a WHERE clause expression.

By convention, the List Manager interface replaces the string "~0" in the userInputFunctionCall with whatever search data the user input.

If userInputFunctionCall is NULL, then the List Manager interface uses the search data input by the user as the right-hand side of the WHERE clause expression.

For example, assume the following scenario:

  • The fieldName is product_code.
  • The searchConditionFunctionCall is UPPER(~0).
  • The userInputFunctionCall is UPPER(~0).
  • The user inputs "MyProduct" for the product number field.

Assuming the searchTable alias is "C", the resulting WHERE clause expression is:

  UPPER(C.product_code) = UPPER('MyProduct')

If the searchConditionFunctionCall is NULL and the userInputFunctionCall is NULL, then the resulting WHERE clause expression is:

  C.product_code = 'MyProduct'
