Standard maintenance columns

Each base table you create should be SQL generation compliant. This means that it should include the standard maintenance columns. Running standard SQL generation methods against a table that does not contain the expected maintenance columns will cause database errors.

Note: Maintenance columns contain data that is automatically updated, or data that is used to control the status of the database record.

The delete_status column supports three values:

  • 0 - active record
  • 1 - logically deleted, this record was deleted directly
  • 2 - logically deleted, the delete was cascaded from another table

Note: Domain tables in OEDB include a record_status column which indicates whether the domain data is active or inactive.
