Structural rules for replace procedures

All stored procedures called in replace mode (called from the middle tier) must be coded to return a result structured according to the standards for that method type. Please review the section on result set interpretation for information on formatting a result set.

Replacing a SQL generation persistence call

Onyx uses a standard structure for any stored procedure that is written to replace a SQL generation persistence call (insert or update method). These procedures must have a parameter for each object property. According to standards, these parameters are ordered according to the order of appearance of properties on the object. The maintenance properties on the object do not need to be included in this list of parameters; however, the stored procedure should be written to fill the maintenance columns. You should always include a user ID parameter for passing the context userID so that the proper information is available for filling the maintenance columns. You should also include the value for onyx_timestamp on updates. The parameter value should be checked against the value in the table. The update should be allowed to continue if the values match. For more information on maintenance properties, see Standard maintenance properties.

Procedures written to replace a SQL generation persistence call must return a result set.
