OPS concepts

The OPS service manages the Fullfillment, Batch, Email Monitor, and Synchronization subsystems. This section focuses on the fulfillment and batch subsystems. The Fulfillment subsystem, processes literature orders by managing the Print and Email engines. The Batch subsystem manages list manager actions. The OPS service and subsystems are configured and monitored by the OPS Administration Tool.

This section describes the overall architecture and the individual components in the OPS service as they relate to the information presented in the OPS customization book. OPS includes components for which Onyx does not support customization such as the Email Monitor and Synchronization subsystems.

The primary components of the OPS server are:

  • OPS Windows service
  • Subsystems
  • OPS Administration Tool

This section describes the following components of OPS:

  • OPS Windows
  • Batch subsystem
  • Fulfillment subsystem
  • Email and Print engines

For email and print literature products the engines use templates.
