Configured Properties

Configured properties

In general, columns in a database table that contain information about an object map to object properties in the OED. Step components usually handle this mapping. However, configured properties provide a means to dynamically map such columns in a custom table to properties of objects contained in the OED. To accomplish such mappings, you add an "internalName" value to the property in the OED to identify the custom column that the property is mapped to. If you do not use configured properties for such mappings, you must provide the mapping in the code that you use to communicate with the database, such as a stored procedure or a SQL statement. While you must manually create a custom SQL table, the Object Designer provides the interface to create configured properties for an object in the OED.

The GetConfiguredProperties method provides a way to list these mappings for any specified object.

Note: You cannot use configured properties with tables shipped with the OEAS application.

The example below shows how the property name "serverAdmin" maps to the custom table column "main.altAdmin" in the database. In this example, main.altAdmin serves to identify the table (main) and the column in the table (altAdmin). You can create the "internalName" string value in any way that helps you identify the table column to be mapped to the "name" value.

XML Example

<properties>

<property>

      <id>97DFB6B9-7B84-42D9-8232-BD52FD4762A9</id>

      <name>serverAdmin</name>

      <description>Name of system administrator</description>

      <dataType>string</dataType>

      <objectType/>

      <content/>

      <collection/>

      <internalName>main.altAdmin</internalName>

      <isKey/>

      <installedBy>Onyx</installedBy>

      <disabled/>

      <propertyConstraints>

             <propertyConstraint>

                   <id>A8C9E338-8B98-4F64-9739-1CCBD132D44E</id>

                   <constraintType>default</constraintType>

                   <name>required</name>

                   <value>0</value>

             </propertyConstraint>

      </propertyConstraints>

</property>

</properties>

To use a configured property, a step component must use the GetConfiguredProperties method to retrieve the list of configured properties from the OED cache. For persisted operations, the step component then translates the property name, which it was passed, to the internalName (database name) in order to determine where the data is stored in the database. For retrieval operations, the step component translates the internal name back to the property name and returns the correct data to the calling client.
