Filter Rule attributes

When you select a filter rule in the Object Designer, the attributes for that filter rule display in the edit area.

The attributes are defined as follows:

Attribute Name

Description

ID

The globally unique identifier (GUID) assigned by the Onyx Enterprise Dictionary. This attribute is read-only.

Rule Text

Type SQL and XSL WHERE clause fragments that you want to add to the database action. These fragments incorporate row level filtering or security capabilities in the SQL generated for the database action.

To execute successfully, rule text must follow these guidelines:

  • The rule text must be well-formed and operate in the proper context. The string can include placeholders for context values, such as siteID and username.
  • The XSL commands should assume that the current XML node being processed is that node whose element name is the name of the relevant object (for example, <company>) in the XML input parameter. Any element values referenced from the context XML should be referenced by their full XPath from the root node of the input XML.
  • Do not include the AND keyword at the start of the filter rule's text. The filter rule text will be surrounded by parentheses and prefixed with the AND keyword. This text manipulation is performed by SQL generation or the stored procedure implementing the filter rule.

Description

Type the description for this filter rule.
