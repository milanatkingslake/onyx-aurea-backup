joinCondition (searchViewDetail)

joinCondition property

The joinCondition property of the searchViewDetail object contains a SQL join clause that relates the search table associated with this searchViewDetail to the base search table of the associated search view.

Data type

string (maximum length)

Access

Read/write

Remarks

The joinCondition property is used only for items with a nodeType of TABLE. This property is NULL for items with a nodeType of FIELD. However it is usually needed for items with a nodeType of TABLE.

Base table node example

For base table nodes (i.e. the node referring to the company_lm_view in the Companies search view), the join condition looks like this:

  B.SITE_ID = ~~

The ~~ gets replaced by the Site ID in the UI, and the "B" is the alias assigned to the company_lm_view search table.

Secondary table node example

For secondary table nodes (i.e. the node referring to the incident_lm_view in the Companies search view), the join condition looks like this:

  B.company_id = F.owner_id

Where "B" is the alias of the base search table (in this case company_lm_view), and "F" is the alias for the secondary table (in this case incident_lm_view).
