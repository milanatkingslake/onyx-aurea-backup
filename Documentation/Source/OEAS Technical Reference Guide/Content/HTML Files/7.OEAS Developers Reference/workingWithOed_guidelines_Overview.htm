Object creation guidelines

As you develop customizations for Onyx Enterprise Application Server (OEAS), Onyx recommends you follow the object creation guidelines presented in this section. OEAS enables rapid customization by providing a set of standard behaviors implemented in its SQL generation engine. Following these object creation guidelines ensures your new objects include the standard behaviors.

Note: The terms business object, logical business object, and object are used interchangeably in this document.

The SQL generation engine provides the ability to communicate with the Onyx Enterprise Database (OEDB) based on metadata stored in OEAS. OEAS can be customized by following some clear rules on the formation of the metadata. These rules enable you to build new objects, specify their correspondence to database schema, and describe their relationships to other entities in the object model. You can add new features to OEAS by specifying OEAS metadata and creating tables in the database.

The SQL generation automates the coding of five standard methods: insert, update, delete, retrieve, and retrieveCollection. Properly structured database tables and OEAS metadata are all it requires to fully implement a new object with any or all of these methods.

Some custom features may not lend themselves to a simple one-to-one correspondence between the object model and database schema. Typically, you develop code to implement complex business rules. While SQL generation's purpose is to minimize the requirement for writing code, it is also designed to be flexible. The SQL generation model uses the same type of metadata to describe the incorporation of both custom COM+ components and stored procedures into the OEAS system. This section presents the rules for incorporating these types of customizations as well.

Metadata repository the Onyx Enterprise Dictionary

The metadata repository for OEAS is called the Onyx Enterprise Dictionary (OED). You view and modify the metadata contained in the OED using the Object Designer.

You can customize the database tier using Microsoft's SQL Server Management Studio. For more information on customizing the database tier, see Customizing the OEDB.
