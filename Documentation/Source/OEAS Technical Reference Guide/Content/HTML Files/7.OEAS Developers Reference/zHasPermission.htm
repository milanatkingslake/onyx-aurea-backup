HasPermission (ISession)
HasPermission method

The HasPermission method of the ISession interface determines whether a user has permission to use a resource.

Syntax

IDL

HRESULT HasPermission(

    [in] const BSTR strSessionID,

    [in] const BSTR strAppName,

    [in] const BSTR strResource,

    [in] const BSTR strSecurityMask,

    [out, retval] VARIANT_BOOL* pbAccess);

IDL parameter descriptions

strSessionID

[in] A string that contains a session ID returned by the authentication component.

strAppName

[in] A string that contains the application name.

strResource

[in] The name of the resource to check.

strSecurityMask

[in] The mask to check against. This is a hex-formatted string representation of the mask. Specifying this parameter is optional.

bAccess

[out, retval] Indicates whether or not the user has access.

Return values

This method returns a boolean value that indicates whether or not the user has access.

When called from a C++ client application, this method returns a standard COM HRESULT or one of the custom HRESULTS listed below. When a Visual Basic client application calls this method and an error occurs, the Err object specifies one of the decimal values listed below and provides a brief description of the error.

Remarks

The HasPermission method uses the resource cache to look up the significant bit for the resource passed in with strResource. It then looks for the bit in the permissions mask to verify that the user can access the resource.

The GetContextXML method returns the permissions mask specified in the strSecurityMask parameter. If the client that calls HasPermission does not have the mask for the specified session, the HasPermission method retrieves the mask from the database.
