Key Map attributes

When you select a key map in the Object Designer, the attributes for that key map display in the edit area.

The attributes are defined as follows:

Attribute Name

Description

Primary Object

Type

Select the type of key map to be stored in this relationship:

  • To map a property in the primary object to a foreign key, select property.
  • Property Name: Select the property used in the relationship. This list includes properties marked for deletion and properties that are disabled, both which are marked as such. Invalid properties are not listed.
  • To map the primary object to a literal value in this relationship, select literal:
  • Literal Value: Type the string value to be mapped. Enter numeric literals or database keywords, such as NULL, without quotes. Enter string literals surrounded in single quotes (apostrophes). A string enclosed in double quotes is interpreted as a database object name, such as table, view, or sequence.

Secondary Object

Type

Select the type of key map to be stored in this relationship:

  • To map a foreign key in the secondary object to a property in the primary object, select property.
  • Property Name: Select the property name that contains the foreign key. This list includes properties marked for deletion and properties that are disabled, both which are marked as such. Invalid properties are not listed.
  • To map the secondary object to a literal value in this relationship, select literal:
  • Literal Value: Type the string value to be mapped. Enter numeric literals or database keywords, such as NULL, without quotes. Enter string literals surrounded in single quotes (apostrophes). A string enclosed in double quotes is interpreted as a database object name, such as table, view, or sequence.
