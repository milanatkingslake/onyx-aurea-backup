OED cache

Data in the OED cache is accessible through the following interfaces:

ICache, ICache2, ICache3 - used by COM DLL components that run in-process with the OEAS COM+ application

DictionaryProxy - used by components and applications that run out-of-process from the OEAS COM+ application

The ICache interfaces are intended for use by custom step components and DictionaryProxy is intended for use by client applications.

The cache provides calling components with access to different types of OED data: object listing, object definitions, empty object instances, steps, and property constraints. The ICache, ICache2, and ICache3 interfaces can access all types of OED data, while the DictionaryProxy interface can access everything except for the steps (the DictionaryProxy interface has no GetSteps method).
