descriptionPath (methodCallDetail)

descriptionPath property

The descriptionPath property of the methodCallDetail object contains an XPath expression, relative to the elementPath, that locates a friendly display string for an output element.

Data type

string

Access

Read/write

Remarks

The methodCallDetail object supports an output having both an ID component and a description component. The returned item can have an identifier (the ID component--such as country.countryCode or company.primaryId) and a user friendly display string (the description component--such as country.description or company.secondaryId). The descriptionPath property is an XPath that locates the output's description component relative to the elementPath. For example, if elementPath specifies rowSet/rows, then the descriptionPath would specify row/c1, where the c1 element contains the description component of the output.

The descriptionPath leads to a single element that is a descendant of an output element. The descriptionPath does not include traversal operations that try to access elements above the output element. If the descriptionPath does not result in a selected node when the method call is executed, the returned output will have an empty description component. If it is NULL, the returned output will not contain a description component. The output node can still contain a text component and/or an ID component.

This property is ignored on input method call details.
