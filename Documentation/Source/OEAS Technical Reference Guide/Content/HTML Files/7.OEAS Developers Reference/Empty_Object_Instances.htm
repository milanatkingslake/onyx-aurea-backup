Empty Object Instances
Empty object instances

An empty object instance is a skeletal representation of a business object. An empty object instance contains only the name of an object and an empty XML element for each of its properties. An empty object instance does not include the object's methods.

Typical uses of empty object instances

You typically use empty object instances to create an object template that is populated later, during run-time, with actual data. Empty object instances are typically used with insert and retrieve methods.

Insert method: For an insert method, you first use the GetObjectInstance method to obtain an empty object instance from the Onyx Enterprise Dictionary (OED) cache. You can then populate the empty instance with property values to create the object that you want to insert in the database. To complete the insert, you call OTM to insert the object in the database.

Retrieve method: Custom step components can, in their retrieve method, obtain an empty object instance and attach it to the XML output that is being returned. The step component can then update the values of the properties in this empty object instance. The primary benefit to using GetObjectInstance is that it saves you the trouble of creating elements in your XML DOM.

You can use Onyx Logical Business Object Tester to get a quick view of objects you create in Object Designer, and to check the order of properties and ensure that you entered them all correctly.
