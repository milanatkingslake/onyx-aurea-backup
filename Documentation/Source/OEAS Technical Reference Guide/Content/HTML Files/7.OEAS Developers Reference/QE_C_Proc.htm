Connection process

The IEBusQE interface establishes a new connection to the Onyx Enterprise Database each time a step component calls one of its methods. The connection is closed before the method returns control to the calling step component. The Query Engine leverages Microsoft's OLE DB technology to manage resource pooling, thereby reducing the execution time necessary to create new connections repeatedly.

Before the Query Engine executes ad-hoc SQL or a stored procedure against the database, it executes a series of Transact-SQL SET statements to configure the existing session. The following table lists the statements and provides a brief explanation of what each does. For additional information about these statements, consult your SQL Server documentation.

Statement Description

SET ANSI_NULLS ON

Causes SELECT statements that use WHERE column_name = NULL or WHERE column_name <> NULL to return zero rows.

SET ANSI_WARNINGS OFF

Prevents the server from raising errors caused by divide-by-zero, string too large for database column, and other similar conditions.

SET ANSI_PADDING OFF

Causes the server to trim trailing spaces in character values before insertion into varchar columns.

SET ANSI_NULL_DFLT_ON OFF

Prevents the insert of NULL into columns when the nullability of the column has not been explicitly specified.

SET QUOTED_IDENTIFIER OFF

Requires that identifiers cannot be quoted and must follow all Transact-SQL rules for identifiers. With this setting, literals can be delimited by either single or double quotation marks.

SET CONCAT_NULL_YIELDS_NULL OFF

Causes the server to discard NULL values that are concatenated to a string.

SET NOCOUNT ON

Prevents the server from returning the number of rows affected by the SQL statement.

SET IMPLICIT_TRANSACTIONS OFF

Causes the server to use autocommit mode for database transactions.

Each statement executed by the Query Engine is subject to a time out period specified by the OEAS Configuration Utility. If a command executed by the Query Engine fails to complete within this time out period, it is aborted and the called IEBusQE method returns an error and an empty result set.

Each of the IEBusQE methods can be configured to limit the total number of rows returned by a command. A few business object methods use this feature for large queries.
