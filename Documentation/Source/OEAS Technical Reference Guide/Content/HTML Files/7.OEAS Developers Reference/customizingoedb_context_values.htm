Context values

Context values are based on a user's settings. The parameters listed in the following table are normally created when context values are needed by a pre, post, or replace stored procedure:

Parameter

Description

@context_site_id

The @context_site_id parameter is used when a record is created. It is also required for modifying domain data since the site_id column would be included in a domain table's primary key. The @context_site_id parameter may also be required to filter data for retrieveList procedures depending on your requirements.

@context_user_id

 

The @context_user_id parameter can be used to maintain the insert_by and update_by columns during insert, update, delete, or saveCollection methods. This parameter may also be used for row level filtering.

@context_language_code

The @context_language_code parameter contains the user's preferred language. This parameter should be used when creating a stored procedure that uses multiple language domain data. The @context_language_code parameter would be used to modify domain data during insert, update, delete, or saveCollection methods, or to filter domain data for retrieve, retrieveCollection, or retrieveList methods .

@context_partner_user

The @context_partner_user parameter may be used to identify whether the @context_user_id user is a partner. Additional security restrictions can be implemented for a method called by a partner user.

@context_otm_id

The @context_otm_id parameter identifies the Onyx Transaction Manager (OTM) GUID value that was generated. This value will be unique for each OTM call.
