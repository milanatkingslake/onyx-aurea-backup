error element

An error element contains the explanation for a statusType value of failure. The information in an error element is intended to help the client troubleshoot the problem that caused the method to fail.

The message element typically contains:

Element Description

errorNumber

A numeric identifier for the error. For errors generated by Onyx components, this number often is tied to a text string in the National Language table.

errorType

Identifies the type of error that occurred. The first table below identifies the error types.

errorDescription

The text associated with the error message. For errors generated by Onyx components, this text comes from the National Language table.

natLangId

The National Language ID that is associated with the error number. Onyx components include this element, but custom components may not.

severity

Errors can have one of two severity values: failure or warning. A value of warning indicates a condition that requires further investigation or is not severe enough to warrant failure of the entire method.

timestamp

The exact time at which the method status was reported.

sourceType and sourceName

Together, these elements describe the source that detected the error. There are four possible values for sourceType. sourceName identifies the type by name. The second table below identifies the values for these parameters.

The errorType element identifies the error as one of four distinct types:

errorType Value Description

oetHresult

A COM call within the component failed.

oetReturnValue

An internal component failed.

oetBusinessLogic

A component discovered a problem with the data submitted by the client.

oetStoredProcedure

The stored procedure used by the step component returned a failure message.

There are four possible values for the sourceType element. The sourceName element identifies the source type by name.

sourceType Value sourceName Description

ostStoredProcedure

A stored procedure name.

ostComponent

The ProgId of a COM component.

ostWinAPI

The name of a method from the Windows API.

ostMisc

Any value that identifies the source.

The error element may also include the associationId attribute.

Example

<error associationId="B49CA0B5-8D3F-48EC-9A09-66E0AAC897FA">

  <severity>failure</severity>

  <errorNumber>9538</errorNumber>

  <errorType>oetBusinessLogic</errorType>

  <errorDescription>Company not found</errorDescription>

  <sourceType>ostMisc</sourceType>

  <sourceName>OnyxCustomerDAC:RetrieveCompany</sourceName>

  <natLangId>709026</natLangId>

  <timestamp>2000-11-19 12:57:34</timestamp>

</error>
