Modifying a property

Existing attributes of properties can be modified for customization. Modifying property attributes can be useful for changing optional properties to required, as this sample demonstrates.

Scenario

This sample demonstrates how to modify an existing property to change it from optional to required. This is useful for making the email address a required entry field.

Guidelines

Changing property attributes can result in unpredictable behavior, or even failure, of the business object. The following guidelines are do's and don'ts for modifying property attributes.

  • Some property attributes must never be modified. Attributes such as Name and Data Type must never be changed.
  • Property attributes that are suitable for modification should be changed only within certain constraints. for example, minimum and maximum value ranges should be set to appropriate values for the given property. Other attributes that can be modified with reasonable consideration are the required/optional attribute and the default value.
  • Write down the attributes of the property you are about to modify before making your changes. If your changes result in incorrect behavior for the property, you will need to restore the attributes to their original settings.
