fulfillmentEngine object

The fulfillmentEngine object enables an administrator to associate a fulfillment subsytem with a COM component that processes literatureProduct orders. The fulfillmentEngine object represents executable binary code that subsystems access through its progId property. Each subsystem uses only one fulfillment engine, but a fulfillment engine may be used by many subsystems. A typical fulfillment engine provides one fulfillmentType such as email, print, or fax by implementing a standard COM interface recognized by the Onyx Batch Processing System.
