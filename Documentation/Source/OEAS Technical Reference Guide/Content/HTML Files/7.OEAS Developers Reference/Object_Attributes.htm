Object attributes

This book describes the object attributes, as displayed in the Object Designer.

Type of OED Metadata

Description

Business Objects

Identifies each business object in the OED with a full object definition.

Properties

Defines properties that describe business objects.

Property Constraints

Defines information about properties used to validate input parameters.

Object Keys

Defines key properties of objects.

Relationships

Defines relationships between business objects.

Key Maps

Maps the properties used to link objects in a relationship.

Object Physical Map

Maps objects and their properties to database tables and columns.

Methods

Defines actions that can be performed on, or by, business objects.

Parameters

Defines additional information that is required by a method to further define its action.

Steps

Defines the steps of a method. Each step is a single unit of instruction and every method is made up of one or more steps.

Procedure Maps

Maps steps to stored procedures. Used for pre-processing, post-processing, or to replace default stored procedures.

Input Maps

Maps a property, literal, or other value to a stored procedure input parameter.

Output Maps

Maps a property, literal, or other value to a stored procedure output parameter.

Sort Properties

Defines which properties and the direction (descending or ascending) that result set data are ordered.
