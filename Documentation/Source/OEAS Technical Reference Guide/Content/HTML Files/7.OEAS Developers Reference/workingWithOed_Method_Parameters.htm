Method Parameters
Parameters

Proper structuring of method parameters is important. These rules will help you ensure that SQL generation works with your object.

All of the standard SQL generation methods take one and only one parameter. The type of this parameter is "object" and the parameter also has an objectType, which should correspond directly to the named businessObject owning the method. The direction of the parameter for all of these methods is "inOut." Only methods of type "retrieveList" and "custom" can have more than one parameter.

A retrieveList method may have as many parameters as desired but the method signature must include one "out" direction parameter of type "object" and objectType "rowSet." The ODM uses this parameter to contain the result set returned from a stored procedure call. It is typically placed last in the signature. All of the other parameters on a retrieveList method must be of simple xml types (e.g., string, i4, boolean, and so on). Only one object parameter is allowed--the rowSet parameter.

Custom methods are not allowed to have any object parameters at all. Custom methods do not return result sets, but return values through "out" or "inOut" parameters. You can use as many parameters with a custom method as you want. These must be simple xml data types.

For the retrieveList rowSet object parameter, there is some additional work you must do to ensure that the ODM knows how to present your result set.

The "ContentOut Definition" requires that you specify an initial national language ID value. This entry tells the ODM what column heading to return. This value should be the message code value of the row in the national_language database table that contains the first column heading needed for your result set. The ODM uses this initial value and the number of columns in the result set to retrieve the desired column headings from the national language database table.

You must also specify the "Column Name" values that you wish the return xml to contain. These should be specified in order of their occurrence in the select statement used by the cursor in your retrieveList procedure.
