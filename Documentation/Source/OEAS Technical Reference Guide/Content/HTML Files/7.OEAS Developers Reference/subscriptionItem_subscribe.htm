subscribe (subscriptionItem)

subscribe method

The subscribe method of the subscriptionItem object creates and updates subscriptionItem instances based on user-initiated requests to subscribe to and unsubscribe from transactional records.

Remarks

User-initiated subscribes and unsubscribes occur when end users specify these actions through an online or offline client user interface. Subscribes and unsubscribes that result from cascade method calls are not user-initiated.

The subscriptionItem object instances are flagged as subscribe (syncAction 3) when inserted. The cascade method creates or updates subscriptionItem instances for records that are related to the subscribed instances. These instances flagged as refresh (syncAction 1). After the sync server downsyncs the transactional records that these subscriptionItem instances represent, their syncActions are set to processed.

When the subscribe method processes a subscribe request to a subscriptionItem instance that exists for the user and transactional object instance, the instance is set to syncAction 3. When subscribe processes an unsubscribe request to a subscriptionItem instance that exists for the user and transaction object instance, the existing subscriptionItem instance is set to syncAction 4.

When the subscribe method succeeds, it returns an empty parametersReturn node.
