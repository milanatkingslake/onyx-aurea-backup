parameters XML

The parameters XML includes the logical interface to the script engine known as the Scripting run-time XML. The scriptSessionXml parameter contains the run-time XML. The scriptSessionXml node must be present under the parameters node and its contents must be well formed XML that represents the run-time XML. The run-time XML is encoded as an escaped text string under the scriptSessionXml node. The engine does not modify run-time XML for the current step, and the OTM does not return the original parameter run-time XML back to the client.

The following sections show the specific parameters used to call each of the logical methods implemented by OnyxScriptEngineStep component.

advanceStep

<parameters>

   <scriptSessionId/>

   <nextSessionStepId/>

   <nextStepId/>

   <scriptSessionXml/>

</parameters>

For more information on these parameters, see the advanceStep topic.

loadStep

<parameters>

   <scriptSessionId/>

   <scriptSessionStepId/>

   <scriptStepId/>

   <scriptSessionXml/>

</parameters>

For more information on these parameters, see the loadStep topic.

resumeSession

<parameters>

   <scriptSessionId/>

   <scriptSessionXml/>

</parameters>

For more information on these parameters, see the resumeSession topic.
