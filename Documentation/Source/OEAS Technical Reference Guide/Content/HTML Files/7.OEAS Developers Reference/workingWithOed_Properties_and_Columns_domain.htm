Properties and Columns (domain)
Properties and columns

Your contract type object requires a key property. This key should be either an auto-generated numeric ID or a user entered natural code. Assume you do not need a meaningful abbreviation to be used elsewhere. Therefore, you elect to use an auto-generated numeric ID. According to the standard, the property is called "primaryId" and the database column is called "contract_type_did."

Besides the ID or key property, this object requires a textual description property. To simplify this example, assume that you do not require multi-language support for the contractType description property.

These properties are what you need to do the work. To see what additional properties are required by SQL generation and/or Onyx standards, see Standard Maintenance Properties. Because you also need a table to persist your object's data, you need to know what database columns are required by SQL generation or suggested by Onyx standards. See Standard Maintenance Columns and also Primary Key Columns.
