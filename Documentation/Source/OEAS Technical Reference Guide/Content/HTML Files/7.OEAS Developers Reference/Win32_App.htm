Win32 application access

In this scenario, the Win32 application calls the authentication process as part of a startup routine. It is important to note that this could be NT authentication, Onyx authentication, or some other form of authentication. Use the IAuthenticateNT interface if the user is logged on to the local domain. Use the IAuthenticateOnyx interface for users who are not logged or cannot log on to the local domain. Once a session has been established, the application holds on to the session ID as a private variable and passes the value with each call to the Onyx Transaction Manager.
