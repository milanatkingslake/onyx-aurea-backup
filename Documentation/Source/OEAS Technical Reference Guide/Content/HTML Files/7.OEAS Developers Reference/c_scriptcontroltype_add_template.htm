Adding the template

The template is part of an XML structure that must conform to the XSL schema. The xsl:template element that you add must be a child of the xsl:stylesheet element. For the sake of clarity and organization, group the template code with the other script control templates.

Having located and backed up script_delivery_step.xsl file, you are ready to add the template to the file.

To add the template to script_delivery_step.xsl:

  1. Open script_delivery_step.xsl in a text editor.
  2. Position the cursor at the first blank line after the line containing:

<!--************************************ the controls themselves

  1. Copy and paste the following XSL template definition into the file.
  1. Continue to the "Adding the template binding" steps.

Code

<!-- customerSearch control -->

<xsl:template name="control_customerSearch">

 <xsl:param name="safePromptId"/>

 <span id="controlPrompt_{$safePromptId}">

  <xsl:attribute name="responseCode">

   <xsl:value-of select="/root/scriptSession/scriptSessionSteps/scriptSessionStep[primaryId = $scriptSessionStepId]/scriptSessionPromptResponses/scriptSessionPromptResponse[scriptPromptId = current()/@primaryId]/responseCode"/>

  </xsl:attribute>

  <xsl:attribute name="value">

   <xsl:value-of select="/root/scriptSession/scriptSessionSteps/scriptSessionStep[primaryId = $scriptSessionStepId]/scriptSessionPromptResponses/scriptSessionPromptResponse[scriptPromptId = current()/@primaryId]/value"/>

  </xsl:attribute>

  <xsl:attribute name="default">

   <xsl:value-of select="@default"/>

  </xsl:attribute>

  <input id="customerId_{$safePromptId}" size="50" disabled="true">

  <!--put in the primaryId for the response made-->

   <xsl:attribute name="responsePrimaryId">

    <xsl:value-of select="/root/scriptSession/scriptSessionSteps/scriptSessionStep[primaryId = $scriptSessionStepId]/scriptSessionPromptResponses/scriptSessionPromptResponse[scriptPromptId = current()/@primaryId]/primaryId"/>

   </xsl:attribute>

  </input>

  <img tabindex="{position()}"

       onmouseup="javascript:this.src='../images/icons/iconopenwindow1.gif';" 

       onmousedown="this.src='../images/icons/iconopenwindow0.gif';" 

       onmouseover="javascript:this.src='../images/icons/iconopenwindow2.gif'" 

       onmouseout="javascript:this.src='../images/icons/iconopenwindow1.gif';"  

       src="../images/icons/iconopenwindow1.gif" align="top"/>    

 </span>

</xsl:template>

Note: The template name must be the same name used for control_type_code in the domain data and for the Javascript class function. The name is case sensitive.

The template does the following:

Declares an input parameter

The stylesheet uses this parameter to pass the ID of the prompt that owns the control.

  <xsl:param name="safePromptId"/>

Defines the HTML

The template provides exemplar text to render the control. For this control, the template includes an input element to display the customer name in a textbox, an img element for a button to launch the customer Search dialog box, and a span element to group the textbox and the button. The span element includes three attributes that the Javascript class references. These are used for storing the values from the array returned by the customer search dialog, and if it were enabled, setting a default value in the control. The template also defines an attribute responsePrimaryId on the input element that binds the control response to the scriptSessionPromptResponse primaryId.
