GetConfiguredProperties (ICache)

GetConfiguredProperties method

The GetConfiguredProperties method of the ICache interface retrieves the configured properties for a logical object. This method has been deprecated.

Note: This release of OEAS supports configured properties as a legacy feature from a prior release of OEAS. If you need to map properties to custom tables, use the Object Physical Maps section of the Object Designer.

Syntax

IDL

HRESULT GetConfiguredProperties(

    [in] BSTR ApplicationName, 

    [in] BSTR ObjectName, 

    [out] SAFEARRAY(VARIANT)* ConfiguredProperties, 

    [out, retval] OEDCacheStatusEnum* Status);

IDL parameter descriptions

ApplicationName

[in] A string containing the logical application name that determines which subset of the cache to use for a query.

ObjectName

[in] A string that contains the name of the object.

ConfiguredProperties

Output. A two-dimensional array of variants (SAFEARRAY) that contains the configured properties for the specified object.

Status

[out retval] An enumeration of type OEDCacheStatusEnum that contains the status of the method call.

Example

The following Visual Basic code demonstrates how to use the GetConfiguredProperties method to determine the property name and internal name of business object properties that are tied to a column in a custom database table.

Dim oCache As OnyxDictionaryCacheLib.DictionaryCache

Dim status As OnyxDictionaryCacheLib.OEDCacheStatusEnum

Dim configuredProperties() As Variant

Dim sPropertyName As String

Dim sInternalName As String

'Set the cache and identify the object and property in the method

Set oCache = New OnyxDictionaryCacheLib.DictionaryCache 

status = oCache.GetConfiguredProperties("Onyx", "incident", configuredProperties())

'Set the variables to the content of the array

If status = ocsSuccess Then

  sPropertyName = configuredProperties(0, ocpPropertyName)

  sInternalName = configuredProperties(0, ocpInternalName)

End If
