incident object

incident object

The incident object enables users to delete, insert, retrieve, and update information about interactions with customers. Examples of customer interactions include support calls, sales opportunities, and requests for service.

Methods

Method Description

delete

Either logically or physically deletes an object from the database. Secondary objects, as defined in the relationship section of the Onyx Enterprise Dictionary, may also be logically deleted or updated depending on the logicalCascadeType elements of its relationships

insert

Persists an object to the database

retrieve

Returns a collection of objects based on the search criteria supplied in the parameters XML

retrieveByWorkNoteId

Searches the database by worknoteId dand returns a list of incident records

retrieveCollection

Returns a collection of object instances

retrieveListByFilter

Searches the database by the filter criteria you pass in and returns a rowSet of incident records

retrieveListByFilterNoDefault

Searches the database by the filter criteria you pass in and returns a rowSet of incident data

retrieveListByIdList

Returns the list of incidents identified by the incidentIdList method parameter

retrieveListByOwner

Searches the database by ownerId and categoryId and returns a list of incident records

retrieveListByPartner

Returns the list of incidents assigned to a partner depending on the input parameters

retrieveListByPartnerAccess

Returns the list of incidents that generates access to a partner through the record identified by the partnerAccessIdList input parameter

retrieveListByWorkTicket

Returns a rowSet of incident information for the specified work ticket

retrieveListExternal

Returns the incident data needed by OCP along with information on the incident's owner and reminder

retrieveListHistory

Retrieves a list, sorted by last update date, of recent incident activity consolidated from: Attachments, Forecasts, Quotes, Tasks, linked Work Notes, and Work Tickets data

retrieveListMobileSearch

Returns the list of objects that meets the specified criteria

retrieveListQuickSearch

Retrieves a list of incident objects based on specified criteria

retrievePartnerListByAccess

Returns the data for the list of partnerGroupAccess and/or userAccess primaryId values

retrievePartnerListOfLeads

Returns the list of incidents assigned to a partner depending on the input parameters

retrievePartnerListOfNewLeads

Returns the list of incidents assigned to a partner depending on the input parameters

retrievePartnerUserList

Returns the data for a partner user - optionality filtered by the user assigned to an incident

retrievePartnerUserListByOwner

Returns the list of incidents for the incident's owner

saveCollection

Performs multiple actions at once: the insert, update, and delete of objects as appropriate

update

Persists modifications to an existing object instance
