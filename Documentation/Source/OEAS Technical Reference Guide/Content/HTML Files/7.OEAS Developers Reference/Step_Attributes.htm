Step attributes

When you select a step in the Object Designer, the attributes for that step display in the edit area.

The attributes are defined as follows:

  • ID: The globally unique identifier (GUID) assigned by the Onyx Enterprise Dictionary. This attribute is read-only.
  • Name: Type the name you want to assign to this step. Step names must be conform to XML naming conventions.
  • Description: Type the description for this step.
  • Step Type: Select the type of step you want to create:
  • Onyx specific com type
    • Step Data: automatically provided
  • com
    • Step Data: Type the ProgID of the desired COM object.
  • method
    • Step Data: In the first drop-down, select the object that contains the method you want to include. In the second drop-down, select the method to include.
  • transaction
    • Step Data: Select the type of transaction step:
      • begin
      • end
  • Additional Data: Enter any additional data needed for this step. For Onyx specific com types, Object Designer provides the basic data structure. Enter valid values where appropriate.
  • Override Template:Do not enter anything in this field; it is reserved for future use. It must be blank.

    • WARNING: If anything is entered in this field, it can break the step.

  • Installed By: Type the name of the vendor or application adding this step.
  • Disabled: Check this box to disable this step.
