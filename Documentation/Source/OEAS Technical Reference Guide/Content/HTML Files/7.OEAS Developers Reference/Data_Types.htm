Data Types
XML data types

The encoding of simple data types adheres to the XML Data Types portion of Microsoft's XML-Schema specifications. The encoding of enumerated types and tokenized types, collectively also known as primitive XML Data types, is dictated by the World Wide Web Consortium (W3C) XML 1.0 Recommendation. The commonly used data types in the OEAS appear in the first table below. The second table lists other valid types not used (but supported in customizations) in a standard OEAS installation. For the most recent specifications, consult Microsoft MSDN and search for: XML Data Types Reference.

Commonly used data types in the OEAS include:

XML Data Type

Description

boolean

0 or 1, where 0 == "false" and 1 =="true".

dateTime

Date in a subset of ISO 8601 format, with optional time and no optional zone. Fractional seconds can be as precise as nanoseconds. For example, "1988-04-07 18:39:09".

See the OEAS-specific information for this type below this table.

float

Real number, with no limit on digits; can potentially have a leading sign, fractional digits, and an exponent. Punctuation as in U.S. English.

Note: OEAS returns all data types as strings, regardless of the actual data type that is stored in the database. Data stored in the database with a data type of "float," when returned as a string, sometimes becomes less accurate because of the way in which the number is rounded during the conversion. For example, if the number 10.1234 is stored in the database as a float data type, a retrieve method may return the number as a string with the value 10.12399999999. To ensure that the number is accurately returned to the calling client, you must first convert the string data type to the appropriate data type that the calling client expects.

int

Number with optional sign. No fractions and no exponent.

i1

Integer represented in one byte. Optional sign.

For example, "1, 127, -128".

i2

Integer represented in two bytes. Optional sign.

For example, "1, 703, -32768".

i4

Integer represented in four bytes. Optional sign.

For example, "1, 703, -32768, 148343, -1000000000".

object

Any type of XML object. This type is also used to represent collections of objects. This type is not part of the Microsoft schema.

string

A string of characters.

When used in the OEAS, the data type dateTime differs from the standard definition of the dateTime data type in the following ways:

  • It does not support fractional seconds.
  • There is a space between the date and the time (instead of the letter 'T'). For example, a valid value for dateTime in OEAS is 1988-04-07 18:39:09, not 1988-04-07T18:39:09.

Other datatypes include:

XML Data Type

Description

bin.base64

MIME-style Base64 encoded binary BLOB.

bin.hex

Hexadecimal digits representing octets.

char

String, one character long.

date (not supported)

Date in a subset ISO 8601 format, without the time data. For example, "1994-11-05".

dateTime.tz (not supported)

Date in a subset ISO 8601 format, with optional time and optional zone. Fractional seconds can be as precise as nanoseconds. For example, "1988-04-07T18:39:09-08:00".

fixed.14.4

Same as "number" but no more than 14 digits to the left of the decimal point, and no more than 4 to the right.

number

Number, with no limit on digits; can potentially have a leading sign, fractional digits, and an exponent. Punctuation as in U.S. English. (Values have same range as most significant number, R8, 1.7976931348623157E+308 to 2.2250738585072014E-308.)

time (not supported)

Time in a subset ISO 8601 format, with no date, and no time zone. For example, "08:15:27".

time.tz (not supported)

Time in a subset ISO 8601 format, with no date but optional time zone. For example, "08:1527-05:00".

r4

Real number, with 7-digit precision; can potentially have a leading sign, fractional digits, and an exponent. Punctuation as in U.S. English. Values range from 3.40282347E+38F to 1.17549435E-38F.

r8

Real number, with 15-digit precision; can potentially have a leading sign, fractional digits, and an exponent. Punctuation as in U.S. English. Values range from 1.7976931348623157E+308 to 2.2250738585072014E-308.

ui1

Unsigned integer. A number, unsigned, no fractions, and no exponent. For example, "1, 255".

ui2

Unsigned integer, two bytes. A number, unsigned, no fractions, and no exponent. For example, "1, 255, 65535".

ui4

Unsigned integer, four bytes. A number, unsigned, no fractions, and no exponent. For example, "1, 703, 3000000000".

uri

Universal Resource Identifier (URI). For example, "urn:schemas-microsoft-com:Office9".

uuid

Hexadecimal digits representing octets and optional embedded hyphens that are ignored. For example, "333C7BC4-460F-11D0-BC04-0080C7055A83".
