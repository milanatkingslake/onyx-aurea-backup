Print engine (OPS)

Print engine

The Print Engine merges the orderBatch XML information using a Word document, which sends it to a print queue for printing. The details of the processing are described in the OPSProcessDocument interface topic.

Merged Data Products

All printed literature products and system documents are the result of merging data from the literature order XML into a print template. The results of this merge are then delivered to a printer queue for physical printing.

Label Products

Label products are a special case of a Merged Data product in that more than one order item can be printed on a page. More information on how the label orders are grouped for printing can be found in the OPS Customization section of this document.

Error Handling

Status is returned to the print engine by the OPSProcessDocument method upon its completion. This status indicates whether the order item processing succeeded or failed along with the Id for the order item. This information is passed back to the Fulfillment Subsystem. The Fufillment Subsystem only updates the status of processed orderDetails in the database.
