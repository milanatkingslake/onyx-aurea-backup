HasPermissions (ISession)
HasPermissions method

The HasPermissions method of the ISession interface determines whether a user has permission to use several specified resources.

Syntax

IDL

HRESULT HasPermissions(

    [in] const BSTR strSessionID,

    [in] const BSTR strAppName,

    [in] const VARIANT vResources,

    [in] const BSTR strSecurityMask,

    [out, retval] BSTR* pstrAccess);

IDL parameter descriptions

strSessionID

[in] A string that contains a session ID returned by the authentication component.

strAppName

[in] A string that contains the application name.

vntResources

[in] A variant array of strings specifying the resources to be checked.

strSecurityMask

[in] A string that contains the mask to check against. This is a hex-formatted string representation of the mask. Specifying this parameter is optional.

strAccess

[out, retval] A XML string that indicates whether or not the user has access to each of the given resources.

Return values

This method returns a XML string that indicates whether or not the user has access to each of the given resources.

When called from a C++ client application, this method returns a standard COM HRESULT or one of the custom HRESULTS listed below. When a Visual Basic client application calls this method and an error occurs, the Err object specifies one of the decimal values listed below and provides a brief description of the error.

Remarks

The HasPermissions method uses the resource cache to look up the significant bits for the resources passed in with vntResource. It then looks for the bits in the permissions mask to verify that the user can access each resource. A value of 0 indicates no access; a value of 1 indicates access.

The format of the XML returned in the strAccess parameter is shown in the following example:

<permissions>

  <Customer.Insert>0</Customer.Insert>

  <Add.Incident.Button>1</Add.Incident.Button>

  <ViewWorknotes>0</ViewWorknotes>

</permissions>

The GetContextXML method returns the permissions mask specified in the strSecurityMask parameter. If the client that calls HasPermission does not have the mask for the specified session, the HasPermissions method retrieves the mask from the database.
