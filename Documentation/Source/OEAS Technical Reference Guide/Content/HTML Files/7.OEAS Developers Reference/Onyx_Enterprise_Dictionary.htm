Onyx Enterprise Dictionary

The Onyx Enterprise Dictionary (OED) is an XML file that contains metadata that defines the structure and content of both the business services tier and the data services tier, as well as the relationships between the two tiers.

The metadata, coded in XML, includes:

Metadata Description

Logical model

Defines the business objects that reside in the business services tier.

Physical model

Defines the tables and columns that reside in the Onyx Enterprise Database (OEDB), as well as the database stored procedures and functions that are used by business object methods.

Relationships

Defines the relationships between business objects.

Logical to physical map

Defines how the logical model is associated with the physical model. Identifies the link between each business object property and its corresponding table column in the database. Also identifies the link between each logical method step and its corresponding database stored procedure or function.

How the OED is used

The Onyx Transaction Manager (OTM) uses the data in the OED to process client requests. The OTM accesses the data from cached version of the OED called the OED cache. The OED cache is loaded into memory the first time a client makes a method call to the OTM. For additional details about the role of the OED in OEAS, see OEAS data flow.

Modifying the OED

If you plan to customize OEAS, you will almost certainly need to modify the OED. You can modify the OED using Object Designer, a design tool in the Onyx Enterprise Studio. When you modify the OED using Object Designer, you are actually modifying a second copy of the OED, or a working copy of the OED; this ensures that the production version of the OED remains isolated from your modifications until you are ready to publish them. When you're satisfied with your modifications, you publish your modifications to the production copy of the OED. The production copy of the OED contains the XML metadata that populates the OED cache for use by the OTM.

At any one time, then, as many as three copies of the OED may exist: the read-only in-memory OED (OED cache), the production copy, and a working copy. The production copy and working copy are physical XML files stored on disk.
