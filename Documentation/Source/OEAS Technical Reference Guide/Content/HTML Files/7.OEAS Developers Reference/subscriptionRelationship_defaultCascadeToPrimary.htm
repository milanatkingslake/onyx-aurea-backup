defaultCascadeToPrimary (subscriptionRelationship)

defaultCascadeToPrimary property

The defaultCascadeToPrimary property of the subscriptionRelationship object indicates whether the primary object of the relationship is automatically subscribed and cascaded when the secondary object of the relationship is subscribed.

Data type

boolean

Access

Read/write

Remarks

When a subscriptionRelationshipRole exists for the relationship and the user who is subscribing belongs to that role, the value of the cascadeToPrimary property overrides the value of defaultCascadeToPrimary property.

Valid values of this property are:

false (0): Do not subscribe the primary object of the relationship.

true (1): Subscribe the primary object of the relationship.
