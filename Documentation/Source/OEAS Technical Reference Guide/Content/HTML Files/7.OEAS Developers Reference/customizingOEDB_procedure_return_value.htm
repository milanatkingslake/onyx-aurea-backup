Procedure return value

The middle tier expects the return value sent back from SQL Server procedures to follow the structural rules described in Replace Procedures.

The following table describes the valid values for the built-in SQL Server return code parameter. The value of the return code parameter determines what is returned in the result set.

Note: Return code parameter values of -1 and 2 specify that the procedure will return a National Language message ID value as part of the result set parameter. You can modify a procedure to return existing National Language messages regarding the result set, the lack of a result set, or some other action that the procedure performed. Additionally, you can customize National Language messages using OES Table Administration.

Value

Status

Description

-2 or less

Success

The procedure successfully completed and is returning a result set. The return code parameter value can be a National Language message ID for the beginning column heading of a result set. This is set within the OED unless the procedure contains logic to determine the value.

-1

Success with message

The procedure successfully completed and a National Language message ID is returned in the result set parameter to provide additional information. There may be many rows in the result set if the return values is -1.

0

Success

The procedure successfully completed with no National Language message ID returned. There may be many rows in the result set if the return values is 0.

1

Failure: database error

The procedure failed with an unanticipated error. A SQL Server error code and message are returned in the result set. There will only be one row in the result set if the return values is 1.

2

Failure: business logic error

The procedure failed with an anticipated error, and a National Language message ID for the error is returned in the result set. There will only be one row in the result set if the return values is 2.

Sometimes a procedure fails by design. For example, you can use a stored procedure to limit user access to data based on return code: a return code parameter value of 2 could return a National Language message ID corresponding to the message “Screen Denied” to unauthorized users.
