Property Constraint

Each property may or may not have property constraints. As the name indicates, these entities constrain how a property behaves. The following options are available for property constraints:

Property Constraint

Description

required

A required property may not persist a NULL value to the data tier. The constraint is checked on both insert and update methods.

Note that on an update method, passing an empty property element does not violate the required constraint, as it constitutes a request to leave the property value unchanged.

default

A default constraint sets a value for a property when the property element is empty or missing in the input XML stream used on the business tier. This constraint is applied only for insert methods.

min or max

The min and max property constraints provide a floor and a ceiling for numeric and date values. In addition, for string properties, these constraints specify the minimum or maximum number of characters allowed. These constraints are checked for both insert and update methods.
