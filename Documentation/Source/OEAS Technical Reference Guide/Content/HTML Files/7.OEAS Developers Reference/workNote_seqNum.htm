seqNum (workNote)

seqNum property

The seqNum property of the workNote object contains the ordinal number of a workNote object in a series of workNote objects comprising a compound workNote object.

Data type

i4

Access

Read-only; for more information, see Read Only attribute

Remarks

The seqNum property is generated automatically by OEAS. When the workNoteText exceeds 1275 characters, the OEAS creates multiple workNote objects with an identical primaryId values and assigns them ordinal numbers (seqNum property) in order to reconstruct the order of workNoteText fragments. If the user enters worknotes with 2700 characters, for example, OEAS creates three workNote objects with the same primaryId but with unique seqNum properties of 1, 2, and 3. The primaryId/seqNum(1) workNote object contains the first 1275 characters of the text. The primaryId/seqNum(2) workNote object contains the second 1275 characters of the text. The primaryId/seqNum(3) workNote object contains the last 150 characters of the text.
