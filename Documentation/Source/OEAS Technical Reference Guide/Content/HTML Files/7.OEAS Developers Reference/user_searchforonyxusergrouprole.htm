searchForOnyxUserGroupRole

The searchForOnyxUserGroupRole method of the user object returns a list of Onyx user, group, and role objects whose ID, name, or description contain the search criteria. The data is returned as a rowSet object in the parametersReturn XML. Certain behaviors are common to all retrieveLists. For a description of these standard behaviors, use the following link.

An integer that specifies whether to return active or inactive user and group data. If the active element of the input parameters XML is empty or NULL, all data are returned, active and inactive. When active contains 1, all active data are returned. If the element contains 0, inactive data are returned. For zero (0), no role data are returned, because all roles are assumed to be active. Remarks

This method supports paging of the result set with the pageSize and startRowIndex attributes on the rowSet parameter.

This method supports the Onyx security component for Cognos security integration.

For a non-exact-match search to succeed, the search string must include appropriate wildcard characters or an exact string match. The search is not case sensitive.

The stored procedure sorts the result set by object type first and then by name. It returns group objects before role objects followed by user objects. Within the object type sort, objects are sorted by name in ascending order.

This method might retrieve only a limited list of data, and not always the entire list of data available. You can use the OEAS Configuration utility to configure the maximum rows returned. For more information, see Paging functionality.
