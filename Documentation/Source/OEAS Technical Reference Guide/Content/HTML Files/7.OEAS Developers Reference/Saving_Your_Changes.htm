Saving your changes

Warning: You cannot undo or undelete changes once they have been saved. Verify that you have not made any accidental or undesired changes before proceeding..

The save operation performs a partial validation on the unsaved changes before applying them to the working copy of the Onyx Enterprise Dictionary (OED). Unsaved data is indicated in the navigation area by an asterisk (*).

To save your changes:

  • Click to save the changes to the working copy of the OED.

    If the save process succeeds with no errors and no additional data to report, a success note appears in the browser status bar. However, if any problems occur during the integrity check, explanations of those problems display in a summary window. Pay close attention to any errors listed. If any problems are found, resolve them now before proceeding. If you do not correct the errors, the Object Designer may automatically mark the object as invalid. Invalid objects are not published to the OED.

    If no critical errors are encountered, your changes are saved to the working copy of the OED. When you are finished modifying and saving changes, validate the OED.

Note: Onyx recommends that you save your work on a frequent basis to avoid data loss due to browser interruptions or other system failures.
