retrieveListByFilter (emailMessage)

retrieveListByFilter method

The retrieveListByFilter method of the emailMessage object returns information about emailMessage instances based on the input parameters.

retrieveList standard method information

Parameters

Parameter Direction Data type Content type Description
associatedWithType in i4   The type of the objects associated with emailMessage instances
associatedWithIdList in string   A list of primary keys of the objects that are associated with emailMessage instances
priority in i2   A single priority to match against
followUp in boolean   A single value to match against followUp
sent in boolean   A single value to match against sent
secondaryId in string   A single value to match against secondaryId
threadId in i4   A single value to match against threadId
subjectText in string   A single value to perform a like search against subject text
fromEmailAddressList in string   A list of values to perform in searches against the "From:" emailContact
recipientEmailAddressList in string   A list of values to perform in searches against the "To:" or "CC:" emailContacts
processedDateStart in dateTime   The start of the range for processedDate (NULL means no start)
processedDateEnd in dateTime   The end of the range for processedDate (NULL means no end)
rowSet out object all(out) The email message information returned by the method

Steps

Step data Step type Description
OnyxDataValidationStep.DataValidation com Validates input parameters
OnyxDM.DataManager com Retrieves a list of emailMessages with a task management filter

Remarks

At minimum, one of the following input parameters must be present:

  • associatedWithIdList
  • fromEmailAddressList
  • recipientEmailAddress
  • followUp

The results of method are sorted by the processedDate column, descending from the most recent date.

An email message that is marked private is not included in the result set unless the current user is the person who set the record to private.

When an email message has a sender or recipients whose emailContact instances are marked private, their names or email addresses will not appear in the from or toList columns unless the current user is the person who set the records to private.
