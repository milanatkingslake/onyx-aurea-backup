stepAdditionalData element

The OTM passes the stepAdditionalData element to the appropriate step component outside of the step package.

Step components use the stepAdditionalData element as a means to know which method to call internally, within the step component. The stepAdditionalData parameter is a string of name/value pairs, separated by semi-colons, that is defined in the Onyx Enterprise Dictionary and passed straight through the OTM to the appropriate step component.

To view the stepAdditionalData for Onyx step components, use the Object Designer to view any of the business objects' step attributes.

The following name/value pairs are commonly used by Onyx step components:

Pair Name Description

stepName

Contains the user friendly name of a step in the logical method.

stepObject

Contains the name of the object containing the step.

stepMethod

Contains the name of the method containing the step.

dataPath

Contains the dataPath value required only for ODM steps on saveCollection or retrieveCollection methods, and it should be set to the name of the collection parameter on the method (eg. dataPath=addresses; for address.saveCollection).

dataPathPrefix

Used only by steps whose type = "method," the dataPathPrefix parameter is pre-pended to the dataPath of the referenced method when the method's steps are merged into the steps for the logical method.

You can use stepAdditionalData in OEAS to send step-specific design time information to step components. The IStep Execute method includes stepAdditionalData as a parameter.
