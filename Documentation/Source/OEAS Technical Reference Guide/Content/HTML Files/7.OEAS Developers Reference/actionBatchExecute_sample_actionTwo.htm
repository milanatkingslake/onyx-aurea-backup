(ActionBatchExecute) process second action
Process second action: incident.insert

The second action creates an incident for the company created in the first action, company.insert. There is no sourceListMap for this action. For each actionReturnMap the value pointed to by the sourceElementPath value is applied to the targetElementPath. In this case company.primaryId returned from the company.insert action, is mapped to the incident.ownerId to be used in incident.insert.

<actionReturnMaps>

   <actionReturnMap>

      <sourceActionId>a0</sourceActionId>

      <sourceElementPath>parametersReturn/company/primaryId</sourceElementPath>

      <targetElementPath>parameters/incident/ownerId</targetElementPath>

   </actionReturnMap>

</actionReturnMaps>

Using the updated methodInputs parameters, the method call is made on the object that logicalObjectName and logicalMethodName point to. In this case an incident.insert call is made to insert a new incident using the ID of the newly inserted company from the first action. Error-handling behavior is determined by the value of onFailure.

XML for insert.incident action

<action>

   <onFailure>stop</onFailure>

   <id>a</id>

   <logicalObjectName>incident</logicalObjectName>

   <logicalMethodName>insert</logicalMethodName>

   <methodInputs>

      <parameters>

         <incident objectType="incident">

            <ownerId null="1"/>

            <ownerType>2</ownerType>

            <categoryId>1</categoryId>

            <typeId>1</typeId>

            <assignedTo>sa</assignedTo>

         </incident>

      </parameters>

      <customData/>

   </methodInputs>

   <actionReturnMaps>

      <actionReturnMap>

         <sourceActionId>a0</sourceActionId>

         <sourceElementPath>parametersReturn/company/primaryId</sourceElementPath>

         <targetElementPath>parameters/incident/ownerId</targetElementPath>

      </actionReturnMap>

   </actionReturnMaps>

</action>
