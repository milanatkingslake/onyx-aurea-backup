cascade (subscriptionItem)

cascade method

The cascade method of the subscriptionItem object applies subscription rules and filters that create or modify subscriptionItem object instances that are related to records to which the specified user is subscribed to or unsubscribing from. This method also processes records related to object instances that are deleted.

Remarks

The cascade method is available to the Sync Service or other custom services that maintain subscription transactions. Cascade uses the following information to determine how to process objects that are related to objects that users subscribe to and un-subscribe from explicitly:

  • object relationships
  • object relationship subscription rules
  • role subscription rules
  • role WHERE statement filters

The cascade method does three primary processing tasks in this order:

  1. Deletion cascade - process subscriptionItems that are marked for deletion by a change capture trigger (syncAction 7) .
  2. Unsubscribe cascade - process subscriptionItems that are marked for unsubscribe (syncAction 4).
  3. Subscribe cascade - process subscriptionItems that have been subscribed to using the subscribe method (syncAction 3) or that need recascading because of a change capture trigger (syncAction 5).

When cascade subscribes related transaction object instances, it flags them as refresh (syncAction 1). When it unsubscribes related instances, it flags them as remove (syncAction 2).

When a user changes a transactional object instance that is subscribed to by anyone (including the user making the change), a change capture trigger sets all corresponding subscriptionItem instances to refresh (syncAction 1). When a user changes a foreign key property of a record (such as contactId, ownerId, and so on), the trigger sets all corresponding subscriptionItem instances to recascade (syncAction 5).

If a user inserts a new child or parent to a transactional object instance that is subscribed to by anyone, the trigger sets the child or parent subscriptionItem instances for all users to recascade (syncAction 5). This causes a subscription of the newly inserted record for all other users who are subscribed to related records.

When the cascade method succeeds, it returns an empty parametersReturn node.
