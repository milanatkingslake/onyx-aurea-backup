Testing fax templates

Testing fax templates consists of the following:

  • Making sure the fax successfully sends
  • Making sure the fax content appears as expected

If the fax fails to send properly, examine the default SMTP header stylesheet configured for the fax subsystem in question. You can identify the default SMTP header stylesheet by looking at the fax subsytem configuration in OPS Administration. Onyx provides two sample default SMTP header stylesheets. The default SMTP header stylesheet must conform to the fax software to which OPS sends the SMTP message. To determine the correct format for the SMTP headers, consult the fax software documentation.

If the fax content appears malformed, examine the XSL stylesheet associated with the literature product in the orderBatch XML. Onyx provides a sample orderBatch output along with the sample fax templates.  

Your tools might not enable you to step through XSL code line by line at runtime, if so testing XSL requires an iterative trial and error approach. This involves using a parser to transform some XML with an XSL stylesheet. You view the parser output to determine whether your transform worked as expected. If not, you edit the stylesheet and run it again. You can use any number of free or proprietary tools generally available.
