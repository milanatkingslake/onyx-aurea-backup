ISession interface

Before the Onyx Transaction Manager (OTM) processes a service request, it instantiates the ISession interface on the SessionManager object and calls the GetContextXML and HasPermission methods. The permissions mask is retrieved with GetContextXML and the session user's access to the requested resource is verified with HasPermission.

A client can also instantiate ISession and call its methods to create and retrieve session variables. All the calls to ISession methods require the strSessionID and strAppName parameters. The strSessionID parameter identifies a session, and strAppName identifies a particular datasource.

Note: Onyx recommends that you use the ISession2 interface, because this newer interface offers more session control.

The Session Manager component exposes the ISession interface. Its methods are listed below:

Method

Description

ClearSessionItems

Clears the values of all session variables

EndSession

Ends a session

GetContextXML

Retrieves session data

GetSessionItem

Retrieves the value of session variable

GetSessionItems

Retrieves all session variables and their values

HasPermission

Determines whether a user has permission to use a resource

HasPermissions

Determines whether a user has permission to use several specified resources

RemoveSessionItem

Removes the value of a session variable

SetImpersonation

Enables a system service or component identity user to impersonate end-users.

SetSessionItem

Saves the value of a session variable
