assignmentExpression object

The assignmentExpression object contains the elements necessary to define an expression that can be used to determine whether or not an OEAS object should be associated with a territory. One or more assignmentExpressions can be added to an assignmentCondition object.

The assignmentExpression object has one key property, primaryId, that uniquely identifies object instances in the database. Once the object is created, the value of this property cannot be changed.

Methods

Method Description
retrieveCollection Returns a collection of objects based on the search criteria supplied in the parameters XML
retrieveListByCompany Returns a list of assignmentExpression records that match the company data
retrieveListByIncident Returns a list of assignmentExpression records that match the incident data
retrieveListByIndividual Returns a list of assignmentExpression records that match the individual data
saveCollection Performs multiple actions at once: inserting, updating, and deleting objects, as appropriate

Properties

Property Description
conditionId The unique identifier of the parent assignmentCondition for this object
insertBy The user ID of the Onyx user who initially saves the object
insertDate The time and date on which the object is initially saved
operator The operator used to calculate the expression
primaryId The unique identifier of the assignmentExpression object
propertyName The name of the object property to be evaluated in the expression
propertyValue1 The first value to be evaluated in the expression
propertyValue2 The second value to be evaluated in the expression
updateBy The user ID of the Onyx user who last saved modifications to the object
updateDate The time and date on which modifications to the object were last saved
