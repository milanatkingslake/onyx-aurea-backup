What to pass to the OTM

When calling the Execute method of the OTM interface (ITransactionManager), you must provide the following parameters:

Parameter Name Required/Optional Data Type Description

applicationName

Required

[in] BSTR

A string that identifies the application to use.

This parameter becomes part of the context element of the step package.

logicalObjectName

Required

[in] BSTR

A string that identifies the business object that contains the method to be called.

The OTM passes this as the LogicalObjectName parameter to the IStep Execute method.

logicalMethodName

Required

[in] BSTR

A string that identifies the business object method to be called.

The OTM passes this as the LogicalMethodName parameter to the IStep Execute method.

sessionID

Required

[in] BSTR

A GUID that identifies the session in use by the client application. This ID is given to the client by an authentication component.

This parameter becomes part of the context element of the step package.

stepAdditionalData

Required

[in] BSTR

A string that contains any special non-XML data intended for the step component.

The contents of this parameter are stored in the OED, and can be modified using the Object Designer.

The OTM passes this as the StepAdditionalData parameter to the IStep Execute method.

paramXML

Optional

[in] BSTR

A well-formed XML fragment that contains the parameters passed to the method.

The OTM includes this parameter in the returnXml node of the step package.

customDataXML

Optional

[in] BSTR

A well-formed XML fragment that contains any special data required by custom COM step components.

The OTM includes this parameter in the returnXml node of the step package.

returnXML

Optional

[out, retval] BSTR

A well-formed XML fragment that contains the XML data returned by the method.

The OTM includes this parameter as an element of the step package.

The OTM processes this data and creates what we call a step package. The step package consists of the following XML elements:

Element Required/Optional Description

context

Required

An XML string that contains user information such as the user ID, preferred language code, and site ID.

parameters

Required

An XML string that contains the input parameters for the specified object and method.

The object/method pair is specified in the stepAdditionalData element.

methodStatus

Required

An XML string that contains the status (success or failure) of the method call.

If the status is a failure, the associationId attribute associates the error with its corresponding message ID.

returnXml

Required

An XML string that contains the output parameters (also known as returnXML).

customData

Required

An XML string that a custom step component can pass to OEAS. OEAS does not process any data contained within this element, preserving its integrity for your customization needs.

The stepAdditionalData element does not become part of the step package. The OTM passes this element directly from the OED to the appropriate step component, but not as part of the step package.
