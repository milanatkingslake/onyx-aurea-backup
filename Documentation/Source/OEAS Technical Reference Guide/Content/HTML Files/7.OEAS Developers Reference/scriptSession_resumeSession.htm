resumeSession (scriptSession)

resumeSession method

The resumeSession method of the scriptSession object retrieves and restarts a previously completed or suspended script session.

Call script engine to resume a script session.

Remarks

Call this method to retrieve a previously saved script session and determine the appropriate place in the script to begin editing or viewing the session. The resumeSession method restarts the session based on the statusId of the scriptSession and completed property of the last scriptSessionStep. If there are no scriptSessionStep objects for the scriptSession, the engine uses the initialStepId from the run-time XML.

For resumeSession, the engine performs the following actions:

  1. retrieves limited transactional and domain data for the specified session
  2. determines which scriptSessionStep or scriptStep to load
  3. calls the loadStep method

Example

This example uses a script which does the following:

  1. Uses a script that prompts the user to search the database for the ID of a caller.
  2. Asks if the user wants to continue or end the script.
  3. If the user chooses to continue, the script asks them if they want to enter an incident for the caller. The user can choose to enter an incident or end the script.
  4. If the user chooses to enter an incident, the script displays the new incident ID.
  5. In the final step, displays the caller's ID and the new incident ID.

The following example XML shows a resumeSession call used to restart a script session that the user suspended earlier. The caller passes in the scriptSessionId and the run-time XML. The run-time XML contains only the context variables.
