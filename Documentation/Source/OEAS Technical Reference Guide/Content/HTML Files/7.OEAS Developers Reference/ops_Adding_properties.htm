Adding properties

Before adding properties, see creating routingPrinter domain object. Typically, you add a property to the logical business object for each column in the corresponding database table with the exception of the site_id column. The properties enable you to map middle tier parameters to specific columns in the database and take advantage of standard SQL generation behaviors in OEAS. Adding an onyxTimestamp property is optional especially for domain data, where the data changes infrequently and collisions are unlikely.
