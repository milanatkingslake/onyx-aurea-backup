statusId (incident)

statusId property

The statusId property of the incident object identifies a predefined milestone of the incident.

Data type

i4

Access

Read/write

Remarks

Milestones for a sales incident can include qualification, needs assessment, and proposal.

The statusId property references an existing status type in the database. Incident categories own status types, therefore the value in the statusId property must reference a child value of the incidentCategory for a given incident.

For example, if the incidentCategory is Sales (categoryId), your status types (statusId) might be any of the following: Open, Closed, Pending. These are default status types. You can add your own using the Reference Table Administration tool.
