Business object attributes

When you select a business object in the Object Designer, the attributes for that object display in the edit area.

The attributes are defined as follows:

Attribute Name

Description

ID

The globally unique identifier (GUID) assigned by the Onyx Enterprise Dictionary. This attribute is read only.

Name

Type the name you want to assign to this object. Object names must be conform to XML naming conventions.

If you change the name of an existing object or method, use the Security Administration tool to establish the access privileges as if the object/method were new. Any security settings previously established for the renamed object/method are not maintained.

Description

Type the description for this object.

Object Type

An integer value automatically assigned by the OED that identifies the type of business object. This attribute is read-only.

Installed By

Type the name of the vendor or application installing this object.

Valid

True if the object is valid. False if the object is invalid. This attribute is read-only and automatically assigned by the OED.

Onyx Version

The Onyx version number of this object. This attribute is read-only.

Custom Version

Type the custom version number you want to assign to this object.

Domain Object

Check this box to indicate this is a domain object. A domain object persists domain data and can only participate in a domain relationship. Domain objects drive look-up lists or drop-down lists on transactional objects.

Apply Partner Security

Check this box to indicate that the object is accessible through Onyx Partner Portal.

Released

This attribute is for documentation purposes only. Check this box to document that the object is no longer under development.

Disabled

Check this box to disable this object. Clear the box to enable the object.
