Property Constraint attributes

When you select a property constraint in the Object Designer, the attributes for that constraint display in the edit area.

The attributes are defined as follows:

Attribute Name

Description

ID

The globally unique identifier (GUID) assigned by the Onyx Enterprise Dictionary. This attribute is read-only.

Type

Select the constraint type to apply to this property:

  • For a default value to be used, select default. This value will be applied to the property when the object is inserted into the database.
  • Default Null: Check this box to indicate the default value is NULL.
  • Value: Type the default value using proper database syntax. This value will only be used for insert methods.

    NOTE: This field will be disable if Default Null is checked.

  • For a minimum inclusive value, select min. This can be used for range checking on numeric and date data types or for the minimum length of strings.
  • Value: Type the minimum value for the property using proper database syntax.
  • For a maximum inclusive value, select max. This can be used for range checking on numeric and date data types or for a maximum length of strings.
  • Value: Type the maximum value for the property using proper database syntax.
  • For a property that must be present in the XML with a value for insert actions, select required. For example, <assignedTo>JohnDoe</assignedTo>. The required constraint is not enforced on update actions.
  • Value: Check this box to indicate that a value is required for this property.

Installed By

Type the name of the vendor or application adding this property constraint.
