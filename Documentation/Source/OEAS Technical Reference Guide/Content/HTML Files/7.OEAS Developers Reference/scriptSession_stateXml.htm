stateXml (scriptSession)

stateXml property

The stateXml property of the scriptSession object contains the variable names and values from script context information and user responses.

Data type

string

Access

Read/write

Remarks

The script engine methods persist the stateXml property based on information in the variables collection of the run-time XML. The engine does not expect anything in this property in the run-time XML, and neither does it return anything in this property. It composes this property from the variables and persists it to the database. When the scriptSession object is retrieved, the engine replaces variable values in the stateXml with any values sent in by the calling application. The engine methods return the stateXml data in the variables collection of the run-time XML and remove the stateXml node from the scriptSession. The reason for this is to avoid duplication and unnecessary network traffic.

The value of this property is an XML document and must be escaped in order to make a scriptSession method call. For more information on escaping XML for the OEAS platform, see the escaping XML topic.
