primaryId

primaryId property

The primaryId property of an object is the unique ID assigned to the object when it is first saved to the database.

This is a generic topic that applies to all properties with the same name. The remarks section may contain details specific to a given object.

Data type

string or i4 (For more information, see the primaryId table at the end of this topic.)

Access

Read-only; for more information, see Read Only attribute.

The value of this property is established by OEAS upon creation of the object. This value cannot be modified.

Remarks

OEAS or the Onyx Enterprise Database generates a unique primaryId value for each object.

The primaryId of domain objects always contains a numeric data type (as shown in the primaryId table). The primaryId of transactional objects always contains a string data type, which is a GUID value.

For creating primaryId values, the OEAS uses the following convention. For columns whose name ends in ID, the OEAS generates a GUID. For columns whose name ends in DID, the database generates a sequential integer value. The database uses the same sequence for all DID columns. For columns whose name ends in CODE, the user enters the value. The OEAS determines how to generate ID values based on the generateId element for each specific property. Valid values for the generateId element include the following.

  • odmGuid--ODM generates a GUID value
  • db--the databse generates a sequential integer
  • empty--the user supplies the value
