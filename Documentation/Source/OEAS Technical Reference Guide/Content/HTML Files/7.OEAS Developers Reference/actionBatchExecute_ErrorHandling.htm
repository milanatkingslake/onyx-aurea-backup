(ActionBatchExecute) Error handling

Error handling

All internal failures and failures due to missing required nodes in the actionBatchXml are flagged as fatal errors and processing is stopped. When OEAS method calls return a failure, the onFailure attribute on the action node in the actionBatchXml determines the flow of control.

If there are no sourceList items but there are sourceListMaps, an error occurs and processing is stopped. If one action fails due to not having permission to execute the call, processing is stopped.

onFailure handling

The actionBatchXml onFailure element on the action node determines how the rest of the actions are processed in the event of a failure in the OEAS method call made for the action. The possible values for onFailure are:

Value Behavior

stop

Stop processing actions for this and all source list items and abort immediately.

continueNextAction

Continue processing with the rest of the actions for the same source list item.

continueNextSourceItem

Stop processing actions for this source list item. Proceed to the next source list item and process its actions.

A specific error is returned if any of the OEAS method calls fail, indicating that one or more actions might not have been executed because of the failure.
