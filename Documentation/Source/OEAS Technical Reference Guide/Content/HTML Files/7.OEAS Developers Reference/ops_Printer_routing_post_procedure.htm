Printer routing

You can implement a printer routing feature using a post procedure by modifying and extending the physical and logical models in the OED. This sample demonstrates how to create a new database table and stored procedure, add a new logical business object, and modify an existing business object to route literature order printable documents to specific printers.

Scenario

For this sample, assume you have a number of literature order items for printable documents that are configured to be printed on a local printer. However, you would rather have these products printed on a printer in the same country of the customer for whom this order is placed. The logic for determining which printer is closest to the customer, and overriding the default printer, is implemented in a post procedure added to the literatureOrderLine saveCollection method. The steps to implement this logic include the following.

In the database...

  • Add a ct_custom_routing_printer_table
  • Add a cp_onyx_custom_routing_printer_stored_procedure

In the OED...

  • Add the new table to the physical model
  • Add the new procedure to the physical model
  • Add a new routingPrinter logical business object
  • Modify the literatureOrderLine saveCollection method

In the OES...

  • Add a new tableAdministrationItem in Table administration
  • Add routingPrinter objects to the Routing Printer table in Table administration

In OEP...

Note: These steps are to test the implementation.
  • Add two new customers with different country codes in their addresses
  • Add a literature order to each customer
  • Create the table and procedure
  • Modify the OED
  • Install the customization
  • Configure and populate the table
  • Test the customization
