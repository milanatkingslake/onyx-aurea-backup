typeId (task)

typeId property

The typeId property of the task object contains a predefined code that identifies the task sub-type.

Data type

i4

Access

Read/write

Remarks

The typeId property references an existing task type in the database. Because task categories own task types, the value in the typeId property must reference a child value of the categoryId for a given task. For more information on configuring task types, consult the Reference Table Administration tool and the Onyx Tools and AdministrationGuide.

For example, if the categoryId is Sales, you can have task types (typeId) General, Product Inquiry, Pricing, Small Account, Medium Account, and Large Account. Keep in mind, though, that these are default task types, and you that can add your own through Reference Table Administration.
