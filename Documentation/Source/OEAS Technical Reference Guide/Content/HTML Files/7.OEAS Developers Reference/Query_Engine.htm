Query Engine

The Query Engine is the component responsible for moving data to and from the Onyx Enterprise Database. ODM uses the Query Engine interface, IEBusQE, to execute ad hoc SQL code or to call stored procedures. The Query Engine is intended for use only by step components in OEAS. It does not have a scriptable interface, and components that do not reside on the OEAS server cannot call it.

The Query Engine is designed for speed and efficiency. It does not use XML, nor does it validate the syntax of the SQL code that it receives. It passes the strings directly to the Onyx Enterprise Database. Any errors that result from an attempt to process the SQL code are returned to the calling component.

Each method in IEBusQE includes an argument for an application name. The Query Engine uses this application name to obtain data source connection information from application server registry settings. This level of abstraction protects components from changes to database settings and supports multiple applications, each having their own database server.

The Query Engine is designed for use only with the database included with this release of the OEAS. Using the Query Engine with different applications to access other database schemas is not supported and may result in unexpected errors.
