actionBatch_statusXml_schema

statusXml schema grid

After performing all the actions in the actions list, OnyxActionBatchExecute collects all the parameters and returnXml for all executed methods, regardless of success or failure and returns it in the statusXml out parameter. The following is a grid representation of the statusXml structure.

View the grid legend

Element

This column contains the local name of each XML node in the statusXml. Indentation signifies hierarchy in the XML structure.

Cardinality

The value in this column indicates how many of the given nodes are expected in the actionBatchXml. One (1) indicates the node is required. One-to-many (1..*) indicates at least one occurrence of the node is required. Zero-to-one (0..1) and zero-to-many (0..*) indicate optional nodes.

Comments

This column contains notable information about the node. For a general description of the node, see the detailed topic by clicking on the local name in the first column.
