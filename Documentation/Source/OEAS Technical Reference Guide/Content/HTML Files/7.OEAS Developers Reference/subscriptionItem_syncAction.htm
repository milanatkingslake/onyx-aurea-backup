syncAction (subscriptionItem)

syncAction property

The syncAction property of the subscriptionItem object contains a code that the sync server uses to track subscribed records.

Data type

ui1

Access

Read/write

Remarks

The following table describes the syncAction codes.

Value

Name

Description

0

Processed

A user subscribed to the object instance, and the staging server has the current version of the instance. It does not need to be synchronized or cascaded.

1

Refresh

The transactional object instance is subscribed, but it needs to be downsynced or resynchronized to the staging server because the instance on the staging server is either out-of-date or the instance has never been downsynced.

2

Remove

A user unsubscribed the transactional object instance. The sync server deletes the instance from the Staging server and then logically deletes the corresponding subscriptionItem object instance. The transactional record is not deleted from the OEDB.

3

Subscribe

A user has subscribed to the transactional object instance by calling the subscribe method. The sync server needs to process the newly subscribed instance by calling the cascade method. After cascade finishes and the records related to the originally subscribed instance are subscribed, cascade sets the syncAction value of all the affected subscriptionItem instances to 1.

4

Unsubscribe

A user unsubscribed the transactional object instance. The sync server needs to call cascade to unsubscribe child instances. The cascade method sets the syncAction of the affected subscriptionItem instances to 2.

5

Recascade

An insert change capture trigger flagged the transactional object instance because a related record has been created that may need to be subscribed to. The difference between syncAction 3 and syncAction 5 is that a recascade does not resubscribe records that were explicitly unsubscribed by an end user. After the cascade method processes a syncAction 5, it sets syncAction to 1.

6

Delete

A user deleted a transactional record or it is a child of a record that has been deleted.

7

DeleteCascade

A delete change capture trigger has flagged the transactional object instance. The transactional record has been deleted from the OEDB, and any child records need to be marked for deletion (syncAction 6).
