Creating pre, post, and sub procedures

Pre and post procedures refer to procedures structured for use with SQL generation. Pre and post procedures are mapped in the Object Designer as procedure maps within a SQL Generation method step.

Sub procedures are called by other stored procedures. For instance, sub procedure may be called by a retrieveList procedure to handle a common string manipulation task.

Pre, post, and sub procedures may need to return national language information or database error information. Onyx recommends that pre, post, and sub procedures return status information using these OUTPUT parameters and not by creating their own result sets. The following three OUTPUT parameters should be used in the procedure when sending information back to the calling procedure:

Parameter Name

Datatype

Default Value

Usage

@message_code

INT

N/A

This variable holds any national language message ids used for returning errors, informational messages, or for retrieveList grid header pointers when a retrieveList may return  result sets whose column headers differ depending on the data returned.

@message_parameter

NVARCHAR(255)

N/A

This variable may be used to return a string that can be inserted into a national language string identified by the @message_code variable. The string in @message_parameter will replace the "~2" placeholder in the national language string.

@error

INT

N/A

This variable is used to return database errors that may be read from the @@error session variable.

A @return_code variable should also be created in the pre, post, or sub procedure. This variable should be used to return the procedure's status using the same rules as replace procedures.
