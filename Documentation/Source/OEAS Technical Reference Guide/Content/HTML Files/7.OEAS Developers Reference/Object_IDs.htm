Object IDs

Knowing the object ID associated with an object is often required when working with OEAS objects, and can help you when generating reports and customizing OEAS and the OEDB.

A number of OEAS objects require that you pass its owner ID before you can insert, retrieve, or update the object. Examples include address, fileLink, incident, reminder, task, and all other objects that have an ownerType property.

When customizing OEAS or OEDB, you can use the ownerType property to designate one object as an owner of another object. To do this, you must know the object ID of the object that you want to designate as the owner.

Instances of OEAS business objects that can, by default, be owned by one or more business objects have an ownerType property. The value of the ownerType property identifies the type of object that owns the object instance. For example, an instance of the internalContact object can be owned by one of several business objects. If you want the internalContact object to be owned by an incident object, you must know that the object ID of the incident object is 6, and assign 6 as the value of the internalContact instance's ownerType property.
