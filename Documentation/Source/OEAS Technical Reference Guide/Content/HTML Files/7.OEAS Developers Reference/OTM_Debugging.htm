Debugging

Finding errors in custom components can be a challenge within a COM+ application system. There are a number of different ways to debug a custom component before deploying it in a production system. Each is explained below.

Run the component in isolation

Before installing the component into a COM+ application, test it as a standalone ActiveX DLL. It may be possible to step through the code of a step component while in a debug session using the Visual Studio IDE. For more information on this, consult Microsoft documentation (http://msdn.microsoft.com).

Change the Onyx Log settings

Using the OEAS Configuration Utility, update the settings for the Onyx Log in the Event Viewer to track every action by the OEAS components.

Run in-process

Step components are intended to run within the same process as their caller (the OnyxEAS COM+ application). Running out-of-process can cause unexpected results.
