Undefined translation message

To support multiple language domain data, the language table includes the column undefined_translation_message. This column provides a generic error message to the end user for values missing a description for that user's preferred language. For instance, suppose that a user's preferred language is French, but the database that they are using does not have a French translation for the description corresponding to the country code "AL." Rather than the user interface showing no value for the description when the user chooses the "AL" country code, it shows "La traduction française n'est pas disponible."

The equivalent message is entered for each language defined in the language table.

Example: undefined_translation_message

The following example shows how the undefined_translation_message would be defined in the language table for the language_code values of "ENG" (English) and "FRA" (French).

language_code

undefined_translation_message

ENG

English translation not available

FRA

La traduction française n'est pas disponible

Note: The message string of the undefined_translation_message column is not a multi-language string itself, since it only needs to be defined once for each language.
