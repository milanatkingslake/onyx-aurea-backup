methodStatus element

The methodStatus element contains information about the current condition of the method. Step components can add information to this element, as needed, to influence the behavior of the remaining steps and to provide information to the calling client. In most cases, steps that complete successfully leave this element untouched.

When the method completes normally, methodStatus contains a single element, statusType, with a value of 'success'.

When an error occurs, or if a step needs to return extended information, methodStatus provides a place for step components to assemble detailed messages for interpretation by the client application. Three elements can appear within the methodStatus element: statusType, message, and error.

Example

The contents of a sample methodStatus element are as follows:

<methodStatus>

    <statusType>failure</statusType>

    <error associationId="65220E99-192E-4906-8179-98CFB3775BD8">

      <severity>failure</severity>

      <errorNumber>9538</errorNumber>

      <errorType>oetBusinessLogic</errorType>

      <errorDescription>Company not found</errorDescription>

      <sourceType>ostMisc</sourceType>

      <sourceName>OnyxCustomerDAC:RetrieveCompany</sourceName>

      <natLangId>709026</natLangId>

      <timestamp>2000-11-15 20:43:51</timestamp>

    </error>

</methodStatus>
