Validation and processing

When customizing OEAS, remember that your custom XML must adhere to the OEAS validation and processing requirements.

Validation

Onyx uses the XML-Schema form of validation only for the Onyx Enterprise Dictionary. The XML in the OEAS middle tier API is not validated against a schema or a DTD, but must be well-formed and comply with the W3C (www.w3.org) XML 1.0 standards.

Processing

All XML in OEAS, including custom XML, is processed by the Microsoft XML Parser component. This component supports the W3C (www.w3.org) XML 1.0 XML DOM and namespaces recommendations.
