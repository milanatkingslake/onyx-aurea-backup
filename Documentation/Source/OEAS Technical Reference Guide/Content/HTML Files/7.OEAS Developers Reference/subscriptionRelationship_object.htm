subscriptionRelationship object

The subscriptionRelationship object provides subscription rules for a specific relationship between two subscribable objects.

Warning: Any modifications to the subscriptionRelationship object instances that are defined for the default installation of OEP-Offline will cause unpredictable behavior in the OEP-Offline subscription service. Accommodating such changes requires significant development of the opSubscriptionItemCascade and opSubscriptionItemCascadeSelectSql stored procedures. The best way to alter subscription rules is to create role-based rules via the subscriptionRelationshipRole object.

Under no circumstance should the definition of the subscriptionRelationship object be modified.

Methods

Method

Description

retrieveCollection

Retrieves a collection of subscriptionRelationships

retrieveListByUser

Retrieves subscription rules for relationship between two subscribable objects

saveCollection

Saves a collection of subscriptionRelationship object instances

Properties

Property

Description

active

Indicates whether the object relationship is available for use by the subscription service

defaultCascadeToPrimary

Indicates whether the primary object of the relationship is automatically subscribed and cascaded when the secondary object of the relationship is subscribed.

defaultCascadeToSecondary

Indicates whether the secondary object of the relationship is automatically subscribed and cascaded when the primary object of the relationship is subscribed.

defaultFilterExpression

Contains an expression used in a WHERE clause that limits the automatically subscribed secondary object instances.

description

Contains text that describes the subscriptionRelationship object instance.

insertBy

Contains the user ID of the Onyx user who initially saves the object.

insertDate

Contains the time and date on which the object is initially saved.

name

Contains a name for the relationship.

oedRelationshipMapId

Contains the GUID that identifies the relationship in the oed_relationship_map table.

primaryId

Contains the unique ID assigned to the object when it is first saved to the database.

primaryObjectRequired

Indicates whether the primary object in the relationship is automatically subscribed when the secondary object is subscribed.

primaryObjectType

Contains the object type enumeration of the primary object in the relationship.

secondaryObjectType

Contains the object type enumeration of the secondary object in the relationship.

subscriptionServiceId

Contains the identifier of the subscription service that uses the relationship.

updateBy

Contains the user ID of the Onyx user who last saved modifications to the object.

updateDate

Contains the time and date on which modifications to the object were last saved.
