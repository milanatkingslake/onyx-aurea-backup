GetSteps (ICache)

GetSteps method

The GetSteps method of the ICache interface retrieves the list of steps that are associated with a particular object and method. Note that this method is optimized for use by the OTM.

Syntax

IDL

HRESULT GetSteps(

    [in] BSTR ApplicationName, 

    [in] BSTR ObjectName, 

    [in] BSTR MethodName, 

    [out] SAFEARRAY(BSTR)* Steps, 

    [out, retval] OEDCacheStatusEnum* Status);

IDL parameter descriptions

ApplicationName

[in] A string containing the logical application name that determines which subset of the cache to use for a query.

ObjectName

[in] A string that contains the name of the specified object.

MethodName

[in] A string that contains the name of the method whose steps are to be retrieved.

Steps

[out] A two-dimensional array of strings (SAFEARRAY) that contains the list of step information associated with ObjectName and MethodName.

Status

[out, retval] An enumeration of type OEDCacheStatusEnum that contains the status of the method call.

Return values

This method returns an enumeration of type OEDCacheStatusEnum that contains the status of the method call. See the OEDCacheStatusEnum topic for a description of individual enumeration values.

Remarks

The code you write to use the GetSteps method is similar to the code for the getPropertyConstraints method. See the getPropertyConstraints method Example section for details.

Note: For an invalid object name or method name, the error message is "Method not found" because of how items are keyed in the cache.
