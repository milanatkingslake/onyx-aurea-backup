Structural Transformation of Objects
Structural transformation of objects

The structural transformation of an object definition to an object instance is described in general terms below. An example is also available below this general outline that shows the transformation of an object definition to an object instance. Each general point below can be viewed in the example. Each bullet corresponds to a row in the example and is labeled accordingly.

In general terms, the transformation is as follows:

  • Parameter names and property names become element tags.

This is shown in all rows of the example below.

  • For simple property types:
  • The property value is encoded into a string and contained in the element text.

This is shown in all rows of the example below.

  • For collection property types:
  • A collection attribute with value of "1" (true) is added to the element.
  • Child elements are created for each instance of the property within the collection. The child elements have a tag name equal to the dataType value. However, this tag is merely a placeholder that is used because these nodes are iterated as a collection.
  • The value for each child element is encoded as specified for simple property types.

This is shown in row D of the example below.

  • For properties of type "object", collection="0" (aggregation relationship 1-1):
  • The objectType is encoded as an attribute of the element.
  • The element contains the properties of that object type.

This is shown in row E of the example below.

  • For properties of type "object", collection="1" (aggregation relationships 0-*, 1-*):
  • The objectType is encoded as an attribute of the element.
  • A collection attribute with value of "1" (true) is added to the element.
  • Child elements are created for each instance of the object within the collection. The child elements have a tag equal to the objectType value.

This is shown in rows F and H of the example below.

  • For properties of type "object", collection="0" (aggregation relationship 0-1):
  • The objectType is encoded as an attribute of the element
  • If the object exists in this instance, the element contains child elements for each of the properties of that objectType.
  • If the object does not exist in this instance, the attribute null="1" is added to the element, and the child property elements are omitted.

This is shown in row G of the example below.
