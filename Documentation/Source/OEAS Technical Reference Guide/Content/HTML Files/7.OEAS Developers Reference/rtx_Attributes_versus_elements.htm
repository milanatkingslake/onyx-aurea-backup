Attributes versus elements

The XML standard provides two basic approaches to encoding data--attribute encoding and element encoding. The following table illustrates the two different approaches for encoding a scriptStep object. The first uses element encoding of the object properties, the second uses attribute encoding of the properties.

Element encoding

Attribute encoding

<scriptStep>

  <primaryId>2</primaryId>

  <name>Driving record</name>

  <description>Gathers information about a driving record</description>

  <headerText>Now let's discuss your driving record.</headerText>

  <footerText />

</scriptStep>

<scriptStep

  primaryId="2"

  name="Driving record"

  description="Gathers information about a driving record"

  headerText="Now let's discuss your driving record."

  footerText=""

/>

Element encoding is the standard for all object instances sent to or returned from OEAS. In order to reduce the size of the run-time XML, Onyx uses attribute encoding of properties for the domain data and script variable sections. Onyx uses element encoding for the transactional data part of the run-time XML, as this enables support for customization of the scripting transactional objects.

Unless otherwise noted, attributes shown in the run-time XML represent properties of the same name that exist on a scripting domain object. For instance, the name attribute on a scriptStep element in the run-time XML holds the data corresponding to the name property of the scriptStep object.
