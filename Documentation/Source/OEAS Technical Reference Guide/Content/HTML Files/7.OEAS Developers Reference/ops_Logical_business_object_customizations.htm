Logical business object customizations

The logical business object customizations for this scenario describe a logical object to represent the new domain data for printer routing and modify an existing object method to use the domain data. You need a logical object for the domain data, so you can use Table Administration to manage routing printer records. You want the system to match a literatureOrderLine with the nearest printer when the literatureOrder is created, so you must modify the saveCollection (insert action) method of the literatureOrderLine object.
