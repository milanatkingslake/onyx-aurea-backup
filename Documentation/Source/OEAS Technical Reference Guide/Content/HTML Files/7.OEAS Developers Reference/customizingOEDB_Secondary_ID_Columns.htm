Secondary ID columns

As noted in Primary Key Columns, transactional tables use a primary key that contains a GUID. GUID values are not particularly legible or user-friendly. It is often preferred to have an identifier for a transactional record that is easier to read and/or memorize, search on, etc. The Onyx transactional tables use a "secondary_id" column for this purpose. The standard secondary_id columns get their values from a pre procedure mapped for the object's insert method or insert action on a saveCollection method. The pre procedure will accept a "natural" value if one is entered by the user, otherwise it automatically generates a unique integer value. If you employ a secondary_id column with the standard mapping, the column should be typed as an integer.
