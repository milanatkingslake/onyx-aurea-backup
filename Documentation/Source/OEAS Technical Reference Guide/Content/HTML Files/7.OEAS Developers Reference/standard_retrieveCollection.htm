retrieveCollection method

A retrieveCollection method returns a collection of object instances based on search criteria.

Parameters

This method accepts a content="any" object as the input parameter and returns a content="all" object as the output parameter. The input XML is passed within the root element named <parameters>.

Parameter Direction Data type Content in Type Content out Type Description

Object name

inOut

object

any

all

On input, the parameter is a collection object consisting of one object.

On return, the collection object consists of objects that satisfy the search criteria.
Steps

All standard methods call the Onyx Data Validation Step for input data validation and either directly or indirectly call Onyx Data Manager to access the data tier. Many methods perform additional steps. For a table that lists the default steps of a specific business object method, refer to its method topic under the business object.

Remarks

The input object contains any combination of property values that is the search criteria for the retrieveCollection method. No search criteria is required, but if the caller does not include the collection object node, the method fails. The collection parameter name is the plural form of the object name for which the retrieveCollection method is defined. For example, the parameter name for a retrieveCollection on the address object is addresses. The items returned are those whose property values match all of the input properties exactly. An empty input node (object or property) matches all values. An input property node set to NULL, matches no values. If no objects meet the search criteria, this method returns an empty collection object. No error is returned in this case.

The following syntax shows a call to retrieveCollection with no search criteria by passing an empty object node in the retrieveCollection parameters.

<parameters>

   <countries>

      <country/>

   </countries>

</parameters>

When you pass no search criteria to a retrieveCollection, it returns all objects in the database of the given type. For performance reasons, you might not want to call retrieveCollection with empty input nodes. In the previous syntax example, retrieveCollection returns all country objects stored in OEDB.

The retrieveCollection method does not match partial strings. For example, if you had three systemParameter objects with the descriptions:

<description>Enables multiple tasks on Service incidents</description>

<description>Enables multiple tasks on Sales incidents</description>

<description>Enables multiple tasks on Support incidents</description>

And you used the search criteria:

<description>Enables multiple tasks</description>

The method would return an empty collection, unless you had an object with the exact string matching "Enables multiple tasks".

You can filter on most object properties with retrieveCollection methods. The few properties that are ignored on retrieveCollections do not have a filter map defined in the OED. You can look in the object physical maps in the object designer to determine which properties are filterable.

When an object has denormalized properties for which filter maps are defined, a retrieveCollection method on a large data set can result in poor performance. Onyx suggests that you use such retrieveCollection methods on small data sets only.

Most OEAS retrieveCollection methods use the embeddedXml feature to improve performance. In certain instances, you might want to disable the feature for a given method. For more information, see Using the embeddedXml feature.

You can limit the maximum rows returned by the retrieveCollection method using the pageSize attribute on the collection object. The following syntax illustrates use of the pageSize attribute. The pageSize attribute is highlighted in the example parameters. Notice in the returnXml that retrieveCollection only returned the first three country objects.
