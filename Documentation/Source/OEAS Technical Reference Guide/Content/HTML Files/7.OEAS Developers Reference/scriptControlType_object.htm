scriptControlType object

scriptControlType object

The scriptControlType object represents functionality associated with a prompt in the user interface such as a textbox, checkbox, or region list. For a complete list of scripted workflow control types and their descriptions, see the Prompts topic. For information on creating custom scriptControlType objects, see the adding a custom scriptControlType scenario.

Methods

Method

Description

retrieveCollection

Returns scriptControlType objects that match search criteria

saveCollection

Performs multiple actions at once: inserting, updating, and deleting objects, as appropriate

Properties

Property

Description

active

Specifies whether the domain data that the object provides is available to the user interface

allowEditableResponses

Specifies whether a response supports additional text in a textbox

allowExtraData

Specifies whether a control uses additional parameters when called

allowInitialValue

Specifies whether a control can be bound to a literal or variable input value

allowMinMax

Specifies whether a control supports minimum and maximum value constraints

allowSize

Specifies whether a control supports custom sizing

allowSort

Specifies whether a control supports sorting of a list or set of responses

allowText

Specifies whether a control supports a text component in its display

allowUserInput

Specifies whether a control allows user input or is read only

controlTypeCode

Identifies a type of control used to collect responses in the user interface

defaultExtraData

Contains a semi-colon delimited list of 'name=' tags

description

Contains a brief explanation of the control

insertBy

Contains the user ID of the Onyx user who initially saved the object

insertDate

Contains the time and date on which the object was initially saved

multiSelect

Specifies whether the control supports multiple response selections at run time

name

Contains a name suitable for display in a user interface

requiresDataType

Specifies whether an application must assign a data type to the control at design time

requiresResponseList

Specifies whether the control uses a list of responses from the run-time XML

updateBy

Contains the user ID of the Onyx user who last saved modifications to the object.

updateDate

Contains the time and date on which modifications to the object were last saved.
