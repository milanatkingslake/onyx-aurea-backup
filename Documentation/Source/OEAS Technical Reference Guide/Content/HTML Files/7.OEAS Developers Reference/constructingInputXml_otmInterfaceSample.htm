Constructing input XML

Before calling the Onyx TransactionManager (OTM), the client application needs to construct the input XML from the user's input. The composition of the input XML parameter depends on the object and method the client application calls. For this sample, we use the company insert method.

To determine the composition of the input XML parameter:

  1. Open the Object Designer. Substitute the name of the server running OEAS for <serverName> in the following URL, http://<serverName>/onyxenterprisestudio_onyx/ObjectDesigner/main.asp.
  2. Under Logical Business Objects, navigate to the object and method (company insert) you want to use. In this case you click to expand company, expand Methods, expand insert, and expand Parameters.
  3. Click on each parameter to get its information. In this case, incident insert has only one parameter which is an incident object.
  4. To determine the content of this sample, find the topic for the insert method of the company object.
  5. Construct an XML string with the root node <parameters>, that includes the method parameters as child nodes.

The following shows the XML input string that the client application submits to the OTM to insert a new company record in the database.

      <primaryContactId/>

      <privateAccess>0</privateAccess>

      <insertBy/>

      <updateBy/>
