insert method

An insert method persists a new object instance to the database.

Parameters

This method accepts an object whose content="all" as the input parameter, and returns a "keysOnly" object as an output parameter. You pass the input XML within the root element named <parameters>.

Parameter Direction Data type Content in type Content out type Description

Object name

inOut

object

all

keysOnly

An object whose content="all" on input.
Steps

All standard methods call the Onyx Data Validation Step for input data validation and either directly or indirectly call Onyx Data Manager to access the data tier. Many methods perform additional steps. For a table that lists the default steps of a specific business object method, refer to its method topic under the business object.

Remarks

For objects with generated object keys, such as GUIDs, the primary object key property node can be empty or not present in the input parameters XML for an insert method on a transactional object.

Non-transactional objects may have a primaryId property or a primary key property with a different name. OEAS uses the value of the property's Generate ID field in the OED to determine how a primary key value is generated. For a value of "odmGuid" a unique GUID value is generated on the middle tier. When the value is "db," the middle-tier expects a unique value to be generated in the data tier. In this latter case, the property should also have an object-physical map that specifies the database function to be called for generating the unique value during the insert operation.

Denormalized properties and maintenance properties, such as insertDate, insertBy, updateDate, updateBy, and onyxTimestamp, are ignored on input, therefore it is unnecessary to include them in the input parameters XML. Also, logical properties that are not required or that have a default value (according to its OED definition) are optional in the input parameters XML.

Example

The following example shows the content of the input parameters and output parametersReturn used to insert an reminder object instance.

Parameters

<parameters>

   <reminder objectType="reminder">

      <primaryId/>

      <ownerId>DE1FB914-1B4F-43C5-9E34-7CEBD6DB0E27</ownerId>

      <ownerType>6</ownerType>

      <userId>sa</userId>

      <message>Make plans for 3/15 site visit.</message>

      <dueBy>2002-2-18 00:00:00</dueBy>

      <recurType/>

      <recurInterval/>

      <alarmActive>1</alarmActive>

      <insertBy/>

      <insertDate/>

   </reminder>

</parameters>

parametersReturn

<parametersReturn>

  <reminder objectType="reminder" content="keysOnly">

    <primaryId>F2230EA8-7EB4-405C-8965-3C4A95E9243E</primaryId>

  </reminder>

</parametersReturn>
