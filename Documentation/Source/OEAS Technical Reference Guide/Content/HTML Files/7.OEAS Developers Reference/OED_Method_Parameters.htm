Method Parameters
Parameter

Parameters, which are owned by a method, provide the OEAS with data that is used by the method. These parameters may be simple variables or they may be objects. When a parameter is an object instance, the method has access to all of the data in the properties on that object. Conceptually, you can treat an object type parameter as a structure that may hold many different kinds of data.

Parameters have specific attributes which include a name that is unique within the method, a description, a direction (in, out, or inOut) that describes the direction of data flow, and a data type, which may be a simple type or the keyword object. Simple parameters may optionally have a default value attribute. Object type parameters must have an object type. These object type parameters also have required attributes called "content in" and "content out."

These content in and content out attributes can actually be collections with their own value and can be one of the following types:

  • all
  • any
  • keysOnly
  • partial

The content in and content out value tells the core components which properties on the object parameter will be accessible. If the value is all, any, or keysOnly, then the list of properties can be determined from the object's definition. In that case, the content in or content out collection can be empty. If the value is partial, then the property list cannot be determined and must be supplied in the collection.

There is one special container object. Methods (typically retrieveList methods) can handle data using a generic container object specially designed for OEAS output. This container object is called a rowSet. The rowSet object is flexible and dynamic - it may hold any number of simple properties or values, which are described by the columnDefinitions collection within the rowSet object. When a method specifies an object parameter with an object type of rowSet, then the list of object properties cannot be determined. In this case, the list of column names for the rowSet must be specified in the contentOut collection.
