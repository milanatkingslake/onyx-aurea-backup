Validating changes to the OED

Important: The validation process does not check whether business objects and step components require the existence of another item. Before publishing your modifications to your OEAS production environment, test them by publishing them to an OEAS test environment.

To validate changes:

  • Click to validate the changes. This process performs an extensive validation.

    If the validation process succeeds with no errors and no additional data to report, a success note appears in the browser status bar. However, if the Object Designer finds any problems during the integrity check, explanations of those problems display in a summary window. Pay close attention to any errors listed. If you do not correct the errors, the Object Designer may automatically mark the object as invalid. Invalid objects are not available to the Onyx Enterprise Application Server (OEAS).

    If any problems are found, resolve them now before publishing your changes. If no problems are found, then it is safe to publish the changes, which results in the OED production copy being overwritten by the OED working copy.
