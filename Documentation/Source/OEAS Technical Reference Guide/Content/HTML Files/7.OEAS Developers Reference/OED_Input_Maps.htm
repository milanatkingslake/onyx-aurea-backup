Input Maps

The input maps collection identifies the stored procedure's parameters and identifies how they should be filled during the SQL Generation process of composing the procedure call. A couple of input map attributes are the name of the procedure parameter being mapped and the "map type" attribute which works in conjunction with another attribute, "map data." The following map types are available for input maps:

Map Type

Description

property

The map type property indicates that the procedure parameter should receive the value of a property on the object containing the method and step that owns the procedure map. This map type applies only when the object exists as a parameter on the containing method (for example, on "insert" or "saveCollection" methods).

methodParameter

A map type of methodParameter indicates that the procedure parameter should receive the value of one of the method parameters of the method that contains the step being executed.

contextVariable

A map type of contextVariable indicates that the procedure parameter should receive a value from the context XML supplied by the core components. The SQL Generation process has several pre-defined context variables for these values. Allowable "map data" values for the contextVariable:

  • vSiteId - the site ID for the application
  • vUser - the user ID of the current user
  • vLanguage - an identifier for the user's preferred language
  • vPartnerUser - identifies if a user should be restricted based on access rules defined for partners

literal

A map type of literal contains a "map data" attribute that can be any string that resolves to a single-valued SQL statement, including a keyword (such as NULL) or a constant that the database can use for the parameter value. This literal value is passed in either as an input parameter for "replace" stored procedures, or it is compiled into the generated stored procedure.

These string literals may include valid XSL commands as well as SQL.

XSL literal

The map type of XSL literal indicates that the input value to a stored procedure (either a "replace" procedure or a generated procedure) is provided by running an XSL statement against the input XML, and passing in the result.
