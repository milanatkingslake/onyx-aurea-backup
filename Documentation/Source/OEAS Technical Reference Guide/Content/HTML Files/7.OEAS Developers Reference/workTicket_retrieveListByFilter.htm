retrieveListByFilter (workTicket)

retrieveListByFilter method

The retrieveListByFilter method of the workTicket object searches the database searches the database by the filter criteria you pass in and returns a rowSet of work ticket records.

retrieveList standard method information

Parameters

Parameter

Data type

Direction

Description

groupIdList

string

in

Filter by an organizational group name or by a semicolon-separated list of group names. This element can be NULL when filtering on assignedTo.

assignedTo

string

in

Filter by the ID of the internal user who is responsible for resolving a work ticket issue. This element can be NULL when filtering on groupIdList.

statusIdList

string

in

Filter by the string representing a predefined status of a work ticket or by a semicolon-separated list of status strings.

priorityIdList

string

in

Filter by the string representing a predefined priority of a work ticket or by a semicolon-separated list of priority strings.

productIdList

string

in

Filter by a description of a product that is associated with the work ticket or by a semicolon-separated list of product strings.

typeIdList

string

in

Filter by the string representing a predefined work ticket type or by a semicolon-separated list of type strings.

keywordList

string

in

Filter by a tag associated with work tickets or by a semicolon-separated list of tags.

dateInsertStart

dateTime

in

Beginning dateTime, in a range of dateTimes, to filter by the time and date on which the object instance was initially saved.

dateInsertEnd

dateTime

in

Ending dateTime, in a range of dateTimes, to filter by the time and date on which the object instance was initially saved.

dateUpdateStart

dateTime

in

Beginning dateTime, in a range of dateTimes, to filter by the time and date on which modifications to an object instance were last saved.

dateUpdateEnd

dateTime

in

Ending dateTime, in a range of dateTimes, to filter by the time and date on which modifications to an object instance were last saved.

includeContact

boolean

in

Switch to include work tickets for which the assignedTo user is listed as the contact for the work tickets. This is the default setting, 1.

Remarks

The retrieveListByFilter search is not case sensitive for the keywordList input parameter.

This method returns non-private records or those owned by the user.

This method supports wild card substitution in search criteria. For more information on wild card substitution, see the retrieveList method (standard behavior) topic.
