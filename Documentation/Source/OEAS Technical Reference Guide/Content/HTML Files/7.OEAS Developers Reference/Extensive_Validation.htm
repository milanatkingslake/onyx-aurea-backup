Extensive validation

You can perform an extensive validation on your modifications to the Onyx Enterprise Dictionary (OED) at any time after you save your work. Each time you validate or publish your modifications, the following steps occur:

  1. All objects are validated for completeness and logical validity. If validation fails, processing does not stop, but the item that fails is saved as invalid. The criteria for determining completeness and logical validity are:
  • ID values for new elements are generated.
  • Presence of all required fields.
  • Maximum length of all fields.
  • Accepted values for all enumerated properties.
  • Valid characters in element names.
  • No duplicate element names.
  1. All valid, non-disabled objects are checked for inter-object references. The following references are checked:
  • Reference via property or parameter to a non-existing object.
  • Reference via property or parameter to an object that is disabled or not structurally valid.
  • Reference via step to a non-existing method.
  • Reference via step to an object that is disabled or not structurally valid.
  • Reference via step to a method that is disabled (at the method level).
  • Reference via step to an object that itself references, directly or indirectly, an object or method that is structurally invalid or disabled.
  • Circular reference in a stepType="method" method call.
  1. Each object that passes steps 1 and 2 is saved as valid, otherwise it is saved as invalid.
