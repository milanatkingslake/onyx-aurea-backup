Navigating the Object Designer

The Object Designer web page is divided into four areas to assist you in navigating through the metadata items and performing your modifications to their attributes. The four areas are:

  • Toolbar
  • Navigation area
  • Context area
  • Edit area

The toolbar appears under the context area and contains the toolbar buttons that control the tasks related to configuring the metadata. Tool tips are provided for each button.

Toolbar table

Icon

Tooltip

Function

Context

Add

Adds a new item to the OED. The item added depends on which item you selected in the navigation area.

Enabled at all times.

Delete

Deletes items from the OED. The item deleted depends on which item you selected in the navigation area.

Enabled only when you select a specific item in the navigation area. Collections, such as properties, relationships, methods, tables, and so on cannot be deleted.

Move up

Moves items up one position in the navigation tree. Be aware that steps are executed in the order they appear in the tree.

Enabled only when you select a Parameter, Method, Object Key, Property, Sort Property, or Step. Collections cannot be moved.

Move down

Moves items down one position in the navigation tree. Be aware that steps are executed in the order they appear in the tree.

Enabled only when you select a Parameter, Method, Object Key, Property, Sort Property, or Step. Collections cannot be moved.

Copy

Copies an item to the clipboard.

Enabled only when items that can be copied are selected.

Paste

Pastes an item from the clipboard.

Enabled at all times.

Save

Saves all modified elements to a working copy of the OED. Removes all unsaved data indicators and validates items against the OED to ensure integrity.

Enabled after any changes have been made.

Validate

Validates all the metadata against the OED, including inter-object validation, and applies any changes resulting from the validation to the working copy of the OED. Any item found not to be valid is saved as invalid.

Enabled at all times, but only performs validation after all changes have been saved.

Publish

Performs the same validation as the Validate button and applies any changes to the metadata that resulted from the validation. In addition, Publish copies the working copy of the OED to the production copy of the OED. The changes are implemented in the application after you refresh the OED cache by restarting the OnyxEASApp COM+ component.

Enabled at all times, but only performs validation and publishing after all changes have been saved.

Help

Launches the Help system.

Enabled at all times.

Navigation area

In the navigation area, the Object Designer displays the hierarchical layout of the OED. You select the object, table, procedure, or other element to display the attributes in the edit area.

Context area

The context area contains the tool name and the name of the OEAS logical application for which you are customizing the metadata.

Edit area

The edit area displays the attributes for the item that you select in the navigation area. You can edit the attributes to create your customization.

Attributes differ from item to item, depending on whether you select an object, property, method, step, relationship, table, and so on.
