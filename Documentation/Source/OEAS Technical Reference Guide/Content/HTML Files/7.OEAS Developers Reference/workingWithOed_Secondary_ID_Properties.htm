Secondary ID properties

The OEAS uses secondary ID properties to provide a more friendly looking identifier. As noted in the previous topic, transactional objects use a primary key that contains a GUID. GUID values are not particularly legible or user-friendly. You often prefer to have an identifier for a transactional object that is easier to read and/or memorize, search on, etc. The Onyx transactional objects use a "secondaryId" property for this purpose. These are mapped to a database stored procedure that accepts a "natural" value if one is entered by the user, otherwise it automatically generates a unique integer value.

Onyx recommends setting the Read Only attribute for secondary ID properties. For more information, see the Read Only attribute section of the Properties topic.
