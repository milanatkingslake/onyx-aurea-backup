saveCollection (workNoteDetail)

saveCollection method

The saveCollection method of the workNoteDetail object performs multiple actions at once: inserting, updating, and deleting object instances, as appropriate.

Remarks

Note: This saveCollection method does not support an update action.

When the logMarkSequenceNumber is NULL and logMark is 1, the insert action takes the logMarkSequenceNumber value generated by OEAS for the first inserted object instance in the input XML and applies that value to all other objects in the workNoteDetails collection tagged with the action="insert" attribute. If the initial insert action in a saveCollection method fails, the remaining actions in the saveCollection are aborted.

When the logMarkSequenceNumber is NULL and logMark is 0, how OEAS determines the logMarkSequenceNumber depends on whether a previous insert caused the generation of a new logMarkSequenceNumber or if an insert is part of a series of inserts using the same OEAS-generated logMarkSequenceNumber.

When a previous insert caused the generation of a new logMarkSequenceNumber:

  • OEAS uses the logMarkSequenceNumber from the previous insert, replacing the NULL with a value from the last OEAS-generated insert.
  • OEAS increments the maximum sequenceNumber that exists for the logMarkSequenceNumber from the previous insert.

When an insert is part of a series of inserts using the same OEAS-generated logMarkSequenceNumber:

  • OEAS uses the maximum logMarkSequenceNumber for the workNoteHeader (in most cases this is the current editing session) and appends the new workNoteDetail to that logMarkSequenceNumber group.
