Procedure Maps
Procedure maps

The procedure maps collection contains individual maps that allow steps to utilize stored procedures on the data tier. A step may be connected to one or more stored procedures in the physical database and each such connection is called a procedure map. A procedure map enables the step to make a specific stored procedure call.

Procedure maps contain a collection of elements and attributes that include the name of the stored procedure to be called. Equally important is the "map type" attribute, which instruct OEAS core components how to call the stored procedure.

The "map type" attribute can contain any of the following three values:

Map type

Description

replace

When a procedure map type is replace, the "replace" procedure becomes the entire activity performed by the step. There may be only one replace map type for a step.

pre

When a procedure map type is pre, the "pre" procedure happens before the normal work of the step. There may be multiple pre procedure maps for a single step. However, you may not combine pre procedure maps with a replace procedure map.

post

For a post map type, the "post" procedure happens after the normal work of the step. There may be multiple post procedure maps for a single step. However, you may not combine post procedure maps with a replace procedure map.

For procedure maps used on "saveCollection" methods, some additional functionality is provided. Each pre or post procedure map may be applied for a specific action (insert or update, for example), or be applied for the entire collection. If the map is applied for the "collection", then it is performed before any action in that collection. If the map is applied for the "collection by owner," then the procedure is called once for each distinct owner in the collection. For example, this can be useful for validating address type when updating multiple addresses for multiple customers simultaneously. If it is applied for a specific action, then it is inserted directly before or after that particular action. For further information on method types, see OEAS methods.

In order to enable the SQL Generation process to compose these procedure calls, the procedure map contains two important collections. One collection is the input maps - these identify stored procedure parameters and how to fill them. The other collection is the output maps - these identify what method parameter(s) receive parameter values or result set data returned by the procedure.

Note: Before a procedure map can be added, it must first be described in the Procedures section of the Physical Model.
