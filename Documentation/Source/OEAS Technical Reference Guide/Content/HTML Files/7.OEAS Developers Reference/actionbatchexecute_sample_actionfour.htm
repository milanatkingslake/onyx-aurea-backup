(ActionBatchExecute) process fourth action
Process fourth action: alert.insert

The fourth action creates an alert. For each actionReturnMap the value pointed to by the sourceElementPath value is applied to the targetElementPath. In this case the incident.primaryId returned from the incident.insert action is mapped to the alert.ownerId to be used in the alert.insert action.

<actionReturnMaps>

   <actionReturnMap>

      <sourceActionId>a</sourceActionId>

      <sourceElementPath>parametersReturn/incident/primaryId</sourceElementPath>

      <targetElementPath>parameters/alert/ownerId</targetElementPath>

   </actionReturnMap>

</actionReturnMaps>

Using the updated methodInputs parameters, the method call is made on the object that logicalObjectName and logicalMethodName point to. In this case an alert.insert call is made to insert an alert using data from the previous incident.insert action. Error-handling behavior is determined by the value of onFailure.

XML for alert.insert

<action>

   <onFailure>stop</onFailure>

   <id>c</id>

   <logicalObjectName>alert</logicalObjectName>

   <logicalMethodName>insert</logicalMethodName>

   <methodInputs>

      <parameters>

         <alert objectType="alert">

            <primaryId/>

            <ownerId/>

            <ownerType>6</ownerType>

            <text>This alert was created using actionBatch.execute!</text>

            <expirationDate>03/30/2010</expirationDate>

            <expired/>

            <insertBy/>

            <insertDate/>

            <updateBy/>

            <updateDate/>

            <onyxTimestamp/>

         </alert>

      </parameters>

      <customData/>

   </methodInputs>

   <actionReturnMaps>

      <actionReturnMap>

         <sourceActionId>a</sourceActionId>

         <sourceElementPath>parametersReturn/incident/primaryId</sourceElementPath>

         <targetElementPath>parameters/alert/ownerId</targetElementPath>

      </actionReturnMap>

   </actionReturnMaps>

</action>
