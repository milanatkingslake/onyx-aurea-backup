How to use embeddedXml

How the embeddedXml element works:

  1. An OEAS retrieveList method calls a step component that typically returns a large list of data from the database.
  2. The step component builds the rowSet data as an XML string, and inserts the data within the embeddedXml element.

The embeddedXml element begins with exactly:

<embeddedXml><![CDATA[

and ends with exactly:

]]></embeddedXml>

    Note that the code must appear exactly as illustrated above: it cannot contain any white space between the embeddedXml element and CDATA section, not even the hard return that is sometimes used when formatting XML for visual display.

    Incorrect usage of this element

    The embeddedXml element in this XML sample is used incorrectly because white space separates it from the CDATA section.

    <parametersReturn>

       <embeddedXml>

    <![CDATA[

             <rowSet objectType="rowSet" rowCount="2" columnCount="9">

                <columnDefinitions collection="1" objectType="columnDefinition">

                   <columnDefinition>

    .

    .

    .

          ]]>

    </embeddedXml>

    <parametersReturn>

  1. The step component passes the XML to the Onyx Transaction Manager (OTM). If it is the last step in the OTM call, the OTM searches the stepPackage XML for the <embeddedXml><![CDATA[ and ]]></embeddedXml> strings and removes them before sending the data to the calling client. The rowSet data between these strings remains intact. No processing is performed on this data while it is contained within the embeddedXml element and CDATA section.

The following example shows the XML after the OTM removes the embeddedXml element and CDATA section. Note that the rowSet data is passed to the calling client as if the embeddedXml element and CDATA section never existed.
