OEAS methods

OEAS methods behave in a standard manner, each method performing the same basic task regardless of which object owns the method.

OEAS has the following standard methods:

  • delete
  • insert
  • retrieve
  • retrieveCollection
  • retrieveList
  • saveCollection
  • update

The Onyx Data Manager (ODM) generates SQL code for all of the standard methods. ODM generates some SQL for retrieveList methods and other custom methods, but not nearly as much as it generates for standard methods.

The business object methods call ODM to use generated SQL and to access replace procedures. Methods  always include an input data validation step prior to ODM step. Some methods call additional steps for preprocessing input data and postprocessing results. And some methods, such as insert on the company object, call other business object methods for additional processing. These six methods have specific and consistent behavior across all OEAS objects, and the following sections provide details about them.

A custom business object, which is an application-specific object that is added to the Onyx Enterprise Dictionary (OED) after OEAS is installed, can use standard methods without requiring any additional coding. When an object and its related information is properly defined in the OED, standard method processing is available to the object by referencing the OnyxDM.DataManager step component in its method definitions.
