XML in OEAS

XML in Onyx Enterprise Application Server (OEAS)

Onyx uses XML to describe all OEAS business objects, to pass parameters to and from the OEAS, and to pass parameters to and from Step Components.

OEAS adheres to specific XML standards when developing XML documents and object definitions. Understanding these XML standards is important if you want to:

  • Write client applications or components that call the Onyx Enterprise Application Server (OEAS). In order to communicate with the Onyx Transaction Manager (OTM), and therefore the entire OEAS, you must send the name of the object method you wish to call, along with its parameters, to the OTM in XML format. The OTM parses the incoming XML string, distributes it to the appropriate components for execution, and returns data to the calling application in an XML string.
  • Write custom steps. The OTM calls each step in the order dictated by the Onyx Enterprise Dictionary (OED), and passes data from step to step in an XML DOM Document.
  • Browse the Onyx Enterprise Dictionary (OED) in its XML form. Although you can view the OED .xml file directly, Onyx supports modifications to the OED only if they are made using the Onyx Object Designer.

ASCII characters

Microsoft XML Parser (MSXML) 6.0, and therefore OEAS, does not accept most American Standard Code for Information Interchange (ASCII) characters whose code value is less than 32. However, the OEDB allows the following ASCII characters:

  • 9 (tab)
  • 10 (line feed)
  • 13 (carriage return)
