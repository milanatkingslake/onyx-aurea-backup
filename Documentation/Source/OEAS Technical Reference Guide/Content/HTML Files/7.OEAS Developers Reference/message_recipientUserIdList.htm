recipientUserIdList (message)

recipientUserIdList property

The recipientUserIdList property of the message object contains one or more userId strings to indicate who receives the message.

Data type

string (maximum length)

Access

Write-only

Remarks

A list of userIds must be semicolon-separated.

The recipientUserIdList element is empty in the parameters on return because it is a write-only property. The recipientUserIdList or recipientGroupIdList properties are required only for message inserts.

Values in either the recipientUserIdList or recipientGroupIdList elements are required on input. Either element can be NULL if the other element contains one or more recipients. Both elements can contain values on input.

OEAS adds the recipient list for a message to the top of the message text.

OEAS creates an instance of the messageDestination object for each userId in the recipient list. The object instances track the read status of the message for each recipient. When a recipient deletes a message, OEAS deletes the corresponding messageDestination object instance. When all messageDestination object instances for a message are deleted, the message object instance is deleted.
