Relationship attributes

When you select a relationship in the Object Designer, the attributes for that relationship display in the edit area.

Note: The Onyx naming convention for object relationships is 'primary object [dash] secondary object [dash] secondary property(s) involved'. For an example see the title bar of the edit pane in the graphic.

The attributes are defined as follows:

Attribute Name

Description

ID

The globally unique identifier (GUID) assigned by the Onyx Enterprise Dictionary. This attribute is read-only.

Name

Type the name you want to assign to this relationship. Relationship names must be conform to XML naming conventions.

Description

Type the description for this relationship.

Relationship Type

Select the type of relationship you want to create:

  • For a relationship to an association object, select association. This relationship only supports delete and restrict logical cascade types.
  • For a relationship to an object that maintains a foreign key property directly in the table that stores the object data, select direct. This relationship supports all logical cascade types.
  • For a relationship to a domain object (typically a look-up list), select domain. This relationship only supports the restrict logical cascade type.
  • For an owner or parent-child relationship, select linkObject if several possible relationships involve the same object properties (e.g., an incident may be owned by a company or an individual). This relationship supports all logical cascade types.
  • For a relationship between tightly coupled objects such as quote and quoteDetail or customerProduct and customerProductDetail, select masterDetail. This relationship supports the delete and restrict logical cascade types.
  • For a direct relationship to one of a number of different objects, select variableDirect. This relationship supports all logical cascade types.

Installed By

Type the name of the vendor or application adding this relationship.

Onyx Version

The Onyx version number of this relationship. This attribute is read-only.

Custom Version

Type the custom version number you want to assign to this relationship.

Released

This attribute is for documentation purposes only. Check this box to document that the relationship is no longer under development.
Primary Object

Object Name

Select the object that you want defined as the primary object in this relationship. The primary object in a relationship is the object whose primary key is involved in the key mapping. Select the object you want to use. This list includes objects marked for deletion and objects that are disabled, both which are marked as such. Invalid objects are not listed.

Cardinality

Select the cardinality of the primary object in the relationship:

  • For a zero to many, select 0-*.
  • For a zero to one, select 0-1.
  • For a one to zero, select 1.
  • For a one to many, select 1-*.
Secondary Object

Object Name

Automatically assigned with the name of the current object, which is the secondary object in this relationship. The secondary object in a relationship is the object containing foreign key referencing the primary key of the primary object.

Cardinality

Select the cardinality of the secondary object in the relationship:

  • For a zero to many, select 0-*.
  • For a zero to one, select 0-1.
  • For a one to zero, select 1.
  • For a one to many, select 1-*.

Logical Cascade Type

Select the cascade rule to apply to the secondary object when an instance of the primary object is deleted:

  • To delete the primary object and cascade the delete to all children, select delete.
  • To delete the primary object and remove links to all children, select null. If the link is stored in it's own table, then the link row will be deleted. If the link is a column inside the table, then the value in the column will updated to NULL.
  • To delete the primary object only if it has no relationships to active objects, select restrict. If the primary object does have relationships to active objects, then neither the primary object nor any children will be deleted.

Object Type Property

Select the secondary object's property that stores the value for the primary object's object type. NOTE: This field is available only if Relationship Type is association or variableDirect.
