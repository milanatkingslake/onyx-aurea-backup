Case conventions

Camel case, the convention of capitalizing the first letter of each full word in a concatenated string of names, appears in at least two forms: lower camel case (businessObject) and upper camel case (BusinessObject). Onyx uses lower camel case as the case convention for all element and attribute names written in XML. For example, the name of a property called company name is written in lower camel case as companyName.

Additional rules apply to the use of acronyms and abbreviations:

  • For names comprised of a single acronym or a common abbreviation, all characters are in lower case (ex. "id," "oeas").
  • When an acronym or abbreviation is appended to another word, the first character is in upper case (ex. "carld," "forOcc").
  • When two acronyms, two abbreviations, or a combination of both are combined, the first is in lower case, and the first letter of the second is capitalized (ex. "occId").
