Properties

The properties collection is a container that holds all of the properties for a business object.

Properties are entities that are owned by a business object and each property encapsulates data closely related to that object. Properties on an object are much like variables in a simple program in that they hold data that is available if requested from the proper context. Business object's also use properties to permanently store data through their connection to the physical model. Object properties have specific attributes which include a name (which must be unique within the business object), a description, a data type, versioning information and some other specialized attributes. The following sections describe in greater detail some of the specialized attributes.

Generate ID attribute

The Generate ID attribute value informs the business tier how unique values are generated for a property when the property is a primary key. The following options are available for this field:

Generate ID Value

Description

odmGuid

A value of odmGuid indicates that a GUID is generated by a COM+ object on the business tier.

dB

A value of dB indicates that a unique key value will be generated in the database (such as with an identity column).

empty

An empty attribute for this value indicates that the property does not require this special treatment.

Read Only attribute

As the name implies, the Read Only attribute governs whether a calling application can change this value on updates. As a best practice, Onyx recommends the following properties have the Read Only attribute set.

  • Standard maintenance properties (except for the active property)
  • Secondary ID properties
  • Denormalization properties

When creating custom properties, keep the following in mind:

  • OEAS ignores supplied values in update actions for Read Only properties.
  • Table Administration does not support editing Read Only properties.

Denormalization Definition attribute

When enabled, the Denormalization Definition attribute contains a join relationship and a target property from another object. The business tier uses this information to retrieve a value from the selected property on another object based on the specified relationship. For example, a product description might be displayed on an order object that only stores a product number - the true storage for the description being "normalized" and held by another object and referenced through its product number.

For more information on the properties, see OEAS Properties.
