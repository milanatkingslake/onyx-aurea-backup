Extended information (Query Engine)

Extended information

Two of the IEBusQE interface methods, ExecuteReturnExtendedInfo and CallStoredProcedureReturnExtendedInfo, return information about the columns that exist in the result set. This information is stored in a two-dimensional Variant array. The second dimension of the array represents the rows of metadata, while the first dimension represents the metadata itself. Note that the dimensions are reversed from their counterparts in the result set array. Onyx step components use this information to generate the column definition information included in rowSet objects.

The columns of data in the extended information array represent the following:

Column number Value Variant sub type Description

0

Column name

String

Result set column name.

1

Column ordinal

Integer

The position of the column of the result set.

2

Bitmask

Integer

A bitmask that describes the column characteristics.

3

Column size

Integer

Maximum possible size of the column in bytes.

4

Data type

String

The XML data type of the column.

5

Precision

Long

For numeric types, this is the maximum precision of the values in the column.

6

Scale

Long

For decimal or numeric types, this is the number of digits to the right of the decimal point for the values in the column.

The XML data type information is provided through a mapping on the application server. The Query Engine uses this data to convert the OLE DB data type to the XML data type.

The Bitmask values are generated by the OLE DB Provider. The values are defined by the DBCOLUMNFLAGS enumerated type. Consult the Microsoft OLE DB documentation for more information about the values of this enumeration.
