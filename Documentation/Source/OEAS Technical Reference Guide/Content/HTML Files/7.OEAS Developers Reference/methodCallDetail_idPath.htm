idPath (methodCallDetail)

idPath property

The idPath property of the methodCallDetail object contains an XPath expression, relative to the elementPath, that locates an unique identifier for an output element.

Data type

string

Access

Read/write

Remarks

The methodCallDetail object supports an output having both an ID component and a description component. The returned item can have an identifier (the ID component--such as country.countryCode or company.primaryId) and a user friendly display string (the description component--such as country.description or company.secondaryId). The idPath property is an XPath that locates the output's ID component relative to the elementPath. For example, if elementPath specifies rowSet/rows, then the idPath would specify row/c0, where the c0 element contains the ID component of the output.

The idPath leads to a single element that is a descendant of an output element. Onyx recommends that the idPath not include traversal operations that try to access elements not descendant of the output element. If the idPath does not result in a selected node when the method call is executed, the returned output will have an empty ID component. If it is NULL, the returned output will not contain an ID component. The output node can still contain a text component and/or a description component.

This property is ignored on input method call details.
