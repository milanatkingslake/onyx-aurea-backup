retrieveListUsersAndGroups

The retrieveListUsersAndGroups method of the user object returns a list of Onyx user and group data corresponding to members of the group with the given ID. The data is returned as a rowSet object in the parametersReturn XML. Certain behaviors are common to all retrieveLists. For a description of these standard behaviors, use the following link.

retrieveList standard method information

Parameters

Parameter

Direction

Data type

Content type

Description

groupId

in

string

—

Value that identifies the parent group object in the database.

active

in

boolean

—

An integer that specifies whether to return active or inactive user and group data. If the active element of the input parameters XML is empty or NULL, all data are returned, active and inactive. When active contains 1, all active data are returned. If the element contains 0, inactive data are returned. If active contains an invalid value, such as 2, the method succeeds, but no data are returned.

Remarks

This method supports paging of the result set with the pageSize and startRowIndex attributes on the rowSet parameter.

This method supports the Onyx security component for Cognos security integration.

The stored procedure sorts the result set by object type first and then by name. It returns group objects before user objects. All objects are sorted by name in ascending order.

This method might retrieve only a limited list of data, and not always the entire list of data available. You can use the OEAS Configuration utility to configure the maximum rows returned. For more information, see Paging functionality.
