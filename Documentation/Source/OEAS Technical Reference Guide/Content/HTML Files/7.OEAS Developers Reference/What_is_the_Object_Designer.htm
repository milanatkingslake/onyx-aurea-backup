What is the Object Designer

The Object Designer is an Onyx design tool that enables you to customize the contents of the Onyx Enterprise Dictionary (OED). When you modify the data in the OED, you are modifying the definitions of the OEAS business objects and physical schema that are defined in the OED. Therefore, when you modify the data in the OED, you modify the way in which a client request can be processed.

Warning: The object designer is a design tool, not an administration or configuration tool. This means that you should, before using the Object Designer, be very familiar with your OEAS system and the way in which the OEAS business objects interact with one another.

The Object Designer directly references either the OnyxEnterpriseDictionary.xml file (the source of the OED) that resides on the application server, or, when you edit the contents of the OED, a working copy of OED called OnyxEnterpriseDictionaryWC.xml. It reads the data stored in that file and, based on its content, displays a complete listing of the object definitions that comprise the OED: business objects, properties, methods, parameters, and steps. The Object Designer also displays a set of customizable attributes specific to these items.

Type of OED Metadata

Description

Logical Business Objects

Each business object implemented in OEAS is represented in the OED with a full object definition, which includes detailed information about the properties, object keys, relationships, object physical maps, and methods.

Physical Model

The OED contains physical schema descriptions for all tables and procedures.

The business objects defined in the OED differ significantly from COM objects. Unlike COM objects, the OEAS business objects are written in XML and are not compiled. Because they are not compiled, they provide a lot of flexibility: you can easily customize them and, because OEAS creates them only when needed, at run-time, your changes can be quickly implemented.
