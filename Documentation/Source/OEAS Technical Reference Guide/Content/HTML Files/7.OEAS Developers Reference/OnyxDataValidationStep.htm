OnyxDataValidationStep

OnyxDataValidationStep component verifies that the XML data passed to it conforms to the rules contained in Onyx Enterprise Dictionary (OED) and OEAS standards for XML. All methods in OEAS must use the OnyxDataValidationStep component as the first step. Omitting OnyxDataValidationStep from a method is not supported and may cause the method to behave in non-standard ways. The OED rules can include:

  • the data type of a property
  • a restriction that limits the minimum or maximum value or length of a property
  • a flag to mark a property as required
  • a flag to mark a property as part of a primary key
  • a default value for a property if none is supplied

The validation step checks the OED for these rules to apply to input XML. This step may modify the XML and continue processing to the next step, or it may return an error to the OTM depending on the results of the validation tests.

All methods must call this step component by adding a step in the OED that references the OnyxDataValidationStep. When you customize OEAS by adding methods or custom components, you use OnyxDataValidationStep to verify that the XML your method or component passes remains valid. You do this by specifying the OnyxDataValidationStep in the method's steps collection as the first step for your component. By default, all Onyx logical methods contain a data validation step as the first step.
