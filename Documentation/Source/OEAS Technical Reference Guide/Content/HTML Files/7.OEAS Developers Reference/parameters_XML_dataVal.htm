parameters XML (data validation)

parameters XML

OnyxDataValidationStep component checks the incoming XML from the OTM and determines if the XML is valid and conforms to OEAS standards. This component performs five validation tests in order depending on the logical object method being called.

  1. cleanup: trims leading and trailing whitespace from elements, removes any elements that are empty and do not have a null="1" attribute or an ignore="1" attribute (ignore attribute has no effect on data validation for properties other than onyxTimestamp)
  2. default: on insert, supplies value specified in OED only if the property or parameter node is missing from the input XML

If the property or parameter node was present in the input XML but removed in the "cleanup" step, then the default will be applied in the "default" step.

  1. primary key: verifies that properties that are part of the object's primary key are supplied for methods that require a primary key
  2. required: for object parameters, verifies that required properties have a value on insert (a default value set previously can satisfy this test), and are not set to NULL on update; for simple parameters, verifies that parameter contains a value or the null="1" attribute

Any property not provided in an update is not changed as part of the update. This constitutes a partial update.

  1. minimum/maximum: Verifies that the property value falls within any declared minimum or maximum constraints in the OED (minimum and maximum constraints represent length for properties with a data type of "string", and represent the value itself for all other data types)
