opPopStringList

@message_code INT OUTPUT,

@message_parameter NVARCHAR(255) OUTPUT,

@error INT OUTPUT,

@string_list NVARCHAR(4000) OUTPUT,

@delimiter NCHAR(1) = N';',

@single_string NVARCHAR(4000) = NULL OUTPUT

@string_list, @single_string

This procedure will pop the first string from a list. The string and the remaining list will be returned.

Parameters:

  • @string_list - The list containing 0-n strings. The return value will be the original list minus the string that has been extracted and placed in the @single_string output parameter.
  • @delimiter - The character that delimits strings in the list.
  • @single_string - Used for output only: the string that has been removed from the list.

opPopulateIntListTable

@message_parameter NVARCHAR(255) OUTPUT,

@error INT OUTPUT,

@id_list NVARCHAR(4000),

@delimiter NCHAR(1) = N';',

@list_id UNIQUEIDENTIFIER = NULL OUTPUT

None

opPopulateStringListTable

@message_code INT OUTPUT,

@message_parameter NVARCHAR(255) OUTPUT,

@error INT OUTPUT,

@string_list NVARCHAR(4000),

@delimiter NCHAR(1) = N';',

@like_search BIT = 0,

@list_id UNIQUEIDENTIFIER OUTPUT

None

This procedure populates a temporary table from a list of string values separated by a delimiter. A '%' will be tacked onto the end if the @like_search parameter is true. This procedure is expected to support strings up to 255 characters - the table is defined as 256 to support like searches. The @list_id parameter can be used later to filter the results if multiple lists have been added to the table.

Expected Table Definition:

CREATE TABLE #string_list_temp(

   nvarchar_string NVARCHAR(256) COLLATE DATABASE_DEFAULT NOT NULL,

   list_id UNIQUEIDENTIFIER NULL

)
