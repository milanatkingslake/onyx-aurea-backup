subsystem Object

subsystem object

The subsystem object enables an administrator to configure a processes for batch execution. The subsystem object encapsulates behavior for setting status, schedule, and error handling for various Onyx Processing Server (OPS) subsystems.

Methods

Method

Description

delete

Deletes a subsystem

insert

Inserts a subsystem

retrieve

Retrieves a subsystem

retrieveCollection

Retrieves a collection of subsystems

saveCollection

Saves a collection of subsystems

update

Updates a subsystem

Properties

Property

Description

active

Specifies whether the domain data the object provides is available to the user interface

adminEmailAddress

Contains the email address where the subsystem sends error message notifications

adminFromEmailAddress

Contains the address that the subsystem uses in the "from" field of its email error message notifications

adminFromEmailDisplayName

Contains a descriptive token associated with the adminFromEmailAddress

alternateId

Contains a system generated GUID that uniquely identifies the subsystem

batchIntervalMinutes

Contains the time in minutes that the subsystem waits before processing a new group of orders

batchSizeOrderLines

Contains the approximate number of literatureOrderLine objects that the subsystem attempts to process at a given batch interval time

description

Contains a customizable description for subsystem objects

emailServer

Contains the name of the email server used for sending error notification emails to the Onyx Batch Processing Server administrator

fulfillmentEngineId

Contains a foreign key to the fulfillmentEngine object associated with the subsystem

governorIntervalMilliseconds

Contains the time in milliseconds between calls to Onyx Transaction Manager (OTM) for list management processing

immediateIntervalMinutes

Contains the time in minutes that the subsystem waits before processing a new group of immediate priority orders

immediatePriorityAlways

Indicates whether the subsystem processes immediate priority orders regardless of operationStartTime and operationStopTime values

insertBy

Contains the user ID of the Onyx user who initially saves the object

insertDate

Contains the time and date on which the object is initially saved

name

Contains an identifier suitable for display in the adminstrative tool

operationStartTime

Contains the military clock time at which the subsystem begins processing

operationStopTime

Contains the military clock time at which the subsystem ends processing

opsServerName

Contains the name of the Onyx Batch Processing Server (OBPS) on which the subsystem COM component code executes

primaryId

Contains the unique ID assigned to the object when it is first saved to the database

progId

Contains the programmatic identifier that the Onyx Batch Processing service uses to load the subsystem COM component

updateBy

Contains the user ID of the Onyx user who last saved modifications to the object

updateDate

Indicates the time and date on which modifications to the object were last saved
