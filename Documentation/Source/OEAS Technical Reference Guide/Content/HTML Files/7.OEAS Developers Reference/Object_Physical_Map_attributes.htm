Object Physical Map attributes

When you select an object physical map in the Object Designer, the attributes for that object map display in the edit area.

The attributes are defined as follows:

Attribute Name

Description

ID

The globally unique identifier (GUID) assigned by the Onyx Enterprise Dictionary. This attribute is read-only.

Property

Select the property that you are mapping to the physical model. This list includes all the object's properties defined in the OED that are not denormalized.

Table

Select the table that will be used in this definition. This list includes all active tables defined in the OED that are not disabled, marked for deletion, or invalid.

Column

Select the column that will be used in this definition. This list includes all columns that belong to the selected table, including columns that are marked for deletion and columns that are disabled, both which are marked as such.

Persistence Map

Enable

Check this box to activate a persistence map. A persistence map is used for updating or inserting data into the database.

Type

Select the type for the persistence map:

  • To enter the property value from the input object in the column without altering it, select Direct.
  • To map a string literal value or a SQL, an XML, or an XSL construct to the property, select Literal.
  • Data: Enter the literal value for the property using proper syntax.
  • To map a property to a database function, select Function.

    Note: Some functions should produce a value during an insert but not during an update, such as the function for the secondaryId property. To handle this case, the database function must be written to accept an existing property value or accept a null. The function can then decide whether a new value should be generated.

    • Data: Enter the function name without parentheses or parameters. For SQL generation, the function is called with the property as a parameter.
  • To map a property to a database function that takes zero (0) or more than one (>1) parameters, select complex function.
  • Data: Select the function name from the drop-down list.

Retrieval Map

Enable

Check this box to activate a retrieval map. A retrieval map is used to select data from the database.

Type

Select the type for the retrieval map:

  • To retrieve the property value from the database column without altering it, select Direct.
  • To map a string literal value or a SQL, an XML, or an XSL construct to the property, select Literal.
  • Data: Enter the literal value for the property using proper syntax.
  • To map to a database function, select Function.
  • Data: Enter the function name without parentheses or parameters.
  • To map a property to a database function that takes zero (0) or more than one (>1) parameters, select complex function.
  • Data: Select the function name from the drop-down list.

 

Filter Map

Enable

Check this box to activate a filter map. A filter map is used for building a WHERE clause that acts on the database.

Type

Select the type for the filter map:

  • To use the property value from the input XML without altering it, select Direct.
  • To map a string literal value or a SQL, an XML, or an XSL construct to the property, select Literal.
  • Data: Enter the literal value for the property using proper syntax.
  • To map to a database function, select Function.
  • Data: Enter the function name without parentheses or parameters.
  • To map a property to a database function that takes zero (0) or more than one (>1) parameters, select complex function.
  • Data: Select the function name from the drop-down list.
