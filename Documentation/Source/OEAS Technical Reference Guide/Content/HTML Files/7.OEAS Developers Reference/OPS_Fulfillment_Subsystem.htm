Fulfillment subsystem (OPS)

Fulfillment subsystem

The Fulfillment Subsystem controls the engines responsible for processing literature orders. The engines merge the order data with template documents to produce email messages and printed documents. Sample templates that can be customized are installed during OPS installation. You can also create new templates for your unique business needs. For more information, see Creating templates.

Fulfillment Subsystem Processing

A literatureOrder entry is created by a Literature Order action by an OEP user. This literatureOrder entry is created which contains a literatureOrderLine collection. The ownerId property of literatureOrderLine entries associate them with their literatureOrder parent.

The fulfillment subsystem processes the literatureOrder as follows:

  1. Calls the retrieveListIncomplete method of a literatureOrderBatch object to retrieve a list of batchIds for previous order batches that were terminated prematurely.
  2. Creates a literatureOrderBatch object that will be used for building a batch of orders for processing.
  3. Selects literatureOrders and their associated literatureOrderLines for a given batchId.
  4. Selects literatureProducts used by the literatureOrderLines of the batch.
  5. Selects all literatureServicePrinters.
  6. Transforms the literatureOrderBatch, literatureOrder, literatureOrderLine, literatureServicePrinter, and literatureProduct XML into an orderBatch XML structure.
  7. Submits the orderBatch XML to the fulfillment engine for processing.
  8. Marks the literatureOrderLine as either "completed" or "error".
