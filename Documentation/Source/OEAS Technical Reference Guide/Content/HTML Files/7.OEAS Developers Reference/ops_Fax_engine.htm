Fax engine

The fax engine creates SMTP messages based on actionBatch input and sends them to a Fax server that supports SMTP. The engine depends on the following functionality to execute properly:

Default SMTP XSL stylesheet

This stylesheet contains the SMTP header information necessary for the Fax server to send a fax. In addition, this stylesheet can output content for the SMTP message body. Most fax software converts the message body into a fax cover page. One default stylesheet corresponds to each fax subsystem configured in OPS Administration.

Literature product file path

The fax engine uses the file path of the literature product to locate the appropriate file (XSL stylesheet, Word document, PDF, JPEG, and so on). The engine creates an attachment to the SMTP message from the file or file output, as the case may be.

Fulfillment type

The fax engine uses fulfillment type to determine whether to attempt data merging or just attach the file as-is to the SMTP message. For data merging, the file path must point to a valid XSL file.

The details of the processing are described in the Fax templates topic in the Customization section of this document.

Merged Data Messages

The fax engine transforms orderBatch XML with an XSL stylesheet to produce optional SMTP headers and fax body content as an attachment to an SMTP message for processing by a Fax server that supports the SMTP protocol.

Error Handling

Status is returned by the fax engine to the fulfillment subsystem upon completion of the processing. This status indicates whether the order item processing succeeded or failed along with the ID for the order item. This information is passed back to the fulfillment subsystem. The fulfillment subsystem only updates the status of processed orderDetails in the database.
