statusType element

The statusType element contains the condition of the method logic. Step components and the OTM only use the statusType element for logic errors and warnings, such as failing to find an expected XML node, and not for hard errors, such as failing to create an XML parser. For hard errors, the OTM and step components return a bad HR code. When the OTM creates the step package, it sets the value of statusType to success. If the OTM or step components encounter a logic error or warning condition during method execution, they set the statusType element to one of three values.

statusType value Description

success

The method was successful, and no errors were reported.

successWithInfo

The method was successful and no errors were reported, but a message is being sent along with the status of success.

failure

The method failed.

Changing the value of statusType

Onyx step components do not change the value of the statusType element unless the situation merits it. This ensures that invalid data is not mistakenly processed, and ensures that valid data is not ignored by other step components or calling applications.

Onyx step components change the value of statusType under the following circumstances:

Under no circumstances does a step component change the value from failure to a success status.

In fact, Onyx step components that encounter a statusType of failure immediately abort, and do not even attempt to execute their programmed actions.
