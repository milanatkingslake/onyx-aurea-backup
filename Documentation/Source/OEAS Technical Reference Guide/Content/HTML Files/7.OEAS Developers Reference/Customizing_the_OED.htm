Managing the OED

Customizations to the Onyx Enterprise Dictionary (OED) are made using the Object Designer, a tool in the Onyx Enterprise Studio (OES). The OED stores the definitions of the business objects, methods, properties, relationships between objects, the physical model and logical-to-physical mappings. Using the Object Designer, not only can you can implement custom business rules for managing and storing customer data, you can also add additional properties to existing objects, modify the behavior (methods and steps), relationships, tables, and so on.

For example, you can modify the company.insert method to perform custom actions like saving data to a custom location, sending an e-mail to a specific user, and so on. You can even create entirely new business objects and integrate them with OEAS to function with the OEAS core components and business objects.

Important: Use the Object Designer to customize the OED. Onyx does not support customizing the OED directly.

The OED can be customized in many ways, you can:

  • Add metadata (business objects, properties, property constraints, methods, parameters, steps, tables, procedures, and so on).
  • Delete metadata (business objects, properties, property constraints, methods, parameters, steps, tables, procedures, and so on).
  • Modify metadata (business objects, properties, property constraints, methods, parameters, steps, tables, procedures, and so on).
  • Rearrange the order of a method's steps.
  • Modify and delete existing object relationships or define new ones.
  • Define the data-tier physical structures.
  • Map business object methods, properties, and relationships to the data-tier.

Before you customize

You need perform the following tasks before you customize a business object or any other metadata:

  • After making changes to the OEDB, update the OED's physical model with additional tables, columns, or procedures.
  • Determine if object relationships need to change or be added.
  • Write a custom step component if necessary.
  • Update the logical model after the physical model reflects the changes.

Although Onyx does not support customization of the default attributes of the OEAS business objects, you can extend their functionality by adding new properties, methods, parameters, and steps. See "Supported/Unsupported OED modifications" for a comprehensive list of supported modifications.

Warning: Before publishing your modifications to your OEAS production environment, test them by publishing them to an OEAS test environment. The changes that you make to the OED can affect the functionality of dependent products such as the Onyx Enterprise Studio and the Onyx Employee Portal.
