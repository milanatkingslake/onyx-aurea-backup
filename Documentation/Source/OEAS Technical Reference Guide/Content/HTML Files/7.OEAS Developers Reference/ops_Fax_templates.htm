Fax templates

In order to use the fax engine provided by the Onyx Processing Server (OPS) you must understand how to use fax templates. You use fax templates to transform the input orderBatch XML and produce HTML or plain text format output. The fax engine faxes the output appropriately.

OPS includes a fax engine that uses fax templates associated with literatureProduct objects to transform one or more literatureOrderLine elements in the orderBatch XML into a fax sent to a customer. When the fax engine fulfills a literature order, it loads the stylesheet referenced in the filePath property of the literatureProduct object. The engine uses a stylesheet to transform all the fax literatureOrderLine nodes in the orderBatch XML. The XSL merges data from the literatureOrderLine to create a personalized fax literature product.

To use fax templates with the OPS you must be familiar with the following:

  • orderBatch XML and logical objects contained within it
  • XPath expressions
  • XSL stylesheets

This book contains topics on using the sample fax templates provided with OPS and writing your own custom fax templates.

Guidelines

OPS uses an in-memory cache to improve the performance of fax template processing. The cache runs in the fax subsystem component. When the subsystem needs to open a fax template, it first looks in the cache. If it's not there, it loads it into the cache from the file path specified in the literatureProduct. This is important to know when you develop fax templates, because your changes, even though correctly saved to the file system, are not available to the subsystem if an older copy is in the cache. To clear the cache and reload the new version, you must shutdown all fax subsystems that might potentially use the template.

You should thoroughly test any fax templates that you create or modify. If your template causes an error, it could result in a malformed fax sent.

The stylesheet processing occurs within the context of a Windows Service with no user interaction. You should make sure that your template does not instantiate any dialog windows requiring user input.
