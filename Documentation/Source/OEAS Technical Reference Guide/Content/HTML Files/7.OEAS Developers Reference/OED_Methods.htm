Methods

The methods collection is a container that holds all of the methods owned by a business object.

Methods are the metadata entities that describe how the object performs work. Specifically, methods contain information defining how the object manipulates, persists, and retrieves data. The method is a complex entity that has a number of subordinate entities and attributes. Together they determine how the object interacts with the physical structures of the data tier.

Methods own parameter entities, which are essentially data conduits. They may be simple or complex. The majority of data a method manipulates is accessed through its parameters. Only global context data, such as the current user's ID, can be accessed without the use of parameters.

Methods also own step entities. Each step describes an individual activity within the scope of work performed by the method. Steps may also be simple or complex.

In addition to these complex entities, methods also have some simple attributes. Each method has a unique name within the object owning it, a description, versioning attributes, and a method type. The method type is very important, as it describes the method's expected behavior to the business tier core components.

The following method types are available:

  • insert
  • update
  • delete
  • retrieve
  • retrieveCollection
  • retrieveList
  • saveCollection
  • custom

For more information about these methods, see OEAS methods.
