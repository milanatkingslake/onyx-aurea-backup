orderBatch XML (OPS)

orderBatch XML structure

You must understand the orderBatch XML structure to customize or create print or email templates. Whether you are using XSL or Open XML Word, you must know how to apply XPath expressions to the orderBatch XML. For more information on XPath, see the W3C Recommendation 16 November 1999 (http://www.w3.org/TR/xpath).

The orderBatch XML structure contains the source XML input that your configured documents use for transformation or substitution rules. The orderBatch XML includes all the information for the literature orders, literature products, configured literature printers, and custom XML data relevant to a batch of orders.

Note: For a complete orderBatch example, you can look in the Utilities directory of the product CD for orderBatchEmailSample.xml and orderBatchPrintSample.xml.

The following XML fragment shows the basic structure of the orderBatch XML. Click a link for more information.

<orderBatch>

    <literatureOrders/>

    <literatureOrderBatch/>

    <customData/>

    <literatureProducts/>

    <literatureServicePrinters/>

</orderBatch>

Handling dateTime data in orderBatch

The OEAS stores all dateTime data as UTC. The fulfillment subsystem passes this data without conversion in the orderBatch XML. If you want to localize dateTime data, you have two options.

  1. You can create your print or email templates to label the data as UTC.
  2. You can create you print or email templates to perform a conversion of the data before outputting it. For email templates you can use the xsl:script element to perform the conversion.
