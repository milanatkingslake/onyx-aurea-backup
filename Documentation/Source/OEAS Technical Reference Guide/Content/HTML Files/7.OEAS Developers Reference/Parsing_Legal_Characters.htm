Parsing Legal Characters

When passing data to the Onyx Transaction Manager (OTM), the authentication components, or the session manager, ensure that your code for the calling application contains logic that verifies that only legal characters are being passed. Otherwise, the characters may not be parsed correctly.

For example, because the following characters have special meaning in XML, you must pass them to the OTM as encoded entities. Otherwise, the Microsoft XML DOM will try to parse them according to their XML meaning.

<

Less-than (begins an XML tag)

>

Apostrophe

&apos;

When customizing OEAS, be aware that the following parameters are frequently passed to the OTM, and therefore must adhere to these standards. OEAS Setup enforces the validity of these items when installing OEAS, but if you implement any of these for your customization, you must make sure that your calling application verifies that they contain only legal characters:

  • OEAS application name
  • user ID
  • user's network account names (DOMAIN\user)
  • OEAS source name
  • language code

To ensure that your customizations remain valid with future versions of OEAS and the Microsoft XML DOM, adhere to the legal characters defined by the World Wide Web Consortium (W3C) in section 2.2 of their document entitled “Extensible Markup Language (XML) 1.0 (Second Edition),” which is currently in recommendation status. You can view the document at http://www.w3.org/TR/2000/REC-xml-20001006.
