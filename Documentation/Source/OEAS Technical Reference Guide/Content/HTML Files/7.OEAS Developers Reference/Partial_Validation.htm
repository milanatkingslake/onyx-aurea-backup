Partial validation

A partial validation is performed each time you save your modifications to the OED and the following steps are performed:

  1. Each modified item is validated against the XML schema for the OED. If this validation fails, the save fails.
  2. The name for each modified item is checked for duplication of an existing object name. If a duplicate name is found, the save fails.
  3. The metadata for each modified object is validated for completeness and logical validity. If validation fails, processing does not stop, but the item that fails is saved as invalid. The criteria for determining completeness and logical validity are:
    • Contains an ID value for each element. If the value is missing, the Object Designer supplies a default value.
    • Contains valid data for all required fields.
    • Defines the maximum length of all fields.
    • Identifies accepted values for all enumerated properties.
    • Provides valid characters in element names.
    • Does not contain duplicate names.
