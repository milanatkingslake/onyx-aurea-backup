Match code functions and procedures

You can customize match code functions and procedures to fit the requirements of your business. This section addresses developers interested in using a custom approach to generate match codes by modifying the default procedures and functions. For overview information on match code functionality, see Match code.

OEAS calls the match code procedures and functions as post procedures on various logical methods. By default, these procedure maps are disabled in the OED. The following methods include match code procedure maps:

  • address.saveCollection (delete, insert, and update actions)
  • company.insert
  • company.update
  • individual.insert
  • individual.update

Before customizing these procedures, become thoroughly familiar with the guidelines in Customizing existing procedures. One can write new procedures or modify the default procedures.  The default functions and procedures are found on the OEAS product CD under Customization Support\Onyx Enterprise Database\Database Scripts. Assuming that you follow the custom naming conventions, you must update the procedure map for the relevant logical method using Object Designer and make sure that the procedure map is not disabled. The match codes are stored in the match_code_value table.
