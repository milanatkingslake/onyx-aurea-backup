Grant access

Unlike previous Onyx releases, members of the public role cannot execute stored procedures. Two new database roles have been created to tighten security. Custom procedures or other database objects that are mapped to object methods must be granted access to the appropriate role.

OnyxPublic

The OnyxPublic role has been created for any database object used to implement .

To grant access to the OnyxPublic role, sign on as the OEDB database owner and execute the following command to grant middle-tier execute access to the database object:

GRANT EXECUTE ON <cpMyProcedure> TO OnyxPublic

Grants will also be necessary for creating functions that return a table:

GRANT SELECT ON <cfMyFunction> TO OnyxPublic

OnyxLMViews

The OnyxLMViews role has been created for to restrict database access for List Management functionality.

To grant access to the OnyxLMViews role, sign on as the OEDB database owner and execute the following command to grant middle-tier execute access to the database object:

GRANT SELECT ON <cvMyListManagementView> TO OnyxLMViews
