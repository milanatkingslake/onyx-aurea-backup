Physical model

The physical model section of the OED contains the metadata that describes the schema of the Onyx Enterprise Database. The following table summarizes the physical model metadata.

Metadata Description

Tables

A collection of OEDB information to facilitate SQL generation

Columns

A collection of database column attributes required by SQL generation

Database actions

Row level filtering and other "where" clause fragments that the SQL generation requires for processing retrieve, update, and delete actions

Procedures and functions

Stored procedure and function prototypes used by Onyx Data Manager

Parameters

Input and output parameters for each procedure or function

The information on stored procedures and functions provides descriptions that are essentially declarations or function prototypes. The physical model must include these descriptions in order for a procedure or function to be called by a business object method. Translation of logical parameters to physical schema parameters for these calls is performed through the logical to physical mapping which is the last section of the OED.

Changes to the physical model section of OED do not automatically translate to OEDB and vice versa. If you make changes in one, you must make the corresponding change in the other.
