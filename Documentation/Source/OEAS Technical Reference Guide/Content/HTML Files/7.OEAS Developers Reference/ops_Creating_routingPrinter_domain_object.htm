Creating routingPrinter domain object

You make the data in ct_custom_routing_printer table available to the middle tier by creating the routingPrinter domain object. This is a domain object, because its data changes infrequently and is referenced by many other objects. Creating this object enables you to specify it as a tableAdministrationItem and hence use Table Administration to manage the routingPrinter data. Before adding a domain object using Object Designer, you should be familiar with the Onyx Object creation guidelines and the Creating domain objects scenario topics.

The following table shows the attributes required for the routingPrinter object. The other attributes are optional. Using Object Designer, add these attributes now.
