Object Naming
Object naming

Object, property, method, and step names are dictated by character requirements and by uniqueness requirements.

Allowed and disallowed characters

Object, property, method, and step names adhere to the W3C XML 1.0 10-February-1998 Recommendation with the additional constraint that periods (.), single quotes ('), and colons (:) cannot be used. With this requirement, names can begin with any letter and can contain any number of alphanumeric characters, underscores (_), or dashes (-). International users should consult the XML recommendation for specifics as to allowed characters in element names.

Important: Name fields in the OED may contain underscores, but the name cannot start with an underscore.

Uniqueness

The following uniqueness requirements are imposed on object, property, method, and step names:

  • Object names in the Onyx Enterprise Dictionary must be unique.
  • Property, method, and step names must be unique within their parent. For example, an object cannot have two properties with the same name, but two different objects can each have a property with the same name.
