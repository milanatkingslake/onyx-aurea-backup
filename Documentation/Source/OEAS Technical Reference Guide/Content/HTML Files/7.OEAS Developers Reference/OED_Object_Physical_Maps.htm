Object Physical Maps

The object physical maps collection is a container owned by a business object that holds all of the mappings that connect the object's properties to physical database columns. This collection holds the elements that describe correspondence between the logical model and the database.

In general, each object physical map is a mapping between a property on an object and a column in a database table. In addition to having attributes for the property, table and column, the map also has attributes that identify function and type. The functions that maps perform are:

  • persistence (SQL insert and update statements)
  • retrieval (SQL select statements)
  • filter (inclusion in WHERE clauses of SQL statements)

Note: There can be more than one map for a single property or column. The same property can have two persistance maps to two different columns that can also be in two different tables. Mapping multi-language support illustrates this point.

The following three map types are available for the functional mapping of a property to a column:

Map Type

Description

complex function

Allows a function to be called if the function either:

  • Accepts more than one parameter, or
  • Accepts a parameter that is different than the property's value

direct

Indicates that a property value is written or read directly from a database column without intervening interpretation of any kind.

function

Indicates that a database function is used in the generation of SQL. The property's value is the function's sole input parameter on persistence and, conversely, the column's value is the function's sole input value if the map is a retrieval map.

literal

When persisting data to the database, the literal type indicates that a literal value is used instead of a property value.

When the map is a retrieval map,  the literal type indicates that a literal value is used instead of a column value.

This type enables you to persist data to columns when no corresponding property exists, or to retrieve a literal value into a property when no corresponding database column exists. The literal value must resolve into a constant, a literal, or a single-valued SQL statement.

XSL literal

Enable you to retrieve a value from the input XML that may not be accessed through a property, context variable, or a method parameter. XSL literals can also include conditional logic by using elements such as the xsl:choose statement.

Note: Persistence maps are used for both insert and update methods within the database. When writing specialized functions for performing such tasks as ID or serial number generation, the database functions mapped here should be written to behave differently on insert versus update. Specifically, they might simply "pass through" a value when a valid value is received for the mapped property but automatically generate a value when the mapped property is input as a NULL value.
