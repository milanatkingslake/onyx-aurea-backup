Mapping multiple language support

Mapping multi-language support objects and tables illustrates an interesting point. OEAS implements multiple language support with multiple tables in the database for a given object in the logical model. The multiple tables include:

  • A base table
  • A multiple language extension table
  • A multiple language view table

The multiple language extension table contains the data that can be localized as well as key data. The base table contains key data and non-localizable data. The view table joins the base and multiple language extension tables into a single table used for specific language data retrieval. It is possible for a single property to have separate mappings to the physical model. Typically, for objects with multi-language support the persistence and filter mappings for a property connect to a column in one table while the retrieval mapping connects the property to a column in the multiple language view. Some properties map to the base table and the view, other properties map to the extension table and the view. A single property should always have its filter and persistence mappings pointed to the same column in a single table.
