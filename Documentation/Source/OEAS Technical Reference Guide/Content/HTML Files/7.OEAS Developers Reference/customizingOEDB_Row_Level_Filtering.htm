Row-level filtering

The stored procedures provided for customization are not implemented through the standard SQL generation methods. For this reason, they do not utilize the row-level filtering feature within the OED.

To implement row-level filtering for the result set of any stored procedure you modify, or any custom stored procedure you create, you must specify the filtering criteria in the WHERE clause of the stored procedure. Filter rules that you create using Object Designer have no affect on your custom stored procedures unless you pass the filter rule into the procedure through a parameter and add it to a dynamic SQL statement.
