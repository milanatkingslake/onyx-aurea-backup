region object

The region object enables users to retrieve a list of the regions for a specific country. Examples of region objects for a countryCode of US (United States) would include each state, or for a countryCode of CA (Canada) would include each province.

The data retrieved by the region object is database domain data that is stored in the region table, and which can be related to a country object. The domain data of the region table can be administered with the Table Administration tool.

Methods

Method

Description

retrieveCollection

Returns a collection of objects based on the search criteria supplied in the parameters XML

retrieveList

Retrieves a list of regions for the provided country code

retrieveListEnumeration

Returns region data that has been denormalized such that the country code is concatenated with the region name in the first column and the region code is returned in the second column

saveCollection

Performs multiple actions at once: inserting, updating, and deleting objects, as appropriate

Properties

Property

Description

active

Specifies whether the domain data the object provides is available to the user interface.

countryCode

Two-letter code designation for the country that contains the region.

insertBy

The user ID of the Onyx user who initially saves the object.

insertDate

The time and date on which the object is initially saved.

onyxTimestamp

Facilitates record locking with the Onyx Enterprise Database.

regionCode

A code or abbreviation for the region.

regionName

Display string for the region.

updateBy

The user ID of the Onyx user who last saved modifications to the object.

updateDate

Indicates the time and date on which modifications to the object were last saved.
