message element

A message element contains generic information intended for the client application. By itself, the existence of a message does not imply an error condition, but messages are often paired with errors in the methodStatus element. Each message has at least two and as many as four elements.

The message element typically contains:

Element Description

messageId

A numeric identifier for the message. This is typically the national language key value for the text of the message. For messages not taken from the national language table, this element is blank.

messageText

The text of the message.

timestamp

The exact time at which the method status was reported.

The message element may also include the associationId attribute.

Example

<message>

  <messageID>74582</messageID>

  <messageText>The data transfer to the ERP system was completed successfully.</messageText>

  <timestamp>2000-12-01 12:34:56</timestamp>

</message>
