epType

epType enumerates the SQL data type used with a parameter for a stored procedure call. Use this enumeration when configuring parameters for the CallStoredProcedure and CallStoredProcedureReturnExtendedInfo methods of the IEBusQE interface.

Array element name (enumeration value)

Corresponding SQL Server data type(s)

Supported data ranges

eptNoType (0)

Reserved for internal use

--

eptReturnCode (1)

Use for SQL Server return codes

(equivalent to the int data type)

eptChar (2)

char

Up to 8,000 characters of non-Unicode data

eptVarChar (3)

varchar

Up to 8,000 characters of non-Unicode data

eptWChar (4)

nchar

Up to 4,000 characters of Unicode data

eptVarWChar (5)

nvarchar

Up to 4,000 characters of Unicode data

eptBSTR (6)

Reserved for future use

--

eptBoolean (7)

bit

0 and 1

eptDateTime (8)

datetime

smalldatetime

datetime: January 1, 1753, through December 31, 9999, with accuracy to milliseconds

smalldatetime: January 1, 1900, through June 6, 2079, with accuracy to seconds

(The Query Engine supports a more restrictive date range and supports accuracy of date times only to seconds)

eptCurrency (9)

money

-2^63 (-922,337,203,685,477.5808) to 2^63 (-922,337,203,685,477.5807), with accuracy to a ten-thousandth

eptSingle (10)

float

Floating precision numbers from -1.79E+38 through 1.79E+38

eptDouble (11)

decimal, numeric

Fixed precision numbers from -10^38+1 through 10^38-1

eptSmallInt (12)

smallint

Integers from 2^15 (-32,768) through 2^15 (32,767)

eptTinyInt (13)

tinyint

Integers 0 through 255

eptInteger (14)

int

Integers from 2^31 (-2,147,483,648) through 2^31 (2,147,483,647)

eptULong (15)

Reserved for future use

--

eptTEXT (16)

text (input parameters only)

2^31-1 (2,147,483,647) characters of non-Unicode data

eptNTEXT (17)

ntext (input parameters only)

2^30-1 (1,073,741,823) characters of Unicode data

Remarks

The members of this enumeration are used only for stored procedure parameters sent through the CallStoredProcedure and CallStoredProcedureReturnExtendedInfo methods. See Result sets for information about the specific data types that are returned in result sets.

Note: The data types that are not supported in result sets are also not supported for use as parameters in stored procedure calls. Using an unsupported data type in a parameter may cause unexpected results.
