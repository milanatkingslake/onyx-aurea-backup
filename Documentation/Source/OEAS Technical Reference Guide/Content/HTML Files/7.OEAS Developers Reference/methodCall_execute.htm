execute (methodCall)

execute method

The execute method of the methodCall object calls the OnyxMethodCallExecute step component with the specified XML input and returns the output XML from the step component.

Remarks

The execute method supports multiple children in a saveCollection method, but the number of children is predetermined by the methodCall designer in the methodCallDetails. For example, if the designer wants to create an individual insert method call that inserts three phone objects for the individual, the designer must create methodCallDetails for three phone objects. When the execute method forms the method call, it orders the methodCallDetails by the sequence number.

Note: The following two examples are intended for instructional purposes only and do not represent a complete set of details for an individual insert method.

inputXml schema

inputXml schema

The input XML must conform to the following schema.

Element/@Attribute(Cardinality)

Cardinality

Comments

root

1

 

 

inputs

1

 

 

input

0..*

 

 

@idProperty

1

You can use one of three values for this attribute:

  • primaryId
  • internalName
  • name

The value determines which methodCallDetail property the step component uses to identify this input. For example, if you want to use primaryId to tell the step component which methodCallDetail defines this input, set idProperty="primaryId".

@idValue

1

Contains the property value of the methodCallDetail corresponding to the idProperty attribute. For example, if the @idProperty="primaryId" and the methodCallDetail defining this input has a primaryId of 100197, set idValue="100197".

@null

0..1

If you set this equal to '1', it specifies that the value of the property identified by @idProperty is NULL.

Note: The execute method does not assume any default values for the input XML.

To add attributes to a method call input detail, use the attributeList property of the methodCallDetail object.

If the output is a list, contains the text node of the element pointed to by the idPath property of the methodCallDetail object. If the output is a scalar value, this attribute is absent.

@description

0..1

If the output is a list, contains the text node of the element pointed to by the descriptionPath property of the methodCallDetail object. If the output is a scalar value, this attribute is absent.

@text

0..1

If the output is a scalar value, contains the text node of the element pointed to by the elementPath property of the methodCallDetail object. If the output is a list, this attribute is absent.

returnXml

0..1

In the event of an error, contains the entire returnXml of the logical method

Note: The execute method does not assume any default values for the output XML.
