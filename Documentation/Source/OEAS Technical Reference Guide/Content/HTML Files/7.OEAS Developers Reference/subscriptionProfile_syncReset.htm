syncReset (subscriptionProfile)

syncReset property

The syncReset property of the subscriptionProfile object indicates whether to recreate the profile's offline datastore.

Data type

boolean

Access

Read/write

Remarks

Warning: A reset causes a user to lose all unsynchronized data in the offline data store. To avoid data loss, coordination with users is recommended to ensure that all data is upsynced prior to the reset.

When syncReset is true, the synchronization server does the following tasks:

  1. Deletes the profile's offline datastore.
  2. Sets all subscriptionItem instances associated with the profile as "dirty." Each instance is set to refresh (syncAction 1), which during synchronization causes a fresh version of the instance to be retrieved from the Onyx Enterprise Database (OEDB).
  3. Synchronizes all object instances identified by the profile's subscriptionItem instances to the offline datastore.
  4. Clears the syncReset flag after the reset occurs.

Valid values of this property are:

  • false (0): (Default) Do not reset the profile.
  • true (1): Reset the profile.
