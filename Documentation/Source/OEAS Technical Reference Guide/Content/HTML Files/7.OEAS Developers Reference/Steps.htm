Steps

Each logical method in the OED is comprised of one or more steps that determine how that method behaves. Each step is defined in the OED with these elements:

Element Description Examples

id

A globally unique identifier (GUID) that identifies the step definition.

Specifies a custom XSL template for the step that overrides the default XSL template. This feature is not supported for this release; this element must be empty.

Onyx

A value of Onyx indicates that this step was installed by Onyx and is not a custom step.

Boolean value that indicates if the step is enabled (0) or disabled (1). A disabled step is not executed as part of the method.

When a client initiates a request to call a method for an object, the Onyx Transaction Manager (OTM) queries the OED cache by calling the GetSteps method for a list of the steps that comprise the method. The OTM then executes those steps in the specified order.
