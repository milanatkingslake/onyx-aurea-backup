statusType element

The statusType element contains the condition of the method logic. Step components and the OTM only use the statusType element for logic errors and warnings, such as failing to find an expected XML node, and not for hard errors, such as failing to create an XML parser. For hard errors, the OTM and step components return a bad HR code. When the OTM creates the step package, it sets the value of statusType to success. If the OTM or step components encounter a logic error or warning condition during method execution, they set the statusType element to one of three values.

statusType Value

Description

success

The method succeeded.

successWithInfo

The method succeeded but returned extra information for the caller.

failure

The method did not succeed.

Onyx step components follow a convention for changing the value of statusType. This convention is intended to prevent a step component from creating a situation where invalid data may be processed, or where informational data might be ignored by a client application.

If the value of statusType is success, step components may change the value to one of the following:

statusType Value

Description

New Value

Reason

successWithInfo

There is a need to add an informational message to the returnXml.

If the value of statusType is successWithInfo, step components may change the value to failure if an unrecoverable error occurs. A step component should not change a value of successWithInfo to success, as a client application may not search for the corresponding message.

If the value of statusType is failure, step components should not change the value for any reason. In fact, Onyx step components that encounter a statusType of failure will not even attempt to execute their programmed actions.
