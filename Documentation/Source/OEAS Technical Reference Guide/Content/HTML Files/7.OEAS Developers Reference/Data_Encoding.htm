Data Encoding
Data encoding

There are two forms of XML data encoding: element encoding and attribute encoding.

Element encoding

OEAS uses element encoding as the standard for all data. Data items include:

  • Simple properties
  • Collections of properties
  • Objects
  • Collections of objects
  • Null properties and objects
  • Missing properties and objects

Attribute encoding

The attributes defined for OEAS and their use are described below:

  • action: Indicates the database action performed on an object. Possible values are, "insert", "update", or "delete". This attribute is only used by the saveCollection method.
  • associationId: Helps you link error messages back to the data that caused the error.
  • collectionId: collectionId can be added to each item in a collection by a calling client, they are then returned in the items so the client can associate the items sent to the items received. This is especially useful when the primary keys for the item are generated as a result of the call, because it allows a unique way of identifying each item. This attribute is only used by the saveCollection method.
  • content: Indicates whether the object is fully populated. Possible values are "all", "any", "keysOnly", or "partial". (The default value is "all".)
  • null: When set to "1", indicates that a property on an object does not contain a value. When omitted, indicates that a property does contain a value (though that value may be the empty string).
  • objectType: Indicates the type of object a given element contains
  • collection: Specifies that the element contains a collection of child objects.
