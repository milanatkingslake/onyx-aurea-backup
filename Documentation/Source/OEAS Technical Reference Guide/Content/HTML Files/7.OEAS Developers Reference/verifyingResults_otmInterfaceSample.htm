Verifying the Results (OTM Interface Sample)
Verifying the results

The OTM returns an XML string with a root node of <returnXml>. If your call to the OTM succeeds, the returnXML includes a child node statusType with the value "success". The parametersReturn child node contains the data pertinent to the record modified or retrieved. The returnXML also includes an otmId node for debugging purposes. You can use the otmId to correlate the returnXml with messages in the Event Viewer. The following shows the returnXml for a successful company insert method.

returnXml

<returnXml>

   <otmId>088BA0B0-36E9-4C76-B07E-855D7BF6AE56</otmId>

   <methodStatus>

      <statusType>success</statusType>

   </methodStatus>

   <parametersReturn>

      <company collectionId="99" objectType="company" content="keysOnly">

         <primaryId>9DE0F89D-4996-4C0C-8F2F-B27FE07DEDFE</primaryId>

         <phones Collection="1" ObjectType="phone" objectType="phone" collection="1"/>

         <addresses Collection="1" ObjectType="address" objectType="address" collection="1"/>

      </company>

   </parametersReturn>

   <customData/>

</returnXml>

You can use the returnXml to display a success message with the new record number to the user interface or pass the returnXml to another step component, but that is not covered in this sample.
