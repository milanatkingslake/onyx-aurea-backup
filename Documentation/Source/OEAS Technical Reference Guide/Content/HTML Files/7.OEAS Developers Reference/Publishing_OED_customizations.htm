Publishing OED customizations

Important: Before publishing your modifications to your OEAS production environment, test them by publishing them to an OEAS test environment. The changes that you make to OED can affect the functionality of dependent products such as Onyx Enterprise Studio and Onyx Employee Portal.

The Object Designer simultaneously publishes all modifications made to the working copy of the OED. You cannot publish only certain portions of the working copy. For this reason, among others, you should make sure that you implement some type of version control over the various copies of the OED.

When you publish your modifications to the Onyx Enterprise Dictionary (OED), the Object Designer:

  • Validates your changes.
  • Overwrites the production copy of the OED with the contents of the working copy of the OED.
  • Creates an abridged OED that is smaller and optimized.
  • Generates the XSL run-time style sheets for method calls.
  • Generates and compiles the OEDB stored procedures.
  • Inserts reference data into the OED tables.
  • Stops COM+. (The OED cache refreshes when the COM+ application restarts.)

After the Object Designer publishes your modifications to the production copy of the OED, you must restart the COM+ application if the Object Designer fails to do so.

You might see a "permission denied" message on the final step of the publish process. This means the credentials the web site is using does not have permission to stop COM+ and this step must be performed by an administrator.
