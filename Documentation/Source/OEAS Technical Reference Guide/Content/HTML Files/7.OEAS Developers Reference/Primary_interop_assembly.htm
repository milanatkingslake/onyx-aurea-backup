Primary interop assemblies

The best practice for obtaining smooth integration of .NET step components with components of OEAS is to reference the signed primary interop assemblies in your Visual Studio project. The IStepStringPIA.DLL interop assembly should be referenced by a .NET step component that implements the IStepString interface.

If .NET applications call Onyx core COM+ components, your .NET project should reference the corresponding interop assembly for the Onyx component:

  • GetAppDataPIA.dll
  • IStepStringPIA.DLL
  • OnyxAuthenticatePIA.DLL
  • OnyxAuthNTPIA.DLL
  • OnyxDictionaryCacheProxyPIA.DLL
  • OnyxEBusQEPIA.dll
  • OnyxEDCachePIA.dll
  • OnyxLogPIA.dll
  • OnyxNatLangCachePIA.dll
  • OnyxNatLangProxyPIA.DLL
  • OnyxSMPIA.DLL
  • OnyxTMPIA.DLL

The primary interop assemblies are located on the OEAS installation CD in the following folder:

CD\Customization Support\Onyx Enterprise Application Server\MS .NET Support

To implement the primary interop assemblies:

  1. Copy the primary interop assemblies that you need from the OEAS installation CD to the \Program Files\Onyx\Primary Interop Assemblies\ folder. You might have to create this folder.
  2. Add the copied primary interop assemblies to the Assembly Cache so that the assembly can be used by any step component in your .NET environment. To add an assembly, use the Microsoft .Net Framework Configuration Tool in Administrative Tools, or use the gacutil.exe utility from the command line as follows:

\<PATH_TO_EXE>\gacutil /i "C:\Program Files\Onyx\Primary Interop Assemblies\IStepStringPIA.dll"

    The <PATH_TO_EXE> depends on your system. You might have to search the file system for the location of the utility, or your .Net Framework installation might not include the utility.

  1. You might need to adjust the .NET step component's security settings for it to execute properly in the .NET environment.
  1. Open the .NET Framework Configuration Tool from the Administrative Tools menu.
  2. Click Runtime Security Policy in the Navigation tree and select Increase Assembly Trust in the pane on the right.
  3. Follow the directions that appear in the wizard to increase the level of trust granted to the .NET step component.
