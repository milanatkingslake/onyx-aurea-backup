Filter Rule

The filter rules collection is a container that holds the filter rule entities that apply to a specific method.

Filter rule entities describe extra conditions applied to SQL operations on a table and are owned by action elements. Filter rules only have two attributes: description and "rule text." The value of the "rule text" attribute is a string literal which can contain embedded XSL commands and SQL statements. After the XSL commands, if any, are processed, the string must resolve into a valid SQL WHERE clause fragment preceded by the SQL keyword "AND."

Important: Onyx recommendeds that multiple conditions following the "AND" keyword be enclosed with a set of parenthesis.

The row level filtering rules are added to the WHERE clauses of SQL commands created by the SQL Generation process during method execution. The filter rules selected for execution depend on the owner action as well as the table that the properties are mapped to in the object physical mapping.

  • All retrieve actions will be applied to retrieve and retrieveCollection methods.
  • All update actions will be applied to the update method and the saveCollection method where the object's action attribute is update.
  • All delete actions will be applied to the delete method for domain objects.

Note: Transactional objects do not support filter rule functionality as of this release.
