systemDocument Object

systemDocument object

The systemDocument object enables an administrator to specify a literatureProduct that is included with every group of orders processed by a subsystem of the Onyx Processing Server (OPS). For example, the literatureProduct objects can include a "Batch Report" product that an administrator can configure to print with every batch of literatureProduct objects processed by the OBPS.

Methods

Method

Description

retrieveCollection

Retrieves a collection of systemDocuments

saveCollection

Saves a collection of systemDocuments

Properties

Property

Description

active

Specifies whether the domain data the object provides is available to the user interface

fulfillmentTypeCode

Indicates whether the system document is an email or a print document

insertBy

Contains the user ID of the Onyx user who initially saves the object

insertDate

Contains the time and date on which the object is initially saved

primaryId

Contains the unique ID assigned to the object when it is first saved to the database

productDescription

Contains the description from the product object

productNumber

Contains a foreign key to the literatureProduct object that represents the systemDocument

subsystemId

Contains a foreign key to the subsystem object that uses the systemDocument

systemDocumentType

Indicates whether the system document is generated per order or per batch

updateBy

Contains the user ID of the Onyx user who last saved modifications to the object

updateDate

Indicates the time and date on which modifications to the object were last saved
