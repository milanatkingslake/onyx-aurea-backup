Fulfillment engines (OPS)

Fulfillment engines

The Email engine and the Print engine produce the literature product for delivery by:

  1. Merging the order information with the email or print templates.
  2. Delivering the resulting document as an SMTP (email) message or to the printer.

The following table lists the fulfillment types supported by OPS.

Fulfillment Type

Description of types

Send as Email Body (supports merging)

Created from an XSL template and merged with data to create an individualized message. The message appears in the body of the email.

Send as Email Attachment (supports merging)

Created from an XSL template and merged with data to create an individualized message. The message is sent as an attachment to the email.

Send as Email Attachment (no merging)

Can be sent as a binary file attachment to an email. Examples include Graphics Interchange Format (GIF) files, Adobe Portable Document Format (PDF) files, or executable program files (.EXE files). This type does not support data merging.

Send as Printed Document

Created from Open XML Word with Mail Merge templates and merged with data to create individualized printed documents. The document is automatically routed to a printer queue for printing. Uses include form letters, graphics, and company literature, mailing labels, and system documents.

Send as Physical Item

Can be physically mailed to a customer. Uses include company apparel, manuals, color brochures, and other packaged products. This type does not support data merging.
