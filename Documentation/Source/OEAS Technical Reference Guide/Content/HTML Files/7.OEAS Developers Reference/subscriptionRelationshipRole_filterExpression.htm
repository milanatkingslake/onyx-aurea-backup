filterExpression (subscriptionRelationshipRole)

filterExpression property

The filterExpression property of the subscriptionRelationshipRole object contains an expression used in a WHERE clause that limits the secondary objects subscribed to according to the roles that the subscribing user is a member of.

Data type

string

Access

Read/write

Remarks

When a user is a member of more than one role, the WHERE clause can have more than one filter expression. The comparison operator is additive for the expressions.

When the user does not belong to a role that has a subscriptionRelationshipRole defined for the subscriptionRelationship to which a subscribed object belongs, the cascade method uses the contents of the defaultFilterExpression property of the subscriptionRelationship object.
