Adding the template binding

Adding the template binding connects the script control template to the script prompt template. The script prompt template contains instructions to be performed for all script prompts, so the stylesheet calls it first. Once the script prompt template has rendered the header and navigation panes, it calls the correct script control template, in order to include the control-specific HTML.

When inserting the template binding into the template, be careful to insert it at the correct point in the template. Inserting the template binding in the wrong location can break the valid structure of the XML. The template binding is an xsl:when element that must be a child of an xsl:choose element.

To add the template binding:

  1. With the script_delivery_step.xsl file open in a text editor, find the following xsl:template element.

<xsl:template match="scriptPrompt" mode="render">

  1. Within the xsl:template element, find the following comment line.

<!--make the selection of the control type and call the template to output the html element-->

<xsl:choose>

  <xsl:when test="@controlTypeCode = 'dropDownListBox'">

  1. Immediately following the closing angle bracket (>) of the xsl:choose element, add a new blank line.
  2. To add the template binding code to the xsl:choose element following this comment, copy and paste the following xsl:when clause into the template at the cursor position.
  1. Save and close the script_delivery_step.xsl file.

Binding code

<xsl:when test="@controlTypeCode = 'customerSearch'">

    <xsl:call-template name="control_customerSearch">

         <xsl:with-param name="safePromptId" select="$safePromptId"/>

    </xsl:call-template>

</xsl:when>

The code calls the custom control template by name, passing it the ID of the prompt that owns it.

Note: The name of the template is case sensitive and must match the name used in the Javascript class definition and the database domain data.
