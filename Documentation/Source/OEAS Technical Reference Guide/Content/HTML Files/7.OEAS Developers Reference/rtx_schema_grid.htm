Run-time XML grid
Master run-time XML grid

The master run-time XML grid contains all possible nodes in the run-time XML. The composition of the run-time XML for any given call to the script engine may contain a subset of the master list. For specific contents of a given method call see the appropriate method topic in this overview.

View Grid explanation

Element/@Attribute

This column contains the local name of each XML node in the run-time XML. Indentation signifies hierarchy in the XML structure. The @ symbol prefixes attribute nodes.

Cardinality

Cardinality indicates how many of the given node are expected in the run-time XML. One (1) indicates the node is required in the run-time XML. One to many (1..*) indicates a required node with no upper limit on the number of nodes present. Zero-to-one (0..1) and zero-to-many (0..*) indicate nodes that are optional in the run-time XML. Required nodes may be children of optional nodes. If this is the case, the child is only required if the parent is present. The master table shows the superset of all possible cardinalities. See the specific method tables for cardinality in a given instance.

Default

If an optional node is missing, the script engine can assume a default value. This column lists the value that the engine provides. Required nodes have no default value. Most optional nodes have default values with the exception of collection elements. If the node has no default value, the column entry is blank. If the default value is NULL, the attribute is not present in the run-time XML unless the caller includes it.

Comments

This column contains any particularly notable information about the node. For a general description of the node, see the detailed topic by clicking on the local name in the first column.

variable

1..*

 

Variable elements consist of two types--scalar or list. A scalar variable contains a single item child. A list variable contains multiple item descendants in a list child. The run-time XML structure depends on the type of variable.

 

 

 

@name

1

 

 

 

 

 

@type

1

 

Valid values include 'scalar' or 'list'. If set to scalar, the variable has an item child and no list child. If set to list, the variable has a list child and no item child.

 

 

 

@stepId

0..1

NULL

This attribute specifies whether the variable is retained in run-time XML for the remainder of the script or is dropped after step completion. It is intended primarily to eliminate the overhead of large list variables.

 

 

 

item

0..1

 

The item child of the variable element is only present if the variable type is scalar. This element has no corresponding LBO property or OEDB schema.
