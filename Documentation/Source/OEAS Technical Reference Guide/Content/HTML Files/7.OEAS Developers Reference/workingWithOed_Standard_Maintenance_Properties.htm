Standard maintenance properties

Your custom business object must include the standard maintenance properties if you want the object to leverage SQL generation's standard methods. If you are only using custom methods, the object does not need to include these properties.

Note: Running the SQL generation standard methods against an object that does not contain the expected maintenance properties generates database errors.

Except for the active property, Onyx recommends setting the Read Only attribute for standard maintenance properties. For more information, see the Read Only attribute section of the Properties topic.

The table shows the onyxTimestamp property as optional. You must include this property if your object is going to support optimistic concurrency control. If concurrency control is not required for the object, you can omit the onyxTimestamp property.

The active property is included on domain objects when you want to control the availability of a record to the user interface. Usually you cannot delete domain data that is currently in use, but you might want to prohibit its use in new records. You use the active property to implement this type of rule.
