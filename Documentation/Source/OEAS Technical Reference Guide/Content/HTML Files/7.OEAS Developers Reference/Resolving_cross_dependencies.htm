Resolving cross-dependencies

When you disable or delete objects and/or methods in the OED, you must resolve all cross-dependencies with these objects and methods.

For example, if you disable an object and save your changes, you must perform additional work to ensure that no other property, method, parameter, or step depends on the existence of that object.

Although you can disable any item in the OED, you need to resolve cross-dependencies only when you disable an object or a method because only objects and methods can reference other items. Properties, parameters, and steps can be referenced by other items, but they cannot themselves reference other items.

Note: Finding cross-dependencies is easy. After you disable/delete an item, click the Validate button on the Object Designer toolbar. The validation process locates all items that are no longer valid.
