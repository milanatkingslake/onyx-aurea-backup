Naming conventions for custom data tier objects

To help you identify custom database objects that you create to support your OEDB customizations, and to help ensure that your custom database objects aren’t overwritten during future OEDB upgrades, we strongly encourage you to use the following naming conventions.

When naming custom data tier objects:

  • Use singular names. For example, use "employee" (singular) instead of "employees" (plural).
  • Use lower case.
  • Use underscores to separate words in custom data tier object names. For example: csu_account_header (Exception: Do not use underscores when naming custom SQL functions or stored procedures.)
  • Use the conventions described in the following tables, including prefixes and suffixes).

    • Prefixes and suffixes used in the naming conventions

  • Prefixes used in these conventions include:
    • "cc" (custom column)
    • "cfk" (custom foreign key)
    • "ci" (custom index)
    • "csf" (custom SQL function)
    • "csp" (custom stored procedure)
    • "csu" (customer specific user table)
    • "fk" (foreign key)
  • Suffixes used in these conventions include:
    • "idx" (index)
    • "lm_view" (list manager view)
    • "ml" (multi-language)
    • "ml_view" (multi-language view)
    • "ref" (reference parameter foreign key - this convention helps you distinguish these keys from other foreign keys)
