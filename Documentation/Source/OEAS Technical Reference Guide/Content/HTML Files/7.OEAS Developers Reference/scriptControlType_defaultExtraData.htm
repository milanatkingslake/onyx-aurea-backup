defaultExtraData (scriptControlType)

defaultExtraData property

The defaultExtraData property of the scriptControlType object contains a semi-colon delimited list of 'name=' tags.

Data type

string

Access

Read/write

Remarks

The defaultExtraData property represents a parameter template for the control. Use this property to store the list of parameter names for which the control needs values at design time. You can use this property to store a default name-value pair or just the 'name' half of a name-value pair. For example, a design tool can display the name-value pair or name tag to the script designer at design time, so the designer knows he or she needs to supply a value for the control to work properly. For example, the product control requires a product type which is represented by 'type=' (without the quotes) in the defaultExtraData property. At design time, the script designer sees the 'type=' string in the Extra Data field of the design tool. The designer needs to provide a value such as 'standard' (without quotes; ex: type=standard). This tells OEAS that this instance of the product control displays only standard products.

If your control requires no extra data to function properly, this property should contain NULL.

To use this property for passing additional information to the control, you must set the allowExtraData property to true (1).
