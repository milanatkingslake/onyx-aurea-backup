actionBatch object

The actionBatch object allows a caller to request multiple method calls to various OEAS business objects in a single call to the Onyx Transaction Manager (OTM). The single method that it exposes, execute, allows the results of a method call to be used as input in a method that follows. For instance, a company's procedures may require that several tasks need to be completed when a certain incident occurs. A single execute call can create the incident, create several tasks owned by the new incident, and then set reminders for the Onyx users who need to complete the tasks.

Methods

Method Description

delete

Deletes the specified object instance

execute

Executes a series of method calls using the OTM

executeForEMS

Documentation is currently not available for this method

insert

Persists an object to the database

retrieve

Returns a single instance of a specified object

retrieveCollection

Returns the collection of object instances based on search criteria

retrieveListByStatusIds

Returns the list of objects according to their status IDs

retrieveNextBatch

Retrieves the properties in a rowSet.

update

Persists modifications to an existing object instance

updateStatus

Updates the actionBatch status without using the onyxTimestamp

Properties

Property Description

actionBatchXML

Contains the information of the OEAS method that calls the batch subsystem to execute the actionBatch

insertBy

Contains the user ID of the Onyx user who initially saves the object

insertDate

Contains the time and date on which the object is initially saved

onyxTimestamp

Facilitates concurrency control with the Onyx Enterprise Database

primaryId

Contains the unique ID assigned to the object when it is first saved to the database

processedDate

Contains the date on which the batch was processed

scheduledProcessDate

Contains the date on which the batch process is expected to begin

statusCode

Contains the status code of the batch such as 0 (Not Processed),
1 (Processing Jobs), 2 (Error) and 3 (Complete) processed by Onyx Processing Service

subsystemId

Contains the ID of the subsystem that processes the actionBatch object

summary

Contains the details of the batch process from Onyx Processing Service

updateBy

Contains the user ID of the Onyx user who last saved modifications to the object

updateDate

Indicates the time and date on which modifications to the object were last saved
