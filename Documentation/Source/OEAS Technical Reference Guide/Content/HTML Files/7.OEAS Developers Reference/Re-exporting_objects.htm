Re-exporting objects

Re-exporting OEAS business objects into OPM

Before you re-export an object into OPM, consider the following:

  • If you change the name of a business object in the OED and re-export the object, be aware that the original exported object (object 1) remains untouched in the Process Designer and a new object appears (object 2) with the new name.
  • You cannot re-export the properties of an object once they are in the OPM library. The re-export process does not override any existing attributes whose names match the properties being exported. Therefore, only new properties of an object are created. To work around this, you can delete the attributes from the library and then re-export the object into OPM.
