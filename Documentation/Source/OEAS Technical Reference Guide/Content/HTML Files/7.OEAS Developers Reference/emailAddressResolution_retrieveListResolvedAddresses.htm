retrieveListResolvedAddresses (emailAddressResolution)

retrieveListResolvedAddresses method

The retrieveListResolvedAddresses method of the emailAddressResolution object resolves input strings to email address records in the database.

Remarks

This method resolves email addresses for an email message. The procedure may be used for email messages picked up by the email monitor subsystem (inbound) or email messages that are created within OEP (outbound). Inbound email messages will resolve a list of email addresses against the company, individual, and users tables. Outbound email messages will resolve a list with a combination of email addresses and a portion of someone's name against the individual, email_address_book_item, and users tables.

This method returns more than one row for a single input string when multiple database records fulfill the search criteria. The sort order of the results is by ordinalPosition followed by objectType. When no records fulfill the search criteria, a single row indicating no match is returned.

For inbound email searches, this method searches against the email addresses of individuals and companies, and the email aliases of Onyx users. Only exact matches are returned in the result set. The matches are not case-sensitive.

For outbound email searches, this method makes like searches against the individual, user, and emailAddressBookItem objects. All like searches are not case-sensitive. A wildcard character is added to the end of input strings. For example, the input string 'Smith, Mary Ann' results in a like searches for 'Smith%' and 'Mary Ann%'.
