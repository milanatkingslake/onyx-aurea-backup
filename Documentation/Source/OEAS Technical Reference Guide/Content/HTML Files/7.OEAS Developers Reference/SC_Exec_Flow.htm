Execution flow

This section outlines the basic execution path that most Onyx step components follow to accomplish their tasks. Since the behavior of step components in the system is not enforced by any of the core business objects, this section also explains many of the conventions that Onyx uses to maintain compatibility with any possible system customizations.

The execution is broken down into three portions:

Topic

Contents

Handle input data

Verify that the input data is in place and complete.

Process

Execute actions using the data.

Handle output data

Assemble the output data for the client, and add any relevant messages.
