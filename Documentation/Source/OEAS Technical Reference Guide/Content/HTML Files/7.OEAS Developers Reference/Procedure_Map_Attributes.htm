Procedure Map attributes

When you select a procedure map in the Object Designer, the attributes for that procedure map display in the edit area.

The attributes are defined as follows:

  • ID: The globally unique identifier (GUID) assigned by the Onyx Enterprise Dictionary. This attribute is read-only.
  • Procedure Map:Select a procedure that has been defined in the Procedure section of the Physical Model. When you select a procedure, the input and output maps are automatically generated for the procedure's parameters. Edit the maps to provide the necessary attributes for each input map and output map.
  • Map Type: Select the type of map for this procedure:
  • To execute the procedure before the operational code generated for the step, select pre.
  • To execute the procedure after the operational code generated for the step, select post.
  • To execute the procedure instead of the SQL code generated for the step, select replace.
  • Action: When the method type is saveCollection and the Map Type is either pre or post, the procedure map lets you specify the saveCollection actions on which the procedure executes. Select collection to execute on all actions; collection by owner to execute the procedure once for each distinct owner in the collection; or select insert, update, or delete to execute on only one or two of the action types.
