Error Reporting

Onyx 360 add-in automatically logs errors and messages to the Windows Event Viewer by default.  System Administrators can optionally set up text-based logging on the user’s local computer to create a diagnostic log file that can then be sent through email.

Onyx 360 supports four different levels of logging for diagnostic purposes.

Log Level Detail
1

Errors only

2

Errors and Warnings

3

Errors, Warnings, and Informational messages

4

Verbose logging – everything is logged

Note: For performance considerations, it is recommended to leave the log level at 1.

Log files are created on a daily basis, so it is easy for System Administrators to set up a retention period to manage the collection of diagnostic data from the Onyx 360 Add-in.  Older files are deleted at the expiration of the retention period.  Default retention period for log files is seven days.

Users may access the log file on their local computer, or send through email it to the System Administrator from the Onyx 360 Settings page through the links available on the bottom of the page:

Open Log

Opens the current log file in Notepad for a user or System Administrator to review.

Report an Issue

Users may click this link to report issues to their System Administrator through email. Clicking the link automatically opens a new email message and attaches the current log file (if exists) to it.

Note: If the System Administrator’s email address is specified in the configuration, this message will be automatically addressed to that email address.
