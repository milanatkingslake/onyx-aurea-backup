Add-in Settings (Administrator)

The application-wide settings that control the behavior of the Onyx 360 Add-in for Outlook are controllable only by a System Administrator.  These settings are typically set during installation time and will not require frequent updates.

Setting Name Description Defaults (if any)

OnyxSiteId

Site ID of the Onyx installation

Set at implementation

OnyxApplicationName

Logical Application Name for the Onyx installation

Set at implementation

EnableOnyxAuthentication

When set to True, allows password based authentication to Onyx.

For Integrated Windows Authentication set this setting as False.

True

OEASAuthWinURL

URL for the Onyx Enterprise Application Server (OEAS) Authentication Web Service (Integrated Windows Authentication)

Set at implementation

OEASWinURL

URL for the Onyx Web Service (Integrated Windows Authentication)

Set at implementation

OEASAuthOnyxURL

URL for the Onyx Enterprise Application Server (OEAS) Authentication Web Service (Onyx user-id/password authentication)

Set at implementation

OEASOnyxURL

URL for the Onyx Web Service (Onyx user-id/password authentication)

Set at implementation

OEPLauncherURL

URL for launching records in Onyx CRM from the Onyx 360 panel in Outlook.

Set at implementation

LogLevel

Controls the verbosity level of log messages.

1-Errors, 2-Warnings, 3-Info, 4-Verbose

1

LogToFile

When set to True, all log entries will be written the Windows Event Log and to a text file in the user's local computer.

%LocalAppData%

When set to False, all log entries will be written only to the Windows Event Log (Source: OnyxOutlook360)

False

LogFileRetentionDays

Determines how far back in days to retain the log files.

7

LogOnyxAdminEmail

System Administrator/IT Support email address for reporting issues.

Set at implementation
