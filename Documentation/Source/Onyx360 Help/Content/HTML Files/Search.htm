Search by Name/Phone/ID

You can perform searches on a Contact’s full name (for example, John Doe) or last name (for example, Doe).  You can perform searches on full or partial Company names (for example, Earth Scents or Earth). 

The Search bar also allows searching by phone number or the contact’s Onyx ID, if known.

The results of the search are displayed in the Customer Panel below the Search Bar.

All name searches are partial match searches – that is, if a portion of a Contact’s last name matches the search query, that record will be included in the search results.  If a portion of a Company’s name matches the search query, the Contact records associated to the Company record will be included in the search results.

As seen in the image above, a search for name Smith will return individual records from the Onyx database with last names beginning with Smith – Smithers, Smithy, and so on along with individual records associated to company records that have the name Smith in them – for example, Smithsonian.

Onyx 360 implements automatic pagination for the search results.  Ten records are displayed per page and links to other result pages are shown at the bottom of the panel for quick access.

Search results are hyper-linked to load the record in Onyx CRM directly from Outlook.  Users will need to be logged on to Onyx for this feature; if a user is not logged on, the user will be prompted for log on credentials.

Note: For performance reasons, the search algorithm excludes searching only on first name. 

For example, the search string of John will include only individual records with last names that start with John (for example, Johnson) and individual records associated with companies with the string John in them.  Individual records with first names starting with John will not be included in the results list.

Tip: Searching on full names (for example, John Doe) will result in more accurate matches!
